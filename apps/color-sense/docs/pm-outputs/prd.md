## 2. 문제 정의

### 핵심 문제
사람들은 자신의 색 감각이 얼마나 좋은지 궁금해하지만, 병원 검사 외에 가볍게 테스트할 방법이 없다.

### 원인 분석

| 원인 | 설명 | 현재 대안의 한계 |
|------|------|------------------|
| 측정 방법 부재 | 색 감각을 수치화할 기회가 일상에 없다 | 이시하라 검사는 색맹 여부만 판별, 감각 "등급"은 안 줌 |
| 비교 욕구 | "나 vs 남" 감각 비교를 하고 싶다 | 기존 테스트는 결과를 공유하기 어려운 구조 |
| 짧은 여가 활용 | 2-3분 짜투리 시간에 할 만한 게임 수요 | 기존 컬러 게임은 UX가 투박하거나 광고 범벅 |

### 시장 맥락
- **바이럴 테스트 트렌드**: MBTI, 성격 테스트 등 "나를 수치화"하는 콘텐츠 지속 인기
- **카톡 공유 루프**: "상위 X%" 결과는 카톡 단톡방에서 대결 욕구를 자극
- **디자이너/크리에이터 층**: 색 감각을 직업 역량으로 인식, 자기 검증 수요 높음
- **경쟁사 약점**: 해외 Kuku Kube 등은 한국어 미지원 + 모바일 UX 부실

## 3. 핵심 가치

> **"당신의 눈은 몇 등급인가요?"**

색상의 미묘한 차이를 구별하는 감각을 20라운드에 걸쳐 테스트하고,
퍼센타일 등급으로 결과를 보여주는 **2분짜리 감각 대결 게임**.

핵심 후크: "상위 X%" 결과 → 공유 → 지인 유입 → 대결 → 재도전

## 5. 코어 루프

```
┌─────────┐     ┌──────────────┐     ┌──────────┐     ┌──────────┐
│  시작    │ ──→ │  게임 (20R)  │ ──→ │  결과    │ ──→ │  공유    │
│  (1탭)   │     │  타일 찾기    │     │  등급+%  │     │  카드    │
└─────────┘     └──────────────┘     └──────────┘     └──────────┘
                       │                    │                │
                       │                    │                │
                  게임오버 시               ↓                ↓
                  결과로 직행          재도전            지인 유입
                                    (시작으로)         (시작으로)
```

### 라운드 구조 (20라운드)

| 라운드 | 그리드 | 색 차이 (deltaE) | 난이도 체감 |
|--------|--------|-------------------|-------------|
| 1-4 | 2x2 (4칸) | 30-20 | 누구나 구분 가능 |
| 5-8 | 3x3 (9칸) | 20-12 | 약간 집중 필요 |
| 9-12 | 4x4 (16칸) | 12-7 | 확실히 어려움 |
| 13-16 | 5x5 (25칸) | 7-4 | 상당히 어려움 |
| 17-20 | 6x6 (36칸) | 4-2 | 극한 난이도 |

### 색상 생성 알고리즘

1. 매 라운드 랜덤 베이스 색상 생성 (HSL 기반)
2. H(색조), S(채도), L(명도) 중 하나를 delta만큼 변형하여 다른 타일 색 생성
3. 베이스 색상의 명도 범위: L 30-70 (너무 어둡거나 밝으면 구분 자체가 불가)
4. 라운드가 올라갈수록 delta 값 감소 → 색 차이 줄어듦

### 점수 계산

```
라운드 점수 = 기본점(100) + 속도 보너스(최대 100)
속도 보너스 = max(0, (남은시간 / 제한시간) × 100)
총점 = 각 라운드 점수의 합
최대 가능 점수 = 4000 (20라운드 × 200점)
```

### 퍼센타일 등급 산정

| 등급 | 점수 범위 | 퍼센타일 | 타이틀 |
|------|-----------|----------|--------|
| S+ | 3600+ | 상위 1% | 초인의 눈 |
| S | 3200-3599 | 상위 5% | 프로 디자이너급 |
| A | 2800-3199 | 상위 15% | 예리한 눈 |
| B+ | 2400-2799 | 상위 30% | 꽤 좋은 감각 |
| B | 2000-2399 | 상위 50% | 평균 이상 |
| C | 1500-1999 | 상위 70% | 보통 |
| D | 1000-1499 | 상위 85% | 조금 아쉬운 |
| F | 0-999 | 하위 15% | 색맹은 아니죠...? |

> 주의: 퍼센타일은 고정 테이블 기반 (MVP에서는 실제 유저 분포 미반영). v2에서 실제 데이터 기반으로 동적 조정.

## 6. MVP 기능 (P0)

### F1. 인트로 화면

**목적**: 앱 가치 전달 + 즉시 시작 유도

- 1줄 카피: "당신의 눈은 몇 등급인가요?"
- 부제: "다른 색 타일을 찾아보세요. 20라운드 도전!"
- CTA: "시작하기" 버튼 1개
- 최고 기록이 있으면 하단에 "내 최고 기록: X점 (상위 X%)" 표시
- 회원가입/로그인 없음. localStorage 기반.

### F2. 게임 화면

**목적**: 핵심 게임플레이

- 상단: 현재 라운드 (예: "Round 5/20") + 프로그레스 바
- 중단: NxN 색상 그리드 (화면 너비에 맞춰 정사각형 타일)
- 하단: 타이머 바 (10초, 시각적 감소)
- 탭 정답 시: 타일 확대 피드백(scale 1.1, 150ms) + 다음 라운드로 전환
- 탭 오답 시: 화면 흔들림(shake, 200ms) + 기회 차감 없이 재시도 가능 (단, 시간은 계속 감소)
- 시간 초과 시: 게임 오버 → 결과 화면으로

### F3. 게임 오버 화면

**목적**: 실패 시 즉각 피드백 + 결과 연결

- "시간 초과!" 메시지
- 정답 타일 하이라이트 표시 (어떤 것이 달랐는지 확인)
- 2초 후 자동으로 결과 화면 전환 (또는 "결과 보기" 탭)

### F4. 결과 화면

**목적**: 성취감 + 공유 동기 부여

- 메인 숫자: 총점 (카운트업 애니메이션, 0 → 최종점수, 1.5초)
- 등급 뱃지: S+, S, A, B+, B, C, D, F
- 퍼센타일: "상위 X%" (큰 텍스트)
- 타이틀: "초인의 눈" 등
- 라운드별 성적 요약: 각 라운드 소요 시간 바 차트
- CTA 1 (Primary): "공유하기"
- CTA 2 (Secondary): "다시 도전" (아웃라인 버튼)
- 하단: "기록 보기" 텍스트 링크

### F5. 기록 화면 (History)

**목적**: 성장 확인 + 재도전 유도

- 최고 기록: 점수 + 등급 + 날짜
- 최근 플레이 목록: 점수, 등급, 도달 라운드, 날짜 (최대 20개)
- 통계: 총 플레이 횟수, 평균 점수, 최고 도달 라운드
- CTA: "다시 도전" 버튼
- 뒤로가기: 인트로 화면

### F6. 공유 카드 (ShareCard)

**목적**: 바이럴 확산

- 모달로 표시
- 카드 이미지 (캡처 가능한 div):
  - 앱 이름 "컬러 감각 테스트"
  - 등급 뱃지 (크게)
  - "상위 X%"
  - 점수
  - "당신도 도전해보세요" + 앱 URL
- 공유 버튼:
  - "카카오톡 공유" (Kakao SDK 또는 URL scheme)
  - "이미지 저장" (html2canvas → download)
  - "링크 복사" (clipboard API)
- 텍스트 공유 포맷:
  ```
  [컬러 감각 테스트]
  나의 색 감각: S등급 (상위 5%)
  점수: 3450점
  당신의 눈은 몇 등급? 👉 {url}
  ```

### F7. 최고 기록 저장

**목적**: 재방문 동기

- localStorage에 저장 (서버 없음)
- 저장 데이터: 점수, 등급, 도달 라운드, 플레이 일시
- 최대 20회 기록 보관 (FIFO)

## 9. 유저 스토리 요약 (상세: user-stories.md)

### 첫 방문 (US-1)
- US-1.1: 앱을 열면 3초 안에 뭘 하는 앱인지 이해
- US-1.2: 한 번 탭으로 바로 게임 시작
- US-1.3: 규칙 설명 없이도 첫 라운드에서 직관적으로 이해

### 게임 플레이 (US-2)
- US-2.1: 다른 색 타일을 찾아서 탭하면 다음 라운드 진행
- US-2.2: 오답 시에도 게임이 끝나지 않고 재시도 가능 (시간 패널티만)
- US-2.3: 난이도가 점진적으로 올라가는 걸 체감
- US-2.4: 현재 진행 상황(라운드, 시간)을 항상 확인 가능
- US-2.5: 시간 초과 시 어떤 타일이 정답이었는지 확인

### 결과 확인 (US-3)
- US-3.1: 점수와 등급을 한눈에 확인
- US-3.2: "상위 X%"로 내 위치 파악
- US-3.3: 라운드별 성적 세부 확인

### 공유 (US-4)
- US-4.1: 결과를 카카오톡으로 공유
- US-4.2: 결과 이미지를 저장해서 인스타 등에 업로드
- US-4.3: 공유받은 사람이 링크 탭 → 바로 시작 가능

### 재방문 (US-5)
- US-5.1: 이전 기록과 비교하며 성장 확인
- US-5.2: 최고 기록 갱신 시 뱃지 업데이트

- **아이콘**: Lucide Icons 통일 (R11 준수)
- **이모지**: UI에 이모지 미사용 (R13 준수). 공유 텍스트에만 한정 사용.

### 애니메이션
- 정답 피드백: scale(1.05) → scale(1), 150ms ease-out (R4 준수 — fadeInUp 미사용)
- 오답 피드백: translateX(-4px, 4px, 0), 200ms (shake)
- 화면 전환: opacity fade, 200ms (모든 화면 동일)
- 점수 카운트업: 0 → 최종값, 1.5초 linear
- 타이머 바: width 100% → 0%, linear (R7 준수 — 장식 파티클 없음)

### 버튼 위계 (R16 준수)
- Primary: bg #e63946, text white, rounded-lg, py-3, font-weight 600
- Secondary: border 1px #d1d5db, text #374151, rounded-lg, py-3, font-weight 500
- Tertiary: text-only, #6b7280, underline on hover

## 11. 기술 요구사항

### 프론트엔드
- React 18+ (단일 페이지)
- 상태 관리: useState/useReducer (외부 라이브러리 불필요)
- 스타일: Tailwind CSS
- 공유 카드: html2canvas (이미지 저장용)
- 저장소: localStorage

### 색상 로직
- HSL 색공간 기반 색상 생성
- deltaE 계산으로 난이도 조절 (CIEDE2000 또는 간이 HSL 차이)
- 화면 밝기/기기 차이를 고려한 최소 delta 보정

### 성능
- 초기 로드: 1초 이내 (번들 100KB 이하 목표)
- 라운드 전환: 100ms 이내
- 타이머 정확도: requestAnimationFrame 기반

### 접근성
- 게임 특성상 색각 이상자를 위한 별도 모드는 v1 미포함
- 터치 타겟: 최소 44x44px (타일 크기)
- 시각적 피드백: 색상 외에 크기 변화(scale)로 정답 표시

## 13. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 기기별 색 재현 차이 | OLED vs LCD에서 난이도 편차 | 최소 delta를 보수적으로 설정 (deltaE 2 이상) |
| 화면 밝기 영향 | 저밝기에서 구분 불가 | 게임 시작 시 "화면 밝기를 올려주세요" 안내 (선택적) |
| 첫 라운드 이해 실패 | 규칙을 모르고 이탈 | 첫 라운드(2x2)가 사실상 튜토리얼 역할 — 매우 쉽게 |
| 20라운드 지루함 | 중간에 이탈 | 그리드 크기 변화 + 라운드 간 텀 최소화(즉시 전환) |
| 색각 이상 유저 불만 | 부정적 리뷰 | 결과 화면에 "이 테스트는 의학적 진단이 아닙니다" 고지 |
