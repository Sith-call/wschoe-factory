# 유저 스토리 맵: 컬러 감각 테스트 (Color Sense Test)

> **Version**: 1.0
> **Last Updated**: 2026-03-21
> **참조**: prd.md

---

## 유저 여정 타임라인

```
첫 방문     →     첫 게임       →     결과 확인     →     공유        →     재방문
(발견/이해)       (플레이)             (성취감)           (바이럴)          (기록/재도전)
```

---

## 1. 첫 방문 (발견)

### US-1.1: 앱 가치 즉시 이해

> 사용자로서, 앱을 열었을 때 3초 안에 "다른 색 타일 찾기 게임"이라는 걸 이해하고 싶다.

**수용 기준:**
- 첫 화면에 핵심 메시지 1줄 표시: "당신의 눈은 몇 등급인가요?"
- 부제로 게임 방식 명시: "다른 색 타일을 찾아보세요. 20라운드 도전!"
- 화면 로드 후 500ms 이내 모든 텍스트 표시 완료
- CTA "시작하기" 버튼이 화면 하단에 즉시 표시

**안티패턴:**
- 온보딩 슬라이드 3장 넘기기 (게임은 바로 시작해야 함)
- 튜토리얼 팝업으로 규칙 설명 (첫 라운드가 곧 튜토리얼)
- 추상적 카피: "당신의 감각을 깨워보세요" (구체적이지 않음)

**페르소나 반응:**
- 김하은: "다른 색 타일 찾기? 바로 해보자"
- 이준서: "20라운드면 2분이면 되겠네. 점심시간에 딱"
- 박영미: "아 이거구나. 별도 설명 없이 바로 되네"

---

### US-1.2: 한 번 탭으로 게임 시작

> 사용자로서, 회원가입이나 설정 없이 "시작하기" 한 번으로 바로 게임을 시작하고 싶다.

**수용 기준:**
- "시작하기" 탭 → 200ms 이내 첫 번째 라운드(2x2 그리드) 표시
- 로그인, 닉네임, 약관 동의 등 사전 단계 0개
- 이전에 플레이한 적 있어도 동일하게 즉시 시작

**안티패턴:**
- "닉네임을 입력하세요" (불필요한 마찰)
- "난이도를 선택하세요" (MVP는 단일 모드)
- 카운트다운 "3, 2, 1..." 연출 (불필요한 대기)

**페르소나 반응:**
- 김하은: "가입 안 해도 되는 거 최고"
- 이준서: "이런 가벼운 게임에 가입은 말도 안 돼"
- 박영미: "버튼 하나면 시작이라 좋네"

---

### US-1.3: 첫 라운드에서 규칙을 직관적으로 이해

> 사용자로서, 별도 설명 없이 첫 라운드를 보자마자 "다른 색을 찾으면 되는구나"를 알고 싶다.

**수용 기준:**
- 첫 라운드(2x2)에서 색 차이가 매우 커서 (deltaE 30) 누구나 즉시 구분
- 화면 상단에 "다른 색 타일을 찾으세요" 텍스트 1줄 표시
- 정답 타일 탭 시 즉각적 시각 피드백 (scale 확대 + 색상 하이라이트)
- 첫 라운드 클리어 후 자연스럽게 2라운드 시작 — 별도 "다음" 버튼 없음

**안티패턴:**
- 반투명 오버레이로 UI 요소 하나하나 설명 (코치마크 과잉)
- 텍스트 설명 2줄 이상 (1줄이면 충분)
- 첫 라운드에 타이머 표시 (부담감 유발) — 단, 내부적으로는 타이머 작동

**페르소나 반응:**
- 김하은: "아 그냥 이거 누르면 되는 거지? 쉽네"
- 이준서: "직관적이다. 설명 필요 없어"
- 박영미: "이런 거 좋아. 복잡하면 바로 닫는데"

---

## 2. 게임 플레이

### US-2.1: 다른 색 타일 찾기

> 사용자로서, 그리드에서 색이 다른 타일 1개를 찾아 탭하면 다음 라운드로 넘어가고 싶다.

**수용 기준:**
- NxN 그리드에서 1개 타일만 다른 색 (나머지는 동일 색)
- 정답 타일 탭 → 150ms scale 피드백 → 100ms 후 다음 라운드 로드
- 라운드 전환 시 그리드 크기, 색상 모두 새로 생성
- 타일 터치 영역은 시각적 크기와 동일 (패딩 없는 정확한 히트 영역)

**안티패턴:**
- 정답 후 "정답입니다!" 모달 (흐름 끊김)
- 라운드 전환에 2초 이상 걸리는 애니메이션 (템포 저하)
- 다음 라운드 버튼을 별도로 누르게 하기 (자동 전환이 맞음)

**페르소나 반응:**
- 김하은: "탭하면 바로 넘어가니까 속도감 있어서 좋아"
- 이준서: "빠른 전환이 핵심이지. 끊기면 안 돼"
- 박영미: "다음 거 바로 나오니까 계속 하게 되네"

---

### US-2.2: 오답 시 재시도

> 사용자로서, 틀린 타일을 탭해도 게임이 끝나지 않고 다시 찾을 기회를 갖고 싶다.

**수용 기준:**
- 오답 타일 탭 → shake 애니메이션(200ms) 피드백
- 게임이 끝나지 않음 — 타이머가 계속 흐르는 것만이 패널티
- 오답 횟수는 내부 기록 (결과 화면에서 참고용으로 표시 가능)
- 오답 타일은 시각적으로 비활성화되지 않음 (다시 탭 가능 — 단, 같은 결과)

**안티패턴:**
- 오답 즉시 게임 오버 (너무 가혹 — 캐주얼 유저 이탈)
- 오답 시 "틀렸습니다" 팝업 (흐름 끊김)
- 오답 시 점수 차감 (시간 소모 자체가 충분한 패널티)

**페르소나 반응:**
- 김하은: "틀려도 계속 할 수 있어서 덜 스트레스"
- 이준서: "속도 보너스가 깎이니까 결국 잘 찾아야 하는 구조네"
- 박영미: "한 번 틀려서 바로 끝나면 화날 텐데, 이건 괜찮네"

---

### US-2.3: 점진적 난이도 증가 체감

> 사용자로서, 라운드가 올라갈수록 "확실히 어려워진다"는 걸 느끼고 싶다.

**수용 기준:**
- 그리드 크기 변화가 시각적으로 명확 (2x2 → 3x3 → 4x4 → 5x5 → 6x6)
- 그리드 크기 변화 시점에 라운드 번호가 강조 (예: "Round 5" 약간 크게 표시)
- 후반부(17-20라운드)에서 색 차이가 "진짜 어렵다"는 반응이 나와야 함
- 각 단계 첫 라운드에서 "새 그리드 사이즈"를 직관적으로 인지

**안티패턴:**
- 난이도가 리니어하게만 증가 (그리드 크기 변화 없이 색만 비슷해짐 → 단조로움)
- 1라운드부터 어려움 (진입 장벽)
- 20라운드가 물리적으로 불가능한 난이도 (deltaE 0.5 등 → 기기 차이로 보이지 않음)

**페르소나 반응:**
- 김하은: "오 5x5부터 진짜 어렵다. 집중해야 돼"
- 이준서: "마지막 4라운드 못 깬 사람 많겠다. 이게 진짜 실력 차이"
- 박영미: "처음엔 쉬워서 할 만했는데 뒤로 갈수록 헉..."

---

### US-2.4: 진행 상황 실시간 확인

> 사용자로서, 현재 몇 라운드인지와 남은 시간을 항상 확인하고 싶다.

**수용 기준:**
- 화면 상단: "Round 5/20" 텍스트 + 프로그레스 바 (filled portion)
- 화면 하단(또는 그리드 아래): 타이머 바 (10초, 색상으로 긴급도 표현)
  - 10-4초: 기본 색 (그레이 또는 블루)
  - 3-1초: 경고 색 (#ffd166 옐로우)
  - 1초 이하: 위험 색 (#e63946 레드)
- 프로그레스 바와 타이머 바가 동시에 보여 혼동 없을 것

**안티패턴:**
- 숫자 타이머 ("7.3초") — 숫자가 빠르게 바뀌면 시선을 빼앗김 → 바 형태가 적합
- 타이머를 화면 상단에 배치 (그리드에서 시선 이동 거리 김)
- 프로그레스 바 없이 라운드 번호만 표시 (전체 중 어디인지 감 안 잡힘)

**페르소나 반응:**
- 김하은: "타이머 바가 줄어드니까 긴장감 있어"
- 이준서: "라운드 프로그레스 있어서 '이제 반 왔다' 알 수 있네"
- 박영미: "빨간색 되면 서둘러야 하는 거지? 직관적이다"

---

### US-2.5: 시간 초과 시 정답 확인

> 사용자로서, 시간이 다 돼서 못 찾았을 때 어떤 타일이 정답이었는지 확인하고 싶다.

**수용 기준:**
- 시간 초과 시 정답 타일이 깜빡임(blink) 또는 테두리 하이라이트로 표시
- 정답 표시 1.5초 → 게임 오버 화면으로 전환
- "라운드 X에서 탈락" 메시지 표시

**안티패턴:**
- 정답 표시 없이 바로 결과 화면 (궁금증 해소 안 됨 → 불만)
- 정답 표시가 5초 이상 (너무 길면 지루)
- "아깝습니다!" 같은 감정 조작 텍스트 (심플하게)

**페르소나 반응:**
- 김하은: "아 저거였어?! 다시 하면 맞출 수 있을 텐데"
- 이준서: "정답 보여주는 거 좋다. 확인해야 다음에 나아지지"
- 박영미: "아 진짜 저게 달랐던 거야? 안 보이는데?"

## 3. 결과 확인

### US-3.1: 점수와 등급 한눈에 확인

> 사용자로서, 게임이 끝나면 내 점수와 등급을 직관적으로 확인하고 싶다.

**수용 기준:**
- 메인 영역에 점수(카운트업 애니메이션, 0→최종값, 1.5초) + 등급 뱃지
- 등급별 색상 구분:
  - S+/S: #e63946 (레드 계열, 프리미엄)
  - A: #2563eb (블루)
  - B+/B: #06d6a0 (그린)
  - C/D: #6b7280 (그레이)
  - F: #9ca3af (연한 그레이)
- 등급 옆에 타이틀 텍스트 ("초인의 눈", "프로 디자이너급" 등)
- 카운트업 완료 후 등급 뱃지가 scale(0.8→1) 등장

**안티패턴:**
- 점수/등급/퍼센타일/타이틀이 동시에 다 보임 (정보 과부하)
- 등급 색상이 앱 primary 색과 동일 (구분 안 됨)
- 모든 등급에 같은 시각적 무게 (S+가 특별해 보여야 함)

**페르소나 반응:**
- 김하은: "S등급이다! 카운트업 올라가는 거 두근두근했어"
- 이준서: "A등급... 한 번 더 하면 S 갈 수 있을 것 같은데"
- 박영미: "B등급이면 나쁘진 않은 거지? 상위 50%라고?"

---

### US-3.2: 퍼센타일로 내 위치 파악

> 사용자로서, "상위 몇 %인지"를 보고 내 감각 수준을 객관적으로 파악하고 싶다.

**수용 기준:**
- "상위 X%" 텍스트가 등급 아래에 큰 폰트로 표시
- 퍼센타일 숫자가 5% 이하일 경우 특별 강조 (색상 또는 크기)
- 하단에 "이 테스트는 의학적 진단이 아닙니다" 고지 (작은 텍스트)

**안티패턴:**
- 퍼센타일을 소수점까지 표시 ("상위 4.7%") — 고정 테이블인데 정확한 척 (v1)
- "하위 X%"로 표현 (부정적 프레이밍 → 공유 의욕 저하)
- 퍼센타일 없이 등급만 표시 (수치가 있어야 비교 가능)

**페르소나 반응:**
- 김하은: "상위 5%! 이거 자랑해야 해"
- 이준서: "상위 15%... 동료 준혁이 상위 5%면 지는 거잖아. 다시 해야지"
- 박영미: "상위 50%면 반은 되는 거니까 나쁘진 않네"

---

### US-3.3: 라운드별 세부 성적 확인

> 사용자로서, 어떤 라운드에서 빨리 찾았고 어디서 오래 걸렸는지 확인하고 싶다.

**수용 기준:**
- 결과 화면 하단 스크롤 영역에 라운드별 소요 시간 바 차트
- 빠른 라운드(3초 이하)는 짧은 바 + 민트색
- 느린 라운드(7초 이상)는 긴 바 + 옐로우/레드
- 게임 오버된 라운드는 X 표시
- 각 라운드 옆에 그리드 크기 표시 (예: "4x4")

**안티패턴:**
- 20개 라운드를 테이블로 나열 (시각적이지 않음)
- 차트가 화면의 50% 이상 차지 (결과 메인 영역을 밀어냄)
- 라운드별 정보가 접혀 있어서 탭해야 보임 (한눈에 파악할 수 있어야)

**페르소나 반응:**
- 김하은: "13라운드부터 시간 많이 걸렸네. 5x5가 약점인가"
- 이준서: "패턴이 보이네. 녹색 계열에서 느린 것 같아"
- 박영미: "그래프로 보니까 재밌다. 어디서 막혔는지 알겠어"

## 4. 공유

### US-4.1: 카카오톡으로 결과 공유

> 사용자로서, 결과를 카카오톡 친구나 단톡방에 바로 공유하고 싶다.

**수용 기준:**
- "공유하기" 탭 → 공유 카드 모달 오픈
- "카카오톡 공유" 버튼 탭 → 카카오톡 앱으로 이동 (URL scheme 또는 Kakao SDK)
- 공유 시 메시지에 등급, 점수, 앱 URL 포함
- 카카오톡 미설치 시 "링크 복사"로 자동 폴백

**안티패턴:**
- Web Share API만 사용 (한국에서 카카오톡 직접 공유가 핵심)
- 공유 전 "닉네임을 입력하세요" 요구 (마찰)
- 공유 성공 후 앱으로 돌아오지 않음 (딥링크 또는 웹뷰 내 처리)

**페르소나 반응:**
- 김하은: "카톡 바로 되면 과 단톡에 보내야지"
- 이준서: "슬랙에 링크 복사해서 올려야겠다"
- 박영미: "카톡으로 보내기 편하네. 직장 단톡에 올려야지"

---

### US-4.2: 결과 이미지 저장

> 사용자로서, 결과 카드를 이미지로 저장해서 인스타그램 스토리 등에 올리고 싶다.

**수용 기준:**
- 공유 카드 모달에서 "이미지 저장" 버튼 탭 → 카드 영역을 PNG로 저장
- 카드 내용: 앱 이름 + 등급 뱃지 + "상위 X%" + 점수 + "도전해보세요" + URL
- 카드 비율 1:1 또는 4:5 (인스타 스토리 호환)
- 저장 완료 시 "저장되었습니다" 토스트 메시지

**안티패턴:**
- 카드에 유저 개인정보 포함 (닉네임도 없으므로 해당 없음)
- 카드 해상도가 낮아서 흐릿함 (2x 해상도 렌더링)
- 저장 후 갤러리로 자동 이동 (앱에 남아 있어야 함)

**페르소나 반응:**
- 김하은: "인스타 스토리에 올려야지. 이미지 저장 가능하면 딱"
- 이준서: "카드 디자인이 깔끔하면 공유 의향 높아짐"
- 박영미: "이미지로 저장해서 카톡 프사에 올려볼까 ㅎㅎ"

---

### US-4.3: 공유 링크로 유입된 유저의 즉시 시작

> 사용자로서, 친구가 공유한 링크를 탭하면 바로 앱이 열리고 게임을 시작할 수 있어야 한다.

**수용 기준:**
- 공유 URL 클릭 → 앱 인트로 화면 직행 (중간 페이지 없음)
- 공유한 사람의 점수가 인트로 화면에 "친구의 기록: X점 (상위 X%)" 로 표시 (쿼리 파라미터 기반)
- "도전하기" CTA 버튼 강조

**안티패턴:**
- 공유 링크가 홈페이지로 가서 앱을 다시 찾아야 함
- "앱 설치" 요구 (웹 앱이므로 불필요)
- 공유한 사람의 정보를 과도하게 노출

**페르소나 반응:**
- 김하은: "링크 눌렀는데 바로 시작할 수 있어서 편해"
- 이준서: "친구 점수 보이니까 이기고 싶은 마음이 드네"
- 박영미: "복잡하지 않고 바로 되니까 해볼 만하네"

---

### US-4.4: 텍스트 형태로 결과 공유

> 사용자로서, 이미지가 아닌 텍스트로도 결과를 간편하게 공유하고 싶다.

**수용 기준:**
- "링크 복사" 버튼 → 클립보드에 텍스트 결과 + URL 복사
- 텍스트 포맷:
  ```
  [컬러 감각 테스트]
  나의 색 감각: S등급 (상위 5%)
  점수: 3450점
  당신의 눈은 몇 등급? {url}
  ```
- 복사 완료 시 "복사되었습니다" 토스트

**안티패턴:**
- URL만 복사 (맥락 없는 링크는 클릭률 낮음)
- 텍스트가 3줄 이상 (너무 긴 메시지는 공유 안 함)
- 복사 성공 피드백 없음 (제대로 됐는지 불안)

**페르소나 반응:**
- 김하은: "트위터에 올리려면 텍스트 복사가 편하지"
- 이준서: "슬랙에는 텍스트가 깔끔해"
- 박영미: "카톡에 붙여넣기 하면 되겠다"

## 5. 재방문

### US-5.1: 최고 기록 확인

> 사용자로서, 앱을 다시 열었을 때 이전 최고 기록을 바로 확인하고 싶다.

**수용 기준:**
- 인트로 화면에 "내 최고 기록: X점 (S등급, 상위 5%)" 표시
- 최고 기록이 없으면 해당 영역 미표시 (빈 상태 텍스트 없음)
- 기록 탭 → 히스토리 화면으로 이동 가능

**안티패턴:**
- "로그인하면 기록을 저장할 수 있습니다" 유도 (localStorage면 충분)
- 최고 기록을 팝업으로 표시 (인트로 화면에 자연스럽게 녹여야 함)
- "지난번보다 X점 올랐어요!" 같은 과도한 격려 (심플하게)

**페르소나 반응:**
- 김하은: "최고 기록 바로 보이니까 '이거 깨야지' 하는 마음이 생겨"
- 이준서: "전 기록 보면 다시 도전하게 되지"
- 박영미: "아 저번에 이만큼 했었구나. 이번엔 더 잘할 수 있을까"

---

### US-5.2: 플레이 기록 목록 확인

> 사용자로서, 과거 플레이 기록을 시간순으로 보고 성장을 확인하고 싶다.

**수용 기준:**
- 히스토리 화면에 최근 20회 플레이 기록 목록
- 각 항목: 점수, 등급 뱃지, 도달 라운드, 날짜
- 최고 기록 항목에 "BEST" 라벨 표시
- 통계 영역: 총 플레이 횟수, 평균 점수, 최고 도달 라운드

**안티패턴:**
- 기록이 없을 때 빈 화면 (안내 메시지 "아직 기록이 없어요. 첫 도전을 시작해보세요!")
- 20개 넘는 기록을 무한 스크롤 (20개면 충분)
- 기록 삭제 기능 (필요 없음. 초기화 원하면 브라우저 데이터 삭제)

**페르소나 반응:**
- 김하은: "점수가 올라가는 게 보이니까 뿌듯해"
- 이준서: "평균 점수가 올라가고 있으면 실력이 는 거지"
- 박영미: "3번 했는데 점수가 비슷하네. 이게 내 한계인가"

---

### US-5.3: 최고 기록 갱신 시 특별 피드백

> 사용자로서, 이전 최고 기록을 넘었을 때 그 사실을 명확하게 알고 싶다.

**수용 기준:**
- 결과 화면에서 최고 기록 갱신 시 "NEW BEST!" 뱃지 표시
- 이전 최고 기록과의 차이 표시 ("이전 기록보다 +320점")
- 뱃지 등장 시 scale 애니메이션 (0.8→1, 300ms spring)

**안티패턴:**
- 컨페티/폭죽 이펙트 (R7 장식 파티클 금지)
- "축하합니다!!!" 모달 팝업 (과잉 반응)
- 기록 갱신 여부를 결과 화면 하단 작은 텍스트로만 표시 (눈에 안 띔)

**페르소나 반응:**
- 김하은: "NEW BEST! 나왔다! 이번엔 진짜 잘했나 보다"
- 이준서: "+320점이면 꽤 올랐네. 이 정도면 공유해야지"
- 박영미: "오 최고 기록! 다음엔 더 할 수 있을까"

---

### US-5.4: 인트로에서 기록 화면 접근

> 사용자로서, 인트로 화면에서 바로 내 기록을 확인할 수 있어야 한다.

**수용 기준:**
- 인트로 화면 하단에 "내 기록 보기" 텍스트 링크 (기록이 있을 때만 표시)
- 탭 → 히스토리 화면으로 전환
- 히스토리에서 "뒤로" → 인트로로 복귀

**안티패턴:**
- 기록 접근을 위해 설정 메뉴를 거쳐야 함 (직접 접근 가능해야)
- "기록" 아이콘만 표시 (텍스트 레이블 병행 필요)
- 기록 화면이 인트로 위에 모달로 뜸 (독립 화면이 적합)

**페르소나 반응:**
- 김하은: "기록 보기가 바로 있어서 편하다"
- 이준서: "지난 기록 확인하고 도전하는 루틴"
- 박영미: "아 여기서 내 기록 볼 수 있구나"

---

### US-5.5: 재도전 시 빠른 루프

> 사용자로서, 결과를 본 후 "다시 도전"을 탭하면 바로 새 게임이 시작되어야 한다.

**수용 기준:**
- 결과 화면 "다시 도전" 버튼 → 200ms 이내 라운드 1 시작
- 인트로 화면을 거치지 않고 바로 게임 화면
- 새 게임은 완전히 새로운 색상 세트로 시작

**안티패턴:**
- 인트로 화면으로 돌아가서 다시 "시작하기" 눌러야 함 (한 단계 불필요)
- "정말 다시 시작하시겠습니까?" 확인 팝업 (게임인데 확인 불필요)
- 이전 게임 데이터가 남아있어 새 게임에 영향 (완전 초기화)

**페르소나 반응:**
- 김하은: "바로 다시 시작! 이번엔 S+ 간다"
- 이준서: "빠른 재시작이 게임에서 제일 중요하지"
- 박영미: "한 번 더 해볼까... 버튼 누르면 바로 되네"
