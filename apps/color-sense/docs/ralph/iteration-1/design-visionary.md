# Design Visionary Critique — Color Sense Test

> "Good design is as little design as possible." — Dieter Rams
>
> 이 앱은 그 원칙에 상당히 가깝다. 하지만 "가깝다"는 것이 "도달했다"는 것은 아니다.

## 화면별 크리틱

### 1. 인트로 화면

**잘한 점:**
- 무채색 배경(#fafafa) 위에 violet 액센트 하나만 사용. UI 크롬이 조용하다.
- Eye 아이콘이 stroke-only SVG로 깔끔하다. 이모지 남용 없음.
- "시작하기" 버튼과 "기록 보기" 텍스트 링크의 위계가 명확하다 (Primary vs Tertiary).
- 최고 기록 영역이 border-top 구분선으로 분리 -- 카드 네스팅 없이 정보를 구분한다.

**문제점:**
- 화면 중간이 완전히 비어 있다. Eye 아이콘 + 제목 + 설명이 상단에 몰려 있고, 버튼이 하단에 있어서 시선이 분산된다. Dieter Rams의 "모든 디테일이 목적을 가져야 한다"는 원칙에서 보면, 이 빈 공간은 의도적 화이트스페이스라기보다 레이아웃의 부재에 가깝다.
- 최고 기록이 없는 첫 방문자에게는 Eye 아이콘 + 두 줄 텍스트 + 버튼뿐이다. 테스트의 기대감이나 동기 부여가 전혀 없다. 간단한 타일 그리드 프리뷰 정도는 있어야 "아, 이런 게임이구나"를 직관적으로 전달할 수 있다.
- best score 표시 시, pt-12 상단 여백이 과하다. 아이콘부터 점수까지의 콘텐츠 블록을 수직 중앙에 더 가깝게 놓는 것이 시각적 무게 중심을 잡는다.

**점수: 6/10**

### 2. 게임 화면

**잘한 점:**
- 게임 배경(#1e1b2e)이 타일 색상의 대비를 극대화한다. 핵심 기능에 복무하는 디자인 결정이다.
- HUD(레벨, 타이머, 프로그레스 바)가 조용하다. text-zinc-400, text-sm으로 주의를 뺏지 않는다.
- 타일에 border 없음, gap 4px. 색각 검사의 원리를 잘 따른다.
- 타이머가 remaining time에 따라 color가 바뀐다(gray -> amber -> red). 기능적 색상 변화다.
- 하단 타이머 바의 색상 변화도 동일 시스템. 일관성 있다.

**문제점:**
- 프로그레스 바가 두 개다. 상단에 레벨 진행 프로그레스 바, 하단에 타이머 바. 정보 중복은 아니지만, 시각적으로 비슷한 요소가 화면 위아래에 있어 산만하다. 상단 프로그레스 바의 존재감이 약하고, 바로 아래 "1 / 20" 텍스트와 역할이 겹친다.
- 타일 그리드가 화면 세로 중앙에 위치하지만, 2x2 그리드일 때 타일이 너무 크다. 각 타일이 약 170px 정도로, 색상 면적이 과도하게 넓어 오히려 색 차이를 쉽게 인지하게 만든다. Lv.1의 의도된 난이도인지 의문.
- "다른 색 타일을 찾으세요" 안내 문구가 Lv.1에서만 보인다. 좋은 판단이지만, 첫 라운드에서 문구가 타일 그리드와 너무 멀리 떨어져 있다 (mb-4 후 flex-1 영역).
- 게임 화면 진입 시 screen-fade-in 애니메이션(150ms opacity)이 있는데, 빠른 라운드 전환에서 이게 인지되지 않는다면 불필요한 코드다.

**점수: 7.5/10**

### 3. 게임 오버 화면

**잘한 점:**
- 극도로 심플하다. 제목 + 라운드 정보 + 결과 보기 버튼. 불필요한 장식 없음.
- 2초 후 자동 전환. 유저를 붙잡지 않는다.

**문제점:**
- 너무 심플해서 "화면"으로서의 존재 이유가 약하다. 2초간 "시간 초과!" 또는 "틀렸습니다!"만 보여주고 넘어가는데, 이 화면이 없이 바로 결과로 가는 것과 체감 차이가 거의 없다. 게임 화면에서 타일에 오답 피드백(shake + red ring)을 주고 약간의 딜레이 후 결과로 전환하는 것이 더 자연스러운 플로우다.
- GameOverScreen과 ResultScreen의 전환이 두 단계로 분리되어 있어, "틀렸습니다!" -> (2초) -> 결과 화면으로 이동할 때 화면이 두 번 깜빡이는 느낌을 줄 수 있다.
- 전체 게임을 클리어(20라운드)한 경우에도 이 화면이 아닌 바로 result로 가는데, timeout일 때만 이 화면을 거친다. 플로우의 비대칭이 사용자를 혼란스럽게 할 수 있다.

**점수: 5/10**

### 4. 결과 화면

**잘한 점:**
- 점수 카운터 애니메이션(0 -> 최종, 800ms, ease-out). 의미 있는 애니메이션이다.
- 통계 영역이 좌측 정렬 + border-bottom 구분선. 카드 네스팅 없이 깔끔하다.
- 버튼 위계가 명확하다: Primary(다시 도전) > Secondary(결과 공유) > Tertiary(기록 보기).
- 하단에 "이 테스트는 의학적 진단이 아닙니다" 면책 조항. 책임감 있는 디자인.
- NEW BEST 배너가 primary-light(#ede9fe) 배경으로 톤인톤 처리. 과하지 않다.

**문제점:**
- 점수 카드(bg-white, border, rounded-xl) 안에 점수 + 등급 뱃지 + 백분위가 들어가고, 카드 밖에 통계가 있다. 시각적 위계는 맞지만, 카드 하단 모서리와 통계 영역 사이의 간격(mt-6)이 살짝 어색하다. 카드가 "여기까지가 메인 결과"라고 선언하지만, 통계도 중요한 정보인데 시각적으로 격하된 느낌.
- 등급 뱃지(S+, A 등)의 rounded-lg가 design-system.md의 "점수 뱃지 → rounded-none (sharp)" 규칙과 다르다. 코드에서는 rounded-lg를 사용 중. 디자인 시스템과 구현이 불일치한다.
- badge-enter 애니메이션(scale 0.8->1, 200ms)이 badge-pop이라는 이름으로 구현되어 있는데, design-system.md에서 "게임 타일 등장: 없음"이라고 했지만 뱃지의 스케일 애니메이션은 명시적으로 허용하지도 금지하지도 않았다. 회색 지대.
- 결과 화면 상단에 "당신의 색 감각"이라는 라벨이 text-tertiary(#a1a1aa)로 있는데, 시각적 존재감이 너무 약해서 거의 보이지 않는다. 있을 거면 제대로 보여주고, 아니면 빼는 것이 Rams적이다.

**점수: 7/10**

### 5. 히스토리 화면

**잘한 점:**
- 좌측 정렬 리스트. R14 규칙을 잘 따른다.
- 뒤로가기 버튼의 터치 타겟이 44x44px. 접근성 기준 충족.
- 최고 기록 카드 + 통계 3열 + 최근 기록 리스트의 정보 위계가 명확하다.
- 날짜, 점수, 레벨 정보가 font-tabular로 정렬된다.

**문제점:**
- 최고 기록 영역이 bg-white + border + rounded-xl 카드 안에 있다. 그 아래 통계 영역은 카드 없이 노출. 다시 그 아래 최근 기록 리스트도 카드 없이 노출. 카드가 하나만 있으니 R15(네스팅 금지)는 통과하지만, 정보 그룹 간의 시각적 구분이 일관되지 않다.
- 통계 3열(총 플레이, 평균 점수, 최고 라운드)이 grid-cols-3으로 균등 분배되는데, "최고 라운드"라는 라벨이 길어서 줄 바꿈이 발생할 수 있다. 특히 큰 폰트 설정 사용자에게.
- 최근 기록의 maxHeight: 320px + overflow-y: auto가 하드코딩되어 있다. 화면 높이가 812px이 아닌 기기에서 잘리거나 남는 공간이 생긴다. flex-1로 남은 공간을 채우는 것이 더 적절하다.
- 0점 기록이 "F" 등급 뱃지와 함께 나열되는데, 0점/R0인 기록이 여러 개 쌓이면 히스토리가 의미 없는 데이터로 가득 찬다. 0점 기록을 필터링하거나 표시 방식을 달리하는 것이 사용자 경험에 도움이 된다.

**점수: 6.5/10**

### 6. 공유 카드 (Share Modal)

**잘한 점:**
- bottom sheet 패턴. slide-up 300ms 애니메이션. 모바일에 자연스럽다.
- 공유 카드 프리뷰가 game-bg(#1e1b2e)를 사용해 앱 아이덴티티를 유지한다.
- 오버레이 클릭으로 닫기 가능. 모달의 기본 UX 충족.

**문제점:**
- 공유 카드의 aspect-ratio: 4/5가 인라인 스타일로 하드코딩. 카드 내부의 텍스트 배치가 고정 비율에 의존하므로 콘텐츠 양에 따라 깨질 수 있다.
- "이미지 저장"과 "링크 복사" 버튼이 동일 스타일(bg-[#f4f4f5] text-[#18181b] rounded-lg). 두 버튼의 위계 차이가 없다. 하나가 주요 액션이라면 시각적 차이를 줘야 한다.
- 이미지 저장 로직이 Canvas API 직접 드로잉이라 실제 카드와 렌더링이 다를 수 있다. 디자인 이슈라기보다 구현 이슈이지만, 사용자가 보는 프리뷰와 저장되는 이미지가 다르면 신뢰를 잃는다.
- 오버레이가 bg-black/50으로, backdrop-blur는 사용하지 않았다. R6 충족. 좋다.

**점수: 6/10**

## 개선 제안 (Top 5)

### 1. 인트로 화면에 게임 프리뷰 추가 (우선도: 높음)
현재 인트로는 아이콘 + 텍스트 + 버튼뿐이다. 첫 방문자가 "이 앱이 뭘 하는 건지" 직관적으로 알 수 없다. 정적인 2x2 또는 3x3 타일 그리드 미리보기를 인트로 중앙에 배치하면 게임의 본질을 즉시 전달할 수 있다. 이것은 장식이 아니라 커뮤니케이션이다.

### 2. 게임 오버 화면 제거 또는 통합 (우선도: 높음)
GameOverScreen은 2초간 "틀렸습니다!" 텍스트를 보여주고 결과로 넘긴다. 이 화면의 존재 이유가 약하다. 게임 화면 내에서 오답/타임아웃 피드백을 강화하고(타일 전체에 붉은 오버레이 + 정답 타일 하이라이트), 1초 딜레이 후 바로 결과 화면으로 전환하는 것이 플로우를 간결하게 만든다. Dieter Rams: "Less, but better."

### 3. 등급 뱃지 radius를 design-system.md에 맞게 sharp(0px)로 통일 (우선도: 중간)
design-system.md에서 "점수 뱃지 -> rounded-none"이라 명시했으나, ResultScreen과 HistoryScreen 모두 rounded-lg를 사용하고 있다. Sharp corner 뱃지는 rounded-xl 카드와의 대비로 리듬감을 만들어준다. 시스템이 의도한 대로 구현해야 한다.

### 4. 히스토리 리스트의 0점 기록 처리 (우선도: 중간)
타임아웃으로 인한 0점/R0 기록이 반복되면 히스토리가 의미 없는 데이터로 오염된다. 0점 기록은 별도로 묶거나("시간 초과 N회"), 최소 점수 이상만 표시하거나, 시각적으로 dimmed 처리하는 것이 좋다.

### 5. 공유 모달의 버튼 위계 구분 (우선도: 낮음)
"이미지 저장"과 "링크 복사" 버튼이 동일 스타일이다. 이미지 저장이 주요 공유 액션이라면 Primary 스타일을, 링크 복사에는 Secondary 스타일을 적용해야 한다. 현재 두 버튼이 모두 bg-gray로 동일하면, 사용자는 어느 것이 주요 액션인지 판단할 수 없다.
