## 4. Critical Issues

### BLOCKER: 오답 시 즉시 게임 오버 (US-2.2)

PRD 명세: "오답 시 게임이 끝나지 않고 재시도 가능 (시간 패널티만)"
현재 구현: 오답 시 phase='wrong' → 게임 종료 → GameOverScreen

이것은 코어 게임플레이를 근본적으로 바꾸는 이슈:
- 캐주얼 유저가 한 번 실수로 즉시 탈락 → 이탈률 급증 예상
- PRD 페르소나 박영미(45세): "한 번 틀려서 바로 끝나면 화날 텐데"
- 게임 완료율 KPI 70% 달성 불가능

### HIGH: 라운드별 성적 바 차트 없음 (US-3.3)

결과 화면에서 라운드별 소요 시간 바 차트가 없어 어디서 잘했고 못했는지 확인 불가.
요약 통계(클리어 레벨, 평균 반응, 총 라운드)만 있어 깊이 부족.

### HIGH: 카카오톡 공유 없음 (US-4.1)

한국 시장에서 바이럴의 핵심 채널인 카카오톡 공유가 없음.
"링크 복사"만으로는 공유율 KPI 25% 달성 어려움.

## 6. Go/No-Go Decision

### ❌ No-Go for Ralph Loop

**이유:**

1. **BLOCKER 이슈 1건**: 오답 즉시 게임 오버 — 코어 게임플레이가 PRD와 정반대로 구현됨. 페르소나 테스트 전에 반드시 수정 필요.
2. **P0 구현율 45%**: 20개 P0 유저 스토리 중 9개만 완전 구현. 최소 70% 이상이어야 Ralph Loop 진입 가능.
3. **공유 기능 부재**: 4개 공유 스토리 중 완전 구현 0건. 바이럴 루프가 앱의 핵심 성장 동력인데 테스트 불가.

### Ralph Loop 진입 전 필수 수정 사항

1. **오답 시 재시도 로직 구현** (US-2.2) — shake 피드백 + 시간만 감소 + 게임 계속
2. **라운드별 바 차트 추가** (US-3.3) — 결과 화면에 라운드별 소요 시간 시각화
3. **카카오톡 공유 추가** (US-4.1) — Kakao SDK 또는 URL scheme
4. **공유 텍스트에 앱 URL 포함** (US-4.4)

### 선택 수정 (Ralph Loop 중 반복 개선 가능)

- 인트로 화면 카피 PRD 일치
- 게임 화면 라이트 모드 전환
- 결과 화면 CTA 버튼 순서 변경 (공유 → Primary)
- 공유 링크 유입 시 친구 기록 표시 (US-4.3)
