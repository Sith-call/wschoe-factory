## 1. 첫 방문

### US-01: 앱 첫 진입
> **"사용자로서, 앱을 열면 로그인 없이 바로 콘텐츠를 볼 수 있어야 한다. 그래야 첫 방문 이탈이 줄어든다."**

**수용 기준:**
- 앱 로드 후 0.5초 내에 홈 화면 렌더링 완료
- 로그인/회원가입 화면 없음
- 홈에서 용어 카드 목록 또는 추천 모델이 즉시 보임
- P0 모델 4개 + 용어 20개가 내장되어 있어 오프라인에서도 동작

**안티패턴:**
- 온보딩 슬라이드 3장 이상 강제 노출
- "시작하기" 버튼 누르기 전까지 콘텐츠가 안 보이는 구조
- 스플래시 화면에서 2초 이상 대기

**페르소나 반응:**
- 민지: "와, 바로 수요-공급이 보인다. 이거 만져볼 수 있는 거야?"
- 정훈: "가입 없이 바로 되네. 점심시간에 써봐야지."
- 수진: "공부 앱인데 광고가 없네. 일단 좋아."

---

### US-02: 홈 화면 탐색
> **"사용자로서, 홈 화면에서 무엇을 할 수 있는지 한눈에 파악하고 싶다."**

**수용 기준:**
- 홈에 3가지 진입점 명확 표시: 용어 카드, 실험실, 내 진도
- 추천/인기 모델 1-2개가 홈에서 바로 접근 가능
- 카테고리 분류가 보임 (미시, 거시, 국제, 정책)
- 네비게이션이 직관적 (하단 탭 또는 명확한 메뉴)

**안티패턴:**
- 홈이 대시보드처럼 숫자/차트로 가득 참 (첫 방문자에겐 의미 없음)
- 모든 기능을 햄버거 메뉴 안에 숨김
- 카테고리 없이 용어 50개를 한 화면에 나열

**페르소나 반응:**
- 민지: "미시경제 카테고리 들어가면 수요-공급 바로 있겠지?"
- 정훈: "'인기 모델: GDP 구성요소' — 이거 먼저 해볼까."
- 수진: "카테고리별로 정리되어 있으니 시험 범위별로 공부할 수 있겠다."

---

### US-03: 첫 용어 카드 발견
> **"사용자로서, 관심 있는 경제 용어를 찾아서 카드를 열어보고 싶다."**

**수용 기준:**
- 용어 목록에서 카드 탭 시 상세 화면으로 전환
- 카드에 한 줄 정의 + 쉬운 설명 + 시각 자료 포함
- "실험실로 가기" 버튼이 카드 하단에 명확히 표시
- 관련 용어 링크가 있어 탐색 가능

**안티패턴:**
- 카드 내용이 교과서 정의를 그대로 복붙 (어려운 용어로 설명)
- 시각 자료 없이 텍스트만 가득
- "실험실" 연결이 없어 읽기만 하고 끝남

**페르소나 반응:**
- 민지: "수요의 법칙: '가격이 오르면 사려는 양이 줄어든다' — 쉽게 써있네."
- 정훈: "그래프가 바로 보이니까 감이 온다."
- 수진: "영어 용어명도 같이 나와서 원서 볼 때 도움 될 듯."

## 2. 첫 학습

### US-04: 수요-공급 곡선 이해
> **"사용자로서, 수요-공급 곡선의 기본 형태와 균형 가격의 의미를 시각적으로 이해하고 싶다."**

**수용 기준:**
- 수요 곡선(우하향)과 공급 곡선(우상향)이 하나의 그래프에 표시
- 균형점(교차점)이 하이라이트 표시
- 균형 가격과 균형 수량이 수치로 표시
- 그래프 아래에 현재 상태를 설명하는 텍스트 패널

**안티패턴:**
- 그래프에 축 라벨이 없음
- 균형점을 찾으려면 사용자가 눈대중으로 교차점을 추정해야 함
- 설명 없이 그래프만 덩그러니 있음

**페르소나 반응:**
- 민지: "교과서 그래프랑 같은 모양인데, 여기선 균형점이 자동으로 표시되니까 좋다."
- 정훈: "아, 가격이 여기서 만나는 거구나."

---

### US-05: 용어 카드에서 실험실로 전환
> **"사용자로서, 용어 카드를 읽다가 '직접 해보기' 버튼을 눌러 인터랙티브 모델로 넘어가고 싶다."**

**수용 기준:**
- 용어 카드 하단에 "실험실에서 직접 해보기" 버튼 존재
- 버튼 탭 시 해당 모델의 인터랙티브 실험실 화면으로 전환
- 실험실에서 "용어 카드로 돌아가기" 뒤로가기 가능
- 전환 시 로딩 없이 즉시 표시

**안티패턴:**
- 용어 카드와 실험실이 별개 섹션이라 찾아가야 함
- 전환 시 전체 페이지 리로드
- 돌아가기가 홈으로 가버림

**페르소나 반응:**
- 민지: "읽다가 바로 실험해볼 수 있으니까 이해가 빨라진다."
- 수진: "이론 읽고 → 실험 → 다시 이론. 이 흐름이 좋다."

---

### US-06: 카테고리별 용어 탐색
> **"사용자로서, 미시/거시/국제/정책 카테고리별로 용어를 분류해서 보고 싶다."**

**수용 기준:**
- 용어 목록 상단에 카테고리 필터 탭 (전체, 미시, 거시, 국제, 정책)
- 카테고리 선택 시 해당 용어만 필터링
- 각 카테고리별 용어 수 표시
- 현재 선택된 카테고리가 시각적으로 구분

**안티패턴:**
- 카테고리가 드롭다운 메뉴 안에 숨겨져 있음
- 필터링 시 화면이 깜빡이며 리렌더
- 카테고리 간 용어가 중복 표시되어 혼란

**페르소나 반응:**
- 민지: "미시경제 시험이니까 미시 카테고리만 모아볼 수 있어서 편하다."
- 수진: "거시경제 파트부터 정리해야지."

---

### US-07: 관련 용어 연결 탐색
> **"사용자로서, 하나의 용어를 보다가 관련된 다른 용어로 자연스럽게 넘어가고 싶다."**

**수용 기준:**
- 용어 카드 하단에 "관련 용어" 섹션 (2-4개)
- 관련 용어 탭 시 해당 카드로 전환
- 뒤로가기로 이전 카드로 복귀 가능
- 관련 용어는 개념적 연관성 기반 (예: 수요 → 탄력성, 균형 가격)

**안티패턴:**
- 관련 용어가 랜덤 또는 알파벳순 (의미 없는 연결)
- 탭했는데 새 탭/창이 열림
- 관련 용어가 10개 이상 나열되어 선택 장애

**페르소나 반응:**
- 민지: "수요 보다가 탄력성으로 넘어갔다. 자연스러운 흐름이네."
- 정훈: "GDP 보다가 인플레이션으로 연결. 이런 식으로 공부하면 맥락이 잡히네."

## 3. 첫 실험

### US-08: 슬라이더로 가격 변수 조절
> **"사용자로서, 수요-공급 모델에서 가격 슬라이더를 움직여 수요량과 공급량이 어떻게 변하는지 실시간으로 보고 싶다."**

**수용 기준:**
- 가격 슬라이더가 화면 하단에 명확히 위치
- 슬라이더 드래그 시 그래프 위의 점(현재 가격선)이 실시간 이동
- 현재 가격에서의 수요량/공급량이 수치로 표시
- 초과수요(가격<균형) 또는 초과공급(가격>균형) 영역이 색상으로 표시
- 균형 가격에 도달하면 "균형!" 피드백

**안티패턴:**
- 슬라이더를 움직여도 그래프가 0.5초 이상 지연
- 수치 표시 없이 그래프만 변함 (정확한 값을 알 수 없음)
- 슬라이더 범위가 너무 좁아 변화가 미미

**페르소나 반응:**
- 민지: "가격을 올리니까 수요량이 줄어든다! 교과서에서 읽은 그대로네. 근데 직접 해보니까 느낌이 다르다."
- 정훈: "초과공급 영역이 빨간색으로 표시되니까, '재고가 쌓이는' 상황이 눈에 보인다."
- 수진: "균형점 찾기 — 이런 문제가 시험에 나오는데, 슬라이더로 풀어보니 직관적이다."

---

### US-09: GDP 구성요소 분해
> **"사용자로서, GDP = C + I + G + (X-M) 각 요소의 슬라이더를 조절해서 GDP 총량이 어떻게 변하는지 보고 싶다."**

**수용 기준:**
- 4개 슬라이더: 소비(C), 투자(I), 정부지출(G), 순수출(X-M)
- 각 슬라이더에 현재 값(조 원) 표시
- 스택 바 차트 또는 파이 차트로 구성 비율 시각화
- GDP 총합이 상단에 크게 표시
- 각 요소 변경 시 설명 패널에 "소비가 증가하면 → GDP가 상승합니다" 동적 텍스트

**안티패턴:**
- 슬라이더 4개가 너무 작아 모바일에서 조작 불가
- 총합만 표시되고 구성 비율은 안 보임
- 요소 간 상호작용 없이 단순 덧셈만 (현실감 부족하지만 P0에서는 허용)

**페르소나 반응:**
- 민지: "정부지출을 확 올려보니 GDP가 늘어났다. 재정정책이 이런 거구나."
- 정훈: "한국 GDP에서 소비 비중이 이렇게 큰 줄 몰랐네."

---

### US-10: 탄력성 실험
> **"사용자로서, 가격 탄력성 모델에서 가격을 변화시켰을 때 수요량 변화율을 관찰하고, 탄력적/비탄력적 재화의 차이를 체감하고 싶다."**

**수용 기준:**
- "재화 유형" 선택 (생활필수품 vs 사치품 등) → 탄력성 계수가 달라짐
- 가격 변화율 슬라이더 → 수요 변화율이 계산되어 표시
- 탄력성 계수(절대값)가 1보다 큰지/작은지에 따라 "탄력적" / "비탄력적" 라벨 표시
- 총수입(가격 x 수량) 변화도 함께 시각화

**안티패턴:**
- 탄력성 숫자만 보여주고 의미 설명 없음
- 재화 유형 선택지가 없어 추상적인 숫자 놀이
- 총수입 변화를 안 보여줘서 "왜 탄력성이 중요한지" 모름

**페르소나 반응:**
- 수진: "생활필수품은 가격 올려도 수요가 안 줄어드는구나. 비탄력적이라 그런 거네."
- 정훈: "유가가 올라도 기름 안 넣을 순 없으니까... 이게 비탄력이구나."

---

### US-11: 인플레이션-통화량 관계 실험
> **"사용자로서, 통화량 슬라이더를 늘렸을 때 물가가 어떻게 변하는지 관찰하고 싶다."**

**수용 기준:**
- 통화량(M) 슬라이더
- 물가 수준(P) 그래프가 통화량에 반응하여 변화
- 화폐수량설(MV=PY) 기반 수식 표시 (선택적)
- "통화량 2배 → 물가 2배" 같은 극단적 케이스를 실험 가능
- 현실 사례 연결: "2020년 코로나 시기 통화량 급증 → 2022년 인플레이션"

**안티패턴:**
- 통화량과 물가의 관계가 항상 선형으로만 표시 (현실과 괴리)
- 그래프에 시간 축이 없어 "즉시 반응"처럼 보임
- 수식만 보여주고 직관적 그래프 없음

**페르소나 반응:**
- 정훈: "아, 돈을 많이 찍으면 물가가 오르는 거구나. 뉴스에서 맨날 말하는 게 이거였어."
- 민지: "MV=PY 공식이 이렇게 작동하는 거구나."

## 4. 심화 학습

### US-12: 곡선 이동 실험 (수요/공급 시프트)
> **"사용자로서, 가격 외 요인(소득, 기호, 기술)이 변하면 수요/공급 곡선 자체가 이동하는 것을 실험하고 싶다."**

**수용 기준:**
- "곡선 이동 모드" 토글 또는 별도 슬라이더 (소득 수준, 생산 기술 등)
- 곡선이 좌/우로 이동하며 새 균형점이 표시
- 이동 전 곡선이 점선으로 잔상 표시 → 비교 가능
- "소득 증가 → 수요 곡선 우측 이동 → 균형 가격 상승, 균형 수량 증가" 설명

**안티패턴:**
- 곡선 이동과 곡선 위의 이동(가격 변화)을 구분하지 않음
- 이동 전 곡선이 사라져서 비교 불가
- 너무 많은 이동 요인을 동시에 조절하게 해서 혼란

**페르소나 반응:**
- 민지: "곡선 위의 이동 vs 곡선 자체의 이동 — 시험에 맨날 나오는 건데, 이제 구분이 된다!"
- 수진: "이동 전후를 점선으로 비교할 수 있어서 좋다."

---

### US-13: 시나리오 모드 체험
> **"사용자로서, '만약 정부가 세금을 올리면?'같은 시나리오를 선택하면 관련 모델이 자동으로 세팅되어 결과를 보고 싶다."**

**수용 기준:**
- 시나리오 목록: 5개 이상 (세금 인상, 금리 인하, 수출 증가, 최저임금 인상, 통화량 확대)
- 시나리오 선택 시 관련 모델 + 변수가 자동 설정
- "결과 보기" 후 변화 전/후 비교
- 사용자가 시나리오 변수를 추가 조작 가능

**안티패턴:**
- 시나리오 결과가 텍스트로만 설명 (그래프 변화 없음)
- 시나리오 실행 후 사용자 추가 조작 불가 (일방적 결과만 보여줌)
- 시나리오가 1-2개뿐이라 금방 끝남

**페르소나 반응:**
- 정훈: "'금리 인하하면?'을 눌렀더니 IS-LM 모델에서 결과가 나온다. 뉴스가 이해된다."
- 수진: "시험 문제가 이런 시나리오 형태인데, 시뮬레이션으로 풀어보니 감이 잡힌다."

---

### US-14: IS-LM 모델 탐색
> **"사용자로서, IS-LM 모델에서 재정정책(IS 이동)과 통화정책(LM 이동)의 효과를 시각적으로 비교하고 싶다."**

**수용 기준:**
- IS 곡선과 LM 곡선이 이자율(Y축)-국민소득(X축) 좌표에 표시
- 정부지출 슬라이더 → IS 곡선 이동
- 통화공급 슬라이더 → LM 곡선 이동
- 균형 이자율과 균형 국민소득이 수치로 표시
- 이동 전/후 비교 (점선 잔상)
- 설명 패널: "정부지출 증가 → IS 우측 이동 → 이자율 상승 + 국민소득 증가"

**안티패턴:**
- IS, LM이 뭔지 설명 없이 곡선만 보여줌
- 곡선 이동 방향이 직관적이지 않음 (화살표 없음)
- 재정정책/통화정책을 동시에 조절 못 함

**페르소나 반응:**
- 수진: "IS-LM 모델이 이렇게 움직이는 거구나. 교과서에서 못 느낀 걸 여기서 체감했다."
- 정훈: "금리 올리면 LM이 왼쪽으로... 경기가 냉각되는 과정이 그래프로 보인다."

---

### US-15: 학습 진도 확인
> **"사용자로서, 내가 지금까지 어떤 용어와 모델을 학습했는지 한눈에 보고 싶다."**

**수용 기준:**
- 열어본 용어 카드 수 / 전체 (예: 12/20)
- 실험해본 모델 수 / 전체 (예: 3/4)
- 카테고리별 완료 막대 그래프
- 최근 학습 이력 리스트 (날짜 + 항목명)
- 진도 데이터는 localStorage에 자동 저장

**안티패턴:**
- 진도 화면이 숫자만 나열 (시각화 없음)
- "100% 완료"만 추적하고 부분 진도는 안 보임
- 이력이 없어서 "어디까지 했더라?" 모름

**페르소나 반응:**
- 민지: "미시경제 80% 했네. 거시경제도 시작해야겠다."
- 수진: "오늘 학습한 게 3개. 일주일이면 다 볼 수 있겠다."

---

## 5. 습관화

### US-16: 재방문 시 이어서 학습
> **"사용자로서, 앱을 다시 열면 마지막으로 본 용어/모델로 빠르게 돌아가고 싶다."**

**수용 기준:**
- 홈 화면에 "이어서 학습하기" 섹션
- 마지막으로 열었던 용어 카드 또는 모델 표시
- 탭 한 번으로 해당 위치로 이동
- localStorage에서 마지막 학습 위치 자동 저장

**안티패턴:**
- 매번 홈에서 처음부터 탐색해야 함
- "이어서 학습" 버튼이 있지만 엉뚱한 곳으로 감
- 브라우저 캐시 삭제하면 모든 진도 날아감 (경고 없이)

**페르소나 반응:**
- 정훈: "어제 GDP 하다 말았는데 바로 이어서 할 수 있네."
- 민지: "시험 전에 매일 조금씩 하는 중. 이어서 하기가 편하다."

---

### US-17: 새로운 모델 발견
> **"사용자로서, 이미 학습한 모델과 관련된 새로운 모델을 추천받아 학습 범위를 넓히고 싶다."**

**수용 기준:**
- 모델 학습 완료 후 "관련 모델" 추천 (1-2개)
- 추천 근거 표시 ("수요-공급을 배웠으니 탄력성도 해보세요")
- 아직 안 해본 모델에 "NEW" 배지
- 모든 모델 완료 시 축하 메시지

**안티패턴:**
- 추천이 랜덤이라 학습 흐름과 무관
- 이미 완료한 모델을 또 추천
- 추천이 팝업/알림으로 학습을 방해

**페르소나 반응:**
- 민지: "수요-공급 다음에 탄력성 추천해주네. 순서가 자연스럽다."
- 수진: "아직 안 한 모델이 뭔지 한눈에 보여서 계획 세우기 좋다."

---

### US-18: 학습 설정 관리
> **"사용자로서, 학습 데이터 초기화나 앱 설정을 관리하고 싶다."**

**수용 기준:**
- 설정 화면에서 학습 진도 초기화 가능 (확인 다이얼로그 필수)
- 학습 데이터 용량 표시
- 앱 버전 정보
- 데이터 초기화 전 "정말 초기화하시겠습니까?" 확인

**안티패턴:**
- 초기화 버튼에 확인 없이 바로 삭제
- 설정 화면이 숨겨져서 찾기 어려움
- 초기화 후 복구 불가능한데 경고 없음

**페르소나 반응:**
- 민지: "새 학기 시작할 때 초기화하고 다시 처음부터 해야지."
- 수진: "시험 끝나고 다시 처음부터 복습할 때 쓸 수 있겠다."
