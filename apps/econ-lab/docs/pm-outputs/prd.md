> 버전: 1.0
> 작성일: 2026-03-22
> 상태: Draft

---

## 1. 문제 정의

### 현재 상황
- 경제학 학습은 텍스트 위주. 교과서나 블로그 글을 읽고 "이해했다고 느끼지만" 실제로 변수 간 관계를 직관적으로 파악하지 못함.
- 수요-공급 곡선을 그림으로 본 적은 있지만, "가격이 오르면 수요가 줄어든다"를 직접 슬라이더로 움직여본 적은 없음.
- 기존 경제학 앱은 퀴즈형(암기 중심)이거나 뉴스 요약형(수동적 소비)이 대부분.

### 핵심 문제
**경제 개념을 "읽는 것"과 "체감하는 것" 사이의 간극.**

가격 탄력성이 -0.5라는 걸 읽는 것과, 슬라이더를 움직여서 가격을 10% 올렸을 때 수요가 5% 줄어드는 걸 눈으로 보는 것은 완전히 다른 학습 경험이다.

### 해결 방향
경제 모델을 인터랙티브 시뮬레이션으로 만들어, 변수를 직접 조작하고 결과를 실시간으로 관찰하게 한다. 물리 실험실처럼, 경제학에도 "실험실"을 만든다.

## 2. 타겟 유저

### 주요 타겟
| 세그먼트 | 설명 | 규모 감각 |
|---------|------|----------|
| 대학생 (교양 경제) | 경제학원론 수강생, 시험 대비 + 이해 목적 | 매 학기 수십만 명 |
| 공시생/취준생 | 경제학 과목 준비, 개념 정리 필요 | 연간 20만+ |
| 재테크 관심 직장인 | 경제 뉴스 이해, 금리/환율/인플레이션 체감 | 가장 큰 세그먼트 |

### 공통 니즈
- 텍스트만으로는 이해가 안 됨 → 시각적 설명 필요
- 개념 간 관계(인과)를 파악하고 싶음 → 인터랙션 필요
- 짧은 시간에 핵심만 → 모바일 친화적 학습 단위

## 3. 페르소나

### 페르소나 1: 민지 (대학생, 22세)
- **배경:** 경영학과 2학년. 경제학원론 수강 중. 중간고사 2주 전.
- **행동:** 강의 노트를 읽었는데 수요-공급 그래프가 "왜 이렇게 움직이는지" 감이 안 옴. 유튜브 강의를 봐도 일방적 설명이라 집중이 안 됨.
- **목표:** 시험 전에 핵심 모델 10개를 "진짜 이해"하고 싶음.
- **좌절 포인트:** "교과서 그래프는 정적이라 변수 하나 바꾸면 어떻게 되는지 상상해야 함."
- **성공 상태:** 슬라이더로 가격을 올려보고 "아, 수요가 이렇게 줄어드는구나"를 체감. 시험에서 그래프 문제를 직관적으로 풀 수 있음.

### 페르소나 2: 정훈 (직장인, 34세)
- **배경:** IT 회사 기획자. 경제 뉴스를 매일 읽지만 "기준금리 인상이 왜 경기를 냉각시키는지" 정확한 메커니즘을 모름.
- **행동:** 점심시간이나 출퇴근길에 10분씩 학습. 긴 강의는 안 봄.
- **목표:** 경제 뉴스에 나오는 용어와 모델을 체감 수준으로 이해하고 싶음.
- **좌절 포인트:** "GDP, CPI, 기준금리... 단어는 아는데 연결이 안 됨."
- **성공 상태:** IS-LM 모델에서 통화 공급 슬라이더를 올려보고 "이자율이 떨어지고 → 투자가 늘고 → GDP가 오르는" 과정을 시각적으로 추적.

### 페르소나 3: 수진 (공시생, 28세)
- **배경:** 7급 공무원 시험 준비 중. 경제학 과목이 약점.
- **행동:** 하루 2시간 경제학 공부. 이론을 암기하지만 응용 문제에서 틀림.
- **목표:** 모델의 "원리"를 이해해서 변형 문제도 풀 수 있게 되고 싶음.
- **좌절 포인트:** "필립스 곡선이 왜 우하향인지 외웠는데, 기대인플레이션이 바뀌면 어떻게 이동하는지 헷갈림."
- **성공 상태:** 시뮬레이션에서 기대인플레이션 슬라이더를 올려보고 곡선이 위로 이동하는 걸 확인. 응용 문제를 자신 있게 풀 수 있음.

## 4. 핵심 기능

### 4.1 경제 용어 카드

용어별 카드 UI. 각 카드에는:
- **용어명** (한/영)
- **한 줄 정의** — 30자 이내
- **쉬운 설명** — 일상 비유 중심, 3-5문장
- **시각 자료** — 정적 그래프 또는 다이어그램 (SVG/Canvas)
- **관련 용어 링크** — 탭하면 해당 용어 카드로 이동
- **"실험실로 가기" 버튼** — 해당 개념의 인터랙티브 모델로 연결

카드는 카테고리별로 분류: 미시경제, 거시경제, 국제경제, 재정/통화정책.

### 4.2 인터랙티브 그래프 실험실

경제 모델의 핵심 그래프를 직접 조작:
- **슬라이더** — 변수 조절 (가격, 수량, 세율, 이자율 등)
- **실시간 그래프 업데이트** — 슬라이더 움직이면 즉시 반영
- **균형점 표시** — 곡선 교차점, 최적점 등 자동 하이라이트
- **변화 방향 화살표** — 이동 전/후를 시각적으로 비교
- **설명 패널** — 현재 상태에 대한 텍스트 해설 (변수값에 따라 동적 변경)

### 4.3 모델 시뮬레이션

보다 복합적인 모델:
- 여러 변수를 동시에 조절
- 인과관계 체인을 단계별로 시각화 (A→B→C)
- "시나리오 모드" — 미리 정의된 시나리오 선택 후 결과 비교
  - 예: "중앙은행이 금리를 1%p 올리면?" → IS-LM 모델에서 결과 시뮬레이션
- 리셋 버튼으로 초기 상태 복귀

### 4.4 학습 진도 추적

- 열어본 용어 카드 수 / 전체
- 실험해본 모델 수 / 전체
- 카테고리별 완료율 (미시, 거시, 국제, 정책)
- 최근 학습 이력 (날짜 + 항목)
- localStorage에 저장, 서버 불필요

## 5. 기술 제약

| 항목 | 결정 | 이유 |
|------|------|------|
| 프레임워크 | React + Vite + TypeScript | 빠른 개발, 컴포넌트 재사용 |
| 상태 관리 | Zustand 또는 React Context | 경량, 충분한 복잡도 |
| 그래프 렌더링 | Recharts 또는 D3.js (SVG) | 인터랙티브 그래프 필수 |
| 데이터 저장 | localStorage | 서버 없음, YAGNI |
| 데이터 | 앱 내장 JSON/TS 상수 | 서버 API 없음, 데모 모드 기본 |
| 배포 | 정적 호스팅 (Vercel/Netlify) | SPA, 서버 불필요 |
| 반응형 | 모바일 우선, 데스크톱 호환 | 출퇴근길 학습 시나리오 |

### 데모 모드
- 앱 실행 즉시 모든 기능 사용 가능 (로그인/회원가입 없음)
- P0 모델 4개 + 용어 20개 이상 내장
- localStorage에 진도 자동 저장

## 6. 성공 지표 (KPI)

### 핵심 지표
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 첫 실험 완료율 | 70%+ | 앱 진입 후 슬라이더를 1회 이상 조작한 비율 |
| 세션 체류 시간 | 5분+ | 평균 세션 길이 |
| 재방문율 (D7) | 30%+ | 7일 내 재방문 비율 |
| 모델 탐색 깊이 | 3개+ | 세션당 평균 탐색 모델 수 |

### 보조 지표
| 지표 | 목표 |
|------|------|
| 용어 카드 열람 수 | 세션당 5개+ |
| 카테고리 완료율 | P0 카테고리 50%+ 완료 유저 비율 20% |
| 학습 진도 페이지 조회율 | 전체 유저의 40%+ |

## 7. 우선순위

### P0 — Must Have (MVP)
- 용어 카드 시스템 (20개 용어, 4개 카테고리)
- 인터랙티브 모델 4개:
  - 수요와 공급 (가격/수량 곡선, 균형점)
  - 탄력성 (가격 변화 → 수요 변화율)
  - GDP 구성요소 (C+I+G+(X-M) 슬라이더)
  - 인플레이션과 통화량 (통화량 → 물가)
- 학습 진도 추적 (localStorage)
- 데모 모드 (즉시 체험 가능)
- 반응형 레이아웃 (모바일 + 데스크톱)

### P1 — Should Have
- 추가 모델 4개:
  - 기회비용과 PPF
  - 비교우위와 무역
  - 승수 효과
  - IS-LM 모델
- 시나리오 모드 ("만약 ~하면?" 프리셋)
- 용어 카드 검색/필터
- 카테고리별 진도 시각화

### P2 — Nice to Have
- 추가 모델 3개:
  - 필립스 곡선
  - 환율과 경상수지
  - 게임 이론 (죄수의 딜레마)
- 학습 퀴즈 모드 (모델 조작 후 질문)
- 다크 모드
- 학습 데이터 내보내기 (JSON)
