## 추가 크리틱 (Iteration 2 이슈)

### 이슈 #1 — "건너뛰기" 버튼 border 가시성 부족 (심각도: 중)

**현상**: `.btn-skip`의 border 색상이 `var(--border)` (#E5DDD2)인데, 앱 배경이 #FBF7F0이라 명도 차이가 극히 작다. 스크린샷에서 건너뛰기 버튼이 거의 텍스트처럼 보인다.

**디자인 시스템 기준**: Secondary 버튼은 `border: 2px solid #2B5F3A` (짙은 녹색)로 정의되어 있다. 현재 `.btn-skip`은 디자인 시스템의 Secondary 스펙과 다르다.

**제안**:
- border 색상을 `var(--text-tertiary)` (#AEA494) 이상으로 올리거나
- 디자인 시스템대로 `var(--secondary)` (#2B5F3A) 녹색 아웃라인 적용
- 건너뛰기는 "다음 개그로 패스"하는 명확한 액션이므로 Secondary 위계가 시각적으로 읽혀야 함

---

### 이슈 #2 — 홈 화면 상단 빈 공간 과다 (심각도: 중)

**현상**: 홈 화면에서 헤더("아재개그 자판기") 아래부터 서브카피+버튼까지 화면의 ~55%가 완전히 비어있다. 자판기 컨셉상 미니멀이 맞지만, 812px 높이에서 이 정도 빈 공간은 "콘텐츠가 로딩 안 됐나?" 하는 착각을 유발한다.

**제안**:
- 버튼+서브카피를 화면의 40~45% 지점에 배치 (현재 50% 정확히 중앙이 아니라 살짝 아래)
- 또는 상단에 "자판기 외형"을 연상시키는 미니멀 라인 일러스트(1색, 60px 정도)를 넣어 시각 앵커 제공
- 어느 쪽이든, 현재 상태보다 의도적인 여백 배분이 필요

---

### 이슈 #3 — 앱 타이틀 크기 디자인 시스템 불일치 (심각도: 하)

**현상**: `.app-title`이 `font-size: 1.5rem` (24px)인데, 디자인 시스템은 앱 타이틀을 `3xl: 36px / 2.25rem`으로 정의했다. Black Han Sans는 자체가 굵고 임팩트 있는 서체라 1.5rem에서도 존재감은 있지만, 디자인 시스템과 불일치.

**제안**: 시스템대로 2.25rem이 과하다면, 최소 1.75rem (28px)으로 올려서 "간판" 느낌 강화. 레트로 자판기 컨셉에서 타이틀은 곧 간판이다.

---

### 이슈 #4 — 화면 전환에 crossfade 없음 (심각도: 하)

**현상**: 메인 <-> 즐겨찾기 전환이 `display: none` 토글로 즉시 전환된다. 디자인 시스템에서 "화면 전환: opacity crossfade 150ms"로 정의했으나 미구현.

**제안**: `.hidden` 클래스에 opacity transition을 추가하거나, 화면 전환 시 간단한 fade 처리. 150ms면 유저가 인지하기 어려울 정도로 짧지만, 있고 없고의 체감 차이는 크다.

---

### 이슈 #5 — 토스트 위치가 카운터와 겹침 (심각도: 하)

**현상**: 토스트가 `bottom: 80px`에 뜨는데, 카운터("오늘 N개 봄")가 화면 최하단에 있어 토스트와 시각적으로 겹치는 순간이 생긴다. 스크린샷에서 "저장됨!" 토스트와 "오늘 1개 봄" 사이 간격이 매우 좁다.

**제안**: 토스트를 `bottom: 120px` 정도로 올리거나, 카운터를 개그 카드 바로 아래로 이동시켜 공간 분리.

### 잘 된 점
- 탠저린 오렌지 + 크림 + 녹색 팔레트가 레트로 자판기 컨셉을 정확히 전달
- 개그 카드 좌측 액센트 바 + 비대칭 radius가 시각적으로 독특
- 버튼의 box-shadow + translateY 눌림 효과가 물리적 피드백 제공
- 복사/저장 버튼 쌍 + 토스트 피드백 = 사용성 개선 확실
- 이모지 없이 SVG 아이콘만 사용 (R13 완벽 준수)

### 우선 개선 순위
1. **건너뛰기 버튼 가시성** — border 색상 강화 (R16 관련, 가장 시급)
2. **홈 화면 여백 밸런스** — 빈 공간 배분 재조정
3. 앱 타이틀 크기 디자인 시스템 일치
4. 화면 전환 crossfade
5. 토스트 위치 미세 조정
