### 주요 발견사항

**S-003 일반형 개그 한번에 표시 미구현:**
PRD/스크린플로우에서는 "일반형 개그(질문-답 구조가 아닌 것)는 한번에 표시"(US-007 인수조건 4번)라고 명시했으나, 현재 코드는 **모든 개그를 질문-답 분리로 처리**한다. `renderSetup()`이 joke type 구분 없이 항상 setup만 먼저 표시하고 카드 탭을 요구한다. 데이터(`jokes.ts`)에도 모든 개그가 setup/punchline 구조로 되어 있어, 사실상 모든 개그가 질문형으로 동작한다. 이는 P1 기능이므로 큰 문제는 아니나, "일반형" 개그의 한번에 표시 경로가 누락된 상태.
