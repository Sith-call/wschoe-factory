## 2. 문제 정의

### 핵심 문제
MZ세대 10명 중 7명이 "돈 관리를 잘 하고 싶다"고 하지만, 실제로 소비 습관을 바꾸는 사람은 10%도 안 된다.

### 원인 분석

| 원인 | 설명 | 현재 대안의 한계 |
|------|------|------------------|
| 자기 인식 부재 | 자신의 소비 패턴을 객관적으로 모른다 | 가계부 앱은 입력이 귀찮아 3일 내 이탈 |
| 실행 동기 부족 | 혼자 절약하면 작심삼일 | 커뮤니티 기반 앱은 비교 스트레스 유발 |
| 성과 불확실 | 변화를 체감하지 못한다 | 단순 지출 추적은 개선 방향을 제시 안 함 |

### 시장 맥락
- **YONO 트렌드**: MZ세대 사이 "You Only Need One" 절약 문화 확산
- **무지출 챌린지**: SNS에서 바이럴 — 실천 도구 부재
- **앱인토스 블루오션**: 1000+ 미니앱 중 게임 레드오션, 금융 유틸리티 블루오션
- **토스 맥락 시너지**: 금융 앱 안에서 소비 개선 도구 → 자연스러운 맥락

## 3. 핵심 가치

> **"진단하고, 도전하고, 증명하라."**

단순 소비 유형 테스트가 아닌, **진단 → 챌린지 → 개선 확인**의 3단계 루프.
"나를 알고, 바꾸고, 확인하는" 소비 습관 개선 앱.

## 4. 3개 핵심 루프

### 루프 1: 진단 (Diagnose)

**목적**: 사용자의 소비 패턴을 객관적으로 파악

- **12문항** 소비 성향 테스트
  - 6개 카테고리: 카페 / 배달 / 쇼핑 / 구독 / 교통 / 유흥
  - 카테고리당 2문항, 4지선다 (1점~4점 스케일)
  - 문항 예시:
    - 카페: "일주일에 카페를 몇 번 가시나요?" → (1) 거의 안 감 / (2) 1-2회 / (3) 3-4회 / (4) 매일
    - 배달: "배달 주문 시 평균 금액은?" → (1) 1만원 이하 / (2) 1-2만원 / (3) 2-3만원 / (4) 3만원 이상
    - 쇼핑: "최근 한 달, 계획에 없던 물건을 산 적이?" → (1) 없음 / (2) 1-2번 / (3) 3-5번 / (4) 셀 수 없음
    - 구독: "현재 구독 서비스 개수는?" → (1) 0-1개 / (2) 2-3개 / (3) 4-6개 / (4) 7개 이상
    - 교통: "택시를 일주일에 몇 번 타시나요?" → (1) 거의 안 탐 / (2) 1-2회 / (3) 3-4회 / (4) 매일
    - 유흥: "한 달 음주 모임 횟수는?" → (1) 0-1회 / (2) 2-3회 / (3) 4-6회 / (4) 7회 이상

- **결과**: 6가지 소비 유형 중 가장 높은 점수 카테고리로 분류

| 유형 | 설명 | 공유 텍스트 이모지 | UI 아이콘 (SVG) |
|------|------|-------------------|-----------------|
| 카페 중독형 | 커피 없이 못 사는 당신 | ☕ | 커피컵 SVG |
| 배달 의존형 | 자취 필수템 = 배달앱 | 🛵 | 배달 바이크 SVG |
| 충동 구매형 | 장바구니가 늘 차 있는 | 🛍️ | 쇼핑백 SVG |
| 구독 과다형 | 매달 빠져나가는 구독료 | 📱 | 스마트폰 SVG |
| 택시 습관형 | 걸어가면 되는데 택시를 | 🚕 | 택시 SVG |
| 유흥 올인형 | 인생은 한 번뿐 YOLO | 🍻 | 맥주잔 SVG |

- **카테고리별 소비 점수 레이더 차트** (6각형)
  - 각 축: 카페 / 배달 / 쇼핑 / 구독 / 교통 / 유흥
  - 점수 범위: 2-8 (카테고리당 2문항, 각 1-4점)
  - 인라인 SVG로 렌더링 (외부 차트 라이브러리 없음)

- **월 예상 낭비 금액 계산**
  - 각 카테고리 점수를 기반으로 예상 월 초과 지출 산출
  - 계산 기준표:

| 카테고리 | 4점 기준 월 추정 낭비 | 점수별 가중치 |
|----------|----------------------|--------------|
| 카페 | 15만원 | 점수 * 1.875만원 |
| 배달 | 20만원 | 점수 * 2.5만원 |
| 쇼핑 | 25만원 | 점수 * 3.125만원 |
| 구독 | 10만원 | 점수 * 1.25만원 |
| 교통 | 15만원 | 점수 * 1.875만원 |
| 유흥 | 20만원 | 점수 * 2.5만원 |

  - 상위 3개 카테고리 점수 합산 → "당신의 월 예상 낭비 금액: ₩XX만원"

### 루프 2: 챌린지 (Challenge)

**목적**: 진단 결과 기반 맞춤형 행동 변화 유도

- 진단 결과에서 **가장 높은 점수 카테고리**의 7일 절약 챌린지 자동 생성

| 유형 | 챌린지 내용 | 일별 절약 추정 | 7일 총 절약 |
|------|-------------|---------------|------------|
| 카페 중독형 | 7일간 카페 3회 이하 | ₩5,000 | ₩35,000 |
| 배달 의존형 | 7일간 배달 2회 이하 | ₩8,000 | ₩56,000 |
| 충동 구매형 | 7일간 장바구니 24시간 규칙 | ₩10,000 | ₩70,000 |
| 구독 과다형 | 사용 안 하는 구독 1개 해지 | ₩5,000 | ₩35,000 |
| 택시 습관형 | 7일간 대중교통만 이용 | ₩7,000 | ₩49,000 |
| 유흥 올인형 | 7일간 음주 1회 이하 | ₩15,000 | ₩105,000 |

- **매일 체크인 시스템**
  - 앱 진입 시 "오늘 지켰나요?" 질문
  - 응답: "지켰어요" / "못 지켰어요"
  - 성공 시: 연속 스트릭 카운트 + 절약 금액 누적 + 햅틱 피드백 (SDK Haptic)
  - 실패 시: 격려 메시지 ("괜찮아요, 내일 다시!") + 스트릭 리셋 없음 (총 성공 일수만 카운트)

- **7일 캘린더 뷰**
  - Day 1 ~ Day 7 수평 표시
  - 각 날짜: 성공(체크 아이콘) / 실패(X 아이콘) / 미완료(빈 원)
  - 현재 날짜 하이라이트

### 루프 3: 증명 (Prove)

**목적**: 변화를 시각화하여 동기 부여 + 바이럴

- **7일 챌린지 완료 후 재진단** 유도
  - 같은 12문항을 다시 풀어서 점수 비교
  - Before/After 비교 카드 자동 생성

- **비교 카드 구성**
  - 메인 유형 카테고리 점수 변화: "카페 소비 점수: 8점 → 5점 (-37.5%)"
  - 전체 레이더 차트 Before/After 오버레이
  - 예상 월 절약 금액: "월 ₩12만원 절약 가능"
  - 챌린지 성공 일수: "7일 중 5일 성공"

- **공유 기능** (SDK Share API)
  - 카카오톡 공유 텍스트:
    ```
    나는 '소비 체질 개선'으로 7일 만에 카페 소비를 37.5% 줄였다!
    ☕ 카페 중독형 → 월 ₩12만원 절약 가능
    나도 진단받기 👉 [앱 링크]
    ```
  - 네이티브 공유 (iOS/Android share sheet)

- **새 챌린지 루프 유도**
  - "다음으로 높은 카테고리 챌린지 시작" 추천
  - "처음부터 재진단" 옵션

## 7. 기술 제약 및 SDK 활용

### 앱인토스 SDK 활용 계획

| SDK | 용도 | 상세 |
|-----|------|------|
| **Storage** | 로컬 데이터 저장 | 진단 결과, 챌린지 상태, 체크인 기록, 히스토리 |
| **Haptic** | 터치 피드백 | 체크인 성공 시 success 패턴, 챌린지 완료 시 notification 패턴 |
| **Share** | 결과 공유 | 카카오톡 텍스트 공유 + 네이티브 share sheet |
| **Clipboard** | 결과 복사 | 공유 텍스트 클립보드 복사 (Share 실패 시 폴백) |
| **Analytics** | 사용 추적 | 진단 완료, 챌린지 시작/완료, 공유 이벤트 |

### 데이터 스키마 (Storage SDK)

```typescript
interface SpendingReformData {
  // 진단 결과
  diagnosis: {
    completedAt: string;       // ISO 8601
    answers: number[];          // 12개 답변 (1-4)
    scores: {
      cafe: number;
      delivery: number;
      shopping: number;
      subscription: number;
      transport: number;
      entertainment: number;
    };
    primaryType: SpendingType;
    estimatedWaste: number;     // 월 예상 낭비 (원)
  } | null;

  // 현재 챌린지
  currentChallenge: {
    type: SpendingType;
    startedAt: string;
    checkIns: (boolean | null)[];  // 7일, null = 미완료
    currentDay: number;            // 0-6
  } | null;

  // 히스토리
  history: {
    diagnosis: DiagnosisResult;
    challenge: ChallengeResult;
    completedAt: string;
  }[];

  // 이전 진단 결과 (Before/After용)
  previousDiagnosis: DiagnosisResult | null;
}

type SpendingType =
  | 'cafe'
  | 'delivery'
  | 'shopping'
  | 'subscription'
  | 'transport'
  | 'entertainment';
```

### Out of Scope (MVP)

| 항목 | 이유 |
|------|------|
| 실제 은행/카드 연동 | 토스 API 제약 — 미니앱에서 접근 불가 |
| 서버 기반 소셜 기능 | 앱인토스 백엔드 불필요 원칙 |
| 토스 로그인/인증 | Console + 백엔드 서버 필요 |
| 인앱 결제 | MVP 범위 초과 |
| 푸시 알림 | 앱인토스 미지원 |

### 폰트

| 용도 | 폰트 | 가중치 |
|------|------|--------|
| 한국어 (본문/제목) | Wanted Sans Variable | 400, 500, 600, 700 |
| 영문/숫자 | Sora | 400, 500, 600, 700 |

- 기존 앱과 중복되지 않는 조합 (Pretendard+Inter, Spoqa+Space Grotesk, Pretendard+Outfit 사용됨)
- H1: 700, H2: 600, H3: 500, Body: 400 — R12 준수

### 애니메이션
- 퀴즈 카드 전환: 좌→우 슬라이드 (translateX)
- 결과 표시: scale(0.95→1) 스프링 이징
- 체크인 성공: 체크마크 드로잉 애니메이션 (SVG stroke-dashoffset)
- 그 외 대부분: 즉시 표시 (애니메이션 없음) — R4 준수

### 기타
- 네비게이션: 선형 플로우 (탭바 없음) — R5 준수
- 버튼: rounded-lg (pill도 sharp도 아닌 중간) — R10 준수
- 아이콘: 인라인 커스텀 SVG — R11 준수
- 이모지: UI 금지, 공유 텍스트에서만 허용 — R13 준수
- 그라디언트 텍스트: 금지 — R2 준수
- backdrop-blur: 공유 카드 모달 1곳만 — R6 준수
