## 유저 여정 타임라인

```
첫 방문    →    진단 (12문항)    →    챌린지 시작    →    매일 체크인    →    7일 완료    →    재방문/재진단
(발견)          (자기 인식)           (행동 시작)         (습관 형성)        (성과 확인)       (반복 루프)
```

## 1. 첫 방문 (발견)

### US-1.1: 앱 가치 즉시 이해

> 사용자로서, 앱을 열었을 때 3초 안에 이 앱이 뭘 하는 건지 이해하고 싶다.

**수용 기준:**
- 첫 화면에 앱의 핵심 메시지가 1줄로 표시: "당신의 소비 체질을 진단하고, 7일 만에 바꿔보세요"
- 하위 메시지로 구체적 혜택 제시: "12문항으로 내 소비 유형 파악 + 맞춤 절약 챌린지"
- 화면 로드 후 1초 이내 모든 텍스트 표시 완료 (로딩 스피너 없음)
- CTA 버튼 "지금 진단하기"가 화면 하단에 고정 표시

**안티패턴:**
- 앱 소개 슬라이드 3장 넘기기 (온보딩 과잉 — 바로 시작하게 해야 함)
- "로그인/회원가입" 먼저 요구 (가입 장벽 = 이탈)
- 추상적 카피: "더 나은 내일을 위해" (구체적이지 않음)

**페르소나 반응:**
- 최지은: "오 이거 뭔지 바로 알겠다. 해볼까?"
- 김태현: "12문항이면 2분이면 되겠네."
- 박소영: "맞춤 절약? 내 상황에 맞는 건가 보자."

---

### US-1.2: 한 번의 탭으로 진단 시작

> 사용자로서, 회원가입 없이 "시작하기" 한 번으로 바로 진단을 시작하고 싶다.

**수용 기준:**
- "지금 진단하기" 버튼 탭 → 즉시 첫 번째 문항 표시
- 로그인, 이름 입력, 약관 동의 등 사전 단계 0개
- 버튼에서 문항 화면 전환까지 300ms 이내

**안티패턴:**
- "시작 전 닉네임을 입력하세요" (불필요한 마찰)
- "이용약관에 동의하세요" 체크박스 (MVP에서 불필요)

**페르소나 반응:**
- 최지은: "가입 안 해도 돼? 바로 시작하자!"
- 김태현: "이런 게 좋아. 빠르게."
- 박소영: "복잡한 거 싫은데, 이건 깔끔하네."

---

## 2. 진단 (12문항)

### US-2.1: 문항별 카테고리와 4지선다

> 사용자로서, 각 문항이 내 어떤 소비 습관에 대한 건지 직관적으로 알고 싶다.

**수용 기준:**
- 각 문항 상단에 카테고리 라벨 표시 (예: "카페 소비")
- 카테고리별 SVG 아이콘 표시 (커피컵, 배달바이크 등)
- 4개 선택지 세로 배열, 탭으로 선택
- 선택 즉시 다음 문항으로 자동 전환 (0.3초 딜레이)
- 뒤로가기 가능 (이전 문항 수정)

**안티패턴:**
- 문항에 카테고리 힌트 없이 "질문 3" 같은 무의미한 라벨
- 선택 후 "다음" 버튼을 별도로 눌러야 함 (마찰 추가)
- 선택지가 너무 길어 스크롤 필요

**페르소나 반응:**
- 최지은: "오 카페 질문이다. 나는... 매일이지 ㅋㅋ"
- 김태현: "택시 질문 나왔는데 4번 누르기 부끄럽네."
- 박소영: "구독 서비스 개수... 세어보니 12개라고?"

---

### US-2.2: 진행 프로그레스로 완료 유도

> 사용자로서, 몇 문항 남았는지 알아서 "곧 끝나겠다"는 느낌을 받고 싶다.

**수용 기준:**
- 화면 상단에 프로그레스 바 표시 (1/12 → 12/12)
- 프로그레스 바 애니메이션: 부드러운 width transition (300ms ease-out)
- 6/12 이후: "벌써 반이나 왔어요!" 메시지 표시
- 10/12 이후: "거의 다 왔어요!" 메시지 표시
- 숫자 인디케이터: "3 / 12" 형식

**안티패턴:**
- 프로그레스 바 없이 문항만 나열 (끝이 안 보이면 이탈)
- "로딩 중..." 표시 (문항 전환에 로딩 불필요)
- 프로그레스 바가 너무 작아서 인지 불가

**페르소나 반응:**
- 최지은: "6번이면 반이네, 금방이겠다."
- 김태현: "10번, 거의 끝이구나."
- 박소영: "12문항이면 2분도 안 걸리겠다."

---

### US-2.3: 결과 로딩 시 재미있는 분석 메시지

> 사용자로서, 결과를 기다리는 동안 지루하지 않고 기대감을 느끼고 싶다.

**수용 기준:**
- 12문항 완료 후 2-3초 로딩 화면 표시 (실제 계산은 즉시, 연출용 딜레이)
- 순차 표시 메시지 (0.8초 간격):
  1. "당신의 지갑을 분석하고 있어요..."
  2. "소비 패턴을 찾아내는 중..."
  3. "맞춤 처방전을 만들고 있어요..."
- 중앙에 원형 프로그레스 애니메이션 (SVG)
- 메시지 전환 시 scale(0.95→1) 애니메이션

**안티패턴:**
- 5초 이상 로딩 (지루함 유발)
- 단순 스피너만 표시 (재미 없음)
- "AI가 분석 중" 같은 과장 (실제로 AI 아님)

**페르소나 반응:**
- 최지은: "ㅋㅋㅋ 지갑 분석이라니. 뭐가 나올까?"
- 김태현: "맞춤 처방전? 궁금하다."
- 박소영: "결과가 궁금해서 기다리는 중."

---

### US-2.4: 결과 화면 — 내 유형 + 레이더 차트 + 예상 낭비 금액

> 사용자로서, 내 소비 유형을 명확히 알고, 어디에 돈을 많이 쓰는지 한눈에 보고 싶다.

**수용 기준:**
- **상단**: 소비 유형 카드
  - 유형 아이콘 (SVG, 64x64)
  - 유형명: "카페 중독형" (font-weight: 700, 24px)
  - 한 줄 설명: "커피 없이 못 사는 당신, 라떼 한 잔이 모이면 해외여행이에요" (font-weight: 400, 14px)
- **중단**: 6각형 레이더 차트 (SVG)
  - 6개 축: 카페 / 배달 / 쇼핑 / 구독 / 교통 / 유흥
  - 차트 면적에 틸 그린(#0d9488) 반투명 fill
  - 각 꼭짓점에 점수 표시
  - 차트 등장: scale(0→1) 스프링 애니메이션 (0.5초)
- **하단**: 월 예상 낭비 금액
  - "월 예상 낭비 금액" 라벨 + "₩32만원" 큰 숫자 (font-weight: 700, 32px)
  - 카테고리별 세부 금액 표시 (접을 수 있는 아코디언)
- **CTA 2개**:
  - "7일 챌린지 시작하기" (primary 버튼)
  - "결과 공유하기" (secondary 버튼)

**안티패턴:**
- 결과가 숫자/텍스트만으로 표현 (시각적 임팩트 부족)
- 레이더 차트 없이 점수 나열 (한눈에 파악 불가)
- 낭비 금액이 비현실적으로 높아 신뢰 저하 (상한선 설정)

**페르소나 반응:**
- 최지은: "헐, 카페 중독형이래. 맞긴 하다... 월 15만원 낭비?"
- 김태현: "유흥 올인형 ㅋㅋㅋ 차트로 보니까 확실하네."
- 박소영: "구독이 제일 높다고? 12개니까... 맞네."

---

### US-2.5: 진단 결과 카카오톡 공유

> 사용자로서, 재밌는 결과를 친구에게 공유해서 같이 해보자고 하고 싶다.

**수용 기준:**
- "결과 공유하기" 버튼 탭 → Share SDK 호출
- 공유 텍스트:
  ```
  나의 소비 유형은 '카페 중독형'!
  ☕ 월 예상 낭비 금액: ₩15만원
  너도 진단받아봐 👉 [앱 링크]
  #소비체질개선 #소비유형테스트
  ```
- Share SDK 실패 시 → Clipboard SDK로 텍스트 복사 + "복사되었습니다" 토스트
- 공유 후 화면은 결과 화면 유지 (화면 이동 없음)
- Analytics 이벤트: `result_share` + `{ method: 'kakao' | 'native' | 'clipboard' }`

**안티패턴:**
- 공유 시 앱에서 나가버림 (토스 미니앱 특성 고려)
- 공유 텍스트가 너무 길어서 읽기 싫음 (3줄 이내)
- 링크 없이 텍스트만 공유 (바이럴 루프 끊김)

**페르소나 반응:**
- 최지은: "이거 재밌다. 친구한테 카톡으로 보내야지."
- 김태현: "우리 팀 슬랙에 올려야겠다 ㅋㅋ"
- 박소영: "인스타 스토리에 올리고 싶은데... 캡처해야겠다."

## 3. 챌린지 시작

### US-3.1: 진단 결과 기반 맞춤 챌린지 자동 추천

> 사용자로서, 내 소비 유형에 딱 맞는 챌린지를 추천받고 싶다.

**수용 기준:**
- 결과 화면에서 "7일 챌린지 시작하기" 탭 → ChallengeIntroScreen 진입
- 화면 구성:
  - 챌린지 타이틀: "7일 카페 절약 챌린지" (유형 기반)
  - 챌린지 규칙: "7일 동안 카페 방문을 3회 이하로 줄여보세요"
  - 예상 절약 금액: "이 챌린지를 완료하면 약 ₩35,000을 절약할 수 있어요"
  - 7일 미리보기 캘린더 (Day 1 ~ Day 7, 모두 빈 원)
- 자동 추천 로직: 진단 점수가 가장 높은 카테고리의 챌린지
- 선택 변경 불가 (MVP) — 추천된 챌린지만 시작 가능

**안티패턴:**
- 챌린지 목록에서 고르게 하기 (선택 과부하, 결정 피로)
- "커스텀 챌린지 만들기" (MVP 범위 초과)
- 예상 절약 금액 없이 챌린지만 제시 (동기 부여 약함)

**페르소나 반응:**
- 최지은: "카페 3회 이하? 할 수 있을까... 해보자."
- 김태현: "음주 1회 이하? 빡세다. 근데 10만원 절약이면..."
- 박소영: "구독 1개 해지? 그건 바로 할 수 있겠다."

---

### US-3.2: 챌린지 규칙 + 예상 절약 금액 미리보기

> 사용자로서, 챌린지를 시작하기 전에 정확히 뭘 해야 하는지, 얼마나 절약되는지 알고 싶다.

**수용 기준:**
- 챌린지 소개 화면 구성:
  - 규칙 카드: 아이콘 + 규칙 텍스트 (1줄)
  - "매일 한 번, 오늘 지켰는지 체크하세요" 안내
  - 예상 절약: "7일 × ₩5,000 = ₩35,000" 계산 과정 표시
  - "도전 시작!" CTA 버튼 (하단 고정)
- CTA 탭 시 → Storage에 챌린지 데이터 생성 + ChallengeScreen 진입

**안티패턴:**
- 규칙이 너무 상세해서 읽기 귀찮음 (1-2줄로 요약)
- 절약 금액이 비현실적으로 높음 (보수적 추정 사용)

**페르소나 반응:**
- 최지은: "하루에 5천원이면... 일주일이면 치킨 한 마리값이네."
- 김태현: "계산 과정이 보이니까 납득이 된다."
- 박소영: "도전 시작 누르면 바로 시작이지? 좋아."

---

### US-3.3: "도전하기" 탭 → 7일 카운트다운 시작

> 사용자로서, 도전 시작 후 D-Day가 카운트되어 긴장감을 느끼고 싶다.

**수용 기준:**
- "도전 시작!" 버튼 탭 시:
  - 현재 시각을 startedAt으로 Storage 저장
  - Haptic SDK: success 패턴 1회
  - ChallengeScreen으로 전환
- ChallengeScreen 구성:
  - 상단: "Day 1 / 7" + 챌린지 타이틀
  - 중단: 7일 캘린더 뷰 (수평 배열)
  - 하단: "오늘 체크인하기" 버튼 (당일에만 활성)
  - 누적 절약 금액 표시

**안티패턴:**
- 시작 후 별다른 피드백 없이 다음 화면 (성취감 부족)
- "정말 시작하시겠습니까?" 확인 다이얼로그 (마찰 추가)

**페르소나 반응:**
- 최지은: "Day 1! 진동이 울리니까 진짜 시작한 느낌."
- 김태현: "7일 캘린더가 보이니까 게임 같다."
- 박소영: "시작했으니 체크인해야지."

## 4. 매일 체크인 (Day 1-7)

### US-4.1: 앱 진입 시 "오늘 지켰나요?" 체크인

> 사용자로서, 앱을 열면 바로 오늘의 체크인을 할 수 있어야 한다.

**수용 기준:**
- 챌린지 진행 중 앱 재진입 시:
  - 아직 오늘 체크인 안 했으면 → ChallengeScreen + "오늘 체크인하기" 버튼 활성
  - 이미 오늘 체크인 했으면 → ChallengeScreen + 오늘 상태 표시 (성공/실패)
- "오늘 체크인하기" 탭 → CheckInScreen 진입
- CheckInScreen 구성:
  - 오늘 날짜 + "Day N" 표시
  - 챌린지 규칙 리마인드: "오늘 카페를 3회 이하로 갔나요?"
  - 두 개 버튼:
    - "지켰어요" (primary 컬러, 좌측)
    - "못 지켰어요" (회색, 우측)
- 날짜 판정: 자정(00:00) 기준 하루 1회 체크인

**안티패턴:**
- 체크인 없이 자동으로 날짜가 지남 (앱 열 이유 없음)
- 체크인 시간을 지정 (유연성 없음)
- "오늘 카페 몇 번 갔는지 숫자를 입력하세요" (입력 마찰)

**페르소나 반응:**
- 최지은: "앱 열면 바로 체크인이네. 간단하다."
- 김태현: "지켰어요/못 지켰어요 두 개만 있으니 빠르다."
- 박소영: "날마다 알아서 물어보는 게 좋다."

---

### US-4.2: 성공 체크인 — 스트릭 + 햅틱 + 절약 금액

> 사용자로서, "지켰어요"를 누르면 즉각적인 보상(피드백)을 느끼고 싶다.

**수용 기준:**
- "지켰어요" 탭 시:
  - Haptic SDK: success 패턴
  - 체크마크 SVG 드로잉 애니메이션 (stroke-dashoffset, 0.5초)
  - "잘했어요!" 메시지 (font-weight: 600)
  - 연속 성공 일수: "N일 연속 성공!" (연속일 경우)
  - 절약 금액 카운터 애니메이션: "+₩5,000" → 누적 금액 업데이트
  - 격려 메시지 (랜덤):
    - "대단해요! 이 페이스라면 7일 완주 거뜬!"
    - "오늘도 성공! 절약한 돈으로 뭐 할 거예요?"
    - "연속 N일! 이미 습관이 되고 있어요."
- 캘린더 뷰에 오늘 날짜 체크 아이콘 표시
- Storage 업데이트: checkIns[day] = true

**안티패턴:**
- 성공 후 별다른 피드백 없이 "확인" (성취감 없음)
- 과도한 축하 이펙트 (컨페티, 사운드 등 — 간결하게)
- 절약 금액이 사라지고 다시 볼 수 없음

**페르소나 반응:**
- 최지은: "진동 오니까 기분 좋다! 5천원 절약했다!"
- 김태현: "3일 연속이면 뭔가 깨기 싫어진다."
- 박소영: "누적 금액 보니까 동기부여 된다."

### US-4.3: 실패 체크인 — 격려 + "내일 다시"

> 사용자로서, "못 지켰어요"를 눌러도 비난받지 않고 내일 다시 도전할 동기를 받고 싶다.

**수용 기준:**
- "못 지켰어요" 탭 시:
  - Haptic SDK: warning 패턴 (가벼운 진동)
  - X 아이콘 표시 (빨간색, 하지만 크기 작게)
  - 격려 메시지 (랜덤):
    - "괜찮아요, 완벽하지 않아도 돼요."
    - "한 번 실패가 끝은 아니에요. 내일 다시!"
    - "습관은 하루아침에 안 바뀌어요. 도전 자체가 대단한 거예요."
  - 절약 금액 변동 없음 (감소시키지 않음)
  - 스트릭 리셋하지 않음 (총 성공 일수만 카운트)
- 캘린더 뷰에 오늘 날짜 X 아이콘 표시 (빨간색, 작게)
- Storage 업데이트: checkIns[day] = false

**안티패턴:**
- "실패했습니다" 같은 단정적 표현 (죄책감 유발)
- 실패 시 벌점, 패널티 (부정적 강화)
- 스트릭 리셋 (실패 1번에 모든 동기 상실)

**페르소나 반응:**
- 최지은: "못 지켰는데 괜찮다고 해줘서... 내일은 진짜 해야지."
- 김태현: "벌점 없는 게 좋다. 압박감 없이."
- 박소영: "도전 자체가 대단하대. 맞는 말이긴 해."

### US-4.4: 7일 캘린더 뷰로 진행 상황 한눈에

> 사용자로서, 7일 동안의 체크인 결과를 한눈에 보면서 전체 진행을 파악하고 싶다.

**수용 기준:**
- 수평 7칸 캘린더 (Day 1 ~ Day 7)
- 각 칸 상태:
  - 성공: 틸 그린 원 + 체크 아이콘 (#0d9488)
  - 실패: 회색 원 + X 아이콘 (#94a3b8)
  - 미완료: 빈 원, 점선 테두리 (#cbd5e1)
  - 오늘: 틸 그린 테두리 하이라이트 + 펄스 애니메이션
- 각 칸 아래에 "월", "화", "수"... 요일 표시
- 캘린더 위에 "Day N / 7" + 남은 일수 표시
- 캘린더 아래에 누적 절약 금액 + 성공 일수 요약

**안티패턴:**
- 세로 스크롤 필요한 캘린더 (한 화면에 다 보여야 함)
- 실패한 날이 빨간색으로 강조 (부정적 시각화)
- 날짜만 표시하고 요일 없음 (직관성 부족)

**페르소나 반응:**
- 최지은: "5개 중에 4개 체크했네. 하나 빠졌지만 괜찮아."
- 김태현: "캘린더로 보니까 게임 스테이지 같다."
- 박소영: "남은 2일! 거의 다 왔다."

## 5. 챌린지 완료

### US-5.1: 7일 완료 축하 + 총 절약 금액

> 사용자로서, 7일 챌린지를 완료하면 달성감과 구체적인 성과를 확인하고 싶다.

**수용 기준:**
- 7일차 체크인 완료 후 → ChallengeCompleteScreen 자동 전환
- 화면 구성:
  - "챌린지 완료!" 타이틀 (font-weight: 700, 28px)
  - 7일 캘린더 전체 결과 표시
  - "7일 중 N일 성공" (font-weight: 600)
  - "총 절약 금액: ₩XX,000" (큰 숫자, 카운트업 애니메이션)
  - 성공률에 따른 메시지:
    - 7/7: "완벽한 한 주! 당신은 진정한 절약 마스터!"
    - 5-6/7: "거의 완벽해요! 이 정도면 습관 변화 시작!"
    - 3-4/7: "절반 이상 성공! 대단한 첫 걸음이에요."
    - 1-2/7: "도전한 것 자체가 대단해요. 다음에 더 잘할 수 있어요."
    - 0/7: "괜찮아요, 인식하는 것이 변화의 시작이에요."
- Haptic SDK: notification 패턴 (축하)

**안티패턴:**
- 0/7 성공해도 "축하합니다!" (신뢰 저하)
- 컨페티, 폭죽 이펙트 과잉 (R7 준수)
- 완료 후 바로 "다시 시작" 유도 (성과 음미 시간 필요)

**페르소나 반응:**
- 최지은: "5일 성공에 2만 5천원 절약! 치킨 사먹자... 아니 저축!"
- 김태현: "7일 중 4일 성공. 반은 넘겼다. 다음에는 더."
- 박소영: "구독 1개 해지했으니 매달 만원씩 절약이네."

### US-5.2: Before/After 비교 카드

> 사용자로서, 챌린지 전후 변화를 시각적으로 비교해서 "정말 바뀌었구나"를 느끼고 싶다.

**수용 기준:**
- 챌린지 완료 화면 하단에 "변화 확인하기" 섹션
- "재진단하고 변화 확인" 버튼 → 12문항 재진단
- 재진단 완료 후 Before/After 카드 자동 생성:
  - 좌(Before) / 우(After) 레이아웃 또는 상하 비교
  - 메인 유형 카테고리 점수 변화: "카페 소비: 8점 → 5점 (-37.5%)"
  - 미니 레이더 차트 2개 (Before 연회색, After 틸 그린)
  - "예상 월 절약 금액: ₩12만원"
- 변화가 없거나 악화된 경우:
  - "아직 변화 전이에요. 한 번 더 도전하면 달라질 거예요."

**안티패턴:**
- 재진단 없이 자동으로 "개선되었습니다" (신뢰 상실)
- Before/After 차이가 너무 극적 (비현실적)
- 악화 시 부정적 표현 (격려 유지)

**페르소나 반응:**
- 최지은: "카페 8점에서 5점? 진짜 줄었다!"
- 김태현: "차트 두 개 비교하니까 확실히 보인다."
- 박소영: "월 12만원이면 1년에 144만원이네!"

### US-5.3: 결과 공유 → 카카오톡 바이럴

> 사용자로서, 챌린지 결과를 친구에게 자랑하고 같이 도전하자고 유도하고 싶다.

**수용 기준:**
- "결과 공유하기" 버튼 → Share SDK 호출
- 공유 텍스트:
  ```
  나는 '소비 체질 개선'으로 7일 챌린지를 완료했다!
  ☕ 카페 소비 37.5% 감소
  💰 예상 월 절약: ₩12만원
  너도 도전해봐 👉 [앱 링크]
  ```
- Share 실패 시 → Clipboard 복사 + 토스트
- 공유 후 결과 화면 유지

**안티패턴:**
- 공유 텍스트에 과도한 이모지 남발
- 공유 텍스트가 5줄 이상 (너무 길면 안 읽음)
- 링크 없이 텍스트만 (바이럴 끊김)

**페르소나 반응:**
- 최지은: "인스타 스토리에 올려야지! 37.5% 줄였다 자랑."
- 김태현: "팀 슬랙에 공유하면 다 같이 하겠다 ㅋㅋ"
- 박소영: "프리랜서 커뮤니티에 공유해야지."

### US-5.4: 새 챌린지 시작 또는 재진단

> 사용자로서, 하나의 챌린지를 완료한 후 다른 카테고리도 개선하고 싶다.

**수용 기준:**
- 챌린지 완료 화면 하단 CTA 2개:
  - "다음 챌린지 시작" → 두 번째로 높은 카테고리 챌린지 자동 추천 → ChallengeIntroScreen
  - "처음부터 재진단" → QuizScreen (12문항 다시)
- 이전 진단 결과는 previousDiagnosis로 Storage 보관 (Before/After용)
- 히스토리에 완료된 챌린지 기록 추가

**안티패턴:**
- 완료 후 갈 곳이 없음 (이탈)
- "챌린지 목록에서 선택" (선택 과부하)
- 이전 기록이 사라짐 (성장 추적 불가)

**페르소나 반응:**
- 최지은: "카페 다음은 배달이네? 해보자."
- 김태현: "유흥 다음은 택시. 대중교통 타보자."
- 박소영: "구독 다음은 카페. 하루 2잔을 1잔으로."

## 6. 재방문 루프

### US-6.1: 챌린지 중 앱 재진입 시 현재 상태 즉시 표시

> 사용자로서, 앱을 다시 열면 내가 챌린지 어디에 있는지 바로 알고 싶다.

**수용 기준:**
- 앱 재진입 시 분기:
  - 진행 중 챌린지 있음 + 오늘 체크인 안 함 → ChallengeScreen (체크인 유도)
  - 진행 중 챌린지 있음 + 오늘 체크인 완료 → ChallengeScreen (진행 상태 표시)
  - 완료된 챌린지 + 새 챌린지 없음 → ChallengeCompleteScreen 또는 IntroScreen
  - 데이터 없음 → IntroScreen
- 로딩 시간: Storage 읽기 포함 500ms 이내

**안티패턴:**
- 매번 IntroScreen부터 시작 (진행 중인 챌린지 무시)
- "어디까지 했더라?" 사용자가 찾아야 함

**페르소나 반응:**
- 최지은: "앱 열면 바로 체크인 화면이네. 편하다."
- 김태현: "3일차라고 알려주니까 바로 알겠다."
- 박소영: "데이터가 저장되어 있으니 안심."

---

### US-6.2: 챌린지 히스토리 확인

> 사용자로서, 과거에 완료한 챌린지 기록을 보면서 성장을 추적하고 싶다.

**수용 기준:**
- HistoryScreen 진입 경로: ChallengeScreen 또는 ChallengeCompleteScreen에서 "기록 보기" 탭
- 화면 구성:
  - 완료된 챌린지 카드 목록 (최근순)
  - 각 카드: 챌린지 유형 + 기간 + 성공 일수 + 절약 금액
  - 총 누적 절약 금액 (상단 요약)
- 최대 10개 히스토리 보관 (Storage 용량 고려)

**안티패턴:**
- 히스토리가 없으면 빈 화면 (안내 메시지 필요)
- 과거 기록 삭제 기능 (사용자가 실수로 삭제)

**페르소나 반응:**
- 최지은: "3개 챌린지 완료! 총 15만원 절약했네."
- 김태현: "기록이 쌓이니까 게임 업적 같다."
- 박소영: "지난 달 대비 얼마나 성장했는지 보인다."

## 7. 엣지 케이스 및 에러 처리

### US-7.1: 체크인 날짜 건너뜀 (앱 미방문)

> 사용자로서, 하루 체크인을 빠뜨려도 챌린지가 종료되지 않아야 한다.

**수용 기준:**
- 체크인하지 않은 날은 자동으로 "미완료" 처리 (실패와 다름)
- 다음 날 앱 열면 밀린 체크인 없이 오늘만 체크인
- 7일 지나면 체크인한 날만 기준으로 성공률 계산
- 캘린더에 미완료 날짜는 빈 원(점선)으로 표시

**안티패턴:**
- 하루 빠뜨리면 챌린지 실패 처리 (너무 엄격)
- 밀린 체크인 소급 입력 (신뢰성 저하)

---

### US-7.2: Storage 데이터 손실

> 사용자로서, 데이터가 사라져도 앱이 깨지지 않아야 한다.

**수용 기준:**
- Storage 읽기 실패 시 → IntroScreen 표시 (새 사용자 플로우)
- 데이터 파싱 에러 시 → 데이터 초기화 + IntroScreen
- "데이터가 초기화되었습니다. 다시 진단해보세요." 안내 메시지

**안티패턴:**
- 에러 시 빈 화면 또는 크래시
- 기술적 에러 메시지 노출 ("JSON parse error")

---

### US-7.3: 진단 중 이탈 후 재진입

> 사용자로서, 진단 도중 앱을 나갔다가 돌아와도 처음부터 다시 하지 않아야 한다.

**수용 기준:**
- 진단 중간 답변은 메모리(state)에만 보관 (Storage 저장 안 함)
- 앱 재진입 시 진단 미완료 → IntroScreen부터 다시 (12문항은 2분 내 완료 가능하므로 재시작 부담 없음)
- 단, 챌린지 진행 중이면 챌린지 화면으로 복귀

**안티패턴:**
- 진단 중간 저장 (복잡성 증가 대비 가치 낮음)
- 재진입 시 "이어서 하시겠습니까?" 다이얼로그 (2분짜리 테스트에 불필요)
