## 화면별 피드백

### 1. 오늘 (메인 화면) - 빈 상태

**좋아진 점**
- 하단 미니 내비("오늘" + "기록")가 투명하고 미니멀하게 자리잡음. 화면 하단에 가볍게 붙어 있어서 콘텐츠 영역을 침범하지 않으면서도 내비게이션 가능함을 알려줌. 기존에는 설정 아이콘(톱니바퀴)만 있어서 "기록 화면은 어디서 가나?" 하는 의문이 있었는데, 이제 탭 하나로 해결됨
- 스크롤 힌트 32px fade가 확실히 효과적. 카드 영역 오른쪽 끝에 자연스러운 그라데이션이 보여서 "더 있다"는 느낌이 바로 듬. 이전 8px는 사실상 없는 것과 같았는데 지금은 적절함
- "이걸로 적기" 버튼이 filled primary(진한 갈색 배경 + 흰색 텍스트)로 변경되어, 오버레이를 열었을 때 시선이 바로 CTA로 감. 이전의 outline 버튼은 subtext와 시각적 무게가 비슷해서 CTA인지 인지하기 어려웠음

**남은 개선점**
- 메인 화면 하반부가 비어 있음. 질문 + 카드 + 입력창이 상단 40%에 몰려 있고, 나머지 60%가 텅 빈 공간. 모바일 한 손 조작 시 엄지 도달 범위(화면 하단 1/3)에 인터랙션 요소가 전혀 없음. 하단 내비가 생겼지만, 핵심 입력 영역이 너무 위에 있어서 한 손으로 쓰기가 여전히 불편
- 질문("한번도 안 해본 거.") 아래 큐레이션 카드까지의 간격이 넓음. 질문과 카드 사이에 불필요한 여백이 있어서 "질문에 대한 힌트"라는 관계가 시각적으로 약함

### 2. 오늘 (저장 후 상태)

**좋아진 점**
- 저장 후 화면이 깔끔하게 전환됨. 큐레이션 카드가 사라지고, 오늘의 기록 카드("산책 30분" + 시간 + 연필 아이콘)가 표시됨. 하루에 하나만 적는다는 앱 컨셉이 UI에서 명확하게 드러남
- **"첫 번째 한 줄."** 누적 통계 문구가 감성적. 숫자("1일째")가 아니라 "첫 번째 한 줄"이라는 표현이 기록의 시작을 축하하는 느낌. 사용이 쌓이면 "스물다섯 번째 한 줄." 같은 형태가 될 텐데, 단순한 카운터보다 정서적 울림이 있음
- "내일 또 다른 질문이 기다려요." 리텐션 메시지가 여전히 효과적. 저장 완료 -> 다음 날 기대감으로 자연스럽게 연결됨
- 토스트("적었다.")가 표시됨. 저장 성공에 대한 피드백이 존재함

**남은 개선점**
- 토스트("적었다.")가 여전히 짧게 보임. 이전에도 지적했듯, 1.5~2초는 유지되어야 인지 가능. 현재는 눈을 깜빡이면 놓칠 수 있는 속도
- "이번 주 N번째 기록" 통계는 기록 화면의 주간 요약("7일 중 1일")과 표현이 다름. "첫 번째 한 줄"(메인) vs "7일 중 1일"(기록) -- 같은 정보를 다른 톤으로 보여주는데, 메인의 감성적 톤이 더 좋음. 기록 화면도 맞추면 일관성이 생길 것

### 3. 수정 플로우

**확인된 동작**
- 저장된 기록 카드의 연필 아이콘을 탭하면 인라인 수정 모드로 전환됨. 텍스트가 입력창에 채워지고, "수정" 버튼(filled primary)과 "취소" 링크가 표시됨
- 글자수(6/100)도 함께 표시되어 수정 시에도 제한을 인지할 수 있음
- "수정" 버튼이 filled primary로 변경되어 "적기" 버튼과 일관됨

**남은 개선점**
- 수정 모드에서 질문은 그대로 보이는데 큐레이션 카드는 사라짐. 수정할 때 다른 힌트를 참고하고 싶을 수 있는데, 카드가 없으면 "산책 30분"을 뭘로 바꿀지 영감이 없음. 큐레이션 카드를 수정 모드에서도 노출하거나, 최소한 원래 선택했던 카드를 표시해주면 좋겠음

### 4. 기록 화면

**좋아진 점**
- 하단 내비에서 "기록" 탭을 탭하면 바로 기록 화면으로 전환됨. 이전에는 설정 옆에 숨어 있어서 진입이 불분명했는데, 탭 내비로 발견성이 크게 개선됨
- 주간 요약 카드("이번 주 / 7일 중 1일 - 산책 30분")가 상단에 있어서 한 눈에 이번 주 현황 파악 가능
- 날짜별 타임라인 구조("4월 2일 목요일" + 수직선 + 기록 카드)가 시각적으로 깔끔함
- 기록 카드에 질문("한번도 안 해본 거.")과 시간(10:48)이 함께 표시되어, 어떤 질문에 어떤 답을 적었는지 맥락이 완성됨
- "..." 메뉴가 여전히 잘 동작함

**남은 개선점**
- 기록이 1개뿐일 때 화면의 70%가 빈 공간. "기록이 쌓이면 여기에 채워질 거예요" 같은 위트 있는 빈 공간 처리가 있으면 좋겠음
- 기록 아이템에 카테고리 태그(만들기/움직이기 등)가 여전히 없음. 기록이 쌓이면 "이번 달은 무슨 카테고리가 많았나" 인사이트를 줄 수 있는 기반 데이터인데 아쉬움

### 5. 설정 화면

**미확인**
- 테스트 도중 브라우저 자동화 도구 한계로 설정 화면 진입에 실패함. 톱니바퀴 아이콘을 탭하면 수정 모드가 활성화되는 등 정확한 히트 영역 확인이 어려웠음
- 알림 description이 추가되었다는 변경사항은 코드 레벨에서 확인이 필요

## 전체 플로우 피드백

**엔트리 작성 플로우 (안정적):**  
메인 진입 -> 큐레이션 카드 탭 -> 오버레이에서 "이걸로 적기"(filled primary) 탭 -> 입력창 자동 채움 -> "적기" 탭 -> 저장 -> "적었다." 토스트 + "첫 번째 한 줄." 누적 통계 + 리텐션 메시지

- 플로우가 매끄럽게 이어짐. "이걸로 적기" 버튼이 filled로 변경되면서 오버레이에서의 다음 행동이 명확해짐
- 저장 후 누적 통계가 보여서 "기록의 의미"가 부여됨. 이전에는 저장하면 그냥 끝이었는데, 이제 "내가 쌓아온 것"의 감각이 생김

**내비게이션 플로우 (신규):**  
오늘 탭 <-> 기록 탭 (하단 내비)

- 2탭 구조가 앱의 두 가지 핵심 기능(오늘 적기 / 쌓인 기록 보기)을 정확히 반영함
- 투명하고 미니멀한 디자인이라 콘텐츠 영역을 해치지 않음
- 다만 설정 진입이 여전히 톱니바퀴 아이콘으로만 가능한데, 화면 구성상 톱니바퀴가 상단 우측에만 있어서 설정에 대한 시각적 무게가 적절함

## 개선 제안

### P0 (즉시 수정 필요)

없음. 핵심 플로우에 블로킹 이슈 없음.

### P1 (다음 반복에 수정)

1. **입력 영역을 화면 하단으로 이동 검토**  
   현재 질문 + 카드 + 입력창이 상단에 몰려 있고, 하단 60%가 빈 공간. 모바일 한 손 조작 시 엄지가 닿는 하단 영역에 입력창을 배치하면 사용성이 크게 개선됨. 질문은 상단, 입력은 하단으로 분리하거나, 전체 레이아웃을 하단 정렬로 조정하는 방안. 이건 큰 구조 변경이므로 신중하게.

2. **알림 설정에 퇴근 시간 선택기 추가**  
   3회 연속 지적. 알림 description이 추가된 것은 좋지만, 실제로 "몇 시에 알림 받을지" 선택할 수 없으면 의미가 반감됨. 앱의 핵심 가치("퇴근 후 루틴")와 직결되는 기능.

3. **토스트("적었다.") 노출 시간 연장 (2초 이상)**  
   2회 연속 지적. 현재 너무 빠르게 사라져서 저장 확인 피드백으로서의 역할이 약함.

### P2 (여유 있을 때)

4. **기록 아이템에 카테고리 태그 표시**  
   2회 연속 지적. 큐레이션을 통해 적은 경우 카테고리("만들기", "움직이기")를 기록 카드에 작은 태그로 표시. 기록이 쌓이면 패턴 인사이트의 기반이 됨.

5. **기록 화면 빈 공간 활용**  
   기록이 적을 때 빈 영역에 가벼운 격려 메시지나 일러스트를 넣으면 앱의 톤을 강화할 수 있음.

6. **연속 기록 보상 요소**  
   3일 연속 기록 시 작은 축하 메시지, 주간 완주 시 요약 카드 등. 현재 "첫 번째 한 줄."이 좋은 시작인데, 이 방향을 확장하면 리텐션에 도움.

## 만족도 점수

**74 / 100** (이전 52 -> 68 -> +6)

### 근거

**컨셉 (8/10, 유지):** 변함없이 좋은 컨셉. 퇴근 후 한 줄 기록이라는 핵심 아이디어가 명확함.

**첫인상 (8/10, 유지):** 이전과 동일하게 깔끔. 하단 내비가 추가되었지만 미니멀해서 첫인상을 해치지 않음.

**핵심 플로우 완성도 (8/10, +1):** "이걸로 적기" 버튼의 filled 스타일 변경으로 CTA 인지가 개선됨. 저장 후 누적 통계("첫 번째 한 줄.")가 추가되어 기록 행위에 의미 부여. 스크롤 힌트 32px로 4번째 카드 인지 가능. 핵심 플로우에서 걸리는 부분이 거의 없어짐.

**편의성 (7/10, 유지):** 하단 내비로 오늘/기록 전환이 쉬워짐. 하지만 입력 영역이 상단에 몰려 있어서 한 손 조작 편의성은 아직 아쉬움. 설정 진입도 자연스럽지만 알림 시간 설정이 없어서 실제 유틸리티는 부족.

**반복 사용 동기 (6/10, +1):** "첫 번째 한 줄." 같은 누적 통계가 감성적이고, "이번 주 N번째"가 기록 화면에도 표시되어 연속성이 생김. 하지만 알림 시간 설정, 연속 기록 보상 등 능동적 리텐션 장치가 아직 부족. 앱을 스스로 열게 만드는 계기(push notification + 타이밍)가 핵심인데 이 부분이 여전히 빈칸.

**종합:** Iteration 3은 마이크로 UX 디테일의 반복. P0 이슈 없이, 스크롤 힌트/CTA 시각 강화/누적 통계/하단 내비 등 이전 피드백의 P1을 착실히 반영함. 핵심 플로우의 완성도가 한 단계 올라갔고, 특히 "첫 번째 한 줄." 같은 감성적 표현이 앱의 톤에 잘 맞음. +6점은 "큰 문제 해결"이 아닌 "전반적 품질 향상"에서 온 점수. 다음 반복에서는 리텐션(알림 시간 설정, 연속 보상)과 레이아웃(입력 영역 위치)에 집중하면 80점 돌파 가능.
