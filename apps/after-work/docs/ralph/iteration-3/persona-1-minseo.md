# Iteration 3 피드백 — 민서 (28세, 대기업 마케터)

> 테스트 환경: http://localhost:5191/ / 375x812 viewport
> 이전 점수: 52% -> 64% -> **이번 평가: 72%**

---

## 1. 홈 화면 (오늘 — 미작성 상태)

**잘 된 점**
- "오늘의 질문" 레이블이 맥락을 잡아줘서 좋다. 처음 들어왔을 때 "뭘 해야 하지?"가 아니라 "오, 질문이 있네"로 자연스럽게 시작됨.
- 큐레이션 카드 3~4장이 가로 스크롤로 보이고, 카테고리(만들기/보기듣기/움직이기/배우기) + 구체적 활동명("레고 조립", "라디오 듣기")이 명확. 영감을 준다.
- 스크롤 힌트(scrollFade) 32px로 확대 — 이전 8px보다 확실히 "더 있구나"가 느껴진다. 개선 체감됨.
- 입력 필드 "한 줄 적기..." placeholder가 부담 없고 좋다.
- "적기" 버튼 비활성 상태에서 텍스트 입력 시 즉시 활성화 — 반응 좋음.

**아쉬운 점**
- 큐레이션 카드 클릭 -> 상세 모달 -> "이걸로 적기" 흐름은 좋은데, 모달에서 돌아온 뒤 입력 필드에 자동 포커스가 안 잡힌다. "이걸로 적기" 누르면 텍스트가 채워지지만 커서가 입력 필드에 없어서 바로 수정/추가가 어렵다.
- 큐레이션 카드 영역과 입력 영역 사이 간격이 너무 없다. 카드 바로 아래 input이 붙어있어서 시각적으로 답답하다.
- 질문("한번도 안 해본 거.")이 매일 바뀐다는 걸 신규 유저는 알 수 없다. "매일 새 질문" 같은 미세한 힌트가 있으면 기대감이 생길 텐데.

## 2. 홈 화면 (오늘 — 작성 완료 상태)

**잘 된 점**
- 작성 후 카드 형태로 내가 쓴 내용 + 시간 + 연필 아이콘 표시 — 깔끔하다.
- "첫 번째 한 줄." / "이번 주 N번째 기록." 같은 누적 피드백 메시지가 뿌듯하다. Iteration 2에 없던 건데, 이게 꽤 큰 리텐션 요소가 될 수 있다.
- "내일 또 다른 질문이 기다려요." 리텐션 메시지 — 자연스럽고 부담 없다.
- 작성 후에는 큐레이션 카드가 숨겨져서 화면이 깔끔하다.

**아쉬운 점**
- 작성 완료 후 토스트 "적었다."가 하단에 뜨는데, 매우 짧게 사라진다. 조금 더 머물러도 좋겠다.
- 수정 모드 진입 시 "수정" 버튼 + "취소" 버튼이 나오는데, 취소의 위치가 카운터(9/100) 옆이라 좀 찾기 어렵다. 좀 더 명확한 위치가 필요.
- 작성 완료 상태에서 아래쪽 절반이 완전히 비어있다. "첫 번째 한 줄." 메시지 아래가 텅 비어 보인다.

## 3. 기록 페이지

**잘 된 점**
- 하단 네비게이션으로 "기록" 탭 접근 — 자연스러운 동선.
- 빈 상태 "아직 없어요." + "첫 한 줄은 오늘 화면에서." CTA 링크 — 명확하고 행동 유도가 잘 된다.
- 뒤로가기 버튼 (화살표) + "기록" 타이틀 — 네비게이션 구조가 명확.

**아쉬운 점**
- 기록이 있을 때의 리스트 뷰를 확인 못 했지만, 코드를 보면 날짜별 그루핑 + WeeklyCard가 있어서 구조는 괜찮아 보인다.
- 빈 상태에서 "아직 없어요."가 화면 중앙보다 살짝 위에 있어서 시각적 무게감이 가벼움. 더 중앙에 있으면 좋겠다.

## 4. 설정 페이지

**잘 된 점**
- "알림 받기" 토글에 "매일 이 시간에 한 줄 쓰기 알림" 설명 추가 — 이전에는 알림이 뭔지 몰랐는데 이제 목적이 명확.
- 섹션 분리(알림/데이터/정보)가 깔끔하다.
- "모든 데이터 삭제"가 빨간색으로 위험 액션 표시 — 적절.

**아쉬운 점**
- 알림 시간을 설정하는 UI가 없다. "매일 이 시간에"라고 써 있는데 "이 시간"이 언제인지, 바꿀 수 있는지 알 수 없다.
- "데이터 내보내기"가 어떤 포맷으로 나오는지 안내가 없다.

## 5. 하단 네비게이션

**잘 된 점**
- 홈(오늘) + 기록 2탭 — 미니멀하고 이 앱에 적합한 구조.
- 투명 배경으로 앱 전체 톤과 어울림.
- 활성 탭 색상 구분 (primary color) — 알아보기 쉽다.

**아쉬운 점**
- 48px 높이에 아이콘+텍스트가 왼쪽 정렬되어 있다. 일반적인 탭바는 중앙 정렬 + 균등 분배인데, 이 앱은 왼쪽에 몰려 있어서 오른쪽이 텅 빈다. 의도적 디자인인지 모르겠지만 어색하다.
- safe-area-bottom padding이 없어서, 홈 인디케이터가 있는 기기에서는 네비게이션이 잘릴 수 있다.

## 개선 제안

### P0 (필수)
1. **하단 네비게이션 레이아웃**: 탭 2개를 `justify-content: space-around` 또는 `space-evenly`로 균등 배치. 현재 왼쪽 몰림이 UX 관례에서 벗어남.
2. **safe-area-bottom 대응**: 하단 네비 + 콘텐츠 영역에 `padding-bottom: env(safe-area-inset-bottom)` 적용. iPhone에서 홈 인디케이터와 겹칠 위험.
3. **알림 시간 설정 UI**: "매일 이 시간에"라고 해놓고 시간 변경 수단이 없으면 유저 혼란. 시간 선택기 또는 최소한 기본값 표시 필요.

### P1 (강력 권장)
4. **"이걸로 적기" 후 입력 필드 자동 포커스**: 큐레이션 카드에서 텍스트를 가져온 뒤 커서가 입력 필드 끝에 위치해야 바로 수정 가능.
5. **작성 완료 후 빈 공간 활용**: 하단 절반이 비어 있다. 간단한 일러스트, 또는 "비슷한 답변" 같은 부가 콘텐츠로 채우면 완성도 상승.
6. **큐레이션 카드와 입력 필드 사이 간격 추가**: 최소 16px spacing 추가로 시각적 여유 확보.

### P2 (있으면 좋은 것)
7. **"매일 새 질문이 와요" 온보딩 힌트**: 최초 1회만 질문 아래 작은 텍스트로 안내하면 재방문 동기 부여.
8. **토스트 표시 시간 연장**: "적었다." 토스트가 너무 빨리 사라짐. 2~3초 유지 권장.
9. **수정 모드 "취소" 버튼 위치**: 입력 필드 아래 오른쪽이 아니라 "수정" 버튼 옆 또는 상단에 배치해서 발견성 향상.
10. **데이터 내보내기 포맷 안내**: "JSON으로 저장됩니다" 같은 한 줄 설명 추가.

**52% -> 64% -> 72% 변화 요약:**
- Iteration 2에서 추가된 "오늘의 질문" 레이블, 카드 스타일, 색상 대비는 그대로 유효하고 기반이 탄탄함.
- Iteration 3의 **누적 피드백 메시지**("첫 번째 한 줄.", "이번 주 N번째")가 이번 가장 임팩트 있는 개선. 마케터로서 이런 소소한 성취감 피드백이 재방문 동기의 핵심이라고 봄.
- **하단 네비게이션** 추가로 기록 탐색 접근성이 생겼지만, 레이아웃 품질이 아직 부족해서 점수 상승폭을 깎음.
- **"이걸로 적기" 버튼 filled primary** 강화는 확실히 눈에 잘 들어옴. 모달 내 CTA로서 적절.
- 80%+ 도달하려면: 네비게이션 레이아웃 정리 + 알림 시간 설정 + 작성 완료 후 빈 공간 활용이 필요.
