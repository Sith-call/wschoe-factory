## 화면별 피드백

### 1. 오늘 (메인 화면)

**좋은 점**
- 첫 화면이 심플하다. 퇴근하고 지친 상태에서 정보량이 적어서 좋음
- "한번도 안 해본 거." 같은 프롬프트가 매일 바뀌는 건 괜찮은 아이디어
- 큐레이션 카드(만들기/보기듣기/움직이기/배우기)가 뭘 쓸지 힌트를 줘서 빈 화면 앞에서 고민하는 시간을 줄여줌
- 100자 제한과 카운터가 보여서 부담 없이 짧게 쓸 수 있다는 느낌
- "적기" 버튼 네이밍이 가볍고 좋음

**개선점**
- **4번째 큐레이션 카드("배우기/위키 서핑")가 375px 뷰포트에서 잘려서 안 보임.** 스크롤 가능하다는 어포던스(affordance)가 전혀 없음. 카드가 3개만 있는 줄 알았음. 살짝 잘리거나 스크롤 인디케이터가 필요
- 적은 뒤에 메인 화면으로 돌아오면 오늘 기록 카드만 딸랑 하나 보임. **다시 적기/수정하기 진입점이 보이지 않음.** 카드를 탭하면 수정 모드로 들어간다는 걸 코드를 봐서 알았지, UI만 보면 절대 모름. 탭 가능하다는 힌트(연필 아이콘, "수정" 텍스트 등)가 필요
- 저장 후 토스트("적었다.")가 뜨는 것 같은데, 테스트 중 너무 빨리 사라져서 확인 어려움
- 큐레이션 카드를 탭하면 오버레이로 상세(subtext)가 뜨는데, **그 내용을 입력창에 자동으로 채워주거나 복사하는 기능이 없음.** 그냥 보여주기만 하면 탭할 이유가 없음. "이거 할래" 버튼이 있어야 함

### 2. 기록 화면

**좋은 점**
- 이번 주 카드가 접히고 펼쳐지는 게 깔끔함. 요일별로 한 줄씩 보여주는 건 좋은 패턴
- "7일 중 1일" 요약이 동기부여가 됨
- 날짜 구분선("4월 2일 목요일")이 읽기 편함
- 기록 아이템에 프롬프트 텍스트가 함께 표시돼서 맥락을 알 수 있음
- 빈 상태("아직 없어요. 첫 한 줄은 오늘 화면에서.")가 자연스러움

**개선점**
- **기록 아이템의 수정/삭제 방법을 전혀 알 수 없음.** 탭하면 수정, 롱프레스하면 삭제 확인이 뜨는 구조인데, 이런 제스처 기반 인터랙션은 발견성(discoverability)이 0에 가까움. 개발자인 나도 코드를 안 봤으면 몰랐을 것
- 주간 카드 펼침 상태에서 빈 요일은 "---"(em dash)만 표시되는데, 뭔가 우울함. "아직" 같은 가벼운 텍스트가 나을 수 있음
- 기록이 많아지면 무한 스크롤인지 페이지네이션인지 불명확. 주 단위로 그루핑되는 것도 아니고 날짜만 나열됨

### 3. 설정 화면

**좋은 점**
- 미니멀하게 필요한 것만 있음. 알림, 데이터 관리, 버전 정보
- "모든 데이터 삭제"가 빨간색으로 위험한 동작임을 알려줌
- 섹션 구분(알림/데이터/정보)이 명확

**개선점**
- **알림 받기 토글이 있지만 실제로 뭘 알림해주는지 설명이 없음.** 몇 시에? 무슨 내용으로? "퇴근 시간에 한 줄 쓰기 리마인더" 같은 설명이 필요
- "데이터 내보내기"를 눌렀을 때 어떤 포맷으로 나오는지 모르겠음 (JSON? CSV? 텍스트?)
- 퇴근 시간 설정이 없음. 앱 컨셉이 "퇴근하면 뭐하지?"인데, 내 퇴근 시간(8-9시)에 맞춰 알림을 보내줄 수 있어야 의미 있음

## 전체 플로우 피드백

**엔트리 작성 플로우:**  
메인 진입 -> 프롬프트 확인 -> 큐레이션 카드 탭(선택) -> 텍스트 입력 -> "적기" 탭 -> 저장됨

- 2탭이면 끝나서 나쁘지 않음 (입력 + 적기)
- 단, 큐레이션 카드를 탭해도 입력에 반영이 안 되는 게 아쉬움

**기록 확인 플로우:**  
메인 -> 기록 아이콘 탭 -> 기록 화면 진입

- 1탭이면 돼서 좋음
- 주간 카드 확인도 1탭 추가

**수정 플로우 (메인에서):**  
카드 탭 -> 수정 모드 -> 텍스트 수정 -> "수정" 탭

- 카드가 탭 가능하다는 걸 모르면 영원히 수정 못 함
- 이건 UX 결함

## 개선 제안

### P0 (즉시 수정 필요)

1. **큐레이션 카드 → 입력 연동 부재**  
   큐레이션 카드를 탭해서 상세를 본 뒤 "이거 할래" 또는 "입력에 넣기" 같은 액션이 없음. 현재는 탭해서 subtext를 보기만 하는데, 그러면 카드의 존재 이유가 없음. 카드를 탭하면 해당 활동 텍스트가 입력창에 자동으로 채워지거나, 상세 오버레이에서 "이걸로 적기" 버튼을 제공해야 함.

2. **오늘 기록 카드의 수정 진입점 부재**  
   저장 후 보이는 카드가 탭 가능하다는 시각적 힌트가 전혀 없음. 연필 아이콘이나 "탭해서 수정" 텍스트를 추가해야 함. 기록 화면의 롱프레스 삭제도 마찬가지로 발견 불가능.

### P1 (다음 반복에 수정)

3. **4번째 큐레이션 카드 잘림 문제**  
   375px에서 카드 4개 중 마지막이 완전히 잘림. 수평 스크롤 가능 여부를 알 수 없음. 마지막 카드가 살짝 보이도록 패딩을 조정하거나, 스크롤 힌트(페이드아웃, 점 인디케이터)를 추가.

4. **알림 설정에 퇴근 시간/알림 시각 커스터마이징 없음**  
   앱의 핵심 컨셉이 퇴근 후인데, 내 퇴근 시간이 언제인지, 알림을 몇 시에 받을지 설정할 수 없음. "알림 받기" 토글 아래에 시간 선택기와 간단한 설명("매일 이 시간에 한 줄 쓰기 알림을 보내드려요")이 필요.

### P2 (여유 있을 때)

5. **기록 화면에서 주 단위 네비게이션 없음**  
   현재는 전체 기록이 날짜순으로만 나열됨. 지난주/지지난주를 넘겨볼 수 있는 UI가 없음. 기록이 쌓이면 스크롤만으로는 불편할 것.

### 근거

**컨셉 (8/10):** "퇴근하면 뭐하지?"라는 질문 자체는 공감됨. 매일 한 줄만 쓰면 된다는 가벼움도 좋음. 개발자 입장에서 코딩 외에 뭘 했는지 기록하는 건 의미 있음.

**첫인상 (7/10):** 화면이 깔끔하고 정보가 적어서 피곤한 상태에서 부담 없음. 베이지 톤 배경이 편안함.

**핵심 플로우 완성도 (4/10):** 큐레이션 카드가 인터랙션 데드엔드(탭해도 입력 반영 안 됨). 수정/삭제 방법을 찾을 수 없음(숨겨진 제스처). 이건 MVP라도 기본이 안 된 것.

**편의성 (5/10):** 입력 → 저장은 빠르지만, 수정이 사실상 불가능에 가까운 UX. 피곤한 상태에서 "이거 어떻게 수정하지?" 하고 포기할 것.

**반복 사용 동기 (4/10):** 알림 설정이 실질적으로 비어 있고, 기록이 쌓였을 때의 보람(주간 리뷰, 월간 요약 등)이 아직 없음. 3일 쓰고 잊어버릴 가능성이 높음.

**종합:** 아이디어와 비주얼은 괜찮지만, 핵심 인터랙션(큐레이션 → 입력, 수정, 알림)이 미완성. 퇴근하고 지친 사람이 "이거 어떻게 쓰는 거야?" 하고 2초 고민하면 그냥 넷플릭스 켬.
