## 화면별 피드백

### 1. 오늘 (메인 화면)

**본 것:**
- 상단에 오늘의 프롬프트 "한번도 안 해본 거." 가 크게 표시됨
- 그 아래 큐레이션 칩 4개 (만들기/레고 조립, 보기듣기/라디오 듣기, 움직이기/계단 오르기, 배우기/위키 서핑)
- 하단에 "한 줄 적기..." 입력창 + "적기" 버튼 + 글자수 카운터 (0/100)

**좋은 점:**
- 프롬프트가 짧고 힘이 있다. "한번도 안 해본 거." 이 한 줄이 생각을 자극한다. 퇴근 후 멍한 상태에서 이 정도 자극은 적절하다.
- 큐레이션 칩이 "만들기 / 레고 조립" 처럼 카테고리+구체적 활동으로 나뉜 것이 좋다. 무엇을 할지 모르는 사람에게 힌트를 준다.
- 배경색이 따뜻한 베이지 톤이라 퇴근 후 눈이 편하다. 원색이었으면 바로 닫았을 것.
- UI가 최소한이라 에너지 소모가 적다. 피곤한 사람한테 중요한 포인트.

**개선점:**
- 큐레이션 칩이 가로 스크롤인데, 4번째 칩("배우기/위키 서핑")이 화면에 잘려서 보이지 않는다. 스크롤 가능하다는 힌트가 전혀 없다. 3개만 보이면 3개만 있는 줄 안다.
- 칩을 클릭하면 뭐가 되는 건지 모르겠다. 클릭했더니 오버레이로 subtext가 나온다는데, 실 테스트에서는 잘 작동하지 않았다. 칩의 역할이 불명확하다 -- "이걸 누르면 뭐가 되는 거지?"
- 프롬프트가 매일 바뀐다는 걸 모른다. 아무 설명도 없다. 내일 다시 들어와야 한다는 동기가 없다.
- 입력창이 화면 중간 아래에 있는데, 존재감이 약하다. "한 줄 적기..."라는 placeholder가 충분히 동기부여가 되지 않는다. 퇴근한 사람한테 "적어"라고 하면 "뭘 적으라고?"라는 반응이 먼저 나온다.
- 글을 저장한 뒤 큐레이션 칩이 사라진다. 다시 보려면? 방법이 없다. 한번 적으면 오늘의 추천을 다시 볼 수 없다.

### 2. 기록 화면

**본 것:**
- 상단에 "기록" 헤더 + 뒤로가기 버튼
- "이번 주" 위클리 카드 (접힌 상태: "7일 중 1일 / 퇴근하고 치킨...")
- 날짜별 기록 ("4월 2일 목요일")
- 각 기록에 텍스트 + 프롬프트 태그 + 시간 표시

**좋은 점:**
- 위클리 카드의 "7일 중 1일" 표시가 간결하다. 일주일 단위로 얼마나 적었는지 한눈에 보인다.
- 날짜 구분선이 "4월 2일 목요일"처럼 자연스럽다.
- 기록 아이템에 프롬프트 태그("한번도 안 해본 거.")가 붙어 있어서 맥락이 보존된다.

**개선점:**
- 기록이 1개뿐인 상태에서 화면이 너무 허전하다. "7일 중 1일"이라고 하면 6일 빠졌다는 죄책감이 든다. 처음 쓰는 사람한테 "잘 시작했다"는 격려가 필요하다.
- 위클리 카드를 클릭해서 펼칠 수 있는지 불분명하다. 시각적으로 "이건 눌러서 펼쳐볼 수 있어"라는 어포던스가 없다. 쉐브론(v) 아이콘이 있긴 한데 너무 작다.
- 기록 수정/삭제 기능이 보이지 않는다. 오타를 고치고 싶으면? 소스코드에는 수정/삭제가 구현돼 있지만, UI에서 접근 경로가 불명확하다.
- 빈 상태("아직 없어요")에서 "첫 한 줄은 오늘 화면에서"라고 안내하는데, 직접 링크가 없다. 탭해서 바로 이동하게 해줘야 한다.

### 3. 설정 화면

**본 것:**
- 알림 섹션: "알림 받기" 토글
- 데이터 섹션: "데이터 내보내기", "모든 데이터 삭제"
- 정보 섹션: "버전 1.0.0"

**좋은 점:**
- 삭제 시 3단계 확인 (모든 데이터 삭제 -> 정말 삭제하시겠어요? -> 되돌릴 수 없어요. 삭제할까요?)은 안전하다.
- 데이터 내보내기 기능이 있다. JSON으로 내보내는 건 개발자적 감각이지만, 있다는 것 자체가 신뢰감을 준다.

**개선점:**
- 알림 토글을 켜도 실제로 알림이 오는지 모르겠다. Web Push인지, 시간 설정은 되는지 -- 테스트할 수가 없다. "알림 시간" 설정이 토글 ON 후에 나타나는 것 같은데, 실 구현이 되어 있는지 의문.
- 설정 화면이 너무 심심하다. "버전 1.0.0" 말고 뭐가 없다. 앱의 성격을 보여주는 한 줄이라도 있으면 좋겠다.
- 네비게이션이 불편하다. 메인 화면 상단 헤더에 기록 아이콘과 설정 아이콘이 있는데, 기록/설정에서 메인으로 돌아가려면 뒤로가기를 눌러야 한다. 하단 탭바가 없어서 화면 간 이동이 불편하다.

## 개선 제안

### P0 (반드시 고쳐야 함)

1. **하단 탭바 추가 또는 네비게이션 개선**
   현재 기록/설정에서 메인으로 돌아가려면 뒤로가기만 가능하다. 세 화면을 자유롭게 오가는 하단 탭바(오늘 / 기록 / 설정)가 필요하다. 모바일 앱에서 이건 기본이다. 매번 뒤로가기를 누르는 건 귀찮아서 결국 안 쓰게 된다.

2. **큐레이션 칩 스크롤 힌트 + 클릭 시 행동 명확화**
   4번째 칩이 잘려서 안 보인다. 그리고 칩을 눌렀을 때 뭐가 되는지 모르겠다. 칩을 누르면 해당 활동이 입력창에 자동 입력되거나, 상세 설명이 바텀시트로 올라오는 등 명확한 반응이 필요하다. "이걸 누르면 이렇게 된다"가 직관적이어야 한다.

### P1 (사용 경험에 큰 영향)

3. **첫 기록 후 격려 / 리텐션 후킹**
   글을 한 줄 적으면 끝이다. "잘 적었다", "내일 또 다른 질문이 올 거예요" 같은 피드백이 없다. 토스트로 "적었다."가 뜨는 건 확인했지만, 그건 확인(ack)이지 격려가 아니다. 첫 기록은 특별하게 대우해줘야 한다.

4. **프롬프트가 매일 바뀐다는 사실 알려주기**
   35개의 프롬프트가 매일 바뀌는데, 사용자는 그걸 모른다. "매일 새로운 질문" 같은 미니 안내나, 기록 화면에서 과거 프롬프트를 보여주면 "내일은 뭘 물어볼까"라는 기대감이 생긴다.

### P2 (있으면 좋음)

5. **기록 빈 상태에서 바로가기 링크**
   "첫 한 줄은 오늘 화면에서."라는 텍스트를 탭하면 오늘 화면으로 이동하게 만들어야 한다. 현재는 안내만 하고 링크가 없다. 사소하지만, 사소한 불편이 이탈로 이어진다.

## 만족도 근거

**컨셉은 좋다. 실행이 반만 됐다.**

"퇴근하면 뭐하지?"라는 질문 자체가 내 상황을 정확히 짚는다. 매일 바뀌는 프롬프트 + 큐레이션이라는 아이디어도 나쁘지 않다. 베이지 톤의 차분한 디자인도 퇴근 후 모드에 맞다.

하지만 실제로 써보면 "그래서?"라는 느낌이 든다.

- 한 줄 적고 나면 그 다음이 없다. 앱이 나한테 아무것도 돌려주지 않는다. 일기장이면 일기장답게 감성적 피드백이라도 주고, 큐레이션 앱이면 큐레이션답게 더 깊은 추천을 해줘야 하는데, 지금은 둘 다 아닌 어중간한 상태.
- 네비게이션이 불편하다. 화면이 3개밖에 안 되는데도 오가는 게 귀찮다. 하단 탭바 없는 모바일 앱은 2026년에 용납하기 어렵다.
- 큐레이션 칩을 눌렀을 때의 경험이 기대 이하다. 활동 제안이 있는데, 그걸 선택하면 기록에 반영되는 것도 아니고, 더 자세한 가이드가 나오는 것도 아니다.
- 리텐션 장치가 없다. 내일 다시 올 이유가 부족하다. "7일 중 1일"이라는 숫자가 압박이지 동기부여가 아니다.

결론: 퇴근 후 피곤한 내가 이 앱을 3일 연속 열 것 같지 않다. 컨셉의 힘으로 첫날은 열겠지만, 두 번째 날부터는 "유튜브가 낫지"가 된다.
