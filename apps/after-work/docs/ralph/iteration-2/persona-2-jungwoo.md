## 화면별 피드백

### 1. 오늘 (메인 화면)

**좋아진 점**
- "오늘의 질문" 라벨이 프롬프트 위에 추가되어 맥락이 명확해짐. 이전에는 "한번도 안 해본 거."가 뜬금없었는데 이제 질문이라는 걸 바로 인지할 수 있음
- 큐레이션 카드 오버레이에 "이걸로 적기" 버튼이 생겨서 카드의 존재 이유가 확실해짐. 오버레이에서 활동명, 카테고리, subtext까지 보여주고 한 번에 입력에 채울 수 있음
- 저장 후 화면에 연필 아이콘이 보여서 수정 가능하다는 걸 시각적으로 알 수 있음. 이전에는 "탭 가능" 여부를 전혀 몰랐는데 큰 개선
- "내일 또 다른 질문이 기다려요." 리텐션 메시지가 저장 후 자연스럽게 표시됨. 내일도 들어와야겠다는 생각이 살짝 듬
- TodayRecord 카드가 border-left + 직각 모서리(radius 0)로 통일되어 기록 화면과 일관성이 생김

**남은 개선점**
- 스크롤 힌트(scrollFade)가 8px로 너무 좁음. 375px 기기에서 보면 4번째 카드("배우기 위키 서핑")가 카드 끝자락만 겨우 보이는 수준. 아예 안 보이는 것보다는 낫지만, 여전히 "스크롤할 수 있다"는 인지가 약함. 최소 24~32px 이상 fade를 주거나, 4번째 카드 앞쪽 20%가 보이도록 카드 폭/패딩을 조정해야 함
- 큐레이션 카드 오버레이에서 "이걸로 적기" 버튼이 outline 스타일(border만 있고 fill 없음)이라 시각적 무게감이 약함. "이걸로 적기"가 이 오버레이의 핵심 CTA인데, 배경색이 없는 ghost 버튼이면 탭 유도력이 떨어짐
- 저장 직후 토스트("적었다.")가 아직도 너무 빠르게 사라짐. 확인할 시간이 부족

### 2. 기록 화면

**좋아진 점**
- MoreHorizontal(...) 아이콘 + 드롭다운(수정/삭제) 도입으로 수정/삭제 발견성이 극적으로 개선됨. 이전에는 롱프레스를 알 길이 없었는데, 이제 점 세 개를 보면 "아, 메뉴가 있겠구나" 하고 바로 탭 가능
- 삭제 시 확인 모달("이 기록을 삭제할까요? 되돌릴 수 없어요.")이 뜸. 위험한 동작에 대한 확인 절차가 있어서 좋음
- 빈 상태 CTA "첫 한 줄은 오늘 화면에서."가 밑줄 링크 형태로 탭 가능해짐. 이전에는 정적 텍스트였는데 이제 오늘 화면으로 바로 이동 가능

**남은 개선점**
- 주간 카드 펼침에서 빈 요일이 여전히 em-dash("---")만 표시됨. 이전에도 지적했는데, "아직"이나 빈 원(circle) 같은 가벼운 표현이 더 나을 것
- 수정 버튼을 눌렀을 때 어떤 상태로 전환되는지 확인하지 못함 (자동화 도구 한계). 수정 플로우가 인라인 에디팅인지 별도 페이지로 이동하는지 명확하지 않음
- 기록 아이템에 카테고리 정보(만들기/보기듣기 등)가 없음. 큐레이션 카드를 통해 적었으면 해당 카테고리 태그가 표시되면 나중에 기록을 훑어볼 때 도움이 됨

### 3. 설정 화면

**이전과 동일**

- 퇴근 시간 설정 여전히 없음
- 데이터 내보내기 포맷 불명

## 전체 플로우 피드백

**엔트리 작성 플로우 (개선됨):**  
메인 진입 -> 큐레이션 카드 탭 -> 오버레이에서 "이걸로 적기" 탭 -> 입력창에 자동 채움 -> "적기" 탭 -> 저장

- 큐레이션 -> 입력 연동이 완성되어 "힌트 보기 -> 바로 적기"가 가능해짐. 피곤한 상태에서 0초 고민으로 기록 가능. 핵심 루프가 작동함
- 다만 오버레이 -> 입력 채움 -> 다시 적기 탭이 3단계인데, 카드를 탭하면 오버레이 없이 바로 채우는 옵션도 있으면 좋겠음 (오버레이는 길게 누를 때만)

**수정 플로우 (개선됨):**  
메인에서: 카드의 연필 아이콘 -> 수정 모드 -> 텍스트 수정 -> "수정" 탭  
기록에서: ... 아이콘 -> 드롭다운 "수정" -> 수정

- 수정 진입점이 명확해짐. 이전의 "히든 제스처" 문제가 해결됨
- 연필 아이콘의 크기(14px)가 살짝 작긴 하지만, 존재 자체가 힌트 역할을 하므로 괜찮음

## 개선 제안

### P0 (즉시 수정 필요)

없음. 이전 P0 이슈가 모두 반영됨.

### P1 (다음 반복에 수정)

1. **스크롤 힌트 강화 (scrollFade 8px -> 32px+)**  
   현재 fade gradient가 8px로 사실상 보이지 않음. 375px에서 4번째 카드가 간신히 엿보이는 정도. fade 폭을 최소 32px로 늘리거나, 카드 간격/패딩을 조정해서 4번째 카드 앞부분이 확실히 드러나도록 해야 함. 또는 첫 로드 시 살짝 좌우로 흔들리는 micro-animation을 넣는 것도 방법.

2. **알림 설정에 시간/설명 추가**  
   알림 받기 토글 아래에 "매일 [시간]에 오늘의 질문을 보내드려요" 설명 + 시간 선택기. 앱의 핵심 가치인 "퇴근 후 루틴"을 지원하려면 필수. 2회 연속 지적.

3. **"이걸로 적기" 버튼 시각적 강화**  
   현재 ghost button(outline)인데, 오버레이의 유일한 CTA이므로 filled 스타일(배경색)로 변경하거나 최소한 색상 대비를 높여야 함. 지금은 subtext보다 시선을 덜 끎.

### P2 (여유 있을 때)

4. **주간 카드 빈 요일 표현 개선**  
   em-dash("---") 대신 "아직" 또는 연한 점(dot) 표시. 2회 연속 지적이지만 치명적이지 않으므로 P2 유지.

5. **기록 아이템에 카테고리 태그 표시**  
   큐레이션을 통해 적은 경우 "만들기", "움직이기" 같은 태그를 RecordItem에 작게 표시. 기록이 쌓이면 "이번 달은 움직이기를 많이 했네" 같은 인사이트 가능.

**편의성 (7/10, +2):** 작성-저장-수정-삭제 전 플로우가 발견 가능하고 동작함. 연필 아이콘, ... 메뉴, 링크형 빈 상태 CTA 등 어포던스가 전반적으로 개선됨. 피곤한 상태에서 "이거 어떻게 하지?"로 이탈할 확률이 크게 줄었음.
