## 화면별 피드백

### 1. 오늘 (메인 화면) — 빈 상태

**좋은 점**
- 앱의 핵심 구조가 안정됐다. 질문 → 큐레이션 카드 → 입력 → 저장. 4번째 이터레이션인데 이 구조가 바뀌지 않았다는 건 처음 설계가 잘 됐다는 뜻이다.
- 하단 네비게이션이 표준 탭바 패턴(space-around)으로 정리되면서 "완성된 프로덕트" 느낌이 생겼다. 이전에는 탭이 왼쪽으로 쏠려 있어서 개발 중인 앱 같았는데, 이 하나의 CSS 변경이 인상을 꽤 바꿨다.
- safe-area-bottom이 적용되어 iPhone X 이후 기기에서의 레이아웃 이슈가 해결됐다. 기술적으로 당연한 건데, 이걸 지적받고 나서야 적용했다는 건 초기 구현이 데스크톱 퍼스트였다는 뜻. 앞으로는 모바일 퍼스트로 시작하는 게 맞다.

**남은 개선점**
- 입력 영역이 여전히 화면 상단에 위치. 한 손(특히 오른손 엄지) 조작 시 화면 상단 40%는 도달하기 어려운 영역이다. 입력 필드를 화면 하단 1/3에 배치하면 접근성이 좋아지지만, 이건 레이아웃 전체 재구성이 필요하므로 큰 결정이다.
- 큐레이션 카드 4번째("위키 서핑")가 스크롤 없이는 보이지 않는 점은 이전과 동일. fade 힌트만으로는 "더 있다"를 전달하기 어렵다.

### 2. 오늘 (저장 후 상태)

**좋은 점**
- "내일 또 다른 질문이 기다려요." 제거가 정확한 판단이었다. 수진이 지적했듯 누적 스탯의 건조한 톤과 감성 문구가 충돌했는데, 이제 "첫 번째 한 줄."만 남아서 앱 전체의 보이스가 일관된다. 이 앱은 칭찬하지 않고, 사실만 말한다. 그게 차별점이다.
- 토스트 2500ms가 적절하다. 2000ms는 눈 깜빡이면 놓칠 수 있었는데, 500ms 추가로 "적었다."를 읽고 이해할 시간이 확보됐다. 이런 미세 조정이 체감 품질을 올린다.
- 기록 카드의 연필 아이콘으로 수정 모드 진입 → 인라인 수정 → "수정" 버튼(filled primary) + "취소" + 글자수 카운터. 수정 플로우가 깔끔하다.

**남은 개선점**
- 저장 후 빈 공간이 여전히 크다. 기록 카드 + 스탯 한 줄 아래가 화면의 50% 이상. 이건 반복 지적이지만, 해결하려면 콘텐츠(인사이트, 과거 기록 등)가 필요하다. 빈 공간 자체는 미니멀리즘으로 볼 수도 있지만, 모바일에서는 "미완성"으로 읽힐 수 있다.

### 3. 기록 화면

**좋은 점**
- 주간 요약 카드의 드롭다운이 인상적이다. 클릭하면 월~일 요일별로 기록/빈칸이 표시된다. "목 — '레고 조립'" 나머지는 대시. 이건 개발자 관점에서 보면 단순한 리스트 렌더링이지만, 사용자 관점에서는 "이번 주에 3번 적었구나 / 빈 날이 4개구나"를 한 눈에 파악할 수 있는 대시보드다.
- 기록 카드에 "더보기" 메뉴(수정/삭제)가 있고, 삭제는 빨간색으로 위험 표시. CRUD가 완성되어 있다.
- 기록에 질문 원문이 함께 표시되어 맥락이 유지된다.

**남은 개선점**
- 기록 아이템에 카테고리 태그가 여전히 없다. 큐레이션 카드에서 "이걸로 적기"를 선택한 경우 카테고리 정보를 저장하고 있을 텐데, 이걸 기록 카드에 작은 라벨로 보여주면 기록에 차원이 하나 추가된다.

### 4. 설정 화면

**좋은 점**
- 알림 토글 ON → "알림 시간 07:00 PM" 즉시 노출. time picker가 있고 시간 변경이 가능하다. 이 기능이 3번의 이터레이션 동안 "없다"고 보고된 건 테스트 범위의 한계였다. 기능 자체는 잘 구현되어 있다.
- 섹션 구분(알림/데이터/정보)이 명확하고, 설정 항목이 적절하다. 미니멀 앱에 맞는 미니멀 설정.

## 개선 제안

### P1 (다음 반복에 수정)

1. **기록 카드에 카테고리 태그 표시**
   3회 연속 제안. 큐레이션 카드에서 선택한 활동은 카테고리(만들기/보기듣기/움직이기/배우기) 정보가 있다. 기록 카드에 작은 태그로 표시하면, 기록이 쌓였을 때 "나는 주로 뭘 하는 사람인가"를 파악할 수 있는 기반이 된다.

2. **저장 후 빈 공간에 미니 인사이트**
   기록 카드 아래의 빈 영역에 "이번 주 기록 현황" 미니 차트(요일별 점 7개)를 표시하면, 저장 후의 "끝났다" 느낌 대신 "쌓고 있다" 느낌이 생긴다. 기록 페이지의 주간 드롭다운과 중복될 수 있지만, 홈에서 즉시 확인 가능하다는 점이 다르다.

### P2 (여유 있을 때)

3. **연속 기록 마일스톤** — 3일 연속, 1주 완주, 한 달 등 마일스톤 달성 시 스탯 문구가 바뀌는 것. "열다섯 번째 한 줄. 한 달 됐다." 같은 특별 문구. 현재의 단순 카운터를 확장하는 자연스러운 방향이다.

## 만족도 점수

**80 / 100** (이전 52 -> 68 -> 74 -> +6)

### 근거

**컨셉 (8/10, 유지):** 퇴근 후 한 줄 기록. 4번째 테스트에서도 컨셉의 매력이 유지된다. 이건 좋은 신호다.

**첫인상 (9/10, +1):** 하단 탭바의 균등 배치 + safe-area 적용이 "완성된 앱" 인상을 준다. 이전에는 레이아웃 어색함이 첫인상을 깎았는데, 이제 그런 요소가 없다.

**핵심 플로우 완성도 (8/10, 유지):** 작성 → 저장 → 피드백까지 걸림 없이 동작한다. "이걸로 적기" filled CTA, 인라인 수정, 글자수 카운터, 토스트 피드백 — 모든 인터랙션이 예상대로 동작한다.

**편의성 (8/10, +1):** 하단 탭바 정상화, safe-area, 알림 시간 설정 확인. 설정에서 시간 선택이 되고, 네비게이션이 표준화되면서 "사용하기 불편하다"는 포인트가 거의 없어졌다.

**반복 사용 동기 (7/10, +1):** "내일 또 다른 질문이 기다려요." 제거 후 스탯만 남긴 것이 오히려 깔끔하다. 기록 페이지의 주간 요일별 뷰가 "빈 칸을 채우고 싶다"는 심리를 자극. 알림 시간 설정도 확인됨. 카테고리 인사이트와 마일스톤이 추가되면 8점 이상 가능.

**종합:** Iteration 4는 "P0 청소" 이터레이션이었다. 하단 네비 레이아웃, safe-area, 감성 문구 제거, 토스트 시간 — 하나하나는 작은 변경이지만 누적 효과가 크다. 기술적으로 블로킹하는 이슈가 없어졌고, UX의 기본기가 갖춰졌다. "매일 쓰겠느냐"에 대해, 알림 설정하고 퇴근 후 푸시가 오면 열어서 한 줄 적을 것 같다. 80점은 "사용할 의향이 있다"는 점수다. 90점이 되려면 기록이 쌓였을 때의 가치(카테고리 인사이트, 트렌드, 마일스톤)가 추가되어야 한다.
