## 화면별 피드백

### 1. 홈 화면 (미작성 상태)

**좋은 점**
- 질문("한번도 안 해본 거.") + 큐레이션 카드 3~4장 + 입력 필드. 이 구성이 4차까지 오면서 안정됐다. 질문-영감-기록의 3단계가 화면 하나에서 끝난다.
- 큐레이션 카드의 카테고리 라벨(만들기/보기듣기/움직이기/배우기)이 활동의 성격을 바로 알려준다. 마케터로서 카테고리 분류는 사용자 인지에 핵심이라고 본다.
- 카드 탭 -> 오버레이 -> "이걸로 적기" 2단계 흐름이 매끄럽다. "이걸로 적기" filled primary 버튼이 눈에 잘 들어온다.

**아쉬운 점**
- 여전히 화면 하단 60%가 빈 공간이다. 질문+카드+입력이 상단에 몰려 있고, 하단 탭바까지의 거리가 멀다. 콘텐츠 밀도가 낮아서 "앱이 아직 덜 만들어진 것 같다"는 인상을 줄 수 있다.
- 4번째 큐레이션 카드("위키 서핑")가 스크롤하지 않으면 보이지 않는다. 카드가 4개라는 것을 모르는 사용자가 있을 수 있다.

### 2. 홈 화면 (작성 완료 상태)

**좋은 점**
- "첫 번째 한 줄." 누적 스탯만 남기고 감성 문구("내일 또 다른 질문이 기다려요.")가 제거됐다. 톤이 통일됐다. 건조하고 담백한 이 앱 톤에 맞다.
- 기록 카드에 내용 + 시간 + 연필 아이콘이 깔끔하게 표시된다. 수정 가능하다는 것이 직관적으로 보인다.
- 토스트("적었다.")가 이전보다 조금 더 오래 보인다. 2500ms면 인지하기에 충분하다.

**아쉬운 점**
- 작성 완료 후에도 빈 공간이 여전히 크다. 기록 카드 + "첫 번째 한 줄." 아래가 화면의 절반 이상이 텅 비어 있다.

### 3. 기록 페이지

**좋은 점**
- 주간 요약 카드("이번 주 / 7일 중 1일 . 레고 조립")가 상단에서 한 눈에 이번 주 현황을 보여준다.
- 드롭다운을 열면 월~일 요일별 기록이 표시된다. "레고 조립"이 목요일에 표시되고 나머지는 대시(--). 한 주를 한 눈에 볼 수 있다. 이건 리텐션 도구로서 강력하다 — "빈 칸을 채우고 싶다"는 심리가 작동한다.
- 기록 카드에 질문("한번도 안 해본 거.")과 시간이 함께 표시돼서 맥락이 완성된다.
- "..." 더보기 메뉴에서 수정/삭제가 가능하다. 삭제가 빨간색으로 위험 표시.

**아쉬운 점**
- 기록이 1~2개일 때 화면이 허전하다. 기록이 쌓이면 해결될 문제지만, 초반 사용자 경험으로는 "이게 다인가?" 느낌.

### 4. 설정 페이지

**좋은 점**
- 알림 토글 ON 시 "알림 시간 07:00 PM"이 바로 아래에 나타난다. 시간 선택이 가능하다. 이걸 3번이나 지적했는데 이미 있었다니 — 토글을 실제로 켜봤어야 했다.
- 섹션 분리(알림/데이터/정보)가 깔끔하다.
- "모든 데이터 삭제" 빨간색 경고 + 2단계 확인 — 안전하다.

**아쉬운 점**
- "데이터 내보내기"의 포맷(JSON? CSV?)에 대한 안내가 여전히 없다. 사소하지만 있으면 좋겠다.

### 5. 하단 네비게이션

**좋은 점**
- space-around 배치로 "오늘"과 "기록"이 좌우 균등하게 자리잡았다. 이전의 왼쪽 몰림이 완전히 해결됐다. 어떤 앱이든 통용되는 표준 탭바 레이아웃이 되었다.
- safe-area-bottom padding 적용으로 iPhone에서 홈 인디케이터와 겹치지 않는다.
- 활성 탭 색상 구분이 명확하다.

## 개선 제안

### P1 (강력 권장)
1. **작성 완료 후 빈 공간 활용** — 기록 카드 아래의 넓은 빈 공간에 간단한 인사이트나 일러스트를 추가하면 완성도가 올라간다. 예를 들어 "이번 주에 가장 많이 한 카테고리" 또는 오늘의 기록과 비슷한 과거 기록 1개 표시.

2. **큐레이션 카드 dot indicator** — 4번째 카드의 존재를 알리기 위해 카드 하단에 dot 3~4개를 추가하거나, 카드 너비를 줄여서 4번째가 살짝 보이게 하면 스크롤 유도가 더 확실해진다.

### P2 (있으면 좋은 것)
3. **"이걸로 적기" 후 입력 필드 자동 포커스 + 커서 위치** — 큐레이션에서 텍스트를 가져온 뒤 입력 필드 끝에 커서가 자동으로 위치하면, "레고 조립 혼자서"처럼 바로 수정/추가할 수 있다.

**52% -> 64% -> 72% -> 80% 변화 요약:**

- Iteration 4에서 P0 이슈 3개(하단 네비 배치, safe-area, 알림 시간)가 모두 해결됨. 이전에 "80% 도달 조건"으로 명시했던 것들이다.
- "내일 또 다른 질문이 기다려요." 제거로 앱 전체의 톤이 일관됐다. 건조하고 담백한 스타일이 처음부터 끝까지 유지된다.
- 토스트 2500ms로 연장되어 저장 확인이 자연스러워졌다.
- 주간 드롭다운의 요일별 기록 뷰는 예상 밖의 리텐션 요소. "빈 칸을 채우고 싶다"는 심리를 자극한다.
- 남은 과제는 빈 공간 활용과 기록 쌓임에 따른 인사이트 — 이건 P1이지 P0이 아니다. "매일 쓰겠느냐"고 물으면, 지금도 쓸 수 있다. 80%.
