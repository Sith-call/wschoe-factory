# 유저 스토리 맵: 퇴근하면 뭐하지?

## 시간축 개요

```
첫 방문 (D0)          1주 (D1~D7)           1달 (D8~D30)
─────────────         ─────────────         ─────────────
앱 열기               반복 기록              축적과 돌아보기
프롬프트 만남          영감 활용/무시          주간 카드 확인
첫 한 줄 적기         기록 습관 형성          패턴 인식
```

---

## Epic 1: 첫 방문 — 30초 안에 한 줄 적기

> 첫 방문 사용자가 앱을 열고, 무엇을 해야 하는지 즉시 이해하고, 한 줄을 적는다.

### US-1.1 앱 첫 화면 진입 [P0]

**As a** 처음 앱을 연 사용자  
**I want to** 앱을 열자마자 무엇을 하면 되는지 이해하고 싶다  
**So that** 튜토리얼 없이 바로 시작할 수 있다

**수용 기준:**
- [ ] 앱 최초 실행 시 온보딩/튜토리얼/회원가입 없이 바로 메인 화면 진입
- [ ] 화면 중앙에 오늘의 프롬프트가 표시됨
- [ ] 프롬프트 아래에 영감 카드 영역과 입력 필드가 보임
- [ ] 3초 이내에 "아, 여기에 뭔가 적으면 되는구나" 이해 가능한 레이아웃

### US-1.2 오늘의 프롬프트 확인 [P0]

**As a** 퇴근길 사용자  
**I want to** 오늘의 질문을 확인하고 싶다  
**So that** 내가 뭘 적으면 될지 방향을 잡을 수 있다

**수용 기준:**
- [ ] 매일 1개의 프롬프트가 화면 상단에 표시됨
- [ ] 프롬프트 톤: 건조하고 짧음. 위로/격려 없음 (예: "퇴근하면 뭐하지?", "오늘 저녁, 뭐 해볼까.", "손이 심심한 밤.")
- [ ] 같은 날 다시 열어도 동일한 프롬프트 유지
- [ ] 자정 기준으로 프롬프트 변경

### US-1.3 영감 카드 둘러보기 [P0]

**As a** 뭘 적어야 할지 모르겠는 사용자  
**I want to** 영감 카드를 넘기며 아이디어를 얻고 싶다  
**So that** 적을 거리가 떠오른다

**수용 기준:**
- [ ] 프롬프트 아래에 영감 카드 3~4장이 가로 스크롤 또는 스와이프 형태로 표시
- [ ] 카드 탭 시 확대 뷰 또는 간단한 부가 설명 표시
- [ ] 카드를 선택하지 않아도 입력 필드 사용 가능
- [ ] 카드 콘텐츠 예시: "동네 산책", "안 읽던 책 10분", "새로운 음악 찾기", "손편지 쓰기"

### US-1.4 한 줄 적기 [P0]

**As a** 적고 싶은 게 떠오른 사용자  
**I want to** 한 줄을 빠르게 적고 저장하고 싶다  
**So that** 생각을 놓치지 않고 기록할 수 있다

**수용 기준:**
- [ ] 화면 하단에 텍스트 입력 필드 1개
- [ ] 최대 100자 제한 (초과 시 입력 불가, 글자 수 카운터 표시)
- [ ] 저장 버튼 탭 시 즉시 저장 (100ms 이내)
- [ ] 저장 후 간결한 완료 피드백 (예: 체크 아이콘 0.5초 표시)
- [ ] 저장 후 입력 필드 초기화
- [ ] 같은 날 이미 적었으면, 기존 기록 표시 + "오늘 이미 적었어요" 상태

### US-1.5 저장 완료 확인 [P0]

**As a** 한 줄을 적은 사용자  
**I want to** 저장이 잘 됐는지 확인하고 싶다  
**So that** 안심하고 앱을 닫을 수 있다

**수용 기준:**
- [ ] 저장 직후 오늘 화면에 작성한 내용이 표시됨
- [ ] 표시 형태: 프롬프트 아래, 입력 필드 위치에 저장된 한 줄이 보임
- [ ] 기록 화면에서도 방금 적은 내용이 바로 확인 가능

## Epic 2: 반복 사용 — 매일 한 줄의 루틴

> 첫 기록 이후 일주일간, 매일 앱을 열고 한 줄을 적는 습관을 형성한다.

### US-2.1 재방문 시 새 프롬프트 확인 [P0]

**As a** 어제도 적은 사용자  
**I want to** 오늘은 다른 프롬프트를 보고 싶다  
**So that** 매일 새로운 자극을 받는다

**수용 기준:**
- [ ] 자정 이후 앱 진입 시 새로운 프롬프트 표시
- [ ] 최소 30개 프롬프트 풀에서 순환 (P1 기능이지만 검증 기준은 P0에서 정의)
- [ ] 프롬프트 반복 주기: 전체 풀 소진 후 재순환

### US-2.2 영감 카드 없이 바로 적기 [P0]

**As a** 적을 게 바로 떠오르는 사용자  
**I want to** 영감 카드를 건너뛰고 바로 적고 싶다  
**So that** 불필요한 단계 없이 빠르게 기록한다

**수용 기준:**
- [ ] 영감 카드를 탭하지 않아도 입력 필드에 바로 접근 가능
- [ ] 영감 카드 영역을 축소하거나 스킵하는 별도 UI 불필요 (자연스러운 스크롤로 해결)
- [ ] 입력 필드 탭 시 키보드 올라옴

### US-2.3 오늘 이미 적은 상태 확인 [P0]

**As a** 앱을 다시 연 사용자  
**I want to** 오늘 이미 적었는지 바로 알고 싶다  
**So that** 중복으로 적지 않는다

**수용 기준:**
- [ ] 오늘 기록이 있으면 입력 필드 대신 작성한 내용이 표시됨
- [ ] "오늘의 한 줄" 형태로 기록 내용 + 시간 표시
- [ ] 수정하고 싶으면 탭하여 수정 모드 진입 가능 (P1)

### US-2.4 기록 목록 조회 [P0]

**As a** 며칠째 기록 중인 사용자  
**I want to** 지금까지 적은 것들을 모아 보고 싶다  
**So that** 내가 뭘 적었는지 돌아볼 수 있다

**수용 기준:**
- [ ] 상단 기록 아이콘 탭 시 기록 화면 진입
- [ ] 최신순으로 날짜 + 프롬프트 + 한 줄 기록 표시
- [ ] 기록이 없는 날은 표시하지 않음 (빈 칸 없음)
- [ ] 스크롤로 과거 기록 탐색

### US-2.5 안 쓴 날 — 죄책감 없음 [P0]

**As a** 어제 앱을 안 연 사용자  
**I want to** 오늘 다시 열었을 때 부담 없이 시작하고 싶다  
**So that** "어제 안 적었네..." 같은 압박을 받지 않는다

**수용 기준:**
- [ ] 연속 기록 카운터, 스트릭 표시 없음
- [ ] 어제 안 적었다는 알림이나 안내 없음
- [ ] 오늘 화면은 항상 동일한 구조: 프롬프트 + 영감 카드 + 입력 필드
- [ ] 기록 화면에서 안 쓴 날은 아예 표시하지 않음 (빈 슬롯 없음)

## Epic 3: 축적과 돌아보기 — 패턴 인식

> 2주 이상 기록이 쌓이면, 사용자가 자신의 패턴을 인식하고 의미를 발견한다.

### US-3.1 이번 주 카드 확인 [P1]

**As a** 일주일 이상 기록한 사용자  
**I want to** 이번 주에 적은 것들을 요약해서 보고 싶다  
**So that** 이번 주 나의 관심사를 한눈에 파악한다

**수용 기준:**
- [ ] 기록 화면 상단에 "이번 주" 카드 표시
- [ ] 카드 내용: 이번 주 기록한 날 수 (예: "7일 중 4일") + 기록한 내용 키워드 또는 요약
- [ ] 기록이 0개인 주에는 카드 미표시
- [ ] 카드 탭 시 이번 주 기록만 필터링하여 표시

### US-3.2 주간 요약 카드 [P2]

**As a** 한 달 가까이 기록한 사용자  
**I want to** 지난주 기록을 돌아보는 요약 카드를 받고 싶다  
**So that** 지난주 내가 뭘 하고 싶어했는지 정리된다

**수용 기준:**
- [ ] 매주 월요일 앱 진입 시 지난주 요약 카드 표시
- [ ] 요약 내용: 기록 수 + 주요 키워드 + 가장 긴 기록 발췌
- [ ] 카드 닫기 가능 (다시 안 보임)
- [ ] 기록이 2개 미만인 주에는 카드 미생성

### US-3.3 캘린더 뷰로 기록 확인 [P2]

**As a** 한 달 이상 기록한 사용자  
**I want to** 달력 형태로 언제 적었는지 보고 싶다  
**So that** 기록 패턴을 시각적으로 파악한다

**수용 기준:**
- [ ] 기록 화면에서 캘린더 뷰 토글 가능
- [ ] 기록한 날에 작은 점 표시
- [ ] 연속 기록 하이라이트 없음 (점만 표시)
- [ ] 날짜 탭 시 해당 날 기록으로 스크롤

## Epic 4: 기록 관리 — 수정, 삭제, 내보내기

> 사용자가 자신의 데이터를 소유하고 관리할 수 있다.

### US-4.1 기록 수정 [P1]

**As a** 오타를 발견한 사용자  
**I want to** 과거에 적은 한 줄을 수정하고 싶다  
**So that** 깔끔한 기록을 유지한다

**수용 기준:**
- [ ] 기록 화면에서 개별 기록 탭 → 수정 모드 진입
- [ ] 수정 후 저장 시 원본 날짜 유지, 수정 시각은 별도 표시하지 않음
- [ ] 100자 제한 동일 적용

### US-4.2 기록 삭제 [P1]

**As a** 특정 기록을 지우고 싶은 사용자  
**I want to** 기록을 삭제할 수 있어야 한다  
**So that** 내 데이터를 내가 통제한다

**수용 기준:**
- [ ] 기록 화면에서 개별 기록 길게 누르기 또는 스와이프 → 삭제 옵션
- [ ] 삭제 전 확인 다이얼로그 (실수 방지)
- [ ] 삭제 후 복구 불가 안내
- [ ] 삭제된 날은 기록 없는 날과 동일하게 처리

### US-4.3 데이터 내보내기 [P2]

**As a** 내 데이터를 백업하고 싶은 사용자  
**I want to** 전체 기록을 파일로 내보내고 싶다  
**So that** 기기 변경이나 데이터 유실에 대비한다

**수용 기준:**
- [ ] 설정 화면에서 "데이터 내보내기" 버튼
- [ ] JSON 형식으로 전체 기록 다운로드
- [ ] 파일 내용: 날짜, 프롬프트, 기록 텍스트, 작성 시각
- [ ] 내보내기 완료 시 공유 시트(Share Sheet) 활용

## Epic 5: 설정 — 최소한의 커스터마이징

> 알림 시간과 데이터 관리 등 최소한의 설정만 제공한다.

### US-5.1 설정 화면 진입 [P2]

**As a** 앱 설정을 변경하고 싶은 사용자  
**I want to** 설정 화면에 접근하고 싶다  
**So that** 알림이나 데이터를 관리한다

**수용 기준:**
- [ ] 메인 상단 설정 아이콘 탭 → 설정 화면 진입
- [ ] 설정 항목: 알림 시간, 데이터 내보내기, 데이터 전체 삭제, 앱 버전 정보
- [ ] 설정 항목은 5개 이하로 유지 (미니멀)

### US-5.2 알림 시간 설정 [P2]

**As a** 퇴근 시간이 일정한 사용자  
**I want to** 매일 특정 시간에 알림을 받고 싶다  
**So that** 기록하는 걸 잊지 않는다

**수용 기준:**
- [ ] 기본값: 알림 OFF
- [ ] 알림 ON 시 시간 선택 (시/분 피커)
- [ ] 알림 텍스트: 건조한 톤 (예: "퇴근했으면 한 줄.", "한 줄 적을 시간.")
- [ ] 로컬 알림 (서버 푸시 아님)
- [ ] 오늘 이미 기록했으면 알림 미발송

### US-5.3 데이터 전체 삭제 [P2]

**As a** 앱을 초기화하고 싶은 사용자  
**I want to** 모든 기록을 한 번에 삭제하고 싶다  
**So that** 처음부터 다시 시작한다

**수용 기준:**
- [ ] 설정 화면에서 "모든 데이터 삭제" 버튼
- [ ] 2단계 확인: "정말 삭제하시겠어요?" → "되돌릴 수 없어요. 삭제할까요?"
- [ ] 삭제 후 앱 첫 방문 상태로 초기화

## Epic 6: 빈 상태와 엣지 케이스

> 모든 상태에서 사용자가 혼란 없이 행동할 수 있다.

### US-6.1 첫 방문 빈 상태 [P0]

**As a** 앱을 처음 연 사용자  
**I want to** 기록이 없어도 자연스러운 화면을 보고 싶다  
**So that** "이 앱 뭐지?" 혼란이 없다

**수용 기준:**
- [ ] 오늘 화면: 프롬프트 + 영감 카드 + 입력 필드 (기록 없어도 동일 구조)
- [ ] 기록 화면: "아직 기록이 없어요" 대신, "첫 한 줄을 적어보세요" 같은 건조한 안내
- [ ] 이번 주 카드: 미표시

### US-6.2 오프라인 상태 [P0]

**As a** 지하철 등 오프라인 환경의 사용자  
**I want to** 인터넷 없이도 정상적으로 앱을 사용하고 싶다  
**So that** 퇴근 지하철에서도 기록할 수 있다

**수용 기준:**
- [ ] 모든 기능이 오프라인에서 동작
- [ ] 네트워크 상태와 무관하게 프롬프트, 영감 카드, 입력, 저장 정상 동작
- [ ] 오프라인 알림이나 배너 표시 불필요 (원래 오프라인 앱이므로)

### US-6.3 자정 전후 기록 [P1]

**As a** 늦은 밤에 기록하는 사용자  
**I want to** 자정 직전에 적은 기록이 오늘 날짜로 저장되길 원한다  
**So that** 날짜 혼동이 없다

**수용 기준:**
- [ ] 기록 저장 시점의 날짜 기준으로 저장
- [ ] 자정 이후 앱 재진입 시 새 프롬프트 + 새 입력 가능
- [ ] 23:59에 적은 기록 → 해당 날짜 기록으로 저장됨
