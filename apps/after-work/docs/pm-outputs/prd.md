# PRD: 퇴근하면 뭐하지?

> 퇴근길에 하루 한 줄, 내가 하고 싶은 걸 적어보는 마이크로 저널

## 1. 문제 정의

### 퇴근 후 시간은 왜 사라지는가

직장인의 퇴근 후 시간은 구조적으로 무의미하게 소비된다. 피로 → 의사결정 회피 → 유튜브/SNS 디폴트 행동 → 자기 전 후회. 이 루프가 매일 반복된다.

**핵심 문제 3가지:**

1. **"뭐 하지?"의 반복**: 퇴근 후 매일 같은 질문을 스스로에게 던지지만, 답을 내리기엔 의지력이 바닥난 상태다. 결국 가장 쉬운 행동(스크롤링)으로 귀결된다.

2. **자기 인식의 부재**: "나는 퇴근 후에 뭘 하고 싶은 사람인가?"라는 질문에 답할 수 있는 사람이 드물다. 하고 싶은 게 뭔지 모르니까, 안 하게 된다.

3. **기록의 부담**: 일기 앱, 감정 기록 앱은 많다. 하지만 대부분 "오늘 하루를 돌아보세요"라는 무거운 질문으로 시작한다. 피곤한 퇴근길에 3줄 이상 쓸 에너지는 없다.

### 기존 해결책의 한계

| 카테고리 | 대표 앱 | 한계 |
|---------|--------|------|
| 무드 트래커 | Daylio | 텍스트 입력 자체를 제거. 생각을 기록하지 않음 |
| 일기 앱 | Day One, 하루콩 | 최소 3줄 이상 작성 유도. 퇴근길 부담 |
| 활동 추천 | 프립, 클래스101 | 커머스. 내 생각이 아닌 상품을 밀어넣음 |
| 습관 앱 | Habitica, 루틴리 | 할 일 관리. "하고 싶은 것"과 "해야 하는 것"은 다름 |

### 이 앱이 필요한 이유

시장에 "건조한 톤 + 한 줄 기록 + 마중물 큐레이션" 조합은 존재하지 않는다. 사용자에게 생각을 강요하지 않으면서도, 매일 한 줄의 기록이 쌓이면 "나는 이런 걸 하고 싶어하는 사람이구나"라는 자기 인식이 형성된다.

## 2. 핵심 가치 제안

> **"매일 한 줄, 내가 하고 싶은 걸 알아가는 가장 가벼운 방법"**

### 가치 계층

| 계층 | 설명 |
|------|------|
| **기능적 가치** | 퇴근 후 할 일을 한 줄로 적는다 |
| **정서적 가치** | 적는 행위 자체로 "나를 위한 시간"이 시작된다 |
| **축적 가치** | 한 줄이 모이면 내가 뭘 좋아하는 사람인지 보인다 |

### 차별화 포인트

1. **한 줄이면 충분하다**: 3줄도 아니고, 이모지 선택도 아니다. 텍스트 한 줄.
2. **마중물, 강요 아님**: 영감 카드 3~4장을 보여주되, 안 골라도 된다. 선택이 아닌 자극.
3. **건조한 톤**: "오늘 하루 수고했어요!" 같은 위로 없음. 앱은 말이 적고 사용자의 생각이 주인공.
4. **죄책감 제로**: 안 쓴 날에 알림을 보내거나 연속 기록을 강조하지 않는다.

## 3. 타겟 페르소나

### 페르소나 1: 민서 (28세, 대기업 마케터)

| 항목 | 내용 |
|------|------|
| **직업** | 대기업 브랜드 마케팅 3년차 |
| **퇴근 시간** | 19:30~20:00 |
| **퇴근 후 패턴** | 지하철 40분 → 집 도착 → 유튜브 틀어놓고 치킨 시킴 → 자기 전 "오늘도 아무것도 안 했네" |
| **핵심 고민** | 회사에서는 늘 남의 브랜드를 키우는데, 나만의 뭔가가 없다는 생각이 자꾸 든다. 글쓰기, 드로잉, 요리... 뭐든 시작하고 싶은데 뭘 해야 할지 모르겠다. |
| **앱 사용 시나리오** | 퇴근 지하철에서 영감 카드 넘기다가 "아 나 오늘 카페 가서 노트북 펴고 아무거나 써보고 싶다" 한 줄 적음 |
| **기대 효과** | 2주 후 기록 보면서 "나 글쓰기 관련 적은 게 많네" 발견 |

### 페르소나 2: 정우 (34세, IT 스타트업 백엔드 개발자)

| 항목 | 내용 |
|------|------|
| **직업** | 시리즈B 스타트업 시니어 개발자 |
| **퇴근 시간** | 20:00~21:00 (변동 큼) |
| **퇴근 후 패턴** | 버스 20분 → 편의점 맥주 → 넷플릭스 → 새벽 1시 취침. 주말에는 "이번 주도 코딩만 했다" 후회 |
| **핵심 고민** | 개발 외에 아무것도 안 하는 삶. 운동도, 취미도, 사람 만나기도 다 귀찮은데, 이러면 안 된다는 건 안다. 근데 뭘 해야 재미있을지 감이 안 온다. |
| **앱 사용 시나리오** | 버스에서 영감 카드 중 "손으로 뭔가 만들기" 보고 "레고 하나 사볼까" 적음 |
| **기대 효과** | 한 달 기록에서 "몸 쓰는 활동"이 반복되는 걸 발견하고 클라이밍 등록 |

### 페르소나 3: 수진 (31세, 공공기관 행정직)

| 항목 | 내용 |
|------|------|
| **직업** | 공공기관 일반행정 5년차 |
| **퇴근 시간** | 18:00 정시 퇴근 |
| **퇴근 후 패턴** | 18:30 집 도착 → 밥 해먹기 → SNS → 유튜브 → 22시 취침. 시간은 있는데 뭘 해야 할지 모름 |
| **핵심 고민** | 정시 퇴근이라 시간은 넉넉한데, 매일 똑같이 흘려보낸다. 자기계발 앱 3개 깔았다 다 삭제. "나는 뭘 좋아하는 사람이지?"가 진짜 고민. |
| **앱 사용 시나리오** | 퇴근 후 소파에서 앱 열고, 영감 카드 안 보고 바로 "오늘은 산책하면서 팟캐스트 듣고 싶다" 적음 |
| **기대 효과** | 주간 요약 카드로 이번 주에 적은 것들 돌아보며 "나 산책 좋아하네" 인식 |

## 4. MVP 기능 목록

### P0 (Must Have — 없으면 앱이 아님)

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 오늘의 프롬프트 | 매일 1개의 건조한 프롬프트 표시. "퇴근하면 뭐하지?" 또는 변형 |
| F-02 | 영감 카드 | 프롬프트 아래 3~4장의 영감 카드. 카테고리 기반 랜덤 노출. 탭하면 확대, 안 골라도 됨 |
| F-03 | 한 줄 입력 | 텍스트 입력 필드 1개. 최대 100자. 저장 버튼 |
| F-04 | 기록 화면 | 날짜별 한 줄 기록 리스트. 최신순 정렬 |
| F-05 | 로컬 저장 | 모든 데이터 로컬 저장 (IndexedDB 또는 localStorage). 서버 없음 |
| F-06 | 빈 상태 처리 | 첫 방문, 기록 없는 날 등 모든 빈 상태에 적절한 안내 |

### P1 (Should Have — 1주 리텐션에 직접 영향)

| ID | 기능 | 설명 |
|----|------|------|
| F-07 | 이번 주 카드 | 기록 화면 상단에 이번 주 기록 요약 카드. 7일 중 기록한 날 수 + 키워드 |
| F-08 | 날짜 네비게이션 | 기록 화면에서 날짜별 스크롤 또는 캘린더 뷰 토글 |
| F-09 | 프롬프트 풀 | 30개 이상의 프롬프트 풀. 매일 다른 프롬프트 노출 (반복 없이 순환) |
| F-10 | 영감 카드 풀 | 50개 이상의 영감 카드 콘텐츠. 카테고리: 몸 움직이기, 만들기, 배우기, 쉬기, 사람 만나기, 먹기, 보기/듣기 |
| F-11 | 수정/삭제 | 과거 기록 수정 및 삭제 기능 |

### P2 (Nice to Have — 1달 리텐션 및 깊이)

| ID | 기능 | 설명 |
|----|------|------|
| F-12 | 주간 요약 카드 | 일요일 밤 또는 월요일 아침에 지난주 기록 기반 요약 카드 생성 |
| F-13 | 설정 화면 | 프롬프트 알림 시간 설정, 데이터 내보내기(JSON), 앱 정보 |
| F-14 | 데이터 내보내기 | JSON 또는 텍스트 파일로 전체 기록 내보내기 |
| F-15 | PWA 지원 | 홈 화면 추가, 오프라인 캐시, 앱 아이콘 |
| F-16 | 알림 | 설정한 시간에 로컬 푸시 알림 1회 (건조한 톤: "퇴근했으면 한 줄.") |
| F-17 | 캘린더 뷰 | 기록 화면에서 월간 캘린더로 기록 여부 시각화 (점 표시, 연속 기록 강조 없음) |

## 5. 성공 지표

### 핵심 지표 (North Star)

| 지표 | 정의 | 목표 (출시 후 4주) |
|------|------|-------------------|
| **주간 기록율** | 주 7일 중 1회 이상 기록한 사용자 비율 | 40% |

### 보조 지표

| 카테고리 | 지표 | 정의 | 목표 |
|---------|------|------|------|
| **활성** | DAU | 일일 앱 오픈 사용자 수 | 측정 시작 (절대 수보다 추이) |
| **활성** | 일일 기록 완료율 | 앱 오픈 후 한 줄 작성까지 완료한 비율 | 60% |
| **리텐션** | D1 리텐션 | 설치 다음 날 재방문 | 50% |
| **리텐션** | D7 리텐션 | 설치 7일 후 재방문 | 30% |
| **리텐션** | D30 리텐션 | 설치 30일 후 재방문 | 15% |
| **참여** | 영감 카드 탭율 | 영감 카드를 1개 이상 탭한 세션 비율 | 35% |
| **참여** | 평균 입력 길이 | 한 줄 입력의 평균 글자 수 | 15~40자 |
| **참여** | 주간 카드 조회율 | 주간 요약 카드를 본 사용자 비율 | 25% |

### 안티 지표 (이것이 높으면 실패)

| 지표 | 의미 | 경계값 |
|------|------|--------|
| 영감 카드 의존율 | 영감 카드 탭 없이 바로 적는 비율이 너무 낮음 | 카드 탭 없이 적는 비율 < 20%이면 경고 |
| 평균 입력 길이 > 80자 | 한 줄이 아닌 일기가 되고 있음 | 80자 초과 비율 > 30%이면 UX 재검토 |
| 알림 OFF 비율 | 알림이 성가시다는 신호 | 50% 이상이면 알림 톤/빈도 재검토 |

## 6. 비기능 요구사항

### 성능

| 항목 | 기준 |
|------|------|
| 앱 초기 로딩 | 2초 이내 (FCP 기준) |
| 한 줄 저장 응답 | 100ms 이내 (로컬 저장) |
| 기록 화면 로딩 | 500ms 이내 (100개 기록 기준) |
| 번들 크기 | 500KB 이하 (gzipped) |

### 호환성

| 항목 | 기준 |
|------|------|
| 브라우저 | Chrome 90+, Safari 15+, Samsung Internet 15+ |
| OS | Android 10+, iOS 15+ (PWA), 데스크톱 브라우저 |
| 화면 | 360px ~ 428px 모바일 최적화. 태블릿/데스크톱은 최대 너비 제한 |

### 데이터

| 항목 | 기준 |
|------|------|
| 저장 방식 | IndexedDB (1차), localStorage (폴백) |
| 데이터 용량 | 1년 기록 기준 1MB 미만 |
| 내보내기 | JSON 포맷, 수동 트리거 |
| 백업 | 없음 (v1). 기기 변경 시 데이터 유실 고지 필요 |

### 접근성

| 항목 | 기준 |
|------|------|
| 폰트 크기 | 본문 16px 이상. 입력 필드 18px |
| 색 대비 | WCAG AA (4.5:1) 이상 |
| 키보드 | 입력 필드 포커스 시 키보드 자동 올림 방지 (사용자가 탭할 때만) |

### 제약사항

| 항목 | 내용 |
|------|------|
| 서버 | 없음. 완전 오프라인 앱 |
| 인증 | 없음. 기기 1대 = 사용자 1명 |
| 분석 | v1에서는 분석 도구 미포함. 로컬 데이터만 |
| 다국어 | 한국어 only (v1) |
