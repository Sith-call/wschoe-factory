# 화면 플로우: 퇴근하면 뭐하지?

## 전체 화면 구조

```
┌─────────────────────────────────────┐
│           오늘 (메인)                │  ← 앱 진입점
│  ┌─────────┐  ┌─────────┐          │
│  │ 기록 ○  │  │ 설정 ○  │          │  ← 상단 우측 아이콘 2개
│  └────┬────┘  └────┬────┘          │
│       │            │                │
│  ┌────▼────┐  ┌────▼────┐          │
│  │ 기록    │  │ 설정    │          │
│  │ 화면    │  │ 화면    │          │
│  └─────────┘  └─────────┘          │
└─────────────────────────────────────┘
```

### 화면 목록

| # | 화면 | 진입 경로 | 상태 수 |
|---|------|----------|---------|
| 1 | 오늘 (메인) | 앱 실행 | 3 (미기록/기록완료/영감확대) |
| 2 | 기록 | 메인 상단 기록 아이콘 | 3 (리스트뷰/캘린더뷰/빈상태) |
| 3 | 이번 주 카드 | 기록 화면 상단 카드 탭 | 2 (접힌/펼친) |
| 4 | 설정 | 메인 상단 설정 아이콘 | 1 |

### 네비게이션 규칙

1. **하단 탭 없음**: 네비게이션은 메인 상단 아이콘 2개로만 구성
2. **뒤로가기**: 기록/설정 → 메인으로 복귀 (시스템 뒤로가기 또는 상단 좌측 화살표)
3. **딥링크 없음**: 모든 진입은 메인 화면부터
4. **전환 애니메이션**: 좌우 슬라이드 (기록: 오른쪽→왼쪽, 설정: 오른쪽→왼쪽)

## 화면 1: 오늘 (메인)

### 목적
하루 한 줄을 적는 핵심 경험. 앱의 유일한 액션 포인트.

### 레이아웃 (위→아래)

```
┌──────────────────────────────┐
│  [기록]              [설정]   │  ← 상단 바 (아이콘 2개)
│                              │
│                              │
│    퇴근하면 뭐하지?            │  ← 오늘의 프롬프트
│                              │
│                              │
│  ┌──────┐ ┌──────┐ ┌──────┐ │  ← 영감 카드 (가로 스크롤)
│  │ 카드1 │ │ 카드2 │ │ 카드3 │ │
│  └──────┘ └──────┘ └──────┘ │
│                              │
│                              │
│  ┌──────────────────────┐    │
│  │ 한 줄 입력...    [저장] │    │  ← 입력 필드
│  └──────────────────────┘    │
│                        78/100│  ← 글자 수 카운터
└──────────────────────────────┘
```

### UI 요소

| # | 요소 | 타입 | 설명 |
|---|------|------|------|
| 1 | 상단 바 | Header | 로고/앱 이름 없음. 우측에 기록 아이콘 + 설정 아이콘만 |
| 2 | 오늘의 프롬프트 | Text | 중앙 정렬. 큰 폰트 (24~28px). 건조한 톤의 질문 |
| 3 | 영감 카드 영역 | Horizontal ScrollView | 카드 3~4장. 가로 스크롤. 카드 크기 균일 |
| 4 | 영감 카드 (개별) | Card | 짧은 텍스트 (2~4단어). 카테고리 아이콘 또는 컬러 태그 |
| 5 | 입력 필드 | TextInput | 단일 행. placeholder: "한 줄 적기..." |
| 6 | 저장 버튼 | Button | 입력 필드 우측 인라인. 텍스트 없을 때 비활성 |
| 7 | 글자 수 카운터 | Text | 입력 필드 하단 우측. "현재/100" 형식. 80자 이상 시 경고색 |

### 인터랙션

| 액션 | 동작 |
|------|------|
| 영감 카드 좌우 스와이프 | 카드 가로 스크롤 |
| 영감 카드 탭 | 카드 확대 뷰 (오버레이). 부가 설명 1줄 표시 |
| 확대 카드 바깥 탭 | 확대 뷰 닫힘 |
| 입력 필드 탭 | 키보드 올라옴. 프롬프트와 영감 카드 위로 밀림 |
| 텍스트 입력 | 실시간 글자 수 카운터 업데이트. 100자 도달 시 입력 차단 |
| 저장 버튼 탭 | 저장 → 체크 아이콘 0.5초 표시 → 기록 완료 상태 전환 |
| 키보드 외부 탭 | 키보드 내림 |

### 상태별 서브뷰

#### 상태 A: 미기록 (기본)
- 프롬프트 + 영감 카드 + 빈 입력 필드
- 이 상태가 기본이며 가장 자주 보이는 화면

#### 상태 B: 기록 완료
```
┌──────────────────────────────┐
│  [기록]              [설정]   │
│                              │
│    퇴근하면 뭐하지?            │  ← 프롬프트 유지
│                              │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │  "카페 가서 노트북    │    │  ← 오늘의 기록 (카드 형태)
│  │   펴고 아무거나 쓰기"  │    │
│  │                      │    │
│  │              19:32   │    │  ← 작성 시각
│  └──────────────────────┘    │
│                              │
└──────────────────────────────┘
```
- 입력 필드 → 기록 카드로 변환
- 영감 카드 영역 숨김 (기록이 주인공)
- 기록 카드 탭 시 수정 모드 진입 (P1)

#### 상태 C: 영감 카드 확대
```
┌──────────────────────────────┐
│                              │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │    동네 산책           │    │  ← 확대된 카드
│  │                      │    │
│  │  가까운 곳부터.        │    │  ← 부가 설명
│  │  이어폰 없이.          │    │
│  │                      │    │
│  └──────────────────────┘    │
│                              │  ← 배경 딤 처리
└──────────────────────────────┘
```
- 모달/오버레이 형태
- 배경 딤 처리 + 카드 중앙 확대
- 바깥 영역 탭 시 닫힘

### 빈 상태 (Empty State)

**첫 방문 빈 상태 = 미기록 상태와 동일**
- 별도 온보딩 없음
- 프롬프트 + 영감 카드 + 입력 필드가 곧 안내

### 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 자정 직전 작성 | 작성 시점의 날짜로 저장. 자정 후 새 프롬프트 표시 |
| 자정 넘어서 앱 유지 중 | 화면 갱신하지 않음. 앱 백그라운드→포그라운드 전환 시 갱신 |
| 100자 정확히 입력 | 카운터 "100/100" 표시. 추가 입력 차단. 삭제는 가능 |
| 빈 문자열 저장 시도 | 저장 버튼 비활성 유지. 공백만 있는 경우도 비활성 |
| 입력 중 앱 종료 | 미저장 텍스트 유실. 드래프트 저장 없음 (한 줄이므로 부담 낮음) |
| 동일 날 2회 저장 시도 | 기록 완료 상태에서는 입력 필드 미표시. 수정만 가능 (P1) |

## 화면 2: 기록

### 목적
지금까지 적은 한 줄들을 시간순으로 돌아보는 화면. 축적의 가치를 느끼게 한다.

### 레이아웃

```
┌──────────────────────────────┐
│  ← 뒤로      기록     [캘린더]│  ← 상단 바 (캘린더 토글 P2)
│                              │
│  ┌──────────────────────┐    │
│  │  이번 주                │    │  ← 이번 주 카드 (P1)
│  │  7일 중 4일 기록        │    │
│  │  산책, 글쓰기, 요리...  │    │
│  └──────────────────────┘    │
│                              │
│  ── 4월 2일 수요일 ──────── │  ← 날짜 구분선
│                              │
│  "카페 가서 노트북           │  ← 기록 항목
│   펴고 아무거나 쓰기"         │
│  퇴근하면 뭐하지?    19:32   │  ← 해당 날 프롬프트 + 시각
│                              │
│  ── 4월 1일 화요일 ──────── │
│                              │
│  "레고 하나 사볼까"           │
│  손이 심심한 밤.      20:15   │
│                              │
│  ── 3월 31일 월요일 ─────── │
│                              │
│  "동네 산책하면서              │
│   팟캐스트 듣기"              │
│  오늘 저녁, 뭐 해볼까. 18:42 │
│                              │
└──────────────────────────────┘
```

### UI 요소

| # | 요소 | 타입 | 설명 |
|---|------|------|------|
| 1 | 상단 바 | Header | 좌측 뒤로 아이콘, 중앙 "기록" 타이틀, 우측 캘린더 토글 아이콘 (P2) |
| 2 | 이번 주 카드 | Card | 기록 화면 최상단. 이번 주 요약. P1 |
| 3 | 날짜 구분선 | Divider | "M월 D일 요일" 형식. 기록 없는 날은 미표시 |
| 4 | 기록 항목 | ListItem | 기록 텍스트 (큰 폰트) + 프롬프트 (작은 폰트, 흐린 색) + 시각 |
| 5 | 스크롤 영역 | ScrollView | 세로 스크롤. 전체 기록 시간순 (최신 상단) |

### 인터랙션

| 액션 | 동작 |
|------|------|
| 세로 스크롤 | 과거 기록 탐색 |
| 기록 항목 탭 | 수정 모드 진입 (P1): 텍스트 수정 가능. 저장/취소 버튼 |
| 기록 항목 길게 누르기 | 삭제 옵션 바텀 시트 표시 (P1) |
| 이번 주 카드 탭 | 이번 주 기록만 필터링하여 표시 (P1) |
| 캘린더 아이콘 탭 | 리스트 ↔ 캘린더 뷰 토글 (P2) |
| 뒤로 | 메인(오늘) 화면으로 복귀 |

### 상태별 서브뷰

#### 상태 A: 리스트 뷰 (기본)
위 레이아웃 참조. 기록이 있는 날만 표시.

#### 상태 B: 캘린더 뷰 (P2)
```
┌──────────────────────────────┐
│  ← 뒤로      기록     [리스트]│
│                              │
│         2026년 4월            │
│  일  월  화  수  목  금  토    │
│            1   2   3   4   5 │
│            ●   ●       ●     │  ← 기록한 날에 점
│  6   7   8   9  10  11  12   │
│                              │
│  ...                         │
│                              │
│  ── 선택한 날짜의 기록 ────── │
│  "카페 가서 노트북             │
│   펴고 아무거나 쓰기"          │
└──────────────────────────────┘
```
- 월간 캘린더 그리드
- 기록한 날에 점 표시 (색상 1가지. 연속/스트릭 하이라이트 없음)
- 날짜 탭 시 하단에 해당 날 기록 표시
- 기록 없는 날 탭 시 하단 영역 비움

#### 상태 C: 빈 상태 (기록 0개)
```
┌──────────────────────────────┐
│  ← 뒤로      기록             │
│                              │
│                              │
│                              │
│         아직 없어요.           │  ← 건조한 안내
│         첫 한 줄은             │
│         오늘 화면에서.          │
│                              │
│                              │
└──────────────────────────────┘
```
- 이번 주 카드 미표시
- 캘린더 토글 미표시

### 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 기록 100개 이상 | 무한 스크롤. 성능 최적화 (가상화 리스트) |
| 기록 수정 중 앱 종료 | 미저장 변경 유실 (수정 전 텍스트 유지) |
| 삭제 후 이번 주 카드 | 삭제 반영하여 카드 업데이트. 이번 주 기록 0개 시 카드 숨김 |
| 같은 달에 기록 1개만 | 캘린더 뷰에 점 1개. 리스트 뷰에 날짜 구분선 1개 + 기록 1개 |

## 화면 3: 이번 주 카드 (기록 화면 내 컴포넌트)

### 목적
이번 주 기록을 요약하여 축적의 가치를 전달한다. 기록 화면 상단에 위치.

### 레이아웃

#### 접힌 상태 (기본)
```
┌──────────────────────────────┐
│  이번 주                   ▼  │
│  7일 중 4일 · 산책, 글쓰기... │
└──────────────────────────────┘
```

#### 펼친 상태 (탭 시)
```
┌──────────────────────────────┐
│  이번 주                   ▲  │
│  7일 중 4일 기록              │
│                              │
│  월  "레고 하나 사볼까"       │
│  화  "동네 산책"              │
│  수  —                       │
│  목  "카페에서 글쓰기"        │
│  금  —                       │
│  토  "새 음악 찾기"           │
│  일  —                       │
└──────────────────────────────┘
```

### UI 요소

| # | 요소 | 타입 | 설명 |
|---|------|------|------|
| 1 | 카드 컨테이너 | Card | 기록 화면 최상단. 배경색 구분 |
| 2 | 기록일 수 | Text | "7일 중 N일" 형식 |
| 3 | 키워드 요약 | Text | 기록에서 추출한 주요 단어 2~3개 (단순 빈도 기반) |
| 4 | 요일별 기록 | List | 펼친 상태에서만 표시. 요일 + 기록 텍스트 (또는 대시) |
| 5 | 토글 아이콘 | Icon | ▼/▲ 화살표 |

### 인터랙션

| 액션 | 동작 |
|------|------|
| 카드 탭 | 접힌/펼친 상태 토글 |
| 요일별 기록 탭 | 해당 날짜 기록으로 스크롤 (기록 리스트 내) |

### 빈 상태
- 이번 주 기록 0개: 카드 자체 미표시
- 이번 주 기록 1개: 카드 표시. "7일 중 1일" + 기록 내용

### 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 월요일 첫 기록 전 | 카드 미표시 (이번 주 기록 0개) |
| 주의 기준 | 월요일~일요일 (ISO 8601) |
| 기록 삭제로 이번 주 0개 | 카드 즉시 숨김 |

## 화면 4: 설정

### 목적
앱의 최소한의 커스터마이징과 데이터 관리.

### 레이아웃

```
┌──────────────────────────────┐
│  ← 뒤로      설정             │
│                              │
│  알림 ─────────────────────  │
│                              │
│  알림 받기              [OFF] │  ← 토글 스위치
│  알림 시간           19:00 > │  ← 알림 ON 시만 표시
│                              │
│  데이터 ───────────────────  │
│                              │
│  데이터 내보내기            > │
│  모든 데이터 삭제           > │  ← 빨간 텍스트
│                              │
│  정보 ─────────────────────  │
│                              │
│  버전                  1.0.0 │
│                              │
└──────────────────────────────┘
```

### UI 요소

| # | 요소 | 타입 | 설명 |
|---|------|------|------|
| 1 | 상단 바 | Header | 좌측 뒤로 아이콘, 중앙 "설정" 타이틀 |
| 2 | 알림 섹션 | Section | 알림 토글 + 시간 설정 |
| 3 | 알림 토글 | Switch | ON/OFF. 기본값 OFF |
| 4 | 알림 시간 | TimePicker | 시/분 선택. 토글 ON 시만 표시 |
| 5 | 데이터 내보내기 | Button/Row | 탭 시 JSON 파일 생성 → 공유 시트 |
| 6 | 모든 데이터 삭제 | Button/Row | 빨간 텍스트. 탭 시 확인 다이얼로그 |
| 7 | 버전 정보 | Text | 앱 버전 번호. 탭 불가 |

### 인터랙션

| 액션 | 동작 |
|------|------|
| 알림 토글 ON | 시스템 알림 권한 요청 → 허용 시 시간 설정 표시 |
| 알림 토글 ON → 권한 거부 | 토글 OFF 복귀 + "설정에서 알림을 허용해주세요" 안내 |
| 알림 시간 탭 | 시간 피커 표시 (시/분) |
| 데이터 내보내기 탭 | JSON 파일 생성 → OS 공유 시트 표시 |
| 데이터 내보내기 (기록 0개) | "내보낼 데이터가 없어요" 토스트 |
| 모든 데이터 삭제 탭 | 1차 확인: "정말 삭제하시겠어요?" [취소/삭제] |
| 삭제 1차 확인 | 2차 확인: "되돌릴 수 없어요. 삭제할까요?" [취소/삭제] |
| 삭제 2차 확인 | 전체 데이터 삭제 → 메인 화면으로 이동 (첫 방문 상태) |
| 뒤로 | 메인(오늘) 화면으로 복귀 |

### 빈 상태
- 설정 화면 자체는 빈 상태 없음 (항상 동일 구조)
- 데이터 내보내기: 기록 0개 시 토스트 메시지로 안내

### 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 알림 권한 이전에 거부됨 | 토글 ON 시 시스템 설정으로 이동 안내 |
| 알림 시간 = 과거 시각 | 다음 날부터 적용 |
| 내보내기 중 앱 종료 | JSON 생성은 동기 처리 (데이터 소량). 문제 없음 |
| 전체 삭제 중 앱 종료 | IndexedDB 트랜잭션 롤백. 재시도 필요 |
| 데이터 1,000개 이상 내보내기 | JSON 파일 크기 1MB 미만. 성능 이슈 없음 |

## 프롬프트 톤 가이드

### 사용하는 톤

| 유형 | 예시 |
|------|------|
| 직접적 질문 | "퇴근하면 뭐하지?" |
| 짧은 제안 | "오늘 저녁, 뭐 해볼까." |
| 감각적 자극 | "손이 심심한 밤." |
| 상황 묘사 | "집까지 20분." |
| 가벼운 도발 | "어제랑 다른 거 하나." |

### 사용하지 않는 톤

| 유형 | 예시 (금지) |
|------|------------|
| 위로/격려 | "오늘 하루 수고했어요!" |
| 친한 척 | "오늘은 뭐 했어~?" |
| 감정 유도 | "오늘 기분이 어때요?" |
| 압박/유도 | "어제는 안 적었네요, 오늘은 적어볼까요?" |
| 과잉 이모지 | 어떤 이모지든 프롬프트에 사용 금지 |
