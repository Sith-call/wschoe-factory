## 우선 보완 항목 (P0) -- 정확성/로직 오류

### P0-1. PPFLab 기회비용 공식 오류 (PPFLab.tsx:122)

**현재 코드:**
```
기회비용: 식량 1단위 = 공산품 {Math.abs(Math.round(pointX / ppfY(pointX) * 10) / 10)}단위
```

**문제:** 원형 PPF (`x^2 + y^2 = R^2`)에서 기회비용은 PPF 접선의 기울기(MRT)로, `dy/dx = -x / sqrt(R^2 - x^2)`이다. 현재 코드는 `x/y`를 계산하고 있는데, 이것은 기울기가 아니라 원점에서의 비율(ray slope)이다. 결과적으로 PPF 끝점 근처에서 기회비용이 과소평가된다.

**수정 공식:**
```typescript
const mrt = pointX / Math.sqrt(MAX * MAX - pointX * pointX);
// 기회비용: 식량 1단위 = 공산품 {mrt.toFixed(1)}단위
```

참고: `x/y = x / sqrt(R^2 - x^2)`이므로 실제로 값이 같아지는 케이스가 있으나, 이는 우연의 일치이며 개념적으로 "접선의 기울기"와 "원점으로부터의 비율"은 다른 의미이다. 학생에게 정확한 개념을 전달하려면 MRT(한계전환율)임을 명시해야 한다.

### P0-2. GDPLab 순수출(NX) 파이차트 왜곡 (GDPLab.tsx:23)

**현재 코드:**
```typescript
{ label: '순수출(NX)', value: Math.max(0.5, NX + 10), color: '#DC2626', pct: 0 },
```

**문제:** NX가 음수일 때 `NX + 10`으로 변환하여 파이차트에 넣고 있다. NX = -3이면 value = 7이 되어, 실제 GDP 기여도(-3)와 시각적 표현(7/총합)이 완전히 불일치한다. 경제학적으로 순수출이 음수이면 GDP에서 차감되어야 하는데, 파이차트에서는 양의 비중으로 표시된다.

**수정 방향:**
- 파이차트 대신 적층 막대그래프(stacked bar)를 사용하여 NX 음수를 차감으로 표현
- 또는 NX 음수일 때 별도의 "수입 초과분" 표시 + 파이차트에서 제외하고 주석 처리
- GDP 총합 계산은 `C + I + G + NX`를 유지하되, 시각화에서 NX < 0일 때 "순수입"으로 명확히 표기

### P0-3. MonopolyLab의 setState in render (MonopolyLab.tsx:31-34)

**현재 코드:**
```typescript
if (profit > maxProfit) {
    setTimeout(() => {
      setMaxProfit(profit);
      if (Math.abs(profit - optimalProfit) < 50) setFoundMax(true);
    }, 0);
  }
```

**문제:** 컴포넌트 렌더링 본문에서 `setTimeout + setState`를 호출한다. React strict mode에서 무한 리렌더링 루프를 유발할 수 있다. `useEffect`로 이동해야 한다.

### P0-4. ComparativeAdvantageLab 동일 기회비용 미처리

**문제:** `aOcWine === bOcWine`일 때 (예: A국 와인10/옷감20, B국 와인20/옷감40) 비교우위가 존재하지 않는다. 그러나 현재 코드는 항상 한쪽에 "특화"를 부여한다. 이 경우 "비교우위 없음 -- 무역 이익 없음"을 표시해야 한다.

### P0-5. MoneySupplyLab(IS-LM) 유동성 함정 미구현

**문제:** IS-LM 모형의 핵심 교훈 중 하나인 유동성 함정(liquidity trap)이 빠져 있다. 이자율이 0% 근처일 때 통화량을 아무리 늘려도 소득이 증가하지 않는 현상은 2008년 이후 실제로 관찰된 중요한 경제 현상이다. 현재 LM곡선은 선형이라 이 효과를 보여줄 수 없다.

## 교육적 개선 제안 (P1)

### P1-1. ConsumerSurplusLab에 생산자잉여(PS) + 총잉여(TS) 추가

소비자잉여만 보여주면 후생경제학의 핵심인 "총잉여 극대화 = 사회적 효율"을 전달할 수 없다. 공급곡선을 추가하고 생산자잉여(PS = 시장가격 - 공급곡선 아래 면적)를 빨간색 삼각형으로 표시하면:
- CS + PS = Total Surplus 공식을 시각적으로 이해 가능
- 가격통제(최고/최저가격)가 DWL을 만드는 실험으로 확장 가능
- ExternalityLab, MonopolyLab과의 개념적 연결 강화

### P1-2. MultiplierLab에 조세승수와 균형예산승수 비교

현재 지출승수(1/(1-MPC))만 보여주는데, 재정정책의 전체 그림을 위해 추가 필요:
- **조세승수** = -MPC/(1-MPC): 감세의 효과는 정부지출보다 작다
- **균형예산승수** = 1: 세금을 걷어 그만큼 지출하면 승수는 정확히 1
- 이 비교를 통해 "같은 100원이라도 정부가 직접 쓰는 것과 감세하는 것의 효과가 다르다"는 핵심 통찰 제공

### P1-3. InflationLab에 72법칙 표시

72법칙은 복리 효과를 직관적으로 이해하는 가장 강력한 도구:
```
구매력 반감 기간 = 72 / 인플레이션율(%)
```
- 인플레이션 3%: 약 24년 후 구매력 절반
- 인플레이션 7%: 약 10년 후 구매력 절반
- 인플레이션 10%: 약 7년 후 구매력 절반

현재 10년 시뮬레이션에 이 수치를 오버레이하면 학습 효과가 크게 증가한다.

### P1-4. ExchangeRateLab에 J-커브 효과 추가

현재 환율 절하가 즉시 무역수지를 개선하는 것으로 모델링되어 있으나, 현실에서는:
1. **단기(0-6개월):** 기존 계약의 수입 결제액 증가로 무역수지 악화
2. **중기(6-18개월):** 수출량 증가 + 수입량 감소로 무역수지 개선

시간축 슬라이더를 추가하여 환율 변동 후 1개월, 6개월, 18개월 시점의 무역수지를 보여주면 실제 정책 분석에 필요한 시간 개념을 교육할 수 있다. 이는 마셜-러너 조건(수출입 수요탄력성 합 > 1이면 절하가 무역수지 개선)과도 연결된다.

### P1-5. MoneySupplyLab을 완전한 IS-LM으로 확장

현재 LM곡선만 이동 가능한데, IS곡선(재정정책)도 이동 가능하게 만들면:
- **통화정책 vs 재정정책** 효과 비교 가능
- IS 이동(정부지출 증가): 소득 증가 + 이자율 상승 (구축효과 시각화)
- LM 이동(통화량 증가): 소득 증가 + 이자율 하락
- 두 정책의 동시 사용(policy mix)도 실험 가능

재정정책 슬라이더를 추가하면 MultiplierLab과의 연결도 강화된다.
