# 감정 처방전 (Mood Pharmacy) — 유저 스토리

## 스크린 플로우

```
[1. Intro] → [2. EmotionSelect] → [3. Intensity] → [4. Context]
                                                         ↓
[8. Feedback] ← [7. Checklist] ← [6. Prescription] ← [5. Analysis]
      ↓
 localStorage 저장 → [8. History 영역] or [1. Intro]로 복귀
```

**조건부 플로우:**
- Intro → EmotionSelect: "시작하기" 탭
- EmotionSelect: 1~3개 멀티셀렉트 → 선택 없으면 "다음" 비활성
- Intensity: 선택된 감정 수만큼 슬라이더 카드 동적 생성
- Analysis: 2~3초 로딩 애니메이션 후 자동 전환
- Checklist: 모든 항목 완료 시 "기록하기" 버튼 활성
- Feedback → 저장 후 Intro로 복귀

---

## US-01: 앱 인트로 및 진입

**AS** 사용자
**I WANT** 앱의 목적을 한눈에 이해하고 진단을 시작할 수 있기를
**SO THAT** 별도 설명 없이 바로 감정 처방 플로우에 진입할 수 있다

### 인수 조건
- [ ] 처방전 병 일러스트 + "오늘의 감정을 처방해드릴게요" 헤드라인 표시
- [ ] "지금 시작하기" CTA 버튼 → EmotionSelectScreen 이동
- [ ] 하단 "이전 처방 기록 보기" 텍스트 링크 → FeedbackScreen(히스토리 영역)으로 이동
- [ ] 첫 방문 시에도 DB/로그인 없이 바로 사용 가능 (데모 모드)

---

## US-02: 감정 선택 (멀티셀렉트)

**AS** 사용자
**I WANT** 현재 느끼는 감정을 1~3개 골라서 선택할 수 있기를
**SO THAT** 복합적인 감정 상태를 정확하게 반영할 수 있다

### 인수 조건
- [ ] 2열 그리드로 감정 카드 표시 (기쁨, 슬픔, 불안, 분노, 무기력, 외로움, 설렘, 지침)
- [ ] 각 카드: Material Symbol 아이콘 + 감정 라벨
- [ ] 최소 1개, 최대 3개 멀티셀렉트 — 선택 시 보라색 테두리 + 체크 뱃지
- [ ] 3개 초과 선택 시도 시 "최대 3개까지" 토스트 또는 선택 불가 처리
- [ ] 0개 상태에서 "다음" 버튼 비활성 (disabled)
- [ ] 프로그레스 바: Step 1/4 (25%)
- [ ] 뒤로가기 → IntroScreen

---

## US-03: 감정 강도 조절 (동적 슬라이더)

**AS** 사용자
**I WANT** 선택한 감정 각각의 강도를 슬라이더로 조절할 수 있기를
**SO THAT** 감정의 세기까지 반영한 맞춤 처방을 받을 수 있다

### 인수 조건
- [ ] US-02에서 선택한 감정 수만큼 카드 동적 렌더링 (1~3장)
- [ ] 각 카드: 아이콘 + 감정명 + 5단계 슬라이더 (1: 살짝 ~ 5: 매우)
- [ ] 슬라이더 기본값: 3 (보통)
- [ ] 현재 단계 텍스트 라벨 실시간 업데이트 ("살짝", "조금", "보통", "꽤", "매우")
- [ ] 프로그레스 바: Step 2/4 (50%)
- [ ] 뒤로가기 → EmotionSelectScreen (선택 상태 유지)

---

## US-04: 상황/맥락 선택 (단일 선택)

**AS** 사용자
**I WANT** 현재 감정의 원인이 되는 상황을 선택할 수 있기를
**SO THAT** 상황에 맞는 구체적인 처방을 받을 수 있다

### 인수 조건
- [ ] 2열 그리드로 상황 카드 표시 (직장/학교, 연인/가족, 대인관계, 건강, 재정, 자기자신, 기타)
- [ ] 단일 선택 — 선택 시 보라색 테두리 + "선택됨" 라벨
- [ ] 각 카드: Material Symbol 아이콘 + 상황 라벨
- [ ] 선택 없이 "다음" 불가
- [ ] 프로그레스 바: Step 3/4 (75%)
- [ ] 뒤로가기 → IntensityScreen (슬라이더 값 유지)

---

## US-05: 감정 분석 로딩 애니메이션

**AS** 사용자
**I WANT** 내 감정 데이터가 분석되는 과정을 시각적으로 확인할 수 있기를
**SO THAT** 처방 결과에 대한 신뢰감과 기대감을 느낄 수 있다

### 인수 조건
- [ ] 다크 배경 (deep purple-black) 전환 — 몰입감 있는 분위기
- [ ] 중앙 원형 프로그레스 링 + 봉투 아이콘 애니메이션
- [ ] "당신의 감정을 분석하고 있어요" → "맞춤 처방전을 작성 중이에요" 텍스트 순차 전환
- [ ] 하단 "감정 분석 완료" 체크 표시 + 단계별 완료 인디케이터
- [ ] 2~3초 후 자동으로 PrescriptionScreen 전환 (사용자 액션 불필요)
- [ ] 프로그레스 바 없음 (전체 화면 몰입 UI)

---

## US-06: 처방전 카드 결과 확인

**AS** 사용자
**I WANT** 내 감정 상태에 대한 진단명과 맞춤 처방 활동을 처방전 형태로 받아볼 수 있기를
**SO THAT** 실제 처방전 같은 경험으로 감정 케어에 동기부여를 받을 수 있다

### 인수 조건
- [ ] 따뜻한 크림색 배경 + 종이 텍스처 처방전 카드
- [ ] 카드 헤더: "처방전" 제목 (Noto Serif KR) + "Prescription" 영문 서브
- [ ] 환자 정보: 환자명("오늘의 나"), 발행일(오늘 날짜), 담당의("Dr. 무드")
- [ ] 진단명: 선택한 감정+강도+상황 조합으로 동적 생성 (예: "복합 스트레스성 무기력증")
- [ ] 처방 항목 3개: 각각 아이콘 + 활동 설명 + 긴급도 태그(즉시/오늘/이번주) + 소요시간
- [ ] 하단 약국 도장 영역 + "처방 실행하기" CTA → ChecklistScreen 이동
- [ ] 처방 내용은 감정+상황 조합에 따라 다르게 매핑 (최소 5가지 조합)

---

## US-07: 처방 체크리스트 실행

**AS** 사용자
**I WANT** 처방받은 활동을 하나씩 체크하며 완료할 수 있기를
**SO THAT** 처방을 실제로 실행하고 성취감을 느낄 수 있다

### 인수 조건
- [ ] 체크리스트 전용 테마 적용 (테라코타 #D67D61, Newsreader 서체)
- [ ] 상단 원형 프로그레스 링: 완료율 실시간 반영 (0% → 33% → 66% → 100%)
- [ ] 격려 문구 표시 ("천천히 하나씩 해봐요.")
- [ ] 처방 항목 3개 카드: 미완료(흰색) / 진행중(보라 테두리 강조) / 완료(녹색 배경 + 취소선)
- [ ] 각 카드 탭 → 완료 토글 (완료 시 체크 애니메이션)
- [ ] 타이머 표시: 각 항목의 권장 소요시간 카운트다운 (선택적)
- [ ] 전체 완료 시 "기록하기" 버튼 활성 → FeedbackScreen 이동
- [ ] 부분 완료 상태에서도 "건너뛰기" 가능

---

## US-08: 처방 후 피드백 및 기분 기록

**AS** 사용자
**I WANT** 처방 실행 후 현재 기분을 기록하고 변화를 확인할 수 있기를
**SO THAT** 감정 변화를 인지하고 처방의 효과를 체감할 수 있다

### 인수 조건
- [ ] "처방 후 기분이 어때요?" 질문 + 5단계 이모지 스케일 (별로~최고)
- [ ] 선택 시 해당 이모지 확대 + 보라색 하이라이트
- [ ] Emotional Shift 영역: Before(선택했던 감정) → After(현재 기분) 시각적 비교
- [ ] 한줄 메모 입력 (선택, 50자 제한)
- [ ] "저장하기" → localStorage에 기록 저장
- [ ] 저장 완료 후 성공 토스트 + IntroScreen 복귀

---

## US-09: 감정 히스토리 조회

**AS** 사용자
**I WANT** 이전 처방 기록을 날짜별로 볼 수 있기를
**SO THAT** 나의 감정 패턴을 파악하고 자기 이해를 높일 수 있다

### 인수 조건
- [ ] FeedbackScreen 하단 "이번 주 기록" 영역에 최근 7일 표시
- [ ] 각 날짜: 원형 아이콘(기분 이모지) + 날짜 + 한줄 요약
- [ ] 기록이 없는 날짜는 빈 원형으로 표시
- [ ] "chevron_right" 아이콘 — 추후 상세 히스토리 확장 가능
- [ ] IntroScreen "이전 처방 기록 보기" 링크로도 접근 가능
- [ ] localStorage에서 데이터 로드

---

## US-10: 데모 모드 및 처방 로직

**AS** 개발자/시연자
**I WANT** DB 없이 완전한 서비스 플로우를 체험할 수 있기를
**SO THAT** 백엔드 구축 전에도 앱의 핵심 가치를 검증할 수 있다

### 인수 조건
- [ ] 모든 데이터 localStorage 기반 (서버 통신 없음)
- [ ] 처방 생성 로직: 감정 조합 + 강도 + 상황 → 진단명 매핑 테이블
- [ ] 처방 활동 풀: 최소 15개 활동, 조건별 3개씩 선택
- [ ] 진단명 생성 규칙: `[상황 수식어] + [주요 감정 기반 증상명]` (예: "업무 과부하성 불안 증후군")
- [ ] 앱 상태 관리: React state (전역 Context 또는 zustand)
- [ ] 히스토리 데이터 구조: `{ date, emotions, intensities, context, mood_after, memo }`
- [ ] 초기 진입 시 샘플 히스토리 3일치 시드 데이터 제공

| 항목 | 소비유형 퀴즈 (비교) | 감정 처방전 |
|------|----------------------|-------------|
| 유저 스토리 수 | 4 | 10 |
| 화면 수 | 4 | 8 |
| 입력 유형 | 단일 선택 | 멀티셀렉트, 슬라이더, 단일선택, 이모지 스케일, 텍스트 |
| 조건부 플로우 | 없음 | 감정 수에 따른 동적 UI, 완료 조건부 버튼 활성 |

| 처방 로직 | 단순 매핑 | 감정x강도x상황 조합 매핑 |
| 테마 | 단일 | 듀얼 테마 (메인 + 체크리스트) |
| 애니메이션 | 없음 | 분석 로딩, 프로그레스 링, 체크 토글 |
