# 수면 일지 (Sleep Log) — Product Requirements Document

> 버전: 1.0
> 작성일: 2026-03-21
> 상태: MVP 정의 완료

---

## 1. 문제 정의

### 배경
한국 직장인의 70%가 수면 부족을 호소하지만, 대부분 "어젯밤 잘 잤나?"를 직감으로만 판단한다. 수면 앱 시장은 대부분 웨어러블 연동 또는 소리 녹음 기반이라 진입 장벽이 높고, "간단히 기록만 하고 싶은" 유저의 니즈를 충족하지 못한다.

### 핵심 문제
1. **인식 부재**: 자신의 수면 패턴을 객관적으로 파악하지 못한다.
2. **요인 미파악**: 어떤 행동이 수면에 영향을 주는지 감으로만 추측한다.
3. **개선 동기 부족**: 수면 문제를 인지해도 "뭘 바꿔야 하는지" 모른다.

### 기회
- 웨어러블 없이 30초 수동 기록만으로도 유의미한 수면 패턴 분석이 가능하다.
- 1주일 분량의 데이터만 모이면 요인-수면질 상관관계를 도출할 수 있다.
- "기록 → 패턴 발견 → 행동 변화"의 자연스러운 행동 변화 루프를 만들 수 있다.

---

## 2. 핵심 가치

> **"기록하면 보인다. 보이면 바뀐다."**

수면 일지는 "진단 도구"가 아니라 "거울"이다. 유저가 자신의 수면을 객관적으로 바라보게 만드는 것이 목표다. 코칭하지 않고, 판단하지 않고, 데이터를 보여준다.

## 3. 타겟 유저

### 페르소나 1: 김민준 (28세, 스타트업 개발자)

| 항목 | 상세 |
|------|------|
| 직업 | 시리즈A 스타트업 백엔드 개발자 |
| 평균 수면 | 5.5시간 |
| 취침 시간 | 새벽 1:30~2:30 (불규칙) |
| 기상 시간 | 8:00 알람 (스누즈 2~3회) |
| 주요 고민 | 카페인 과다 → 불면 → 피로 → 카페인 악순환. "커피를 줄여야 하는 건 아는데, 안 마시면 오후에 쓰러질 것 같다." |
| 원하는 것 | "내 수면 패턴이 어떤지 객관적 데이터로 보고 싶다. 커피를 줄이면 진짜 수면이 나아지는지 확인하고 싶다." |
| 기술 친화도 | 높음 — 데이터 기반 의사결정 선호, 차트/통계에 익숙 |
| 앱 사용 맥락 | 아침 출근 준비 중 (8:00~8:30), 한 손으로 빠르게 기록 |
| 실패 시나리오 | 입력이 3단계 넘으면 "귀찮다" → 이탈. 데이터가 쌓이기 전에 인사이트가 없으면 흥미 상실. |

**민준의 하루 (As-Is):**
- 08:00 — 알람, 스누즈 3번
- 08:20 — 겨우 기상, "또 피곤하다" 한숨
- 09:00 — 출근, 아메리카노 1잔
- 14:00 — 점심 후 졸음, 아메리카노 추가
- 18:00 — 야근 시작, 에너지 드링크
- 01:30 — 침대에 누웠는데 카페인 때문에 잠이 안 옴
- 02:30 — 겨우 잠듦

**민준의 하루 (To-Be, 앱 사용 2주 후):**
- 08:00 — 기상, 앱에서 30초 기록 (어제: 6시간, 질 3, 요인: 카페인)
- 08:05 — 주간 차트 확인: "카페인 없는 날 수면 질 3.8 vs 있는 날 2.1"
- 09:00 — 오전 커피 한 잔만 마시기로 결심
- 00:30 — 평소보다 1시간 일찍 취침
- 07:00 — 자연 기상, "확실히 다르다"

**서연의 핵심 저니:**
- 1주차: 기록 시작 → 주중/주말 수면 시간 격차 시각화

- 3주차: 행동 변화 시도 — 주중 취침 30분 앞당기기
- 4주차: 결과 확인 — "주중 6.5시간으로 개선, 주말 8시간으로 격차 축소"

| 원하는 것 | "어떤 날 잘 자고 어떤 날 못 자는지 요인을 알고 싶다. 기록은 최대한 간단하게." |
| 기술 친화도 | 낮음 — 앱 복잡하면 바로 삭제. 버튼 2~3번 탭으로 끝나야 함 |

**재호의 핵심 발견 (앱 사용 3주 후):**
- "음주한 날 수면 질 평균 1.8, 안 한 날 4.1 → 회식 다음 날 피로 원인 확인"
- "야식 먹은 날 수면 질 2.5 → 저녁 이후 음식 자제"
- "운동한 날 수면 질 4.5 → 주 3회 운동 목표 설정 동기 부여"

## 4. 유저 저니 맵

### 4.1 전체 여정 (5단계)

```
발견 → 온보딩 → 첫 기록 → 패턴 발견 → 행동 변화
(Day 0) (Day 0) (Day 1) (Day 7) (Day 14+)
```

### 4.2 감정 곡선

| 단계 | 유저 감정 | 핵심 행동 | 리스크 |
|------|-----------|-----------|--------|
| 발견 | 호기심 ("수면 기록? 해볼까") | 앱 설치 | "또 수면 앱이야?" → 차별점 미인지 |
| 온보딩 | 기대감 ("간단하네") | 목표 설정 + 데모 데이터 확인 | 설정 과다 → 이탈 |
| 첫 기록 | 성취감 ("30초만에 끝이네") | 아침 기록 완료 | 다음 날 잊음 → 리마인더 필요 |
| 패턴 발견 | 놀라움 ("카페인 영향 이렇게 크네") | 인사이트 확인 | 데이터 부족 → 인사이트 부정확 |
| 행동 변화 | 동기 부여 ("확실히 달라졌다") | 수면 습관 개선 | 슬럼프 → 기록 중단 |

### 4.3 터치포인트별 상세

**발견 단계:**
- 채널: 카카오톡 공유 카드, 블로그 후기, 동료 추천
- 첫인상: "웨어러블 없이 수면 패턴을 알 수 있다고?"
- 전환 포인트: "30초면 기록 끝" 메시지

**온보딩 단계 (3 스텝, 총 45초 이내):**
- 스텝 1: 가치 제안 ("수면을 기록하면 패턴이 보입니다") → 시작하기
- 스텝 2: 수면 목표 설정 (슬라이더, 기본 7.5시간)
- 스텝 3: 평소 취침/기상 시간 (시계 피커)
- 완료 시: 최근 7일 샘플 데이터 자동 생성 → 앱이 비어 보이지 않게
- 탈출구: 각 스텝에 "건너뛰기" 옵션

**첫 기록 단계 (30초 이내):**
- 트리거: 아침 알림 ("좋은 아침! 어젯밤 수면을 기록해볼까요?")
- 행동: 4단계 탭 플로우 (취침 → 기상 → 질 → 요인)
- 보상: "기록 완료!" + 오늘의 수면 요약 카드
- 가변 보상: 첫 기록 시 "첫 기록 축하!" 메시지

## 6. 데이터 모델

### 6.1 SleepRecord

```typescript
interface SleepRecord {
  id: string;                    // UUID
  date: string;                  // YYYY-MM-DD (기록 대상 날짜)
  bedtime: string;               // HH:mm (취침 시간, 전날 기준)
  wakeTime: string;              // HH:mm (기상 시간)
  duration: number;              // 분 단위 수면 시간 (자동 계산)
  quality: 1 | 2 | 3 | 4 | 5;   // 수면 질
  factors: SleepFactor[];        // 수면 요인 목록
  createdAt: string;             // ISO datetime
  updatedAt: string;             // ISO datetime
  isDemo: boolean;               // 데모 데이터 여부
}

type SleepFactor =
  | 'caffeine'     // 카페인
  | 'exercise'     // 운동
  | 'alcohol'      // 음주
  | 'stress'       // 스트레스
  | 'late_meal'    // 야식
  | 'screen_time'  // 스크린타임
  | 'none';        // 없음
```

### 6.2 UserSettings

```typescript
interface UserSettings {
  goalHours: number;            // 목표 수면 시간 (6~9, 0.5 단위)
  defaultBedtime: string;       // 기본 취침 시간 HH:mm
  defaultWakeTime: string;      // 기본 기상 시간 HH:mm
  reminderEnabled: boolean;     // 알림 ON/OFF
  reminderTime: string;         // 알림 시간 HH:mm
  demoMode: boolean;            // 데모 모드
  onboardingCompleted: boolean; // 온보딩 완료 여부
}
```

### 6.3 저장소

- MVP: **localStorage** (DB 없이 바로 체험 가능한 데모 모드)
- 키: `sleep-log-records`, `sleep-log-settings`
- 향후: IndexedDB 또는 클라우드 동기화

- **테마**: 라이트 모드 (DESIGN_RULES.md R9 준수)

  - Primary: Teal 계열 (#0d9488, 수면/편안함 연상)
  - Surface: #fafbfc (밝은 회백색)
  - Text: #1e293b (진한 네이비 그레이)
  - Accent: #6366f1 (인디고, 밤/수면 연상 — CTA에 제한적 사용)
  - Quality 색상: teal opacity 10%~100% (수면 질 1~5)
- **폰트**: Pretendard (한국어) + Inter (영문/숫자)
- **아이콘**: Lucide 아이콘 세트 통일 (이모지 절대 사용 금지)
- **Border-radius**: 메인 카드 16px, 내부 요소 8px, 버튼 12px (rounded-xl)

### 7.2 인터랙션 원칙
- **입력 최소화**: 모든 입력에 기본값 제공, 탭 한 번으로 선택
- **피드백 즉시**: 모든 인터랙션에 즉각적 시각 반응
- **진행 표시**: 기록 플로우에 단계 표시 (dot indicator)
- **애니메이션**:
  - 화면 전환: 좌→우 슬라이드 (기록 플로우)
  - 차트: scale(0.95→1) 등장
  - 나머지: 즉시 표시 (fadeInUp 금지)

### 7.3 네비게이션 구조
- **상단 세그먼트 컨트롤** 4개: 홈 / 캘린더 / 인사이트 / 설정
- 바텀 탭바 사용 안 함 (DESIGN_RULES.md R5 준수)
- 기록 플로우: 홈에서 CTA → 모달 또는 풀스크린 오버레이
