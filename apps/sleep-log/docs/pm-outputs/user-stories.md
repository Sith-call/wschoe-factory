## 1. 첫 방문 — 온보딩 (Day 0)

### US-1.1: 앱의 가치를 즉시 이해하기

> **유저로서**, 앱을 처음 열면 이 앱이 뭘 해주는지 3초 안에 이해하고 싶다.

**수용 기준:**
- 첫 화면에 핵심 메시지: "수면을 기록하면 패턴이 보입니다"
- 부제: "30초 기록으로 나만의 수면 인사이트를 발견하세요"
- CTA 버튼: "시작하기"
- 화면에 텍스트 3줄 이하, 시각적 일러스트 또는 아이콘 1개

**안티패턴 (하지 말 것):**
- 회원가입/로그인 요구
- 퍼미션 요청 (알림 등은 나중에)
- 3개 이상 온보딩 캐러셀
- 긴 설명 텍스트

**페르소나별 반응:**
- 민준: "깔끔하다. 데이터 기반이라니 끌린다."
- 서연: "심플하네. 일단 시작해보자."
- 재호: "글자가 적어서 좋다. 복잡하면 나갈 뻔했다."

**화면 상세:**

### US-1.2: 수면 목표 설정하기

> **유저로서**, 나의 수면 목표를 설정하고 싶다.

**수용 기준:**
- 화면 제목: "하루에 몇 시간 자고 싶으세요?"
- 슬라이더 또는 +/- 버튼: 6시간 ~ 9시간, 0.5시간 단위
- 기본값: 7.5시간 (한국 성인 권장 수면)
- 선택된 시간 크게 표시: "7.5시간"
- 하단: "다음" 버튼 + "건너뛰기" 텍스트 링크

**인터랙션 상세:**
1. 화면 진입 → 슬라이더 중앙(7.5시간)에 위치
2. 좌우 드래그 또는 +/- 탭 → 값 변경 → 큰 숫자 실시간 업데이트
3. "다음" 탭 → 다음 스텝
4. "건너뛰기" 탭 → 7.5시간 기본값으로 저장 후 다음 스텝

**페르소나별 설정:**
- 민준: 7시간 (현실적 목표, 현재 5.5시간에서 점진적으로)
- 서연: 7.5시간 (기본값 유지)
- 재호: 7시간 (이미 7시간이지만 질을 높이고 싶어서 유지)

**화면 상세:**
```
[GoalScreen]
┌─────────────────────────┐
│  ← (뒤로)    건너뛰기   │
│                         │
│  하루에 몇 시간         │
│  자고 싶으세요?         │
│                         │
│        7.5시간          │
│   (큰 숫자, semibold)   │
│                         │
│  6h ───●─────────── 9h  │
│       (슬라이더)        │
│                         │
│  성인 권장 수면 시간:   │
│  7~9시간               │
│                         │
│  ┌───────────────────┐  │
│  │      다음         │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

### US-1.3: 평소 수면 스케줄 입력하기

> **유저로서**, 평소 취침/기상 시간을 알려주고 싶다.

**수용 기준:**
- 화면 제목: "평소 몇 시에 주무세요?"
- 취침 시간 피커: 기본 23:00
- 기상 시간 피커: 기본 07:00
- 각 피커: 시간 드럼 롤러 (30분 단위) 또는 +/- 30분 버튼
- "완료" 버튼 → 온보딩 종료 + 데모 데이터 생성 + 홈으로 이동

**데모 데이터 생성 로직:**
- 온보딩 완료 시 최근 7일 분량의 샘플 수면 기록 자동 생성
- 설정한 취침/기상 시간을 기반으로 +-30분 랜덤 변동
- 수면 질: 2~5 랜덤 (자연스러운 분포)
- 요인: 랜덤 배정 (카페인 40%, 스트레스 30%, 운동 20%, 없음 50%)
- `isDemo: true` 플래그로 표시

**페르소나별 입력:**
- 민준: 취침 01:30, 기상 08:00 → "5.5시간으로 계산되네... 적다"
- 서연: 취침 00:30, 기상 06:30 → "6시간... 목표 7.5시간이랑 차이 크다"
- 재호: 취침 23:00, 기상 06:00 → "7시간, 일단 양은 충분"

**화면 상세:**

## 2. 첫 기록 — 아침 기록 플로우 (Day 1)

### US-2.1: 30초 안에 수면 기록 완료하기

> **유저로서**, 아침에 일어나서 30초 안에 어젯밤 수면을 기록하고 싶다.

**수용 기준:**
- 4단계 플로우: 취침 → 기상 → 수면 질 → 요인
- 각 단계 1탭(선택)으로 완료 가능
- 상단 프로그레스: 4개 dot indicator
- 총 소요 시간: 30초 이내

**트리거:**
- 홈 화면 "오늘의 기록" 카드의 CTA 버튼
- (향후) 아침 리마인더 알림

**핵심 원칙:**
- "귀찮으면 기록 안 한다"가 최대 리스크
- 모든 입력에 기본값 제공 → 탭만으로 진행 가능
- 뒤로가기로 수정 가능
- 중간 이탈 시 데이터 유실 없음 (마지막 완료 시 저장)

### US-2.2: 취침 시간 기록하기

> **유저로서**, 어젯밤 몇 시에 잤는지 기록하고 싶다.

**수용 기준:**
- 화면 제목: "어젯밤 몇 시에 잤나요?"
- 시간 피커: 드럼 롤러 (15분 단위)
- 기본값: 최근 기록의 취침 시간 (첫 기록 시 온보딩 설정값)
- +/- 15분, +/- 30분 빠른 조절 버튼
- "다음" 버튼

**인터랙션 상세:**
1. 화면 진입 → 기본값 표시 (예: "23:00")
2. 드럼 롤러 스크롤 또는 +/- 버튼 → 시간 변경
3. "다음" 탭 → Step 2로 슬라이드 전환

**엣지 케이스:**
- 자정 넘긴 취침: 01:30처럼 표시, "어젯밤" 레이블 유지
- 극단적 시간 (04:00 이후): "새벽 늦게 주무셨군요" 메시지
- 시간 범위: 18:00 ~ 06:00 (다음 날)

**페르소나별 시나리오:**
- 민준: 기본값 01:30 → 실제 02:00으로 변경 → "+30분" 한 번 탭
- 서연: 기본값 00:30 → 맞아서 바로 "다음"
- 재호: 기본값 23:00 → 맞아서 바로 "다음" (1초 소요)

### US-2.3: 기상 시간 기록하기

> **유저로서**, 오늘 몇 시에 일어났는지 기록하고 싶다.

**수용 기준:**
- 화면 제목: "오늘 몇 시에 일어났나요?"
- 시간 피커: 드럼 롤러 (15분 단위)
- 기본값: 현재 시간 (반올림, 08:07 → 08:00)
- 자동 계산: 총 수면 시간 실시간 표시
- 수면 시간 표시: "6시간 30분 잤어요"

**인터랙션 상세:**
1. 화면 진입 → 현재 시간 기본값 + 자동 계산된 수면 시간 표시
2. 시간 변경 → 수면 시간 실시간 업데이트
3. "다음" 탭 → Step 3로 슬라이드 전환

**자동 계산 로직:**
- 취침 23:00, 기상 07:00 → 8시간 0분
- 취침 01:30, 기상 08:00 → 6시간 30분
- 자정 교차 자동 처리

### US-2.4: 수면 질 직관적으로 평가하기

> **유저로서**, 수면의 질을 직관적으로 평가하고 싶다.

**수용 기준:**
- 화면 제목: "수면의 질은 어땠나요?"
- 5단계 선택 (가로 배열, SVG 아이콘)
  - 1: 최악 (거의 못 잠) — SVG: 깨진 달 (cracked moon)
  - 2: 나쁨 (자주 깸) — SVG: 구름 낀 달 (cloudy moon)
  - 3: 보통 — SVG: 반달 (half moon)
  - 4: 좋음 (숙면) — SVG: 보름달 (full moon)
  - 5: 최고 (개운) — SVG: 빛나는 별 (shining star)
- 선택 시: 아이콘 확대(scale 1→1.2) + 등급 텍스트 표시
- 기본 선택: 없음 (반드시 선택해야 다음 진행)

**이모지 사용 절대 금지 (DESIGN_RULES.md R13):**
- 수면 질 아이콘은 반드시 SVG로 구현
- 일관된 스타일의 line/stroke 아이콘
- teal 계열 색상으로 통일

**인터랙션 상세:**
1. 5개 아이콘 가로 배열, 모두 비활성(회색) 상태
2. 아이콘 탭 → 선택된 아이콘 teal 색상 + scale(1.2) + 레이블 표시
3. 다른 아이콘 탭 → 이전 선택 해제 + 새 선택 활성
4. "다음" 버튼 활성화 (선택 후)

**페르소나별 선택 패턴:**
- 민준: 주로 2~3 (카페인 + 늦은 취침)
- 서연: 주중 2~3, 주말 4~5
- 재호: 회식 날 1~2, 평일 3~4

**화면 상세:**
```

│  ← (뒤로)               │
│  ● ● ● ○  (3/4)        │

│                         │
│                         │
```

### US-2.5: 수면 영향 요인 기록하기

> **유저로서**, 수면에 영향 준 요인을 기록하고 싶다.

**수용 기준:**
- 화면 제목: "어젯밤 수면에 영향을 준 것이 있나요?"
- 7개 태그 칩 (복수 선택 가능):
  - 카페인 (caffeine)
  - 운동 (exercise)
  - 음주 (alcohol)
  - 스트레스 (stress)
  - 야식 (late_meal)
  - 스크린타임 (screen_time)
  - 없음 (none)
- 각 태그: 텍스트 칩, 비활성(outline) ↔ 활성(filled teal) 토글
- "없음" 선택 시 나머지 모두 해제
- 나머지 선택 시 "없음" 해제
- "완료" 버튼 → 기록 저장

**인터랙션 상세:**
1. 7개 칩 모두 비활성 상태
2. 칩 탭 → 토글 (비활성↔활성)
3. "없음" 탭 → 나머지 6개 해제 + "없음" 활성
4. 다른 칩 탭 → "없음" 해제 + 해당 칩 활성

## 3. 1주 사용 — 패턴 발견 (Day 7)

### US-3.1: 이번 주 수면 패턴 확인하기

> **유저로서**, 이번 주 수면 패턴을 한눈에 보고 싶다.

**수용 기준:**
- 홈 화면 주간 바 차트:
  - X축: 월~일 (7일)
  - Y축: 수면 시간 (0~12시간)
  - 바 색상: 수면 질 기반 teal 강도 (1=10% opacity → 5=100%)
  - 목표선: 점선으로 목표 시간 표시
  - 기록 없는 날: 빈 공간 (바 없음)
- 주간 평균 텍스트:
  - "이번 주 평균 6.3시간 (목표 대비 -1.2시간)"
  - "수면 질 평균 3.2/5"

**페르소나별 차트 해석:**
- 민준: 월~목 5~6시간(낮은 바, 연한 색), 금 4.5시간(가장 낮고 연함), 토일 8시간(높고 진함) → "주중/주말 격차가 눈에 보인다"
- 서연: 비슷한 패턴이지만 금요일 취침이 가장 늦어서 토요일 바가 가장 낮음
- 재호: 수요일(회식) 바가 진하지만 질이 낮아서 연한색 → "7시간 잤는데 왜 연한색이지?" → 질이 낮았다는 것 인식

**차트 인터랙션:**
- 바 탭 → 해당 일 상세 (시간, 질, 요인) 툴팁 표시
- 좌우 스와이프 → 이전/다음 주 (향후 기능)

---

### US-3.2: 요일별 수면 패턴 발견하기

> **유저로서**, 요일별 수면 패턴 차이를 발견하고 싶다.

**수용 기준:**
- 인사이트 탭에서 요일별 분석 표시
- 자동 생성 인사이트 예시:
  - "금요일 평균 취침 시간이 01:30으로 가장 늦어요"
  - "일요일 수면 시간이 9.2시간으로 가장 길어요 (주중 평균 대비 +2.8시간)"
  - "월요일 수면 질이 2.3으로 가장 낮아요"
  - "목요일 수면 질이 4.1로 가장 높아요"

**인사이트 생성 로직:**
- 최소 데이터: 7일 이상 기록 시 활성화
- 같은 요일 데이터가 2건 이상일 때 해당 요일 통계 생성
- 가장 높은/낮은 값의 요일을 자동 추출

**데이터 부족 시:**
- 7일 미만: "아직 데이터가 부족해요. N일 더 기록하면 패턴을 분석해 드릴게요."
- 부족 날짜 강조: 캘린더에서 빈 날짜 표시

---

### US-3.3: 수면 질 영향 요인 알기

> **유저로서**, 수면 질에 영향을 주는 요인을 알고 싶다.

**수용 기준:**
- 인사이트 탭 "요인 분석" 섹션:
  - 각 요인별 수면 질 비교 카드
  - "카페인을 섭취한 날 평균 수면 질: 2.1 (미섭취 시: 3.8) → 차이: -1.7점"
  - "운동한 날 평균 수면 질: 4.2 (안 한 날: 3.0) → 차이: +1.2점"
  - "음주한 날 평균 수면 질: 1.8 (안 한 날: 3.9) → 차이: -2.1점"

**분석 로직:**
- 각 요인에 대해: 해당 요인 있는 날의 평균 질 vs 없는 날의 평균 질
- 차이 절대값 기준 정렬 (영향 큰 순)
- 양/음 방향 표시: 긍정 요인(운동) vs 부정 요인(음주, 카페인)
- 최소 데이터: 해당 요인 3회 이상 기록 시 표시

**시각화:**
- 각 요인 카드에 수평 바 차트:
  - 왼쪽(음): 부정 영향 (빨간/주황 계열)
  - 오른쪽(양): 긍정 영향 (초록/teal 계열)
  - 바 길이: 영향 크기 비례

**페르소나별 핵심 인사이트:**
- 민준: "카페인 → -1.7점. 커피 끊으면 진짜 달라질까?"
- 서연: "스트레스 → -1.5점. 주중 스트레스 관리가 수면의 핵심이네."
- 재호: "음주 → -2.1점. 회식 날 수면은 확실히 망가진다."

### US-4.1: 월간 수면 캘린더로 패턴 보기

**수용 기준:**
- 캘린더 탭: 7x5(6) 그리드 (일~토)
- 각 날짜 셀:
  - 기록 있음: 수면 질 기반 teal 색상 강도

    - 질 2: teal 30% opacity

    - 질 4: teal 75% opacity
    - 질 5: teal 100% opacity (진함)
  - 기록 없음: 빈 셀 (배경색만)
  - 오늘: teal 테두리 2px

1. 날짜 셀 탭 → 바텀시트 열림

   - 요인: 태그 칩
   - "수정" 버튼 (기록 편집)
3. 빈 날짜 탭 → "이 날의 수면을 기록해볼까요?" + "기록하기" CTA

**페르소나별 캘린더 패턴:**
- 민준: 주중 연한색, 주말 진한색의 반복 → "주중/주말 패턴이 캘린더로 보니까 더 명확하다"

- 재호: 수요일(회식 날)이 주기적으로 연함 → "회식 패턴이 달력에 찍힌다"

## 5. 습관화 — 리텐션 (Day 60+)

### US-5.1: 기록 연속일(스트릭) 유지하기

> **유저로서**, 기록 연속일을 유지하는 동기를 갖고 싶다.

**수용 기준:**
- 홈 화면 상단: "12일 연속 기록 중" + 스트릭 아이콘 (SVG)
- 스트릭 계산: 연속으로 기록한 일수 (하루라도 빠지면 리셋)
- 스트릭 깨지면: "다시 시작해요! 이전 최고: 12일"
- 마일스톤: 7일, 14일, 30일, 60일, 100일 달성 시 특별 메시지

**마일스톤 메시지:**
- 7일: "일주일 연속! 벌써 패턴이 보이기 시작했어요."
- 14일: "2주 연속! 수면 습관이 자리 잡고 있어요."
- 30일: "한 달 연속! 당신의 수면 데이터는 이제 아주 풍부해요."
- 60일: "60일 연속! 수면 전문가 수준의 기록이에요."
- 100일: "100일! 대단합니다. 수면 기록이 완전한 습관이 되었네요."

**스트릭 복구 정책:**
- 당일 자정까지 기록하면 스트릭 유지
- 다음 날 아침까지도 전날 기록 가능 (12시간 유예)
- 유예 시간 지나면 스트릭 리셋

## 6. 엣지 케이스 및 예외 처리

### 6.1 데이터 관련

| 상황 | 처리 |
|------|------|
| 같은 날 중복 기록 | 최신 기록으로 덮어쓰기 (이전 기록 유지 안 함) |
| 과거 날짜 기록 | 캘린더에서 빈 날짜 탭 → 해당 날짜로 기록 가능 |
| 미래 날짜 기록 | 불가 |
| 취침 > 기상 시간 | 자정 교차로 판단 (23:00→07:00 = 8시간) |
| 수면 시간 0분 | 경고: "취침과 기상 시간을 확인해주세요" |
| 수면 시간 14시간+ | 경고: "14시간 이상이에요. 맞나요?" + 확인/수정 |

### 6.2 온보딩 관련

| 상황 | 처리 |
|------|------|
| 온보딩 중간 이탈 | 다음 앱 실행 시 이어서 진행 |
| 온보딩 전부 건너뛰기 | 기본값으로 설정 (7.5시간, 23:00, 07:00) |
| 데모 데이터 삭제 | 설정에서 "데모 데이터 삭제" 또는 "데이터 초기화" |

### 6.3 표시 관련

| 상황 | 처리 |
|------|------|
| 데이터 0건 (온보딩 직후) | 데모 데이터 표시 (isDemo 배지) |
| 주간 차트에 빈 날 | 해당 바 미표시 + 아래 "기록 없음" 표시 |
| 인사이트 데이터 부족 | "N일 더 기록하면 인사이트를 볼 수 있어요" |
| 월 전환 시 데이터 없는 월 | 빈 캘린더 + "이 달의 기록이 없어요" |
