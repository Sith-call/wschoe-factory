# Ritual Maker — PRD (Product Requirements Document)

> 일상 루틴을 RPG 스킬트리로 게이미피케이션하는 앱

**버전:** 1.0
**작성일:** 2026-03-20
**상태:** Draft

---

## 1. 문제 정의 (Problem Statement)

### 배경

한국 20-30대 직장인은 자기계발과 건강 관리에 대한 높은 관심을 가지고 있지만, 실제로 루틴을 꾸준히 유지하는 비율은 매우 낮다. 기존 습관 트래커 앱들은 단순한 체크리스트 방식으로, 초기 동기 부여는 가능하지만 2주 이상 지속률이 급격히 떨어진다.

### 핵심 문제

1. **동기 부여 부족:** 단순 체크리스트는 반복할수록 지루해진다. 루틴을 지켜야 할 '이유'가 점점 약해진다.
2. **성장 실감 부재:** 매일 같은 일을 반복하지만, 내가 얼마나 성장했는지 체감하기 어렵다.
3. **사회적 연결 부족:** 혼자 하는 루틴은 외롭다. 포기해도 아무도 모른다.
4. **회고 습관 없음:** 한 주를 돌아보며 패턴을 파악하는 습관이 없어, 같은 실패를 반복한다.

### 가설

> RPG 게이미피케이션 요소(캐릭터 성장, 스킬 해금, 보스 배틀, 길드)를 루틴 관리에 결합하면, 기존 습관 트래커 대비 **4주 이상 지속률을 2배 이상** 높일 수 있다.

## 2. 타겟 유저 (Target Users)

### 주요 타겟

| 항목 | 내용 |
|------|------|
| **연령** | 25-35세 |
| **직업** | 직장인, 프리랜서, 대학원생 |
| **특성** | 자기계발에 관심 있지만 루틴 유지에 실패한 경험이 있는 사람 |
| **게임 성향** | RPG, 육성 시뮬레이션 장르에 친숙한 사람 |
| **디지털 리터러시** | 높음 (앱 사용에 거부감 없음) |

### 유저 페르소나

#### 페르소나 A — "김도현" (28세, 스타트업 개발자)

- 매일 아침 운동, 독서, 사이드 프로젝트를 하고 싶지만 퇴근 후 넷플릭스만 본다
- 습관 앱을 3개 이상 깔아봤지만 2주를 못 넘겼다
- 게임에서는 캐릭터를 만렙까지 키우는 끈기가 있다
- "내 루틴도 캐릭터 키우듯 할 수 있으면 좋겠다"

#### 페르소나 B — "이수진" (31세, 마케팅 매니저)

- 명상, 저널링, 운동을 루틴으로 만들고 싶다
- 혼자 하면 작심삼일, 친구와 함께하면 좀 더 오래 간다
- 경쟁보다는 함께 성장하는 느낌을 좋아한다
- "같이 하는 사람이 있으면 더 잘 할 수 있을 텐데"

#### 페르소나 C — "박준혁" (26세, 신입 사원)

- 사회초년생으로 좋은 습관을 만들고 싶다
- 게임처럼 보상이 즉각적이고 시각적인 것을 좋아한다
- 스킬트리, 레벨업 같은 성장 시스템에 몰입한다
- "내가 얼마나 성장했는지 눈에 보이면 좋겠다"

## 4. 핵심 기능 (Key Features)

### 4.1 캐릭터 생성 & 클래스 선택

**설명:** 앱 시작 시 사용자는 4가지 클래스 중 하나를 선택한다. 각 클래스는 루틴 성향을 반영하며, 추천 퀘스트와 스킬트리가 달라진다.

| 클래스 | RPG 이름 | 루틴 성향 | 추천 퀘스트 예시 |
|--------|---------|----------|---------------|
| 전사 | Warrior | 체력형 | 아침 운동, 스트레칭, 걷기 10000보 |
| 마법사 | Mage | 지식형 | 독서 30분, 강의 수강, 사이드 프로젝트 |
| 힐러 | Healer | 감성형 | 명상, 저널링, 감사 일기, 충분한 수면 |
| 레인저 | Ranger | 탐험형 | 새로운 장소 방문, 새로운 레시피 시도, 사진 촬영 |

**세부 사항:**
- 간단한 성향 테스트(5문항)로 클래스 추천 제공
- 추천과 상관없이 원하는 클래스 자유 선택 가능
- 캐릭터 이름 설정 (닉네임)
- 캐릭터 기본 외형 선택 (프로필 아이콘 스타일)

### 4.2 일일 루틴 체크인 (퀘스트 시스템)

**설명:** 하루를 세 시간대로 나누어 퀘스트 형식으로 루틴을 관리한다.

| 시간대 | 퀘스트 슬롯 | 시간 범위 |
|--------|------------|----------|
| 아침 퀘스트 | 최대 3개 | 05:00 - 11:59 |
| 오후 퀘스트 | 최대 3개 | 12:00 - 17:59 |
| 저녁 퀘스트 | 최대 3개 | 18:00 - 04:59 |

**세부 사항:**
- 클래스별 기본 추천 퀘스트 3세트 제공
- 커스텀 퀘스트 추가/수정/삭제 가능
- 퀘스트 완료 시 경험치(EXP) 획득
- 콤보 시스템: 연속 달성 시 EXP 보너스 (3일 1.5배, 7일 2배, 30일 3배)
- 알림: 각 시간대 시작 시 푸시 알림 (선택적)

### 4.3 RPG 스킬트리

**설명:** 루틴 반복 횟수에 따라 스킬을 해금하는 성장 시스템. 클래스별로 고유한 스킬트리를 가진다.

**스킬트리 구조 (예시 — 전사 클래스):**

```
Lv1 기초체력 ─── Lv5 근력강화 ─── Lv15 철벽방어
      │                │
      └── Lv3 유연성 ── Lv10 지구력 ─── Lv20 불굴의의지
                                              │
                                        Lv30 [궁극기] 전장의지배자
```

**해금 조건:**
- Lv1: 해당 카테고리 퀘스트 3회 완료
- Lv5: 해당 카테고리 퀘스트 15회 완료
- Lv10: 해당 카테고리 퀘스트 30회 완료 + 7일 연속 달성 1회
- Lv15: 해당 카테고리 퀘스트 50회 완료
- Lv20: 해당 카테고리 퀘스트 80회 완료 + 14일 연속 달성 1회
- Lv30: 해당 카테고리 퀘스트 120회 완료 + 30일 연속 달성 1회 (궁극기)

**스킬 효과:**
- 보스 배틀 시 공격력/방어력 보너스
- 추가 EXP 보너스
- 특별 칭호 획득
- 캐릭터 외형 변화 (이펙트, 장비 등)

### 4.4 주간 보스 배틀 (회고 시스템)

**설명:** 매주 일요일, 한 주의 루틴 달성률을 기반으로 보스와 전투하는 회고 시스템.

**보스 배틀 흐름:**

1. **보스 등장:** 매주 다른 테마의 보스 (예: "게으름의 드래곤", "미루기의 골렘", "유혹의 마녀")
2. **전투 준비:** 이번 주 달성률 확인 → 공격력으로 환산
3. **회고 단계:**
   - 이번 주 잘한 점 1가지 작성 → 추가 공격력
   - 아쉬운 점 1가지 작성 → 방어력 보너스
   - 다음 주 다짐 1가지 작성 → 치명타 확률 증가
4. **전투 실행:** 애니메이션으로 전투 결과 표시
5. **보상 지급:** 승리/패배에 따른 보상

**전투력 계산:**
- 기본 공격력 = 주간 퀘스트 달성률 x 100
- 스킬 보너스 = 해금된 스킬에 따라 추가
- 회고 보너스 = 회고 작성 시 +20%
- 보스 HP = 주차에 따라 점진적 증가 (성장 체감)

### 4.5 길드 시스템

**설명:** 친구들과 함께 루틴을 실천하는 소셜 기능.

**기능 목록:**
- 길드 생성 (최대 10명)
- 초대 링크로 친구 초대
- 길드 내 주간 랭킹 (EXP 기준)
- 길드원 퀘스트 달성 현황 타임라인
- 길드 보스 배틀: 길드원 전체 달성률로 협동 보스전
- 응원 보내기 (이모지 리액션)

### 4.6 데모 모드 (3일 시뮬레이션)

**설명:** 회원가입 없이 앱의 전체 경험을 3일치 데모 데이터로 체험할 수 있는 모드.

**데모 시나리오:**
- Day 1: 캐릭터 생성 → 첫 퀘스트 수행 → Lv1 스킬 해금
- Day 2: 콤보 시스템 체험 → 추가 퀘스트 도전 → 스탯 성장 확인
- Day 3: 미니 보스 배틀 체험 → 주간 회고 맛보기 → 길드 샘플 확인

**데모 데이터 구성:**
- 미리 생성된 샘플 캐릭터 (전사 클래스, Lv5)
- 3일치 퀘스트 히스토리
- 해금된 스킬 2개
- 샘플 길드 (NPC 길드원 3명)
- 미니 보스 배틀 1회

## 5. 화면 흐름 (Screen Flow)

### 5.1 전체 화면 구성

```
[스플래시] → [데모/시작 선택]
                │
        ┌───────┴───────┐
        │               │
   [데모 모드]     [회원가입/로그인]
        │               │
        └───────┬───────┘
                │
         [성향 테스트] → [클래스 선택] → [캐릭터 생성]
                                              │
                                         [메인 홈]
                                              │
                ┌──────┬──────┬──────┬────────┘
                │      │      │      │
           [퀘스트]  [스킬트리] [보스배틀] [길드]
                │      │      │      │
                └──────┴──────┴──────┘
                              │
                         [프로필/설정]
```

### 5.2 주요 화면 상세

#### 1) 스플래시 & 온보딩
- 앱 로고 + 짧은 슬로건 ("당신의 일상이 모험이 된다")
- "데모로 체험하기" / "모험 시작하기" 버튼

#### 2) 성향 테스트
- 5개의 간단한 상황 기반 질문
- 각 질문에 4개 선택지 (각 클래스에 매핑)
- 결과 화면에서 추천 클래스 표시

#### 3) 캐릭터 생성
- 클래스 선택 (추천 클래스 하이라이트)
- 캐릭터 이름 입력
- 프로필 아이콘 선택 (클래스별 4종)
- "모험 시작!" 버튼

#### 4) 메인 홈
- 상단: 캐릭터 정보 (이름, 클래스, 레벨, EXP 바)
- 중앙: 오늘의 퀘스트 카드 (아침/오후/저녁 탭)
- 하단: 콤보 카운터 + 빠른 통계
- 하단 탭바: 홈 / 스킬트리 / 보스배틀 / 길드 / 프로필

#### 5) 퀘스트 화면
- 시간대별 탭 (아침/오후/저녁)
- 퀘스트 카드 리스트 (체크박스 + 퀘스트명 + EXP)
- 완료 시 체크 애니메이션 + EXP 획득 팝업
- 커스텀 퀘스트 추가 버튼 (+)

#### 6) 스킬트리 화면
- 트리 형태의 스킬 맵 (노드 그래프)
- 해금된 스킬: 밝게 표시 + 이펙트
- 미해금 스킬: 어둡게 + 해금 조건 표시
- 스킬 탭 시 상세 정보 모달

#### 7) 보스 배틀 화면
- 보스 일러스트 + HP 바
- 이번 주 달성률 요약
- 회고 작성 폼 (잘한 점/아쉬운 점/다짐)
- 전투 애니메이션
- 결과 화면 (보상 표시)

#### 8) 길드 화면
- 길드 정보 (이름, 멤버 수)
- 멤버 리스트 (레벨, 주간 EXP)
- 주간 랭킹
- 길드원 활동 타임라인
- 길드 보스전 버튼 (주말)

#### 9) 프로필 화면
- 캐릭터 상세 정보 (스탯, 칭호)
- 달성 통계 (전체 달성률, 총 퀘스트 수, 최장 콤보)
- 설정 (알림, 계정, 테마)

## 7. MVP 범위 (MVP Scope)

### MVP에 포함 (Phase 1 — 4주 개발)

| 기능 | 상세 | 우선순위 |
|------|------|---------|
| 캐릭터 생성 | 클래스 선택 (4종) + 이름 + 아이콘 | P0 |

| 주간 보스 배틀 | 달성률 기반 전투 + 회고 작성 | P0 |
| 데모 모드 | 3일치 시뮬레이션 데이터 (DB 없이 체험) | P0 |

## 8. 기술 고려사항

### 프론트엔드
- React (SPA) 또는 React Native (추후 모바일 확장)
- 스킬트리 시각화: Canvas 또는 SVG 기반 노드 그래프
- 보스 배틀 애니메이션: Lottie 또는 CSS 애니메이션

### 데이터 모델 (핵심 엔티티)

```
User
├── character_name
├── class_type (warrior | mage | healer | ranger)
├── level
├── total_exp
└── created_at

Quest
├── user_id
├── title
├── time_slot (morning | afternoon | evening)
├── is_custom
├── exp_reward
└── is_completed

Skill
├── user_id
├── skill_id
├── class_type
├── unlock_condition
├── is_unlocked
└── unlocked_at

BossBattle
├── user_id
├── week_number
├── weekly_completion_rate
├── reflection_good
├── reflection_bad
├── reflection_goal
├── result (win | lose)
└── rewards

Guild
├── guild_name
├── owner_id
├── member_ids[]
└── created_at

ComboStreak
├── user_id
├── current_streak
├── longest_streak
└── last_completed_date
```

### 데모 모드 구현
- 로컬 스토리지 기반 (서버 불필요)
- 미리 정의된 JSON 데이터로 3일치 시뮬레이션
- 데모 → 실제 계정 전환 시 데이터 마이그레이션 지원
