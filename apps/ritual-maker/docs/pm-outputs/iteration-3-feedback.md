## 총평

오, 이번에는 진짜 게임이 된 느낌이다. 스킬트리에서 "탭하여 해금!"이 뜨고, 실제로 탭하면 파티클이 팡! 터지면서 "해금 완료! 패시브 효과 적용됨"이 뜨는 순간 쾌감이 확 온다. 쿠키런에서 쿠키 스킬 찍을 때 그 느낌이랑 비슷하다. 그리고 보스전에 진짜 몬스터가 생겼다! 슬라임이 물방울 흘리면서 꿈틀거리고, 드래곤은 불 뿜고, 피닉스는 불꽃 날개가 일렁인다 — 이전에 이모지 동그라미였던 거에 비하면 천지차이다. 루틴 편집도 프로필에서 할 수 있게 됐고, 퀘스트 완료할 때 파티클도 터진다. 전체적으로 "이건 진짜 게임이다"라는 몰입감이 확 올라왔다. 근데 아직 부족한 부분도 있다.

## 3. 몰입감 (25%)

### 스킬트리 재미 (8%)
**점수: 8/10** (이전: 6/10, +2)

이건 이번 이터레이션에서 가장 큰 변화다. 이전에는 스킬트리가 "보기만 하는 도감" 수준이었는데, 이제 진짜 게임이 됐다. 레벨 조건을 충족하면 "탭하여 해금!" 텍스트가 녹색 반짝임(animate-pulse)과 함께 뜨고, 탭하면:

1. 카드 배경이 앰버 그라데이션으로 바뀌면서 1.02배 확대
2. 8개 파티클(금색/주황/에메랄드)이 랜덤 위치에서 팡 터짐
3. "해금 완료! 패시브 효과 적용됨" 텍스트가 slideUp 애니메이션으로 등장
4. 1.5초 후 원래 "적용 중" 상태로 전환

이게 쿠키런에서 스킬 찍을 때 느낌이랑 진짜 비슷하다! handleUnlockSkill에서 레벨 체크 + 선행 스킬(requires) 체크까지 하는 것도 확인했다. 스킬 트리 상단에 "3/12 해금됨" 카운터가 있어서 진행률도 보인다. Tier 1(초급, 녹색) → Tier 2(중급, 파랑) → Tier 3(고급, 보라) → Tier 4(전설, 금색) 컬러 구분도 "레전드급 스킬 찍고 싶다"는 욕구를 자극한다.

아쉬운 점: 패시브 효과가 실제로 게임 로직에 반영되는지는 코드에서 확인 안 됨. skills 배열에 unlocked: true만 세팅하고, 실제 XP 계산이나 스탯에 패시브가 적용되는 로직은 아직 없는 것 같다. "패시브 효과 적용됨"이라고 표시되지만, 실제로는 적용 안 되는 건 좀 아쉽다.

### 보스배틀 긴장감 (7%)
**점수: 7.5/10** (이전: 6/10, +1.5)

SVG 보스 일러스트가 추가되면서 보스전이 확 살아났다. 인트로 화면에서 140px 크기 BossAvatar가 둥실둥실 떠있고, 전투 화면에서는 60px로 HP바 옆에 붙어있다. 공격하면 damaged=true가 되면서 animate-shake로 보스가 흔들린다 — 이 타격감이 진짜 좋다.

슬라임은 물방울이 뚝뚝, 고블린은 눈이 반짝, 드래곤은 불꽃이 일렁, 피닉스는 날개 불꽃이 출렁거린다. 각 보스마다 고유한 성격이 비주얼로 드러나서, "이번 주 보스가 뭘까?" 하는 기대감이 생긴다. 이전에 이모지+CSS 원이었던 것과는 비교가 안 된다.

HP바 색상이 체력에 따라 빨강→주황→초록으로 변하는 것도 전략적 긴장감을 준다. "한 번 더 공격하면 깰 수 있을 것 같은데!" 하는 느낌이 든다.

여전히 전투 자체가 회고 텍스트 입력 → 공격 한 번 구조라서 "배틀"이라기엔 단조롭긴 하다. 보스 공격 패턴이나 방어/회피 같은 전투 메카닉이 있으면 더 재밌을 텐데.

## QA 이슈

### 통과 항목
- TypeScript 컴파일 에러 없음
- 스킬 해금 로직 정상: 레벨 체크 + 선행 스킬(requires) 체크 + unlocked 상태 토글
- BossAvatar SVG 렌더링 정상: 4종류(slime/goblin/dragon/phoenix) 각각 고유 SVG + 애니메이션
- 보스전 타격 시 damaged={attackAnim} → animate-shake 적용 정상
- 프로필 > 루틴 편집 → RitualSelectScreen → handleUpdateRituals 플로우 정상
- 기존 캐릭터/레벨/로그 유지하면서 rituals만 교체하는 로직 정상
- 퀘스트 완료 파티클 6개 생성 + 컬러 4색 순환 정상

### 발견된 이슈
1. **패시브 효과 미적용**: 스킬 해금 시 "패시브 효과 적용됨"이라고 표시되지만, handleCompleteRitual에서 해금된 스킬의 패시브를 XP 계산이나 스탯에 반영하는 로직이 없다. UI와 실제 로직 불일치.
2. **스킬 해금 비용 없음**: 스킬 포인트나 골드 같은 해금 비용 없이 레벨만 충족하면 무료 해금. 전략적 선택의 깊이가 부족하다.
3. **RitualSelectScreen 재사용 시 기존 선택 미반영**: 프로필에서 루틴 편집할 때 RitualSelectScreen이 뜨지만, 현재 선택된 루틴이 체크 상태로 표시되는지 확인 필요. 코드상 onComplete만 받고 initialRituals props가 없어서, 빈 상태에서 처음부터 다시 고르는 것으로 보임.
4. **길드 멤버 아바타 불일치**: DEMO_GUILD의 avatar가 여전히 이모지 문자열. 플레이어는 ClassAvatar SVG를 사용.
5. **친구 초대 버튼**: 여전히 onClick 핸들러 없음.

### 개선됨 (Iteration 2 → 3)
1. **스킬트리 해금**: 보기 전용 → 탭으로 실제 해금 + 파티클 애니메이션 + "패시브 적용됨" 피드백
2. **보스 몬스터 비주얼**: 이모지/CSS 원 → SVG 일러스트 4종 (슬라임/고블린/드래곤/피닉스) + 각각 고유 애니메이션
3. **루틴 편집**: 온보딩 후 변경 불가 → 프로필 > 설정 > 루틴 편집으로 언제든 재선택 가능
4. **퀘스트 완료 피드백**: XP 텍스트만 → 6개 컬러 파티클 burst 추가
5. **보스전 타격감**: 정적 이모지 → SVG 보스가 공격 시 흔들리는 shake 애니메이션

### 아직 부족한 것
1. **알림 시스템** — 3번째 이터레이션인데 여전히 없음 (가장 시급!)
2. **패시브 실제 적용** — UI는 "적용됨"이라고 하지만 로직상 미반영
3. **루틴 자유 입력** — 여전히 16개 라이브러리에서만 선택 가능
4. **스토리/세계관** — 서사 부재
5. **공유 기능** — 캐릭터 카드/보스전 결과 이미지 공유 없음
6. **이탈 방지** — 복귀 보상, 스트릭 보호 여전히 없음

## 개선 가설 Top 3 (다음 이터레이션)

### 1. 알림 시스템 (예상 임팩트: +6~8점)
- 시간대별 기본 알림 (아침 7시, 점심 12시, 저녁 8시)
- Web Notification API 또는 Service Worker 기반
- "내일 보스전 D-1!" 같은 게임 이벤트 알림
- 사용성 "알림 설정" 1→6~7점, 재방문 의도에도 영향
- **3번째 이터레이션인데 아직 없는 건 심각한 결함. 최우선 구현 필요.**

### 2. 패시브 효과 실제 적용 (예상 임팩트: +3~5점)
- 해금된 스킬 패시브가 XP 계산, 스탯 보너스, 스트릭 보호 등에 실제 반영
- 예: "집중의 기운" 해금 → 지력 루틴 XP +10% 실제 적용
- 스킬트리 해금의 의미가 "비주얼 뱃지"에서 "실제 게임 메카닉"으로 승격
- 몰입감 "스킬트리 재미" 8→9점 가능

### 3. 스토리/세계관 도입 (예상 임팩트: +4~6점)
- 보스별 짧은 스토리 텍스트 (왜 이 보스를 물리쳐야 하는지)
- 온보딩에 세계관 인트로 (30초 내)
- 보스 처치 후 다음 장(chapter) 예고
- 몰입감 "스토리/세계관" 3.5→6~7점 가능

## 친구 추천 여부

**N (아직 추천하지 않음)**

솔직히 이번에 꽤 많이 좋아졌다. 스킬트리 해금은 진짜 게임하는 느낌이었고, 보스 몬스터 SVG는 기대 이상이었다. 루틴 편집 기능도 필수였는데 들어왔고. 근데... 알림이 없는 습관 앱을 친구한테 추천할 수는 없다. "이거 진짜 재밌는데, 알람 안 울려서 매일 스스로 기억하고 열어야 해"라고 하면 친구가 "그럼 의미 없잖아"라고 할 게 뻔하다. 67점이면 "보류" 중에서도 위쪽이긴 한데, 80점(추천 기준)까지는 아직 13점 남았다. 알림만 들어오면 진짜 훅 올라갈 것 같은데.
