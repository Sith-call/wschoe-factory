# Ritual Maker - Design Visionary Review (Iteration 1)

**Reviewer:** Design Visionary
**Date:** 2026-03-20
**Benchmark:** Duolingo, Forest, Habitica, Spotify, Nike Run Club

---

## 총점: 38 / 100

솔직히 말하겠다. 이건 "프로토타입"이 아니라 "이모지로 때운 MVP"다.
세계적 수준의 게이미피케이션 습관 앱과 비교하면, 지금 상태는 2015년 해커톤 제출작 수준이다.

---

## 차원별 평가

### 1. 첫인상 (First Impression): 35/100

다크 테마에 보라-앰버 그라디언트 배경은 나쁘지 않다. "RPG 감성"을 시도하고 있다는 건 느낀다. 하지만 앱을 처음 열었을 때 드는 생각은 "이게 뭐지?"다.

**문제점:**
- 캐릭터 카드가 앱의 핵심인데, 캐릭터 비주얼이 이모지 하나(`⚔️`)다. Habitica는 풀 픽셀아트 아바타를 제공한다. Duolingo는 Duo라는 상징적 캐릭터가 있다. 여기엔 아무것도 없다.
- "모험가 Lv.3"이라는 텍스트만으로는 몰입감이 0이다
- 홈 화면 레이아웃이 단조롭다. 정보 나열형. Spotify 홈처럼 시각적 위계가 명확하지 않다

### 2. 차별화 (Differentiation): 30/100

Habitica가 이미 "RPG + 습관"이라는 카테고리를 10년 전에 선점했다. Ritual Maker가 Habitica와 뭐가 다른가?

**현재 차별점으로 보이는 것:**
- "보스 배틀에서 회고를 통해 공격" -- 컨셉은 괜찮다
- 시간대별 퀘스트 (아침/오후/저녁) -- Forest의 타이머 방식보다 구조적

**그러나:**
- 이 차별점들이 시각적으로 전혀 드러나지 않는다. 보스 배틀 화면을 보라 -- 초록색 원이 "나태의 슬라임"이다. 진심인가? Habitica의 보스 일러스트와 비교해보라.
- 스킬트리가 리스트다. "트리"라는 이름이 붙었는데 트리 형태 UI가 아니라 그냥 리스트. 리그 오브 레전드, 패스 오브 엑자일의 스킬트리를 한 번이라도 봤다면 이렇게 만들 수 없다.

### 3. 시각적 완성도 (Visual Polish): 40/100

**잘 된 것:**
- 다크 테마의 전반적 톤은 일관적이다. `#0a0615` 베이스, 보라/앰버/에메랄드 악센트
- 스탯 바의 색상 분리 (빨강-보라-초록-노랑)는 가독성이 좋다
- 카드 border에 `border-white/10`으로 미묘한 구분선 -- 기본은 갖췄다
- 배경에 미세한 파티클/그라디언트 애니메이션 -- 노력의 흔적

**문제점:**
- 타이포그래피가 단조롭다. Noto Sans KR 하나로 모든 걸 해결하려 한다. RPG 앱이라면 타이틀/레벨/스탯에 다른 톤의 서체가 필요하다. 최소한 weight 변주라도 더 과감하게
- 스페이싱이 기계적이다. 모든 카드가 같은 리듬. 시각적 호흡이 없다
- 퀘스트 화면 -- 오후 퀘스트 1개만 있을 때 화면의 60%가 빈 공간이다. 이 비어 있는 공간을 어떻게 활용할지 전혀 고민하지 않았다
- 프로필 화면이 데이터 테이블처럼 보인다. 숫자 나열. 감동이 없다

### 4. 감정적 연결 (Emotional Connection): 30/100

Duolingo는 스트릭을 잃을 때 Duo가 우는 모습을 보여준다. 사용자가 감정적으로 끌리게 만든다. Forest는 나무가 자라는 시각적 보상이 있다. Nike Run Club은 달리기를 끝내면 축하 애니메이션이 터진다.

Ritual Maker에서 루틴을 완료하면? 텍스트가 바뀐다. `0/1` -> `1/1`. 끝.

**구체적 문제:**
- 보상 순간에 도파민이 없다. XP 획득 시 `+30 XP` 텍스트만 뜬다. 파티클 이펙트, 사운드, 진동, 뭐라도 있어야 한다
- 레벨업 순간이 존재하지 않는다. 레벨이 올라가도 아무 일도 일어나지 않는다
- 스트릭 `3일`이 그냥 숫자다. Duolingo처럼 불꽃이 커지거나, 특별한 프레임이 생기거나 하는 변화가 없다
- 보스 처치 후 `🏆` 이모지 하나. Nike Run Club의 달리기 완료 화면과 비교하면 처참하다

### 5. 디테일 수준 (Detail Level): 35/100

**있는 것:**
- `fadeIn`, `slideUp` 기본 애니메이션
- `active:scale-95` 터치 피드백
- `transition-all` 기본 트랜지션
- `shake` 애니메이션 (보스 전투)
- `breathe` 배경 파티클

**없는 것:**
- 화면 전환 애니메이션. screen state만 바꾸고 즉시 렌더. 좌우 슬라이드, 페이드 전환 등이 전무
- 스크롤 기반 인터랙션. 패럴럭스, 스크롤 시 헤더 축소 등 전무
- 제스처 인터랙션. 스와이프로 퀘스트 완료, 좌우 스와이프 네비게이션 등 전무
- 프로그레스 애니메이션. 스탯 바가 처음부터 채워진 채로 렌더된다. 로딩 시 0에서 현재값까지 차오르는 애니메이션이 있어야 한다
- 해프틱 피드백 연동
- 사운드 디자인 (RPG 앱인데 효과음이 없다?)

### 6. 에셋 밀도 (Asset Density): 25/100

이것이 가장 치명적인 문제다. **커스텀 비주얼 에셋이 단 하나도 없다.**

- 모든 아이콘이 이모지: `⚔️🏠🌳🏰👤🔥⚡💪🧠✨🎯🌙☀️`
- 보스 몬스터가 CSS 원(`bg-green-500 rounded-full`)이다. 농담이 아니다.
- 캐릭터 아바타가 없다
- 배경 일러스트가 없다
- 스킬트리 아이콘이 없다
- 로딩 화면이 없다
- 빈 상태(empty state) 일러스트가 없다
- 성취 뱃지 비주얼이 없다

Duolingo에는 수백 개의 커스텀 일러스트가 있다. Habitica에는 수천 개의 픽셀아트 에셋이 있다. Ritual Maker에는 0개가 있다.

## 작동하는 것 (What Works)

1. **다크 RPG 컬러 팔레트:** `#0a0615` 베이스 + 보라/앰버/에메랄드 조합은 RPG 분위기에 맞다. 이 컬러 시스템은 유지하되 깊이를 더해야 한다
2. **정보 구조:** 홈 -> 퀘스트 -> 스킬트리 -> 보스배틀 -> 길드 -> 프로필. IA 자체는 논리적이다
3. **시간대별 퀘스트 분류:** 아침/오후/저녁으로 나누는 것은 생활 리듬에 맞는 좋은 구조다
4. **보스 배틀 회고 메커니즘:** "답변으로 데미지를 입힌다"는 게이미피케이션 아이디어는 독창적이다

## 평범한 것 (What's Mediocre)

1. **캐릭터 카드:** 구조는 있지만 비주얼 임팩트가 부족. 이모지 아이콘 + 텍스트로는 RPG 몰입이 안 된다
2. **길드 랭킹:** 기본적인 리더보드 기능은 있지만, 시각적으로 그냥 리스트. 경쟁/협력의 긴장감이 없다
3. **퀘스트 화면의 탭 구조:** 동작하지만 영감이 없다. 탭 디자인이 너무 기본적이다

## 극적으로 개선이 필요한 것 (What Needs Dramatic Improvement)

1. **에셋 부재:** 이모지 의존도 100%. 이건 치명적이다
2. **보상 경험 부재:** 루틴 완료, XP 획득, 레벨업 -- 모든 보상 순간이 텍스트 변경으로만 처리됨
3. **보스 비주얼:** 초록색 원은 "나태의 슬라임"이 아니라 CSS 에러처럼 보인다
4. **스킬트리 UI:** 트리가 아니라 리스트. 이름값을 못 한다
5. **화면 전환:** 즉시 교체. 앱이 아니라 웹페이지처럼 느껴진다

## 구체적 실행 권고 (Actionable Recommendations)

### P0 - 당장 해야 할 것

1. **보스 몬스터 비주얼 교체:** CSS 원 대신 SVG 일러스트 또는 최소한 고품질 이모지 조합 + 그림자/글로우 이펙트. 혹은 AI 생성 일러스트(Midjourney/DALL-E)를 활용하라
2. **루틴 완료 보상 애니메이션 추가:** 체크 시 파티클 이펙트 + XP 숫자가 떠오르는 `floatUp` 애니메이션 (이미 keyframe 정의되어 있는데 안 쓰고 있다!)
3. **캐릭터 아바타 시스템:** 클래스별 최소 1개씩 SVG 아바타. 전사/마법사/힐러/레인저 각각 시각적 정체성 필요
4. **빈 화면 개선:** 퀘스트가 1개일 때, 나머지 공간에 동기부여 메시지, 오늘의 팁, 또는 캐릭터 일러스트를 보여줘라

### P1 - 다음 이터레이션

5. **스킬트리 비주얼 리디자인:** 수직 리스트가 아닌, 실제 트리/그래프 형태의 UI. 노드 간 연결선, 잠금/해제 상태의 시각적 차이, 글로우 이펙트
6. **화면 전환 애니메이션:** `framer-motion` 또는 CSS transition으로 화면 간 슬라이드/페이드 전환 구현
7. **레벨업 축하 화면:** 레벨 변경 감지 시 전체 화면 축하 모달. 별/파티클 효과 + 새로운 칭호 표시
8. **스탯 바 진입 애니메이션:** `width: 0%`에서 시작하여 현재값까지 1초간 채워지는 애니메이션

### P2 - 품질 도약

9. **사운드 디자인:** 루틴 완료 시 효과음, 레벨업 팡파르, 보스 전투 BGM (Web Audio API 또는 Howler.js)
10. **캐릭터 커스터마이징:** 레벨/업적에 따라 장비 변경, 외형 변화
11. **스트릭 비주얼 진화:** 1일=불씨, 7일=작은불, 30일=대형 화염. 숫자만이 아닌 시각적 변화
12. **길드 챌린지 시각화:** 진행 바, 참여 멤버 아바타, 실시간 느낌의 업데이트 피드

## 결론

게이미피케이션 아이디어 자체(시간대별 퀘스트, 회고 기반 보스전, 스탯 성장)는 가능성이 있다. 하지만 현재는 **"좋은 아이디어를 이모지로 프로토타이핑한 상태"**에 불과하다.

유저가 이 앱을 열었을 때 "RPG를 하고 있다"고 느끼게 만들려면, 최소한 캐릭터 비주얼, 보스 일러스트, 보상 이펙트 이 세 가지는 반드시 있어야 한다. 이것 없이 출시하면, 유저는 10초 만에 "이거 그냥 할일 체크앱이네"라고 판단하고 떠난다.

**한 마디로: 뼈대는 있다. 하지만 살이 없다.**
