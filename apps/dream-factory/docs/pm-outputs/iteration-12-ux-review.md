## 플로우 분석

### Time to First Value
- **앱 첫 실행**: 온보딩 3단계 (3탭) → IntroScreen → "꿈 기록하기" 탭 → SceneBuilder → 장소+날씨 선택(최소 2탭) → "다음 단계로" → EmotionScreen → 감정 1개 선택 → "꿈 해석받기" → AnalysisScreen(3초 대기) → InterpretationScreen
- **총 단계**: 최소 8단계, 약 25-35초
- **재방문 시**: 온보딩 스킵, IntroScreen → 기록하기 → 4단계 → 해석 도착. 약 15-20초.
- **판정**: 재방문 플로우는 적절하다. 초회 플로우의 온보딩 3단계는 "빠르게 기록하고 싶은" 유저에게 약간 마찰이지만, 꿈 기록 앱 특성상 수용 가능 범위.

### 마찰 포인트
1. **SceneBuilder에서 탭 전환 인지 부담**: 장소/날씨/인물/오브젝트 4개 탭이 있지만, 인물과 오브젝트는 선택하지 않아도 진행 가능. "필수 vs 선택"이 불명확하여 유저가 모든 탭을 채워야 하는지 혼란.
2. **AnalysisScreen에서 뒤로가기 불가**: 분석 화면(3초)에서 "잘못 선택한 것 같은데" 하고 뒤로 가고 싶어도 방법이 없다. `onComplete`이 타이머로 자동 호출되며, 뒤로가기 버튼도 없다.
3. **PatternScreen 진입 경로 혼란**: IntroScreen 하단 네비의 "통계" 버튼이 `onGoGallery`에 연결되어 있다(코드 확인: line 206, `onClick={onGoGallery}`). 통계를 눌렀는데 갤러리로 가는 버그.

### 이탈 위험 지점
1. **SceneBuilder에서 12개 장소 선택**: 3열 그리드에 12개 옵션이 스크롤 없이 표시되는데, 하단 옵션(지하, 미지의 장소)이 CTA 버튼 아래에 가려질 수 있다.
2. **Dream Story 마지막 카드에서 "돌아가기"만 있고 "공유" 없음**: Wrapped 경험의 핵심인 "공유하기" CTA가 빠져 있다.

## 스크린별 UX 이슈

### IntroScreen
- **통계 버튼 버그**: 하단 네비 "통계" 버튼이 Gallery로 연결됨 (line 206: `onClick={onGoGallery}`). 3탭 네비인데 갤러리와 통계가 같은 곳으로 감.
- **"최근 기록된 조각" 라벨**: `text-[10px]`으로 매우 작고, `tracking-[0.2em]`으로 문자 간격이 넓어 가독성 저하.
- **터치 타깃**: 하단 네비 버튼들의 터치 영역은 적절(아이콘+라벨 포함 약 48x52px). 양호.

### SceneBuilderScreen
- **진행 상태 표시 불일치**: 상단에 "단계 1/3"이 표시되지만, `currentStep` 계산 로직에서 place와 weather 탭이 모두 step 1로 계산됨. 유저가 날씨 탭으로 이동해도 진행률이 변하지 않아 "진행 중인지" 불확실.
- **선택 피드백 우수**: `glow-ring-animate` + `scale-[1.02]` + 테두리 변화가 즉각적이고 명확함. 이 부분은 잘 설계됨.
- **다음 단계 버튼 비활성 상태**: `canProceed = scene.place !== null && scene.weather !== null`로 명확하게 구현. 비활성 시 시각적으로도 구분됨(`cursor-not-allowed` + 색상 변경). 좋음.

### EmotionScreen
- **감정 카드 터치 타깃**: 2x4 그리드, 각 카드 `p-5` + `gap-4`. 실제 카드 크기 약 170x100px. 충분한 터치 영역.
- **최대 2개 선택 제한**: `emotions.length < 2` 조건으로 3번째 선택이 무시됨. 하지만 유저에게 "최대 2개"라는 안내가 없다. 3번째를 탭해도 아무 반응이 없어서 "고장났나?" 할 수 있음.
- **선명도 슬라이더**: 기본 HTML range input에 커스텀 스타일링. 20px thumb, 충분한 터치 영역. 라벨도 있어서 양호.

### AnalysisScreen
- **강제 3초 대기**: `setTimeout(onComplete, 3000)`. 뒤로가기 불가. 에러 복구 불가. "잘못 선택했는데 어떡하지" 상황에서 유저가 할 수 있는 것이 없다.
- **모션 경험**: 수정구슬 글로우 + 단계별 텍스트 전환이 "기다림을 아름답게" 만든다. UX 관점에서 3초는 수용 가능 범위.

### InterpretationScreen
- **스크롤 필요**: 카드 내용(제목 + 배지 + 해석 텍스트 + 상징 해설 + fortune + personalInsight)이 길어서 상당한 스크롤 필요. `aspect-[2/3.2]` 카드 안에 `overflow-y-auto`가 있지만, 모바일에서 "카드 안에서 스크롤"은 혼란을 줄 수 있다.
- **저장 후 피드백 부재**: "저장하기"를 누르면 즉시 Gallery로 이동하지만, "저장되었습니다" 같은 확인 메시지가 없다.
- **상징 해설 테이블**: 양쪽 정렬(`justify-between`)로 symbol/meaning이 표시되는데, 긴 텍스트에서 줄 바꿈 시 정렬이 깨질 수 있다.

### ShareScreen
- **공유 기능 미구현**: 3개 버튼 모두 `alert()`으로 "추후 업데이트 예정"을 표시. 핵심 기능이 placeholder.
- **뒤로가기만 가능**: 하단 네비에서 다른 화면으로의 이동이 제한적(기록/갤러리/통계가 있지만 onClick이 불완전).

### GalleryScreen
- **검색 기능**: 제목, 키워드, 상징, 메모, 해석 텍스트를 모두 검색 가능. 우수.
- **필터 시스템**: 감정/장소/오브젝트 필터가 horizontal scroll로 제공. 색상 구분(primary/teal/amber)이 직관적.
- **삭제 확인 다이얼로그**: 있음. 실수 방지에 양호.
- **빈 상태 처리**: "아직 기록한 꿈이 없어요" + 기록하기 CTA. 양호.

### PatternScreen
- **최소 데이터 가드**: `dreams.length < 3` 시 빈 상태 표시(코드에서 "5개 이상"이라고 쓰지만 실제로는 3개 조건). 텍스트와 코드 불일치.
- **Dream Story 진입**: "이번 달의 꿈 이야기 보기" CTA가 상단에 명확하게 위치. 발견 가능성 우수.
- **데이터 내보내기**: JSON 다운로드 버튼이 화면 하단에 있음. 파워 유저에게 유용.

## 치명적 UX 문제 TOP 3

### 1. IntroScreen 하단 네비 "통계" 버튼 버그 (심각도: High)
**문제**: `onClick={onGoGallery}`로 연결되어 있어 통계를 누르면 갤러리로 이동.
**영향**: 유저가 "통계"를 탭했는데 갤러리가 열리면 즉각적인 불신감.
**해결**: `onClick={onGoGallery}` → Pattern 화면으로 연결하는 `onGoPattern` prop 추가.

### 2. AnalysisScreen에서 뒤로가기 불가 (심각도: Medium)
**문제**: 3초 타이머가 자동으로 진행하며, 뒤로가기 방법이 전혀 없다.
**영향**: "감정을 잘못 선택했는데" 상황에서 유저가 어떤 행동도 할 수 없음.
**해결**: 화면 구석에 작은 "X" 또는 뒤로가기 제스처(좌측 스와이프) 추가.

### 3. 감정 선택 최대 2개 제한 안내 부재 (심각도: Medium)
**문제**: 2개 이상 선택 시 아무 반응 없이 무시됨. 시각적/텍스트 안내 없음.
**영향**: "앱이 고장났나?" 오인 가능. 특히 감정이 복합적인 꿈에서 3개 이상 선택하려는 유저에게 마찰.
**해결**: 헤드라인 아래에 "(최대 2개)" 텍스트 추가, 또는 2개 선택 후 나머지를 시각적으로 비활성화.

## 개선 제안 (우선순위)

1. **IntroScreen 하단 네비 "통계" 버튼 연결 수정** -- 즉시 수정 가능한 버그. 0.5시간.
2. **감정 선택 "(최대 2개)" 안내 텍스트 추가** -- 미스터치 방지. 0.5시간.
3. **AnalysisScreen에 뒤로가기 옵션 추가** -- 에러 복구 경로 확보. 1시간.
4. **극소 폰트 사이즈 최소 11px로 상향** -- text-[9px] → text-[11px], text-[10px] → text-[11px]. 1시간.
5. **viewport meta 수정** -- `width=430`이 아닌 `width=device-width`로 변경하여 다양한 디바이스 대응. 단, 430px 고정 디자인이라면 현재 방식도 의도적일 수 있음.
