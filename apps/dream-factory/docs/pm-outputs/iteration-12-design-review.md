## Stitch 레퍼런스 vs 실제 구현 비교

Stitch 레퍼런스 이미지(8장)와 실제 앱 스크린샷, 그리고 소스코드를 교차 검증했다.

### 주요 차이점

| 화면 | Stitch 레퍼런스 | 실제 구현 |
|------|----------------|----------|
| Intro | 수정구슬 + 풍경 이미지 | SVG 별자리 + concentric rings (코드로 생성) |
| SceneBuilder | 2열 카드, 이미지 배경 카드 | 2-3열 Material Icons, 배경 없음 |
| Emotion | 감정 카드에 아이콘 원형 배경 | Material Icons + glass-card (유사) |
| Analysis | 수정구슬 + 글로우 | 동일 구조 유지 (swirl + 글로우) |
| Interpretation | 풍경 이미지 상단 + 카드 레이아웃 | DreamIconComposition (아이콘 조합) + 카드 |
| Share | 카드 + 공유 버튼 3개 | 유사 구조 |
| Gallery | 카드 그리드 + 4탭 하단 네비 | 2열 masonry + 3탭 하단 네비 |
| Pattern | 차트 대시보드 + 4탭 하단 네비 | 유사 대시보드 + Dream Story CTA + 3탭 네비 |

**핵심 관찰**: Stitch 레퍼런스에는 실제 풍경 이미지(바다, 산 등)가 사용되었으나, 구현에서는 Material Symbols 아이콘만으로 모든 비주얼을 처리하고 있다. 이것이 "비주얼 리치니스" 점수에 가장 큰 영향을 미친다.

## 7차원 평가

### 1. 색상 시스템 일관성 (15%)

M3 기반 색상 토큰이 `tailwind.config`에 체계적으로 정의되어 있다. `surface-dim`, `primary-container`, `on-surface-variant` 등 37개 색상 토큰. 감정별 그라데이션(`gradient-peace` ~ `gradient-anger`)이 8개 모두 정의. 일관되게 사용되고 있다.

**점수: 85/100** -- 체계적이고 일관됨. 다만 일부 하드코딩된 색상값(rgba 직접 사용)이 토큰 시스템을 벗어남.

### 2. 타이포그래피 위계 (15%)

4단계 폰트 패밀리: `headline`(Noto Serif KR), `body`(Be Vietnam Pro), `label`(Be Vietnam Pro), `dream-quote`(Cormorant Garamond). 크기는 `text-4xl`(헤드라인) ~ `text-[9px]`(라벨)까지 넓은 범위. 위계 자체는 명확하나 `text-[9px]`, `text-[10px]`가 과도하게 많이 사용됨 -- 가독성 이슈.

**점수: 75/100** -- 위계는 명확하나 극소 사이즈 남용이 디자인 품질을 낮춤.

### 3. 컴포넌트 패턴 통일 (10%)

하단 네비게이션이 모든 화면에 존재하나, 구현이 화면마다 미세하게 다르다. IntroScreen은 `left-0` 없이 `w-full`, GalleryScreen은 `left-0 w-full`로 미세 차이. 카드 컴포넌트도 `glass-card`, `glass-card-subtle`, `bg-surface-container-low` 등 3가지 패턴이 혼재. `card-radius`(1.5rem)와 `rounded-xl`(3rem)이 혼용됨.

**점수: 70/100** -- 대체로 통일되어 있으나 디테일에서 불일치.

### 4. 인터랙션/모션 완성도 (15%)

- `screen-enter` (fadeInUp 0.4s): 모든 화면 적용 -- 일관됨
- `glow-ring-animate` (선택 피드백): SceneBuilder, EmotionScreen에 적용
- `emotion-pulse` (선택된 감정 카드 맥동 2s infinite): 적용됨
- `breathe` (별자리 호흡): IntroScreen, AnalysisScreen에 적용
- `card-reveal` (카드 등장 0.8s rotateY): InterpretationScreen에 적용
- `constellation-draw` (선 그리기 3s): IntroScreen, AnalysisScreen에 적용

모션이 체계적이고 목적에 맞게 사용됨. 다만 모든 모션이 CSS 기반이라 인터랙티브 반응형 모션(스크롤 연동, 제스처 연동)은 없다.

**점수: 78/100** -- CSS 모션은 잘 정의되어 있으나 인터랙티브 모션 부재.

### 5. 다크 테마 완성도 (15%)

전체 앱이 다크 테마 전용(`<html class="dark">`, 배경 #120e31). M3 다크 팔레트가 일관 적용. 대비 비율은 대체로 양호하나, `text-on-surface-variant/40`(rgba 0.4 opacity)이나 `text-white/20` 같은 극저 불투명도 텍스트가 다수 존재 -- 이는 다크 배경에서 거의 보이지 않을 수 있다.

**점수: 78/100** -- 기본 체계는 우수하나 극저 opacity 텍스트의 가시성 이슈.

### 6. 에셋 품질/밀도 (15%) -- NEW

**이것이 이번 리뷰의 핵심 감점 항목이다.**

모든 비주얼 에셋이 Material Symbols Outlined 아이콘에 의존한다. 화면별 고유 일러스트레이션, 사진, SVG 아트워크가 전무하다.

- IntroScreen: SVG 별자리 = 원(circle) 6개 + 선(line) 7개. 이것은 "별자리"라기보다 "기하학적 도형 몇 개"이다.
- DreamIconComposition: 장소/날씨/오브젝트 아이콘을 그라데이션 배경 위에 배치. 구조는 스마트하지만, 결과물은 "아이콘 3개가 떠있는 색판"이다.
- Dream Story 카드: 텍스트 + 그라데이션 배경만. 비주얼 페이로드 없음.
- Gallery 카드: 감정 그라데이션 상단 띠(1.5px) + 텍스트. 시각적 밀도가 낮다.

Stitch 레퍼런스에서는 풍경 이미지가 사용되어 시각적 밀도가 높았으나, 구현은 아이콘만으로 처리하여 "프로토타입" 느낌이 강하다.

**점수: 45/100** -- 고유 비주얼 에셋이 없고, Material Icons에 전적으로 의존.

### 7. 비주얼 리치니스 (15%) -- NEW

화면이 "완성되고 풍부한" 느낌인가, "비어 있고 템플릿 같은" 느낌인가?

- IntroScreen: 별자리 + 호흡 애니메이션 + 글로우 블러가 어느 정도 공간을 채운다. 하지만 중앙 비주얼이 "투명한 원 4개"이기 때문에 Stitch 레퍼런스의 수정구슬 이미지 대비 임팩트가 약하다.
- SceneBuilder: 3열 그리드에 아이콘 카드가 정렬되어 있다. 기능적으로 깔끔하지만 시각적으로 "설문조사 양식"이다.
- AnalysisScreen: 이 앱에서 가장 리치한 화면. 수정구슬(글로우 + 스월 + 반사광) + 별자리 배경 + 단계별 텍스트. 몰입감 있다.
- InterpretationScreen: DreamIconComposition 덕분에 상단에 컬러풀한 영역이 있고, 카드 레이아웃이 타로 카드 느낌을 준다. 중간 수준.
- PatternScreen: 데이터 시각화(도넛 차트, 바 차트, 선 그래프, 워드 클라우드, 컬러 팔레트)가 다양하다. 데이터가 있을 때는 상당히 풍부하다.
- ShareScreen: 카드 + 3개 공유 버튼. 레퍼런스와 유사하지만 카드 비주얼이 아이콘 기반.

**점수: 58/100** -- AnalysisScreen과 PatternScreen은 리치하지만, 나머지 화면은 아이콘 + 텍스트 + 그라데이션의 반복으로 단조롭다.

## 디자이너 코멘트

기존 일관성 중심 평가(84점)에서 에셋 품질과 비주얼 리치니스를 추가하면 70점으로 내려간다. 이것이 현실적인 점수다.

이 앱의 디자인 시스템은 우수하다 -- 색상 토큰, 폰트 위계, 모션 패턴이 체계적이다. 문제는 **"시스템은 좋은데 콘텐츠가 없다"**는 것이다. Material Icons 위에 아무리 좋은 글로우와 그라데이션을 입혀도, 사용자가 보는 것은 결국 "아이콘"이다.

### 80점 달성을 위한 제안

1. **DreamIconComposition에 장소별 SVG 일러스트 도입**: 최소 8개 장소에 대해 심플한 라인 아트 SVG를 만들면 "아이콘이 떠있는 색판"이 "장면 일러스트"로 변한다.
2. **Gallery 카드에 비주얼 썸네일 추가**: 현재 텍스트만 있는 카드에 DreamIconComposition 미니 버전이라도 추가하면 밀도가 올라간다.
3. **Dream Story 카드에 데이터 비주얼 삽입**: 텍스트만이 아니라, "가장 자주 찾아온 장소" 카드에 해당 장소 아이콘이 크게 들어가거나, 감정 변화 카드에 미니 그래프가 들어가면 프레젠테이션 슬라이드에서 "Wrapped"로 진화한다.
