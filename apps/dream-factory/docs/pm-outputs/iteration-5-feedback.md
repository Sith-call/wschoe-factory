# 꿈 공장 — Iteration 5 사용자 피드백: 박지훈 (Persona 2)

**평가일**: 2026-03-20
**페르소나**: 박지훈 (32세, 프리랜서 UX 리서처, ENTP, 파워 유저)
**평가 맥락**: 트위터에서 "AI 없이 꿈 해석해주는 앱"이라고 해서 깔아봄. Oura Ring으로 수면 트래킹하고 노션에 꿈 메모하는 습관 있음. 패턴 분석 + 자각몽 활용 목적.

## 1. 스크린별 반응 (박지훈 시점)

### 01. 인트로 (IntroScreen)

"첫인상은 좋다. 비주얼 퀄리티가 높고 '어젯밤 무슨 꿈 꿨어?'라는 카피도 자연스럽다. 온보딩 3단계도 간결해서 불필요한 마찰이 없고. 다만 최근 기록한 꿈 미리보기가 1개만 뜨는 건 아쉽다 — 2~3개 캐러셀이면 '다시 들어올 이유'가 훨씬 강해질 텐데. 갤러리/기록/통계 탭이 하단에 있는 건 표준적이라 좋다."

### 02. 장면 구성 (SceneBuilderScreen)

"이건 재밌다. 장소 12개, 날씨, 인물, 오브젝트를 조합하는 방식이 꿈 일기를 '구조화된 데이터'로 만들어주니까 분석할 거리가 생긴다. 탭 네비게이션도 직관적이고, 미리보기 아이콘 스트립이 선택 상태를 피드백해주는 것도 좋고. 다만 UX적으로 — 장소가 3열 그리드인데 12개면 스크롤이 상당히 길다. 카테고리별로 접히는 구조가 나을 수도 있어. 그리고 '다음 단계로' 버튼 위치 때문에 하단 카드 2개가 가려지는 거 보인다. 고정 푸터 높이 계산이 좀 아쉽다."

### 03. 감정 선택 (EmotionScreen)

"감정 8개, 최대 2개 선택, 선명도 슬라이더, 메모 — 입력 구조가 합리적이다. 선명도를 5단계로 정량화한 건 나중에 패턴 분석할 때 쓸모 있을 거라 기대된다. 하지만 메모가 80자 제한이라고? 꿈 설명을 80자에 담기엔 너무 짧다. 최소 200자는 돼야 한다. 그리고 감정을 2개까지만 선택 가능한 건 '복합 감정'을 표현하기엔 제한적이다. 꿈에서 느끼는 감정은 3~4개가 겹치는 경우가 많은데."

### 04. 분석 (AnalysisScreen)

"로딩 트랜지션인데 '상징을 해독하는 중...'이라는 카피가 기대감을 만든다. 오브 애니메이션도 분위기에 맞고. 다만 실제로 어떤 '분석'이 이루어지는 건지 궁금해진다. 이건 단순히 매핑 테이블 조회 아닌가? 나중에 이 의구심이 확인되면 신뢰도가 크게 떨어질 것 같다."

### 05. 해석 카드 (InterpretationScreen)

"여기서 기대와 현실의 갭이 생긴다. 카드 비주얼은 확실히 예쁘다. 타로 카드 느낌의 레이아웃, 아이콘 컴포지션, 키워드 뱃지, 상징 해설 — 형식은 잘 갖춰져 있다.

문제는 해석 텍스트의 깊이다. `generateInterpretation` 함수를 뜯어보면 결국 `PLACE_TEXTS[place][variant]` + `WEATHER_TEXTS[weather][variant]` + `EMOTION_TEXTS[emotion][variant]`를 이어 붙이는 구조다. 장소별 3개, 날씨별 3개, 감정별 3개 문장을 조합하면 최대 27가지인데, 오브젝트나 인물이 추가되더라도 근본적으로는 '독립적인 문장 나열'이다. 문장 간 논리적 연결이 없어서 읽으면 '그냥 각 요소 설명을 붙여놓은 것'이라는 느낌이 든다.

상징 해설(`symbolReadings`)도 오브젝트별 5개 의미 중 하나를 랜덤 선택하는 건데, 같은 오브젝트를 반복 선택하면 '아 이게 그냥 돌아가면서 나오는구나' 하고 패턴이 보일 거다. 3~4회 정도 쓰면 '매핑 테이블이 뻔하네'라는 생각이 들 것 같다.

`personalInsight`가 그나마 차별화 포인트인데 — 반복 상징 감지, 감정 아크 추적, 이전 꿈 제목 연결 — 이 세 가지를 하는 건 좋다. 하지만 인사이트 텍스트가 '최근 꿈에서 X가 자주 나타나고 있어요' 수준이면 깊이가 부족하다. '왜' 자주 나타나는지, 심리학적으로 어떤 의미가 있는지까지 가야 진짜 인사이트다."

### 06. 공유 (ShareScreen)

"공유 카드 디자인은 괜찮다. 해석 텍스트 첫 문장만 뽑아서 인용 형식으로 보여주는 건 센스 있다. 근데 '이미지 저장'이 alert('추후 업데이트'), 카카오톡도 alert — 이건 MVP라 이해하지만, 실제 이미지 렌더링(html2canvas 같은)이 없으면 공유 기능의 가치가 0이다. SNS에 텍스트 링크를 올릴 사람은 거의 없다."

### 07. 갤러리 (GalleryScreen)

"갤러리 UI는 나쁘지 않다. 감정별 그라디언트 컬러 코딩이 시각적으로 구분감을 주고, 필터 칩도 잘 작동한다. 메이슨리 레이아웃도 적절하다. 그런데 — 검색이 없다. 꿈을 30개, 50개 쌓으면 특정 꿈을 찾기가 불가능하다. 키워드 검색이나 날짜 범위 필터가 필요하다. 그리고 삭제 기능도 보이지 않는다."

### 08. 패턴 분석 (PatternScreen)

"이 앱을 깔아본 핵심 이유인데... 솔직히 기대에 미치지 못한다.

**꿈 기록 빈도 (14일 달력)**: 개념은 좋은데 구현이 아쉽다. dot calendar가 14일 고정인데, 월간 뷰나 확장이 안 된다. 기록 습관을 '스트릭'으로 보여주면 동기 부여가 될 텐데.

**자주 나오는 상징 TOP 5**: 이건 괜찮다. 장소+오브젝트를 합쳐서 빈도를 보여주는 건 의미 있다. 근데 bar width가 하드코딩된 `[90, 65, 55, 40, 25]`라서 실제 데이터 비율을 반영하지 않는다. 1위가 12회, 2위가 11회여도 바 길이 차이가 크게 보인다. 이건 데이터 시각화로서 misleading이다.

**감정 분포 (도넛 차트)**: 시각적으로는 예쁜데, 중앙에 '84%'라고 쓰여있는 게 의미 불명이다. 코드를 보면 `emotionDist.reduce((s, e) => s + e.pct, 0)` — 이건 항상 100%에 수렴할 텐데 왜 84%가 나오지? 아마 반올림 이슈인 것 같은데, 이렇게 의미 없는 숫자를 중앙에 크게 표시하면 '분석'의 신뢰도가 떨어진다.

**꿈 선명도 추이**: 여기가 가장 문제다. `dotHeights`가 `[20, 45, 30, 60, 50, 80, 70]`으로 하드코딩되어 있다. 실제 vividness 데이터(1~5)를 전혀 반영하지 않는 장식용 차트다. SVG path도 `d='M 24 74 L 80 49...'`로 고정값이다. 이건 '데이터 시각화'가 아니라 '데이터처럼 보이는 일러스트레이션'이다. UX 리서처 입장에서 이런 건 용납하기 어렵다.

**패턴 인사이트**: `getPatternInsights` 함수를 보면, 3회 이상 반복되는 오브젝트/감정/장소를 감지하고 평균 선명도가 4 이상이면 루시드 드리머 코멘트를 주는 정도. 구현 자체는 나쁘지 않지만 출력이 한 줄짜리 문장 하나. 인사이트가 '최근 바다 상징이 자주 등장해요'에서 끝나면 나도 그건 알아. '그래서 뭔데?'가 없다."

## 3. Top 3 Pain Points

### Pain 1: 패턴 차트가 데이터를 반영하지 않는 장식용 시각화

선명도 추이 차트의 dot 높이와 SVG path가 하드코딩이고, TOP 5 상징 바 너비도 `[90, 65, 55, 40, 25]` 고정값이다. 데이터 기반 앱에서 데이터가 반영 안 되는 차트는 치명적. 사용자가 이걸 발견하면 (파워 유저는 반드시 발견함) 앱 전체의 신뢰도가 무너진다.

**심각도**: Critical
**영향**: 앱 핵심 가치 제안(패턴 분석) 자체의 신뢰 붕괴

### Pain 2: 해석 텍스트의 조합 로직이 금방 드러남

장소 3개 + 날씨 3개 + 감정 3개 문장의 독립적 나열. 문장 간 논리적 연결 없음. 같은 조합을 다시 선택하면 비슷한 해석이 나옴. 3~4회 사용 후 "이건 그냥 랜덤 조합이구나" 감이 옴.

**심각도**: High
**영향**: 리텐션 D3~D7에서 이탈 위험

### Pain 3: 데이터 포터빌리티 전무

내보내기(Export) 기능 없음. 삭제 기능 없음. localStorage 의존이라 브라우저 데이터 삭제 시 전체 기록 소실. 파워 유저가 3개월치 데이터를 쌓았는데 날아가면 → 즉시 삭제.

**심각도**: High
**영향**: 장기 사용자 신뢰 및 데이터 축적 동기

## 4. Top 3 Delights

### Delight 1: 구조화된 꿈 입력 UX

장소/날씨/인물/오브젝트를 아이콘 기반으로 선택하는 방식이 텍스트 일기보다 훨씬 빠르고, 자동으로 분석 가능한 데이터가 된다. 이건 기존 꿈 일기 앱에 없는 진짜 차별점이다. 아침 반쯤 깬 상태에서 아이콘만 탭하면 되니까 진입 장벽이 낮다.

### Delight 2: 반복 상징/감정 아크 감지 (personalInsight)

히스토리 기반으로 반복 상징을 감지하고, 최근 3개 꿈의 감정 흐름(negative→positive 전환 등)을 분석하는 건 좋은 시작점이다. 이게 더 깊어지면 진짜 가치가 생긴다.

### Delight 3: 타로 카드 스타일의 해석 카드 비주얼

InterpretationScreen의 카드 레이아웃이 정말 예쁘다. 아이콘 컴포지션 + 그라디언트 + 키워드 뱃지 + 상징 해설 구성이 "결과물을 받는 경험"으로서 만족스럽다. 캡처해서 공유하고 싶은 느낌은 확실히 있다 (공유 기능이 실제로 작동하면).

## 5. PM 가설 및 개선 방향

### 가설 1: 차트를 실제 데이터 기반으로 교체하면 D7 리텐션이 15%+ 개선될 것

- 선명도 추이: `vividnessTrend` 배열의 실제 값을 높이에 매핑 (예: `(vividness / 5) * maxHeight`)
- TOP 5 상징 바: 실제 count 비율로 너비 계산 (`(count / maxCount) * 100%`)
- 도넛 차트 중앙: 무의미한 퍼센트 대신 총 기록 수 또는 가장 빈번한 감정 표시
- **검증**: A/B 테스트로 하드코딩 vs 실제 데이터 차트 비교

### 가설 2: 해석 텍스트에 요소 간 관계 로직을 추가하면 "해석 신뢰도" 체감이 2배 이상 오를 것

- 현재: 각 요소별 독립 문장 나열
- 제안: 조합별 시너지 텍스트 추가 (예: forest + rain → "비에 젖은 숲은 정화와 성장이 동시에 일어나는...")
- 오브젝트 의미를 5개에서 15개로 확장하여 반복 노출 지연
- **검증**: 동일 조합 3회 반복 시 해석 텍스트 유사도 측정

### 가설 3: JSON 내보내기 + 삭제 기능을 추가하면 파워 유저 리텐션이 확보될 것

- JSON/CSV 내보내기 (꿈 기록 전체)
- 개별 꿈 삭제 + 확인 다이얼로그
- 갤러리 키워드 검색 (해석 제목 + 상징 키워드 기반)
- **검증**: 내보내기 클릭률로 파워 유저 비율 측정

### 가설 4: 패턴 인사이트를 심리학 기반 해석으로 확장하면 자기이해 도구로서의 가치가 생길 것

- "바다가 자주 나타나요" → "바다는 융 심리학에서 무의식의 총체를 상징합니다. 최근 감정적으로 깊은 곳을 탐색하려는 욕구가 있을 수 있어요."
- 주간/월간 리포트 생성 (감정 아크 요약 + 반복 상징 + 선명도 추세)
- **검증**: 인사이트 카드 '유용했어요' 반응 수집

### 가설 5: 선명도 데이터를 Oura Ring 등 수면 트래커와 연동하면 차별화된 포지셔닝이 가능할 것 (장기)

- 수면 단계(REM/Deep)와 꿈 선명도 상관관계 시각화
- 수면 시간대별 꿈 패턴 분석
- **검증**: 수면 트래커 사용자 비율 조사 후 우선순위 결정

## 6. 한 줄 요약

> **"데이터 앱이라고 약속해놓고 데이터를 장식으로 쓰는 건 파워 유저에게 가장 큰 배신이다. 차트를 진짜로 만들고, 해석에 깊이를 더하면 매일 쓸 수 있는 앱이 된다."**
