## Iteration 1 피드백 반영 확인

### 1. "수정하기"가 실제 수정 화면으로 이동하는가?

**부분 반영 (70%)**

Iteration 1에서 "캘린더 DayDetail 바텀시트의 수정하기 버튼이 `setSelectedDate(null)`만 호출해서 실제로 수정이 안 된다"고 피드백했다. 이번에 확인한 결과:

- `CalendarPage`에서 `onEditDate` prop을 받아서 `DayDetail`의 `onEdit` 콜백에 연결했다.
- `App.tsx`에서 `onEditDate` 핸들러가 해당 날짜의 기록 유형(morningLog/nightLog)에 따라 `MorningLogPage` 또는 `NightLogPage`를 overlay로 열고, `editDate`를 전달한다.
- `MorningLogPage`와 `NightLogPage` 모두 `editDate` prop을 받아 해당 날짜의 기존 기록을 불러온 상태에서 수정 화면을 보여준다.

**수정 기능 자체는 동작하게 되었다.** 다만 아쉬운 점:
- 수정 화면에 "수정 모드"라는 표시가 없어서, 새 기록을 쓰는 건지 기존 기록을 고치는 건지 화면만 봐서는 구분이 안 됨. 타이틀이 그냥 "아침 기록"/"밤 기록"으로 동일함.
- morningLog와 nightLog가 둘 다 있는 날짜를 탭하면 morningLog 수정만 열림. nightLog도 같이 수정하고 싶을 때 진입 경로가 없음.

### 2. 인사이트 제품 섹션이 3개로 분리되었는가?

**완전 반영 (100%)**

Iteration 1에서 제품 인사이트가 "좋은 제품/주의가 필요한 제품" 2개로만 분류되어 있다고 지적했는데, 이번에 확인한 결과 3개 섹션으로 분리되었다:

1. "피부에 좋은 제품" (`impact > 0`, `usedDays >= 3`)
2. "주의가 필요한 제품" (`impact < 0`, `usedDays >= 3`)
3. "아직 판단하기 이른 제품" (`usedDays < 3`)

신제품 마니아인 나한테 "아직 판단하기 이른 제품" 카테고리는 진짜 유용함. 새 제품 사서 2~3번 써봤을 때 성급한 판단을 막아주고, "더 써봐야 한다"는 메시지를 주니까 데이터 기반 스킨케어 습관에 도움이 됨.

## 새로운 피드백

### 피드백 1: 메모 입력이 textarea로 개선됨 — 하지만 홈 카드에 수정 버튼이 여전히 없음

Iteration 1에서 지적했던 "메모 input이 단일 줄"이 `textarea rows={3}`으로 바뀐 건 확인함. 좋은 개선.

그런데 홈 화면에서 오늘 기록을 완료하면 카드에 점수/키워드/메모가 표시만 되고, "수정하기" 버튼이 없음. 오늘 아침에 3점으로 기록했는데 점심에 보니까 "아 사실 2점이었는데" 하고 고치고 싶을 때, 캘린더로 가서 오늘 날짜를 탭하고 바텀시트에서 수정하기를 눌러야 함. 홈 카드에서 바로 수정으로 진입할 수 있는 경로가 필요함.

### 피드백 2: 캘린더 기록률 표시가 여전히 "N일"만 보여줌

Iteration 1에서 "기록률이 N일로만 표시되고 비율(%)이 없다"고 피드백했는데, 이번에도 `기록률: {recordedDays}일`로 동일함. "21/31일 (68%)" 같은 형태가 더 동기부여된다고 분명히 적었는데 반영 안 됨. 특히 나같이 매일 기록하려고 노력하는 사용자에게 비율은 중요한 동기 요소임.

### 피드백 3: 키워드에 여전히 "모공" 관련 항목 없음

Iteration 1에서 가장 강하게 요청한 "모공 벌어짐", "피지 과다" 같은 지성 피부 키워드가 여전히 추가되지 않음. 현재 11개 키워드(촉촉, 건조, 트러블, 뾰루지, 각질, 번들, 칙칙, 화사, 붉음, 당김, 매끈) 중 "번들"이 그나마 지성 관련이지만, 모공이 벌어진 느낌과 번들거리는 건 다른 증상임. 지성 피부 사용자로서 핵심 고민인 "모공"을 기록할 수 없으면 이 앱의 가치가 절반으로 줄어듦. `ALL_KEYWORDS`에 `'pore'`(모공), `'oily'`(피지) 같은 키워드가 반드시 필요함.

### 피드백 4: 생활 변수에 "카페인", "새 제품 사용" 변수가 여전히 없음

Iteration 1에서 "카페인", "물 많이 마심", "새 제품 사용" 변수를 요청했는데 `ALL_VARIABLES`가 10개 그대로임. 특히 "새로운 제품 처음 사용" 변수는 신제품 마니아인 나한테 트러블 원인 추적의 핵심 단서가 될 수 있음. 현재 제품 선택만 하면 "이미 쓰던 건지, 오늘 처음 쓴 건지" 구분이 안 되니까, 며칠 후 트러블이 났을 때 "그때 새 제품 때문인가?"를 역추적하기 어려움.

### 피드백 5: 온보딩 Step 2에서 피부 타입 미선택 시 "다음" 버튼 비활성화 미반영

Iteration 1에서 "피부 타입을 선택 안 해도 다음 버튼이 활성화되어 있고, 건너뛰기 버튼과 의도가 모호하다"고 피드백했는데, 현재 코드를 보면 Step 2의 "다음" 버튼에 `disabled` 조건이 없음 (Step 1만 `disabled={step === 1 && !name.trim()}`이 걸려 있음). "건너뛰기"가 있으니 "다음"은 최소 1개 선택 시 활성화되어야 의미가 구분됨. 아니면 건너뛰기를 없애고 다음 버튼을 항상 활성화하되 "선택하지 않으면 일반 피부로 설정돼요" 같은 안내를 붙이거나.

## 만족도 점수

**65 / 100**

### 근거

**개선된 것 (+3점, 62 → 65):**
- 수정 기능이 실제로 동작하게 됨. 캘린더에서 과거 기록 수정 가능 (+3)
- 인사이트 제품 섹션이 3개로 분리되어 신제품 실험 추적이 명확해짐 (+3)
- 메모 입력이 textarea로 바뀌어 여러 줄 작성 가능 (+1)
- `editDate` prop으로 MorningLogPage/NightLogPage 모두 수정 모드 지원 (+1)

**여전히 부족한 것 (이전 피드백 미반영, -8점 유지):**
- 모공/피지 키워드 미추가. 지성 피부 핵심 고민을 기록할 수 없음 (-10 유지)
- 생활 변수 "카페인", "새 제품 사용" 미추가 (-3 유지)
- 캘린더 기록률에 비율(%) 미표시 (-2 유지)
- 온보딩 Step 2 validation 미반영 (-1 유지)
- 홈 카드에서 오늘 기록 수정 진입 경로 없음 (-3 유지)
- 인사이트 기간 필터 여전히 없음 (-5 유지)
- 부위별 피부 기록 여전히 불가 (-8 유지)

**결론:**
수정 기능과 인사이트 3분할은 잘 반영됐고 체감 가능한 개선임. 하지만 Iteration 1에서 "Top 5 개선 제안" 1번으로 올린 핵심 피드백(모공/피지 키워드, 피부 고민 선택)이 아직 반영되지 않은 게 가장 크게 아쉬움. 지성 피부 사용자가 "모공"을 기록할 수 없는 스킨케어 앱은, 아무리 구조가 잘 짜여 있어도 "내 앱이 아닌 느낌"이 듦. 이 부분이 해결되면 70점대 중반은 줄 수 있을 것 같음.
