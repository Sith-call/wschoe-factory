## Iteration 1 피드백 반영 확인

### 1. 인사이트 요약이 홈에 노출되는가?

**반영됨.** 홈 화면 하단에 "이번 주 발견" 카드가 추가되었음. `bestProduct`(가장 좋았던 제품 + 점수 영향)와 `worstVariable`(가장 나빴던 변수 + 점수 영향)이 한 줄씩 표시됨. 이전에 "인사이트 탭에 들어가야만 분석 결과를 볼 수 있다"는 피드백이 정확히 반영됨. 다만 데이터가 충분하지 않으면(제품 3회 미만 사용 등) 이 카드 자체가 안 보이므로, 신규 유저에게는 여전히 인사이트의 존재를 모를 수 있음.

### 2. "수정하기"가 실제 동작하는가?

**반영됨.** 이전에는 `onEdit`이 `() => setSelectedDate(null)`로만 되어 있어 시트만 닫혔는데, 이제 CalendarPage에서 `onEditDate` prop을 통해 해당 날짜의 기록 유형(아침/밤)에 따라 적절한 편집 오버레이(`morningLog` 또는 `nightLog`)가 열림. `editDate` state를 통해 해당 날짜의 기존 데이터가 폼에 pre-fill되는 구조. 실제 수정 후 저장이 동작하는 흐름이 완성됨.

### 3. 저장 완료 피드백이 있는가?

**반영됨.** App.tsx에서 `showToast('기록이 저장되었어요')` 토스트 메시지가 아침/밤 기록 저장 시 모두 노출됨. 하단에 2초간 표시되는 다크 배경 토스트. 이전에 저장 후 아무 피드백 없이 화면이 닫히던 것이 개선됨.

### 추가 확인: 메모 textarea 변경

**반영됨.** MorningLogPage의 메모 입력이 `<textarea rows={3}>`로 변경되어 여러 줄 입력이 가능해짐. "턱에 좁쌀 2개 올라옴, 어제 라면 먹었는데 그거 때문인 듯" 같은 긴 메모를 편하게 적을 수 있게 됨.

## 새로운 피드백 (Iteration 2)

### 1. 캘린더 기록률에 퍼센트 여전히 없음

CalendarPage의 월간 통계에서 `기록률: {recordedDays}일` 형태가 그대로임. Iteration 1에서 "15일 / 31일 (48%)" 같은 비율 형태를 요청했는데 반영되지 않았음. 이번 달 며칠 중 며칠을 기록했는지, 얼마나 꾸준했는지를 퍼센트로 보고 싶음. 특히 취준 기간에 피부 관리 꾸준히 하고 있다는 성취감이 중요한데, "기록률: 15일"만으로는 잘하고 있는 건지 감이 안 잡힘.

### 2. 캘린더에서 기록 없는 날 클릭 시 여전히 무반응

CalendarGrid에서 `hasRecord`가 false인 날은 `onDateSelect`를 호출하지 않음. 빈 날을 눌러도 아무 반응이 없어서, 과거 기록을 소급 입력할 방법이 없음. "이 날은 기록이 없어요 -- 지금 기록하기" 같은 빈 상태 바텀시트가 떠서 과거 날짜에 대해 아침/밤 기록을 할 수 있으면 좋겠음. 어제 기록을 깜빡한 날이 있으면 다음날이라도 채워넣고 싶음.

### 3. 홈 "이번 주 발견" 카드가 탭 연결 없음

인사이트 요약이 홈에 노출된 건 좋은데, 이 카드를 탭하면 인사이트 탭으로 이동하는 인터랙션이 없음. 단순 텍스트 노출에 그치고 있어서, "오 밀가루가 안 좋았다고? 더 자세히 보고 싶은데" 하는 순간에 직접 하단 탭바에서 인사이트를 찾아 눌러야 함. 카드 자체를 탭하면 인사이트 탭으로 자동 전환되면 탐색 플로우가 훨씬 자연스러울 것.

### 4. 밤 기록 저장 후 아침 기록 유도 없음

밤 기록을 저장하면 토스트("기록이 저장되었어요")가 뜨고 홈으로 돌아감. 여기서 "내일 아침에 피부 체크 알림 받을래요?" 같은 유도가 없음. 밤에 루틴 기록하고 다음날 아침에 피부 상태를 기록해야 이 앱의 핵심 루프(밤 루틴 -> 아침 피부 연결)가 완성되는데, 아침 기록을 잊어버릴 가능성이 높음. 푸시 알림은 어렵더라도, 최소한 토스트 대신 "저장 완료! 내일 아침에 피부 점수도 기록해봐" 같은 안내 모달이 있으면 리텐션에 도움이 될 것.

### 5. DayDetail에서 아침-밤 순서 여전히 역순

Iteration 1에서 "전날 밤 루틴 -> 오늘 아침 결과" 순서로 뒤집자고 제안했는데, DayDetail 컴포넌트를 보면 여전히 `morningLog`가 먼저 렌더링되고 그 아래에 `prevRecord?.nightLog`가 나옴. 앱의 핵심 메시지가 "피부는 어제의 결과"인데, 상세 뷰에서 원인(밤 루틴)이 결과(아침 피부) 아래에 위치하면 인과 흐름이 직관적이지 않음. "전날 밤 루틴 -> (화살표/연결 텍스트) -> 오늘 아침 피부" 순서로 배치해야 컨셉이 일관됨.

### 근거

**개선된 부분 (+10점, 62 -> 72):**
- 홈에 인사이트 요약("이번 주 발견") 노출: 매일 앱을 열 이유가 생김. 기록만 하는 앱에서 피드백을 주는 앱으로 한 단계 진화. (+4점)
- "수정하기" 기능 완성: 캘린더에서 과거 기록을 실제로 수정할 수 있게 됨. 미완성 기능이 해소됨. (+3점)
- 저장 완료 토스트: 기록 저장 시 시각적 확인이 생겨서 "저장됐나?" 불안감 해소. (+1점)
- 메모 textarea 변경: 여러 줄 메모 가능. 피부 상태를 자세히 적을 수 있게 됨. (+2점)

- 캘린더 빈 날 소급 기록 불가 + 기록률 퍼센트 미표시: 2회 연속 미반영. (-5점)
- DayDetail 인과관계 순서 미반영: 2회 연속 미반영. 컨셉과 UI의 불일치. (-4점)
- 홈 인사이트 카드 탭 인터랙션 없음: 정보 노출은 됐지만 탐색 연결이 끊김. (-3점)
- 밤-아침 기록 연결 유도 부재: 핵심 루프 완성을 돕는 장치가 없음. (-3점)
- 생활 변수 기본값(프리셋) 설정 여전히 없음: 앞머리를 매일 눌러야 하는 번거로움. (-3점)
