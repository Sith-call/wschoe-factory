# Persona Feedback: 김수빈

- **나이**: 27세 직장인
- **피부 타입**: 건성, 환절기 건조
- **핵심 니즈**: "새 제품 썼는데 맞는 건지 판단이 안 돼"
- **주요 변수**: 야근/스트레스, 회식(음주), 운동 유무
- **테스트 일시**: 2026-03-21
- **테스트 환경**: 375x812 viewport (iPhone 크기), localhost:5186

## 화면별 피드백

### 1. 온보딩 (OnboardingPage)

**Step 1 - 소개 + 이름 입력**
- "피부는 어제의 결과예요" 카피가 직관적이고 공감됨. 내가 딱 원하는 게 이거 -- 어젯밤 뭘 발랐는데 그게 맞는 건지 결과를 보고 싶은 것.
- 이름 입력은 간단하지만, 닉네임인지 실명인지 가이드가 없어서 살짝 망설임.
- "다음" 버튼이 이름 미입력 시 disabled 처리되어 있어서 좋음.

**Step 2 - 피부 타입 선택**
- 건성/복합성/지성/민감성 4가지만 제공. 나는 "건성+민감성" 복수 선택이 가능해서 좋음.
- 하지만 "건조한 건 맞는데 T존은 좀 번들거려" 같은 부분 건조 케이스를 표현하기 어려움.
- "건너뛰기"가 있어서 급할 때 좋지만, 피부 타입은 인사이트에 활용되는지 불명확. 선택해도 이후 어디에서 반영되는지 보이지 않음.

**Step 3 - 제품 등록**
- 카테고리 칩(클렌징/토너/세럼 등 9종) 선택 후 제품명 입력하는 방식.
- 나는 지금 새 세럼 하나를 테스트 중인데, 기존 루틴 제품도 다 등록해야 하나? 라는 의문. "최소 몇 개 등록하세요" 같은 안내가 없음.
- 제품명을 직접 타이핑해야 함. 자동완성이나 인기 제품 추천이 없어서, 정확한 제품명을 기억해야 하는 부담.
- "데모 데이터로 체험하기" 링크가 있어서 앱 느낌을 먼저 볼 수 있는 건 좋음.

### 2. 홈 화면 (HomePage)

- 상단에 잎 아이콘(제품 관리)과 설정 아이콘이 있는데, 잎 아이콘이 "제품 관리"라는 것을 직감하기 어려움. 아이콘만으로는 의미 전달 부족.
- "안녕, 수빈님 / 오늘 피부는 어때?" 인사말이 친근함.
- "오늘 아침 피부" 카드와 "오늘 밤 루틴" 카드 두 개가 명확하게 구분되어 있음. 아침은 결과 기록, 밤은 원인 기록이라는 구조가 직관적.
- 아침 기록 전일 때 "아침 피부 기록하기" 버튼이 채워진 스타일, 밤 기록은 outline 스타일 -- 우선순위가 시각적으로 보여서 좋음.
- 주간 요약(WeeklySummary) 차트가 아래에 있는데, 첫 사용 시에는 데이터가 없어서 "기록을 시작하면 여기에 추이가 나와요"라는 빈 상태 안내가 나옴. 적절함.
- 연속 기록 일수(streak) 표시가 동기부여가 됨.

### 3. 아침 기록 (MorningLogPage)

- "오늘 아침 피부는 어때?" 질문 후 1~5점 원형 점수 선택. 색상으로 시각 피드백(빨강~초록)이 있어서 직관적.
- 점수 라벨(최악/별로/보통/좋아/꿀피부)이 한국어로 잘 되어 있음.
- 키워드 칩: 촉촉/건조/트러블/뾰루지/각질/번들/칙칙/화사/붉음/당김/매끈 -- 11개. 건성인 나한테 "건조", "당김", "각질"이 자주 쓸 것 같은 키워드인데, 키워드 순서가 고정이라 매번 찾아야 함.
- 메모는 input type="text"로 한 줄짜리. "새 세럼 써봤는데 다음날 건조함이 좀 줄었다" 같은 긴 메모를 쓰기엔 부족함. textarea가 더 적합해 보임.
- 저장 안 하고 닫으려 하면 confirm 다이얼로그가 뜸. 좋음.

### 4. 밤 기록 (NightLogPage)

- "오늘 밤 뭘 발랐어?" 질문이 캐주얼하고 좋음.
- "어젯밤과 동일" 버튼이 있어서 루틴이 비슷할 때 빠르게 기록 가능. 이건 정말 편리.
- 제품 선택 시 카테고리별로 그룹핑되어 표시됨. 등록한 제품이 칩으로 나와서 탭만 하면 됨.
- 밤 기록에서 바로 "제품 추가"도 가능.
- 생활 변수: 밀가루/매운 음식/음주/운동/수면 부족/앞머리/스트레스/야근/마스크/생리전 -- 10개. 야근, 스트레스, 음주가 내 주요 변수인데 다 있어서 좋음.
- 하지만 "회식"이라는 변수가 없음. 회식은 음주 + 밀가루 + 늦은 귀가의 복합 변수인데, 각각 따로 선택해야 함. 커스텀 변수 추가 기능이 없음.
- 밤 기록에 "몇 시에 잤는지" 수면 시간 기록이 없음. 수면 부족은 있지만 주관적 판단이라 데이터로 활용하기 어려움.

### 5. 캘린더 (CalendarPage)

- 월 네비게이션(이전 달/다음 달)이 있고, 점수별 배경색으로 한 달 피부 상태를 한눈에 볼 수 있음.
- 날짜 아래 작은 점(dot)으로 점수 색상 표시. 시각적으로 깔끔.
- 날짜 탭하면 바텀시트(DayDetail)로 해당 날 기록 확인 가능.
- DayDetail에서 "전날 밤 기록"도 함께 보여줘서 원인-결과 연결이 됨. 이건 핵심 가치에 잘 맞음.
- 이번 달 평균 점수와 기록률이 하단에 표시됨.
- 기록률이 "X일"로만 표시되고 "%"가 아님. "이번 달 30일 중 20일 기록 (67%)" 같은 형태가 더 의미 있을 것.
- DayDetail에서 "수정하기" 버튼이 있지만 onEdit이 단순히 setSelectedDate(null)로 닫기만 함. 실제 수정 기능이 동작하지 않음.

### 6. 인사이트 (InsightPage)

- 7일 미만 데이터면 프로그레스 바와 함께 "X일 더 기록하면 분석을 볼 수 있어요" 안내. 동기부여 장치로 좋음.
- "제품 분석" / "변수 분석" 세그먼트 탭이 있음.
- 제품 분석: 사용 시 vs 미사용 시 평균 점수를 바 차트로 비교. 이게 내가 가장 원하는 기능 -- "새 세럼이 효과가 있는지"를 데이터로 보고 싶었음.
- "아직 판단하기 이른 제품" 카테고리가 있어서 3회 미만 사용 시 섣부른 판단을 방지. 좋음.
- 변수 분석: "음주 다음날, 피부 점수가 평균 X점 낮아져요" 같은 문장형 인사이트 제공. 직관적.
- InsightCard에 impact 수치(+/-)와 색상(녹/적) 표시. 데이터 시각화가 잘 되어 있음.
- 다만, "이 제품을 계속 써야 할까?"에 대한 명확한 추천(recommendation)은 없음. 수치만 보여주고 해석은 사용자 몫.

### 7. 제품 관리 (ProductListPage)

- 카테고리별 그룹핑, 사용 횟수 표시, 보관/복원/삭제 기능이 있음.
- "보관" 개념이 있어서 안 쓰는 제품을 삭제 대신 보관할 수 있음. 나중에 다시 쓸 때 복원 가능.
- 영구 삭제 시 confirm 다이얼로그가 뜸.

### 8. 설정 (SettingsPage)

- 내 정보(이름, 피부 타입), 데모 모드 토글, 데이터 내보내기(JSON), 데이터 초기화, 기록 통계 표시.
- 프로필 수정 기능이 없음. 이름이나 피부 타입을 변경할 수 없음.
- 데이터 초기화 시 이중 confirm이 있어서 실수 방지. 좋음.
- 데이터 내보내기가 JSON -- 일반 사용자에게는 의미 없는 포맷. 엑셀이나 이미지 리포트가 더 유용할 것.

### 9. 하단 탭바 (TabBar)

- 홈/캘린더/인사이트 3탭. 텍스트만으로 구성(아이콘 없음).
- 아이콘 없이 텍스트만 있으니 좀 밋밋함. 하지만 기능이 3개뿐이라 혼란은 없음.
- 활성 탭에 하단 밑줄 + 색상 변경으로 현재 위치 표시. 적절.

## 개선 제안 (5개)

### 1. "이 제품 계속 써도 될까?" 판정 기능 추가
현재 인사이트는 수치만 보여주고 해석은 사용자에게 맡김. "새 제품 써봤는데 맞는 건지 판단이 안 돼"가 내 핵심 고민인데, 앱이 "이 제품은 사용 시 평균 점수가 0.5점 높아요. 2주 더 써보면 확실한 판단이 가능해요" 같은 문장형 추천을 제공하면 훨씬 실용적. 인사이트 페이지에 "판정 중 / 효과 있음 / 효과 없음 / 판단 불가" 같은 상태 라벨이 있으면 좋겠음.

### 2. 메모 입력을 textarea로 변경 + 제품별 메모 분리
아침 기록의 메모가 한 줄짜리 input인데, "세 세럼 바르고 자고 일어났더니 건조함이 좀 줄었다" 같은 관찰 기록을 남기기엔 너무 짧음. 또한 밤 기록에는 메모 기능 자체가 없어서, 제품 사용 시 느낌("이 세럼 좀 끈적였다")을 기록할 수 없음. 밤 기록에도 메모를 추가하고, 둘 다 textarea로 바꿔야 함.

### 3. 커스텀 생활 변수 추가 기능
현재 10개 고정 변수(밀가루/매운 음식/음주 등)만 제공. "회식", "카페인 과다", "환절기", "에어컨", "신제품 테스트" 같은 개인별 변수를 추가할 수 없음. 내 경우 "야근 후 회식"이 하나의 패턴인데, 이걸 표현하려면 야근 + 음주 + 밀가루를 각각 눌러야 함. 커스텀 변수를 만들 수 있으면 더 정확한 분석이 가능함.

### 4. 과거 기록 수정/추가 기능 구현
캘린더 DayDetail에 "수정하기" 버튼이 있지만 실제로 수정 화면으로 이동하지 않음 (닫기만 됨). 또한 과거 날짜에 기록을 추가할 수 없음. 야근하고 기록 못 한 날이나, 실수로 잘못 입력한 날을 보정할 수 없어서 데이터 정확도가 떨어짐.

### 5. 아침 키워드 자주 쓰는 순 정렬 또는 즐겨찾기
키워드 11개가 항상 같은 순서로 나옴. 건성 피부인 나는 "건조", "당김", "각질"을 거의 매일 쓰는데 매번 찾아야 함. 최근 자주 선택한 키워드를 상단에 배치하거나, 피부 타입(건성) 기반으로 관련 키워드를 우선 노출하면 기록 속도가 빨라짐. 30초 기록이 목표라면 이런 디테일이 중요.

### 근거

- 아침(결과) / 밤(원인) 분리 구조가 핵심 가치를 잘 반영 (+15)

- "어젯밤과 동일" 빠른 복사 기능으로 반복 기록 부담 감소 (+8)
- 생활 변수에 야근/스트레스/음주가 포함되어 내 주요 변수 커버 (+7)
- 캘린더 DayDetail에서 전날 밤 기록을 함께 보여주는 원인-결과 연결 (+5)
- 데모 모드로 앱 체험 가능 (+5)
- 온보딩 카피와 전체 UI 톤이 따뜻하고 부담 없음 (+5)
- 접근성(aria-label, 최소 44px 터치 타겟) 고려 (+5)

**아쉬운 점 (-)**
- 내 핵심 질문("이 제품 맞는 건가?")에 대한 직접적 답변/판정 기능 부재 (-12)
- 과거 기록 수정 불가 -- 수정 버튼은 있으나 미구현 (-8)
- 밤 기록에 메모 없음 + 아침 메모가 한 줄짜리 input (-7)
- 커스텀 변수 추가 불가 (-5)
- 제품 등록 시 자동완성/추천 없이 수동 입력만 가능 (-5)
- 피부 타입 선택이 이후 경험에 반영되지 않음 (키워드 우선순위, 인사이트 등) (-3)
- 탭바에 아이콘 없이 텍스트만 -- 시각적 밀도 부족 (-2)
- 데이터 내보내기가 JSON 포맷만 지원 (-2)
- 프로필(이름/피부타입) 수정 불가 (-1)

**총평**: 기본 구조와 핵심 아이디어(밤 루틴 기록 -> 아침 피부 결과 비교)는 훌륭. 데이터를 수집하는 UX는 잘 되어 있으나, 수집된 데이터를 기반으로 "그래서 이 제품 써도 돼?"라는 사용자의 핵심 질문에 명확하게 답해주는 마지막 한 발이 부족함. 기록의 편의성(메모 확장, 키워드 정렬, 과거 수정)도 보완이 필요.
