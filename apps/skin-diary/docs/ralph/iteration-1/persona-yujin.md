# 페르소나 피드백: 서유진

| 항목 | 내용 |
|------|------|
| 이름 | 서유진 |
| 나이 | 24세 |
| 직업 | 대학생 / 취준생 |
| 피부 타입 | 복합성, 턱 주변 트러블 |

## 화면별 피드백

### 1. 온보딩 (OnboardingPage)

**좋은 점:**
- "피부는 어제의 결과예요"라는 첫 문구가 공감됨. 밤에 바른 것과 다음 날 피부 상태를 연결하겠다는 컨셉이 바로 이해됨.
- 3단계 프로그레스 바가 단순하고 부담 없음.
- "데모 데이터로 체험하기" 버튼이 있어서 일단 어떤 앱인지 먼저 볼 수 있는 게 좋음.
- 피부 타입 복수 선택 가능한 점 (복합성이니까 건성+지성 같이 선택할 수 있음).

**아쉬운 점:**
- Step 2에서 피부 타입 4가지(건성, 복합성, 지성, 민감성)만 있음. "복합성+민감성"처럼 복합적인 경우가 많은데, 선택 후에 어디에 활용되는지 전혀 안 보임. 온보딩에서 받아놓고 인사이트나 홈에서 피부 타입 기반 코멘트가 전혀 없음.
- Step 3 제품 등록 시 카테고리 칩이 9개(클렌징~기타)인데 한 줄에 다 안 들어가서 줄바꿈됨. 375px 뷰포트에서 "아이크림", "선크림", "마스크팩", "기타"가 2번째 줄로 밀림 — 작지만 지저분해 보임.
- 제품명 입력란에 "제품명을 입력하세요"라고만 되어 있는데, 처음 쓰는 사람은 브랜드명까지 써야 하는지 헷갈림. "예: 아이소이 트러블케어 세럼" 같은 예시가 있으면 좋겠음.

### 2. 홈 (HomePage)

**좋은 점:**
- "안녕, 유진님 오늘 피부는 어때?" 인사말이 자연스러움.
- 아침 피부 카드와 밤 루틴 카드가 명확히 분리되어 있어 오늘 뭘 해야 하는지 바로 보임.
- 주간 추이 차트(WeeklySummary)가 SVG 라인 차트로 깔끔함. 점수 1~5 범위 + 요일 표시.
- 연속 기록 streak 표시가 동기부여됨.

**아쉬운 점:**
- 상단 아이콘이 왼쪽 잎사귀(제품 관리), 오른쪽 톱니바퀴(설정)인데 잎사귀 아이콘만으로 "제품 관리"를 알기 어려움. 처음 들어왔을 때 뭔지 몰라서 안 누를 것 같음.
- 밤 기록 버튼이 아침 기록 전에는 outline 스타일(border만 있는 버튼)로 되어 있음. 의도는 "아침 먼저 기록해"일 수 있지만, 나는 밤에 앱을 열 가능성이 높은데 이때 밤 기록 버튼이 너무 약해 보임.
- 주간 차트에서 기록이 없는 날은 점이 빠지고 선이 끊김. 빈 날에 회색 점이라도 찍혀 있으면 "아 이날 안 했구나"가 더 직관적일 듯.
- 주요 키워드("촉촉, 트러블, 건조" 등)가 차트 아래에 텍스트로만 나옴. 키워드별 빈도 바 차트 같은 게 있으면 내 피부 경향을 더 빨리 파악할 수 있을 것 같음.

### 3. 밤 기록 (NightLogPage)

**좋은 점:**
- "오늘 밤 뭘 발랐어?" — 대화체 톤이 편하고 자연스러움.
- "어젯밤과 동일" 버튼이 있어서 매일 같은 루틴이면 한 번에 끝남. 이거 진짜 편함.
- 제품이 카테고리별로 그룹핑되어 있어서 토너 > 세럼 > 크림 순서대로 선택하기 편함.
- 생활 변수(밀가루, 매운 음식, 음주, 운동, 수면 부족, 앞머리, 스트레스, 야근, 마스크, 생리전)가 10개로 적절함. "앞머리" 변수가 있는 게 신기하고 좋음 — 이마 트러블이랑 연관 분석 가능.
- 저장 안 하고 나가려 하면 confirm 다이얼로그가 뜸.

**아쉬운 점:**
- 제품 선택 시 칩 형태인데, 제품명이 길면("아이소이 트러블케어 세럼") 칩이 한 줄을 거의 다 차지함. 제품 6개 이상이면 스크롤이 길어질 것 같음.
- 생활 변수에 "앞머리"가 있는데 이게 on/off 토글임. 나처럼 항상 앞머리인 사람은 매번 눌러야 해서 번거로움. 기본값 설정 같은 게 있으면 좋겠음.
- 밤 기록에 "아무것도 안 발랐어" 같은 옵션이 없음. 귀찮아서 스킵한 날도 기록하고 싶은데, 제품 0개 선택 + 변수만 선택하고 저장하면 되긴 하지만 의도적으로 아무것도 안 바른 날과 그냥 기록 안 한 날의 구분이 안 됨.

### 4. 아침 기록 (MorningLogPage)

**좋은 점:**
- 1~5점 원형 점수 셀렉터가 직관적. 색상도 빨강(최악) → 초록(꿀피부)으로 자연스러움.
- "어떤 느낌이야?" 키워드 칩(촉촉, 건조, 트러블, 뾰루지, 각질, 번들, 칙칙, 화사, 붉음, 당김, 매끈) 11개가 피부 상태를 잘 커버함.
- 메모 필드가 선택사항이라 부담 없음.
- 점수 미선택 시 저장 버튼 비활성화 — 실수 방지.

**아쉬운 점:**
- 턱 주변 트러블이 주 고민인데, "트러블"과 "뾰루지"의 차이가 뭔지 모르겠음. 부위별(이마, 볼, 턱, 코) 선택이 있으면 더 정밀한 인사이트가 나올 텐데.
- 점수 1~5 레이블이 "최악, 별로, 보통, 좋아, 꿀피부"인데, 375px에서 5개 원이 꽉 차서 라벨 텍스트가 좁음. 큰 문제는 아니지만 1점과 2점 사이의 미묘한 차이를 구분하기 어려움 — 반점(0.5점 단위) 지원이 있으면 좋겠음.
- 메모 입력이 `input type="text"` 한 줄짜리임. "턱에 좁쌀 2개 올라옴, 어제 라면 먹었는데 그거 때문인 듯" 같은 긴 메모는 textarea가 더 적절함.

### 5. 캘린더 (CalendarPage)

**좋은 점:**
- 월 네비게이션(이전/다음 달)이 간단.
- 기록이 있는 날에 점수 색상 배경 + 하단 도트가 표시되어 한눈에 기록 현황 파악 가능.
- 날짜 클릭 시 DayDetail 바텀시트가 올라오는 UX가 자연스러움.
- 이번 달 평균 점수 + 기록률(N일) 표시.

**아쉬운 점:**
- 기록률이 "기록률: 15일" 형태인데, "15일 / 31일 (48%)" 같은 비율 형태가 더 직관적. 퍼센트를 보고 싶음.
- 기록이 없는 날을 클릭하면 아무 반응이 없음 (hasRecord 체크). 빈 날 클릭 시 "이 날은 기록이 없어요 — 지금 기록하기" 같은 안내가 있으면 과거 기록도 할 수 있을 텐데.
- DayDetail에서 "수정하기" 버튼이 있지만 `onEdit`이 `() => setSelectedDate(null)`로 되어 있어서 실제로 수정 기능이 없고 시트만 닫힘. 수정 기능이 동작하지 않음.
- DayDetail에서 "전날 밤 기록"을 보여주는데, 시간순으로 "전날 밤 루틴 → 이 날 아침 피부"로 배치하는 게 인과 관계를 더 잘 보여줄 것 같음. 현재는 아침 먼저 나오고 밤이 아래에 나옴.

### 6. 인사이트 (InsightPage)

**좋은 점:**
- "제품 분석" / "변수 분석" 세그먼트 컨트롤 구분이 명확.
- 7일 미만이면 "아직 데이터를 모으고 있어요" + 프로그레스 바로 기대감 형성.
- 제품별 "사용 시 vs 미사용" 비교 바 차트가 핵심 가치를 잘 전달함. 이게 이 앱을 쓰는 이유임.
- "밀가루 다음날, 피부 점수가 평균 0.8점 낮아져요" 같은 문장형 인사이트가 직관적.

**아쉬운 점:**
- "피부에 좋은 제품" / "주의가 필요한 제품" / "아직 판단하기 이른 제품" 분류가 좋지만, 제품이 많아지면 카드가 쭉 나열되어 스크롤이 길어짐. 가장 임팩트 큰 제품 1~2개를 하이라이트하는 요약 카드가 상단에 있으면 좋겠음.
- 변수 분석에서 "앞머리"나 "수면 부족" 같은 내 주요 변수가 데이터가 적으면 아예 안 보임. "아직 3회 미만" 이라는 안내라도 있으면 "더 기록해야겠다"는 동기가 될 텐데.
- 인사이트 결과의 신뢰도나 데이터 수가 너무 적을 때 통계적으로 유의미한지에 대한 안내가 없음. "사용 3회"로 인사이트를 주면 오해의 소지가 있을 수 있음.

## 개선 제안

### 1. 부위별 트러블 기록 추가
아침 기록에 "트러블 부위" 선택(이마, 볼, 턱, 코 등)을 넣으면 턱 주변 트러블이 주 고민인 유저에게 훨씬 정밀한 인사이트를 줄 수 있음. "밀가루 먹은 다음날 턱 트러블 확률 70%"  같은 분석이 가능해짐.

### 2. 생활 변수 기본값(프리셋) 설정
앞머리, 마스크 같은 "항상 해당"되는 변수를 프로필에서 기본 체크로 설정할 수 있게 하면, 매번 밤 기록 시 반복 선택하는 번거로움이 줄어듦. 특히 앞머리는 자르기 전까지 매일 해당되므로.

### 3. 인사이트 요약 카드 + 홈 노출
인사이트 탭에 들어가야만 분석 결과를 볼 수 있는데, 홈 화면 하단에 "이번 주 인사이트: 밀가루 먹은 날 피부 점수 -0.8" 같은 한 줄 요약을 노출하면 매일 앱을 열 동기가 더 강해짐. 지금은 기록만 하고 인사이트 탭을 안 볼 수도 있음.

### 4. 아침 메모를 textarea로 변경 + 사진 첨부
현재 한 줄 input인데, 피부 상태를 더 자세히 적고 싶을 때 부족함. textarea로 바꾸고, 장기적으로 피부 사진 첨부 기능이 있으면 시각적 변화 추적이 가능해짐. 사진 비교가 되면 "세럼 바꾸고 2주 후 턱 트러블 줄었다"를 눈으로 확인 가능.

### 5. 캘린더 DayDetail에서 인과관계 시각화
현재 DayDetail에서 "아침 피부" → "전날 밤 기록" 순서로 나오는데, 이를 "전날 밤 루틴 → 오늘 아침 결과"로 뒤집고, 점수와 밤 루틴 사이에 화살표나 "그 결과" 같은 연결고리를 넣으면 앱의 핵심 컨셉("피부는 어제의 결과")이 캘린더에서도 자연스럽게 전달됨.

### 근거

**잘 된 부분 (40점 기여):**
- 핵심 컨셉("밤 루틴 → 다음 날 피부")이 명확하고, 온보딩 첫 화면에서 바로 이해됨.
- 밤 기록 + 아침 기록 분리가 자연스러움. "어젯밤과 동일" 퀵카피 기능이 실용적.
- 인사이트 페이지의 제품별/변수별 비교 분석이 앱의 핵심 가치이며 잘 동작함.
- 생활 변수에 "밀가루", "앞머리" 같은 항목이 있어서 내 실제 피부 고민 변수를 추적할 수 있음.
- 데모 모드가 있어서 앱 가치를 온보딩 전에 체험 가능.
- 전체적인 디자인 톤(따뜻한 베이지 계열, Gowun Batang 세리프 헤딩)이 차분하고 좋음.

**아쉬운 부분 (-38점):**
- 부위별 기록 불가: 내 핵심 고민인 "턱 주변 트러블"을 특정할 수 없어서 인사이트 정밀도가 낮음. (-12점)
- 반복 변수 매번 선택: 앞머리를 매일 눌러야 하는 번거로움. 기본값 설정 부재. (-6점)
- 캘린더 수정 미작동: DayDetail "수정하기"가 실제로 수정을 못 함 (시트만 닫힘). 미완성 기능. (-8점)
- 인사이트 홈 미노출: 기록만 하고 인사이트 탭을 안 들어갈 수 있음. 피드백 루프 약함. (-5점)
- 메모 UI 제한: 한 줄 input으로는 상세 기록이 어려움. (-3점)
- 온보딩에서 수집한 피부 타입이 이후 어디에도 활용되지 않음. 형식적 정보 수집. (-4점)
