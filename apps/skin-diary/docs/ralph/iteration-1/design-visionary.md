# Design Visionary Critique — 피부 일지 (Skin Diary)

**Iteration:** 1
**Date:** 2026-03-21
**Reviewer:** Design Visionary Agent
**Target Aesthetic:** Personal diary/journal — warm, soft, intimate

---

## 디자인 점수: 8 / 10

이 앱은 "AI가 만든 전형적인 앱"에서 상당히 벗어나 있다. 디자인 시스템이 잘 세워져 있고, 일기장 콘셉트에 충실하며, 17개 안티패턴 규칙을 대부분 잘 지키고 있다. 세리프 헤딩(Gowun Batang), warm cream 배경(#fdf8f4), dusty rose primary(#c2847a)의 조합이 정체성을 만들어낸다. 텍스트 전용 하단 탭바는 의도적 차별화로 효과적이다.

부족한 2점은 아래 크리틱에서 다룬다.

## 화면별 크리틱

### 1. 온보딩 (OnboardingPage)

**잘한 점:**
- 3단계 프로그레스 바가 명확하고, R17(로딩 목적감)의 정신에 부합
- "피부는 어제의 결과예요" — 세리프 헤딩이 일기장 톤을 정확히 잡음
- 버튼 위계가 명확: Primary(다음) / Tertiary(건너뛰기, 데모 체험)
- Step 3의 카테고리 pill 칩 + 입력 필드 조합이 깔끔

**아쉬운 점:**
- Step 1 인트로 텍스트가 좌측 정렬이지만, 첫 인상으로는 약간 밋밋함. 일기장 콘셉트를 강화할 수 있는 시각적 장치(예: 수첩 질감 표현, 얇은 ruling line)가 없어 "빈 화면에 텍스트" 느낌
- "Step 1 / 3" 라벨이 영문인데, 나머지 UI는 모두 한국어. 통일성 문제
- Step 2 피부 타입 선택지가 2x2 grid인데, 4개만 있어 공간이 넓어 보임. 선택 시 피드백(체크마크 등)이 색상 변경뿐이라 다소 단조로움

### 2. 홈 (HomePage)

**잘한 점:**
- 상단 바: LeafIcon(제품관리) + SettingsIcon(설정)이 양 끝에 배치 — 미니멀하고 깔끔
- 인사감("안녕, 유진님\n오늘 피부는 어때?")이 세리프 폰트와 결합되어 일기장 느낌 강화
- 아침/밤 카드가 `bg-sd-primary-light`(#f5ebe8)로 따뜻한 톤 유지
- 아침이 기록되면 밤 카드 버튼이 Secondary(아웃라인)로 바뀌는 위계 전환 — R16 준수
- 주간 요약(WeeklySummary)의 SVG 차트가 인라인으로 잘 구현되어 있고, 색상 도트가 점수별로 다름

**아쉬운 점:**
- "N일 연속 기록 중" 텍스트가 `text-center`로 중앙 정렬됨 — R14 위반 소지. 다만 짧은 단일 통계치이므로 허용 범위 내
- 아침/밤 카드가 모두 동일한 배경색(#f5ebe8) + 동일한 radius(16px) + 동일한 구조여서, 시각적으로 "아침"과 "밤"의 구분이 약함. Sun/Moon 아이콘 차이만으로는 부족할 수 있음
- 상단 LeafIcon이 "제품 관리"를 의미한다는 것이 직관적이지 않음 (icon-only 버튼의 한계)

### 3. 캘린더 (CalendarPage)

**잘한 점:**
- 월 네비게이션(ChevronLeft/Right + 세리프 제목)이 일기장의 달력 넘기기 느낌
- CalendarGrid의 점수별 배경색(20% opacity) + 하단 도트가 정보를 과하지 않게 전달
- 오늘 날짜에 `border-2 border-sd-primary` — 적절한 강조
- 월 통계(평균, 기록률)가 간결하게 하단에 배치

**아쉬운 점:**
- 캘린더 셀의 `aspect-square`가 작은 화면에서 도트가 잘 안 보일 수 있음
- 기록이 없는 날을 눌러도 아무 반응이 없는데, 시각적 피드백 부재 (해당 날짜에 기록하러 가는 CTA가 있으면 좋겠음)
- DayDetail 바텀시트의 "수정하기" 버튼이 있지만, 실제로 편집으로 연결되지 않고 시트만 닫힘 (`onEdit={() => setSelectedDate(null)}`)

### 4. 인사이트 (InsightPage)

**잘한 점:**
- 데이터 부족 시 프로그레스 바 + "N일 더 기록하면" 메시지 — 빈 상태 처리가 훌륭
- 세그먼트 컨트롤(제품 분석 / 변수 분석)이 R5(네비게이션 차별화)에 부합
- InsightCard의 사용 시/미사용 비교 막대 그래프가 직관적
- 영향도 표시(▲/▼/—)가 색상+기호로 명확

**아쉬운 점:**
- 빈 상태("아직 데이터를 모으고 있어요") 화면이 `items-center text-center`로 중앙 정렬 — 예외적 허용이지만, 일기장 콘셉트와는 살짝 동떨어짐
- InsightCard가 여러 개 나올 때 모두 동일 구조(이름 + 막대 + 영향도)라서 monotonous할 수 있음. "가장 큰 발견" 같은 하이라이트 카드가 있으면 좋겠음

### 5. 아침 기록 (MorningLogPage)

**잘한 점:**
- "오늘 아침 피부는 어때?" — 친근한 일기장 톤의 세리프 제목
- ScoreSelector의 1-5 원형 버튼 + 점수별 색상이 직관적이고 예쁨
- KeywordChips의 선택/미선택 상태 전환이 즉시(0ms) — R4 준수
- 메모 필드가 "(선택)"으로 부담 없이 제공

**아쉬운 점:**
- 메모 입력이 `<input type="text">`(단일 라인)인데, 일기장이라면 `<textarea>`(다중 라인)가 더 자연스러움
- 저장 후 피드백(토스트 메시지 등)이 코드상 보이지 않음 — 저장 성공 여부를 알기 어려움

### 6. 밤 기록 (NightLogPage)

**잘한 점:**
- "어젯밤과 동일" 퀵카피 버튼 — 반복 기록의 마찰을 줄이는 UX
- 제품 선택 + 생활 변수를 점선 구분선(`border-dashed`)으로 나누는 것이 일기장 느낌
- ProductSelector에서 카테고리별 그루핑이 잘 되어 있음

**아쉬운 점:**
- "오늘 밤 뭘 발랐어?" / "오늘 하루 어땠어?" — 이 두 질문이 일기장의 "질문-응답" 포맷을 잘 살렸지만, 시각적으로는 두 섹션이 너무 비슷함(동일한 H2 + 칩 리스트)
- 제품 추가 인라인 폼이 열리면 `bg-sd-primary-light` 배경이 되는데, 이미 전체 페이지가 `bg-sd-bg`(비슷한 톤)이라 구분이 약할 수 있음

### 7. 제품 관리 (ProductListPage)

**잘한 점:**
- 카테고리별 그루핑 + 사용 횟수 표시가 실용적
- "보관" → "보관된 제품 (N)" 토글 → "복원/삭제" 흐름이 잘 설계됨
- 새 제품 추가 폼이 인라인으로 열리고, 취소/추가 버튼 위계(Secondary/Primary)가 명확

**아쉬운 점:**
- 제품 수가 많아지면 단순한 리스트가 길어질 수 있음. 검색이나 필터 기능 부재

### 8. 설정 (SettingsPage)

**잘한 점:**
- iOS 스타일 그룹화된 리스트 아이템 — 깔끔하고 예측 가능한 패턴
- 데모 모드 토글 스위치 커스텀 구현이 적절
- 기록 통계(총 기록 일수, 최장 스트릭, 사용 제품 수)가 Crimson Pro 숫자 폰트로 표시
- "데이터 초기화"가 `text-sd-danger`로 시각적 경고

**아쉬운 점:**
- 전반적으로 가장 "일기장" 느낌이 약한 화면. 표준 설정 UI라서 콘셉트와 동떨어짐 — 이건 어쩔 수 없는 부분이기도 함

### 9. DayDetail 바텀시트

**잘한 점:**
- 모달 오버레이(bg-black/30)와 함께 바텀시트가 올라오는 패턴이 모바일 네이티브 느낌
- 상단 그랩 바(w-10 h-1 bg-sd-border rounded-full)가 있어 닫기 가능 시사
- 점수 뱃지(원형 + score 색상 배경)가 컴팩트하면서 정보 전달 효과적

**아쉬운 점:**
- "수정하기" 버튼이 실제 편집으로 연결되지 않음 (기능 미완성)

## 개선 제안 (Top 5)

### 1. 아침/밤 카드의 시각적 차별화 (우선도: 높음)

현재 두 카드 모두 `bg-sd-primary-light`(#f5ebe8) + 동일 radius + 동일 구조. 밤 카드에 미묘한 차별화 — 예를 들어 아침 카드는 현재 톤 유지, 밤 카드는 살짝 더 진한 배경(#efe3de 정도) 또는 좌측에 문 아이콘 색상 border-left — 를 주면 스캔 시 구분이 쉬워진다. 일기장에서도 "아침 일기"와 "저녁 일기"는 다른 펜 색으로 쓰지 않는가.

### 2. 메모 입력을 textarea로 변경 (우선도: 중간)

MorningLogPage의 메모가 `<input type="text">`(단일 라인)인데, 일기장 앱이라면 여러 줄 쓸 수 있는 `<textarea>`가 콘셉트에 맞다. 2-3줄 높이로, 줄 수에 따라 자동 확장되는 `auto-resize`가 이상적. 일기장에서 한 줄만 쓰라는 법은 없다.

### 3. 저장 성공 피드백 (우선도: 중간)

아침/밤 기록 저장 후 어떤 시각적 피드백도 없이 화면이 닫힌다. 일기장을 덮을 때의 만족감이 빠져 있다. 간단한 토스트("기록했어요") 또는 홈으로 돌아왔을 때 카드가 채워지는 것만으로도 충분하지만, 저장 직후 0.5초간 체크마크나 짧은 메시지가 있으면 기록의 완결감이 생긴다.

### 4. 온보딩 "Step N / 3" 한국어화 (우선도: 낮음)

"Step 1 / 3"을 "1단계 / 3"이나 "1 / 3 단계"로 바꾸면 전체 UI의 한국어 일관성이 유지된다. 사소하지만, 영문이 섞이면 "외국 앱 번역판" 느낌이 나서 일기장의 내밀한 분위기가 깨진다.

### 5. 일기장 질감 강화 — ruling lines 또는 texture hint (우선도: 낮음)

현재 "일기장" 콘셉트는 주로 세리프 폰트 + warm cream 배경으로 표현되는데, 조금 더 밀어붙여도 좋다. 예를 들어:
- WeeklySummary 차트 아래에 얇은 가로 ruling line(일기장 줄)을 장식적으로 추가
- 온보딩 첫 화면에 수첩 모서리 느낌의 미세한 그림자
- 다만 이것은 "장식"이 되지 않도록 주의해야 하며, R7(장식 파티클 금지)을 위반하지 않는 선에서 구현해야 함

## 총평

피부 일지는 디자인 시스템의 의도가 코드까지 일관되게 관통하는, 잘 만들어진 앱이다. "일기장" 콘셉트를 세리프 타이포그래피, warm cream 팔레트, 텍스트 전용 탭바, 최소한의 애니메이션으로 구현한 것이 효과적이다. 17개 안티패턴 규칙을 전수 준수한 것은 특히 인상적이다.

핵심 약점은 반복되는 시각 구조(아침/밤 카드 동일, InsightCard 동일)에서 오는 단조로움이다. "일기장"이라는 콘셉트를 텍스트와 색상 너머의 촉각적(tactile) 레벨까지 끌어올리면 8점에서 9점으로 갈 수 있다.
