## Iteration 3 피드백 반영 확인

### 1. 캘린더 기록률 퍼센트 표시

**반영됨 (4회차에 드디어).** CalendarPage line 41-42에서 `daysInMonth`를 `new Date(year, month + 1, 0).getDate()`로 계산하고, `recordPercent`를 `Math.round((recordedDays / daysInMonth) * 100)`로 산출함. Line 76에서 `기록률: {recordedDays}/{daysInMonth}일 ({recordPercent}%)` 형태로 렌더링됨. 3회 연속 미반영 이후 4회차에 반영. "15/31일 (48%)" 같은 형식으로 해당 월의 총 일수 대비 기록 비율이 한눈에 보임.

### 2. 빈 날 밤/아침 선택

**반영됨.** DayDetail 컴포넌트에서 `!record`인 경우(line 28-61), "아침 피부 기록하기" 버튼과 "밤 루틴 기록하기" 버튼 두 개가 모두 표시됨. `onEdit`은 아침 기록, `onEditNight`는 밤 기록으로 분리되어 있고, App.tsx line 165-168에서 `onEditNightDate` 핸들러가 `setOverlay('nightLog')`를 호출함. 이전에 아침 기록만 가능했던 반쪽짜리 소급 기록이 완전한 형태가 됨.

### 3. DayDetail 인과관계 순서 (전날 밤 루틴 -> 오늘 아침 피부)

**반영됨 (4회차에 드디어).** DayDetail line 75-87에서 `prevRecord?.nightLog`가 먼저 렌더링되고, line 89-91에서 "그 결과"라는 연결 텍스트가 표시된 후, line 93-117에서 `record.morningLog`가 나옴. "전날 밤 루틴 -> 그 결과 -> 아침 피부" 순서로, 앱의 슬로건 "피부는 어제의 결과"와 UI가 일치함. 3회 연속 미반영 이후 4회차에 반영. 점선 구분선과 "그 결과" 텍스트가 인과 흐름을 시각적으로 잘 연결해줌.

### 4. 홈 수정 버튼

**반영됨.** HomePage line 103-106에서 아침 피부 기록이 있는 경우(`hasMorning`) "수정" 버튼이 표시되고, `onEditMorning`을 호출함. 밤 루틴도 마찬가지로 line 144-148에서 "수정" 버튼이 표시되어 `onEditNight`를 호출함. App.tsx line 147-148에서 `onEditMorning`과 `onEditNight`가 오늘 날짜 기준으로 각각 morningLog, nightLog 오버레이를 여는 핸들러로 연결됨. 기록 완료 후에도 홈에서 바로 수정할 수 있는 진입점이 확보됨.

### 5. 밤 기록 저장 후 아침 유도 메시지

**여전히 미반영.** App.tsx line 186에서 밤 기록 저장 시 `showToast('기록이 저장되었어요')`만 표시됨. "저장 완료! 내일 아침에 피부 점수도 기록해봐요" 같은 핵심 루프 리마인드 메시지가 3회 연속 미반영. 토스트 문구 수정만으로 해결 가능한 항목인데 계속 누락됨.

### 6. 생활 변수 기본값(프리셋) 설정

**여전히 미반영.** NightLogPage에서 밤 기록 시 매번 수동으로 생활 변수를 선택해야 함. 설정에서 "기본 생활 변수"를 미리 체크해두는 기능이 4회 연속 미반영. "어젯밤과 동일" 복사 버튼은 있지만 제품까지 다 복사하는 문제도 그대로.

## 최종 피드백 (Iteration 4)

### 긍정적 변화

핵심 미반영 항목 3개가 이번에 한꺼번에 해결됨. 기록률 퍼센트, 빈 날 밤/아침 선택, DayDetail 인과관계 순서가 모두 반영되면서, 앱의 핵심 가치인 "어제의 루틴이 오늘 피부를 만든다"가 UI 전체에서 일관되게 전달됨. 특히 DayDetail에서 "전날 밤 루틴 -> 그 결과 -> 아침 피부" 흐름이 점선 구분선과 연결 텍스트로 시각화된 점이 좋음. 홈 화면의 수정 버튼도 기록 완료 후 "아 아까 잘못 넣었는데" 상황에서 바로 진입할 수 있어서 실용적임.

### 아쉬운 부분

1. **밤 기록 저장 토스트에 아침 유도 메시지 없음 (3회 연속 미반영):** 토스트 문구 한 줄만 바꾸면 되는 건데 여전히 "기록이 저장되었어요"로 끝남. 핵심 루프(밤 루틴 기록 -> 다음날 아침 피부 체크)를 자연스럽게 유도하는 장치가 없어서, 사용자가 밤 기록만 하고 아침 기록을 까먹을 확률이 높음.

2. **생활 변수 기본값 프리셋 4회 연속 미반영:** 나처럼 앞머리를 늘 내리고 다니는 사람은 매일 "앞머리" 변수를 수동 선택해야 함. 사소하지만 매일 반복하면 귀찮음이 쌓이는 부분.

3. **부위별 트러블 기록 불가:** 턱 주변 트러블이 내 핵심 고민인데, 피부 점수가 전체 얼굴 단위라서 "턱은 안 좋은데 이마는 괜찮은 날"을 구분할 수 없음. 인사이트의 근본적 한계. 대대적인 기능 추가가 필요한 건 알지만, 없으면 결국 "내 문제"를 추적할 수 없음.

4. **밤-아침 기록 시간순 안내 없음:** 아침 기록 없이 밤 기록만 해도 아무 안내가 없음. "오늘 아침 피부는 아직 기록 안 했어요. 먼저 기록할까요?" 같은 부드러운 넛지가 있으면 핵심 루프가 강화될 텐데.

## 만족도 점수

**82 / 100**

### 근거

**개선된 부분 (+6점, 76 -> 82):**
- 기록률 퍼센트 반영: 4회째에 드디어. "15/31일 (48%)" 형태로 동기부여 효과 있음. (+1점)
- 빈 날 밤/아침 선택 가능: 소급 기록이 완전한 형태가 됨. 아침만 가능하던 한계 해소. (+2점)
- DayDetail 인과관계 순서 반영: 전날 밤 루틴 -> "그 결과" -> 아침 피부. 앱 컨셉과 UI 일치. (+2점)
- 홈 수정 버튼: 기록 후 바로 수정 가능한 진입점 확보. (+1점)

**여전히 아쉬운 부분 (-18점):**
- 부위별 트러블 기록 불가: 내 핵심 고민을 추적할 수 없는 근본적 한계. (-7점)
- 밤 기록 저장 토스트 아침 유도 메시지 3회 연속 미반영: 간단한 수정인데 계속 누락. (-3점)
- 생활 변수 기본값 프리셋 4회 연속 미반영: 매일 같은 변수 반복 선택. (-3점)
- 밤-아침 기록 시간순 안내 없음: 핵심 루프 이탈 가능성. (-2점)
- 인사이트 부위별 분석 없음: 전체 피부 점수 기반이라 "턱 트러블과 밀가루의 관계" 같은 세밀한 분석 불가. (-3점)

### 최종 코멘트

4회 반복 끝에 82점까지 올라왔고, 앱의 핵심 루프(밤 루틴 기록 -> 다음날 아침 피부 확인 -> 인사이트 확인)가 UI 전반에서 일관되게 작동하게 됨. 기록하는 행위 자체는 이제 불편하지 않음. 다만 "내 턱 트러블에 진짜 영향을 주는 게 뭔지"를 알아내려면 부위별 기록이 필요한데, 이건 앱의 다음 버전에서 풀어야 할 과제. 현재 상태에서도 매일 쓸 만한 앱이라고 느껴짐. 80점 이상이면 커밋 가능.
