## Iteration 2 피드백 반영 확인

### 1. 캘린더 빈 날 소급 기록 가능한가?

**반영됨.** CalendarGrid에서 `onDateSelect`가 모든 날짜에 대해 호출됨 (이전에는 `hasRecord`가 false인 날은 무시). DayDetail 컴포넌트에서 `!record`인 경우 "이 날은 기록이 없어요" 메시지와 함께 "이 날 기록하기" 버튼이 표시됨. 이 버튼을 누르면 `onEdit(date)`가 호출되어 App.tsx의 `onEditDate` 핸들러를 통해 해당 날짜에 대한 아침 기록 오버레이가 열림. 2회 연속 미반영이었던 항목이 드디어 해결됨. 다만 빈 날 바텀시트에서 "아침 기록"만 열리고, "밤 기록"을 선택할 수 있는 옵션은 없음 (App.tsx line 158: record가 없으면 무조건 `morningLog` 오버레이).

### 2. 인사이트 카드 탭 연동이 되는가?

**반영됨.** HomePage의 "이번 주 발견" 카드가 `<button onClick={onNavigateToInsight}>`로 변경됨. "자세히 보기" 텍스트도 추가되어 탭 가능하다는 시각적 힌트가 있음. App.tsx에서 `onNavigateToInsight={() => setActiveTab('insight')}`로 인사이트 탭으로 자동 전환됨. 이전에 단순 텍스트 노출에 그쳤던 것이 탐색 플로우까지 연결됨.

### 3. 데모 데이터 로드가 정상 작동하는가?

**정상 작동.** App.tsx의 `useEffect`에서 데모 모드가 ON이고 records가 비어있으면 자동으로 `loadDemoData()`를 호출함. `handleLoadDemo`도 온보딩에서 사용 가능. 데모 데이터는 30일치 기록(아침+밤)이 생성되며, 6개 제품과 다양한 생활 변수 패턴이 포함됨. 프로필도 '유진 / 복합성'으로 자동 설정됨.

### 미반영 확인: 기록률 퍼센트

**여전히 미반영.** CalendarPage line 72-74에서 `기록률: {recordedDays}일` 형태가 그대로임. 3회 연속 미반영. "15일 / 31일 (48%)" 형태를 요청했으나 반영되지 않음.

### 미반영 확인: DayDetail 인과관계 순서

**여전히 미반영.** DayDetail 컴포넌트에서 `morningLog`가 먼저 렌더링되고(line 62-85), 그 아래에 `prevRecord?.nightLog`가 나옴(line 87-102). 3회 연속 미반영. "전날 밤 루틴 -> 오늘 아침 피부" 순서로 뒤집어야 컨셉("피부는 어제의 결과")과 일치함.

## 새로운 피드백 (Iteration 3)

### 1. 빈 날 소급 기록에서 밤 기록을 선택할 수 없음

캘린더에서 빈 날을 클릭하면 바텀시트에 "이 날 기록하기" 버튼이 하나만 있고, 이걸 누르면 무조건 아침 기록 오버레이만 열림. App.tsx의 `onEditDate` 핸들러에서 record가 없는 경우 `setOverlay('morningLog')`로 고정되어 있기 때문. 소급 기록이라는 건 그날 밤 루틴도 채워넣고 싶은 건데, 아침 기록만 가능하면 반쪽짜리임. "이 날 기록하기" 버튼 대신 "아침 피부 기록" / "밤 루틴 기록" 두 버튼을 제공하거나, 아침 기록 저장 후 바로 밤 기록으로 이어지는 플로우가 필요함.

### 2. 기록률 퍼센트 3회 연속 미반영

CalendarPage 월간 통계에서 여전히 `기록률: 15일`만 표시됨. 3번째 이터레이션인데도 반영이 안 됨. 이번 달이 31일인지 28일인지에 따라 같은 15일이라도 의미가 다름. `기록률: 15 / 31일 (48%)` 형태로 해당 월의 총 일수 대비 기록 일수 비율을 보여주면 됨. JavaScript의 `getDaysInMonth`가 이미 CalendarGrid에서 사용되고 있으니 값 자체는 바로 쓸 수 있음.

### 3. DayDetail 인과관계 순서 3회 연속 미반영

DayDetail에서 아침 피부(결과)가 위에, 전날 밤 루틴(원인)이 아래에 위치하는 구조가 3회 연속 그대로임. 앱의 슬로건이 "피부는 어제의 결과"인데, 상세 뷰에서 결과를 먼저 보여주고 원인을 나중에 보여주면 인과 흐름이 역순임. `prevRecord?.nightLog` 블록을 `record.morningLog` 블록 위로 옮기고, "전날 밤 루틴" 아래에 화살표나 "그 결과" 같은 연결 텍스트를 넣으면 컨셉과 UI가 일치함.

### 4. 밤 기록 저장 후 아침 기록 유도 여전히 없음

Iteration 2에서 제안한 "내일 아침에 피부 점수도 기록해봐" 안내가 여전히 없음. 밤 기록 저장 시 App.tsx line 179에서 `showToast('기록이 저장되었어요')`만 2초간 표시되고 끝남. 토스트 메시지를 "저장 완료! 내일 아침에 피부 점수도 기록해봐요" 정도로 바꾸기만 해도 핵심 루프(밤 루틴 -> 아침 피부 체크) 리마인드 효과가 있을 것. 큰 공수 없이 토스트 문구만 수정하면 되는데 2회 연속 미반영.

### 5. 생활 변수 기본값(프리셋) 설정 여전히 없음

매번 밤 기록할 때마다 NightLogPage의 VariableChips에서 자주 해당하는 변수(예: 앞머리)를 손으로 눌러야 함. "어젯밤과 동일" 복사 버튼은 있지만, 이건 제품까지 다 복사해버려서 변수만 프리셋으로 두고 싶은 경우에 맞지 않음. 설정에서 "기본 생활 변수"를 미리 체크해두면 밤 기록 페이지 진입 시 자동 선택되어 있는 기능이 필요함. 3회 연속 미반영.

### 근거

**개선된 부분 (+4점, 72 -> 76):**
- 캘린더 빈 날 소급 기록 가능: 3회째에 드디어 반영. 어제 깜빡한 기록을 다음날 채워넣을 수 있게 됨. 다만 아침 기록만 가능한 한계 있음. (+2점)
- 인사이트 카드 탭 -> 인사이트 탭 전환: 홈에서 "이번 주 발견"을 탭하면 인사이트 탭으로 이동. "자세히 보기" 힌트 텍스트도 있어서 탐색 플로우가 자연스러워짐. (+2점)

**여전히 아쉬운 부분 (-24점):**
- 부위별 트러블 기록 불가: 내 핵심 고민인 "턱 주변 트러블"을 위치 특정할 수 없음. 인사이트의 근본적 한계. (-8점)
- 기록률 퍼센트 3회 연속 미반영: 간단한 수정인데 계속 누락됨. (-3점)
- DayDetail 인과관계 순서 3회 연속 미반영: 컨셉과 UI의 불일치가 지속됨. (-3점)
- 빈 날 소급 시 밤 기록 선택 불가: 소급 기록이 아침만 가능하면 반쪽. (-3점)
- 밤 기록 저장 후 아침 유도 메시지 2회 연속 미반영: 토스트 문구 수정만으로 해결 가능한데 미반영. (-2점)
- 생활 변수 기본값 프리셋 3회 연속 미반영: 매일 같은 변수를 반복 선택하는 번거로움. (-3점)
- 밤-아침 기록 시간순 강제 없음: 아침 기록 없이 밤 기록만 해도 아무 안내가 없어서 핵심 루프가 끊길 수 있음. (-2점)
