## Iteration 2 제안 반영 체크

### 1. 토스트 애니메이션 polish --- NOT ADDRESSED

**이전:** `opacity: 1` 고정, transition 선언은 있으나 실제 페이드 없음.
**현재:** App.tsx line 226에 여전히 `style={{ opacity: 1, transition: 'opacity 200ms' }}`로 동일. 토스트가 즉시 나타나고 즉시 사라진다.

토스트가 기능적으로 동작하므로 감점 요소는 아니지만, 페이드인/아웃이 추가되면 저장 완결감이 더 부드러워진다. 우선도 낮음으로 재분류.

### 2. "이번 주 발견" 카드 톤 통합 --- RESOLVED

**이전:** `bg-white border border-sd-border` --- 홈의 warm tone과 이질적인 차가운 순백.
**현재:** HomePage.tsx line 170에 `style={{ backgroundColor: '#fdf8f4' }}`로 변경됨. 아침 카드와 동일한 warm cream 톤으로 통합.

이것은 정확히 제안한 방향이다. 홈 화면 전체가 warm tone(`#fdf8f4`, `#f0e8e4`, `#f5ebe8`)으로 일관되게 유지되어, 스크롤 시 차가운 순백 카드가 눈에 걸리는 문제가 해결되었다. `border border-sd-border`는 유지하여 카드 경계가 명확하되, 배경 톤이 주변과 어울린다.

### 3. 기록 없는 날짜 터치 시 CTA --- RESOLVED

**이전:** DayDetail에서 `!record`일 때 `return null` --- 아무 반응 없음.
**현재:** DayDetail.tsx lines 27-48에서 빈 날짜 바텀시트가 완전히 구현됨:
- "이 날은 기록이 없어요" 안내 텍스트
- "이 날 기록하기" Primary 버튼 (bg-sd-primary, 흰 텍스트)
- 바텀시트 핸들 바(10x1 rounded pill) + 반투명 오버레이

이것은 UX 완결성 측면에서 중요한 개선이다. 캘린더에서 빈 날짜를 터치했을 때 "아무 반응 없음"이 "안내 + 행동 유도"로 바뀌어, 과거 기록 보충의 진입점이 생겼다. 빈 상태 처리의 원칙(항상 다음 행동을 제안)에 부합한다.

### 4. confirm() 대신 커스텀 모달 --- NOT ADDRESSED

MorningLogPage, NightLogPage, SettingsPage, ProductListPage 총 5곳에서 여전히 브라우저 기본 `confirm()` 사용 중. 앱의 warm serif 톤과 이질적이지만, 우선도가 낮았으므로 합당한 판단.

### 5. 인사이트 "하이라이트 발견" 카드 --- NOT ADDRESSED

InsightPage의 제품/변수 카드가 동일 InsightCard 컴포넌트로 반복되며, 최상위 발견에 대한 시각적 강조는 없음. 기능적으로 3그룹 분류(좋은/주의/판단 이른)가 효과적이므로, 미반영이 치명적이지 않다.

## 화면별 재평가

### 1. 홈 (HomePage)

**현재 상태: 우수**

홈은 이 앱의 핵심 화면으로, 현재 완성도가 높다:

- **인사 메시지:** 시간대별 인사(`좋은 아침이에요` / `좋은 오후예요` / `오늘 하루 수고했어요`) + 사용자 이름 조합. 아침 기록 완료 시 `"유진님, 오늘도 기록했어요"`로 전환. 따뜻하고 개인화된 톤.
- **아침/밤 카드:** `#fdf8f4` vs `#f0e8e4` 톤 차별화 유지. Sun/Moon 아이콘 색상도 아침(`#d4956a` warm orange) vs 밤(`#8a7a9e` muted purple)로 구분. 카드 간격 `mb-3`(12px)으로 적절 --- 너무 좁지도, 넓지도 않다.
- **밤 카드 버튼 위계:** 아침 미기록 시 `border border-sd-primary text-sd-primary`(Secondary), 아침 기록 완료 시 `bg-sd-primary text-white`(Primary). 이 조건부 전환이 "아침 먼저 기록하세요"라는 암묵적 가이드를 제공.
- **이번 주 피부 차트:** SVG 라인 차트가 깔끔. 점선 그리드 + 색상별 점수 도트(`#e8a0a0`~`#7ac27a`) + 요일 라벨. 차트 하단에 평균 + 주요 키워드 표시.
- **이번 주 발견 카드:** `bg-[#fdf8f4]`로 warm tone 통합 완료. `border border-sd-border`로 구분 유지. "자세히 보기" 텍스트가 인사이트 탭으로의 네비게이션을 자연스럽게 유도.

**아쉬운 점:**
- 아이콘(LeafIcon, SettingsIcon)의 색상이 `text-sd-text`(#3d2c2c)로 통일되어 있는데, 이것은 기능적으로 충분하지만 약간 무거운 느낌. `text-sd-text-secondary`(#8b7e7e)가 더 가벼울 수 있다. 단, 탭 타겟 크기(`min-w-[44px] min-h-[44px]`)는 접근성 기준 충족.
- 스트릭 표시(`N일 연속 기록 중`)가 `text-center`로 중앙 정렬. R14 경계선이지만, 단일 통계 라인이므로 예외 허용 범위 내.

### 2. 캘린더 (CalendarPage)

**현재 상태: 양호~우수**

- 색상 코딩이 직관적: 점수별 배경색(`#e8a0a0`~`#7ac27a`)으로 한 달의 피부 상태를 한눈에 파악 가능.
- 점수 도트(작은 원)가 날짜 아래에 표시되어, 기록 유무 + 점수를 이중으로 전달.
- 오늘 날짜(21)가 `border border-sd-primary` 테두리로 명확히 강조.
- 월간 통계(이번 달 평균: 3.2, 기록률: 20일)가 좌측 정렬로 깔끔.

**아쉬운 점:**
- 캘린더 하단에 넓은 빈 공간이 있다. 월간 통계 아래에 추가 인사이트(예: "이번 달 가장 좋았던 날", "기록을 빼먹은 날")를 넣으면 정보 밀도가 올라간다.
- 빈 날짜(22일 이후)와 기록 있는 날짜의 시각적 대비가 뚜렷하여, 미기록 날짜가 "구멍"처럼 보인다. 이것은 동기 부여 측면에서 효과적이지만, 미래 날짜(22~31일)에도 동일한 빈 스타일이 적용되어 "아직 안 온 날"과 "놓친 날"의 구분이 없다.

### 3. DayDetail 바텀시트

**현재 상태: 우수 (빈 날짜 CTA 추가로 크게 개선)**

- 기록 있는 날: 아침 점수(색상 원 + 점수 텍스트) + 키워드 + 메모 + 전날 밤 기록 + 생활 변수(pill chip) + 수정하기 버튼. 정보 구조가 체계적이고, 점선 구분선(`border-dashed`)이 섹션을 자연스럽게 구분.
- 빈 날짜: "이 날은 기록이 없어요" + "이 날 기록하기" Primary CTA 버튼. 단순하고 명확.
- 바텀시트 핸들 바(`w-10 h-1 bg-sd-border rounded-full mx-auto`)가 모바일 관례를 따름.

### 4. 인사이트 (InsightPage)

**현재 상태: 양호**

- 세그먼트 컨트롤(제품 분석 / 변수 분석)이 깔끔. active 상태(`bg-sd-primary text-white`)와 inactive(`bg-white text-sd-text-secondary`) 대비가 명확.
- 제품 3그룹 분류(좋은/주의/판단 이른) 유지.
- 변수 분석 탭에서 `sentenceTemplate` 문장("매운 음식 다음날, 피부 점수가 평균 0.8점 낮아져요")이 데이터를 인간적 문장으로 전달.

**아쉬운 점:**
- InsightCard가 반복되면서 시각적 단조로움 존재. 최상위 발견 카드에 약간의 차별화(배경 tint, 더 큰 impact 숫자)가 있으면 스캔 효율이 올라간다.
- 빈 상태의 중앙 정렬은 예외적 허용 범위 내.

### 5. 아침 기록 (MorningLogPage)

**현재 상태: 양호**

- 점수 선택기(1~5)가 원형 버튼으로 가로 배열. 각 점수 아래 라벨(최악/별로/보통/좋아/꿀피부). 선택 시 `bg-sd-primary text-white` 활성화.
- 키워드 칩("촉촉", "건조", "트러블" 등)이 `rounded-full` pill 형태로 다중 선택 가능.
- textarea(3줄) + "특이사항이 있다면 적어봐" placeholder. 일기장 콘셉트에 부합.
- 하단에 인사이트 요약이 한 줄씩 표시되어 기록 동기를 부여.

**아쉬운 점:**
- "닫기"/"저장하기" 버튼의 위치. 상단 X 버튼이 닫기, 하단 full-width가 저장. 구조는 합리적이나, confirm() 다이얼로그가 warm 톤과 이질적(기존 지적 사항).

### 6. 밤 기록 (NightLogPage)

**현재 상태: 양호**

- "어젯밤과 동일" 퀵카피 버튼이 상단에 위치. `border border-sd-primary text-sd-primary`(Secondary 스타일).
- 제품 리스트가 카테고리별(토너/세럼/에센스/크림/선크림/마스크팩)로 그루핑.
- 생활 변수("밀가루", "매운 음식", "음주" 등)가 칩 형태로 토글 가능.
- 하단에 인사이트 요약 표시.

## 개선 제안 (Iteration 3 -> 4)

### 1. 홈 상단 아이콘 색상 경량화 (우선도: 중간)

현재 LeafIcon(제품 관리)과 SettingsIcon(설정)이 `text-sd-text`(#3d2c2c)로 본문 텍스트와 동일한 무게감. 이 아이콘들은 보조 기능 진입점이므로 `text-sd-text-secondary`(#8b7e7e) 또는 `text-sd-text-tertiary`(#b5aaaa)로 가볍게 처리하면, 화면의 시선이 중앙 인사 메시지와 카드로 더 자연스럽게 향한다.

### 2. 캘린더 미래 날짜 vs 과거 빈 날짜 구분 (우선도: 중간)

현재 캘린더에서 22일(미래)과 가령 기록을 빼먹은 날이 동일하게 보인다. 미래 날짜에 `text-sd-text-tertiary` 또는 `opacity-40`을 적용하면, "아직 안 온 날"과 "기록을 놓친 날"이 구분되어 캘린더의 정보 정확도가 올라간다.

### 3. 토스트 페이드인/아웃 (우선도: 낮음)

2회 연속 미반영이므로 우선도를 "낮음"으로 재분류한다. 현재 토스트는 기능적으로 충분하다. 다만 `setTimeout`으로 `opacity: 0 -> 1 -> 0` 트랜지션을 추가하면 200ms 작업으로 polish가 올라간다.

### 4. 인사이트 최상위 카드 시각 강조 (우선도: 낮음)

InsightPage에서 첫 번째(가장 임팩트가 큰) 카드에 약간의 차별화를 주면 좋다. 예: `bg-sd-primary-light` 배경 tint, 또는 impact 숫자를 `text-lg` 대신 `text-xl`로 키우기. 단, 현재 3그룹 분류가 충분히 효과적이므로, 이것은 "있으면 좋은" 수준.

## 총평

Iteration 3은 **핵심 UX 결함 제로** 상태에 도달했다. Iteration 2에서 지적한 두 가지 중요 이슈(인사이트 카드 톤 불일치, 빈 날짜 무반응)가 정확히 해결되어, 앱 전체에 "빈틈 없는" 느낌이 형성되었다.

특히 빈 날짜 CTA 구현은 단순한 디자인 수정이 아니라 UX 로직 개선이다. 캘린더가 "조회 전용"에서 "행동 유도"로 전환된 것은 앱의 사용 가치를 높인다.

8.5점에서 9점으로의 상승은 "대부분 올바른 결정을 내리고 있으며, 남은 개선 사항이 polish 수준"이라는 뜻이다. 9.5점 이상으로 가기 위해서는 토스트 애니메이션, 아이콘 경량화, 미래/과거 날짜 구분 같은 세부 마감이 필요하지만, 이것들은 모두 "기능적으로 충분한 것을 더 세련되게 만드는" 작업이다.

이 앱은 warm diary 콘셉트를 일관되게 유지하면서, 데이터 기반 인사이트를 자연스럽게 녹여낸 좋은 사례다.
