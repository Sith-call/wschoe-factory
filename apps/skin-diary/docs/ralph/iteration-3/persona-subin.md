## Iteration 2 피드백 반영 확인

### 1. verdict 근거 문장이 추가되었는가?

**반영됨 -- 만족.**

`getProductVerdict()` 함수가 완전히 개선됨. 이전에는 "계속 사용하세요" / "주의가 필요해요" / "더 지켜봐야 해요" 세 마디 뿐이었는데, 이제 각 verdict에 구체적 수치 근거가 붙음:

- `"계속 사용하세요 — 사용한 7일간 평균 3.8점 (미사용 시 3.1점)"`
- `"주의가 필요해요 — 사용한 5일간 평균 2.9점 (미사용 시 3.5점)"`
- `"아직 뚜렷한 차이가 없어요 — 4일간 평균 3.3점 (미사용 시 3.2점)"`
- 사용 횟수가 3회 미만일 때: `"아직 2회만 사용했어요. 1회 더 사용하면 판단할 수 있어요"`

사용 일수, 사용 시 평균 점수, 미사용 시 평균 점수가 한 문장에 다 들어가서 "왜 이 제품이 좋다고 하는지"를 바로 이해할 수 있음. I2에서 요청한 "근거 포함 문장"이 정확히 반영됨. 신뢰감이 확 올라감.

### 2. 홈 인사이트 탭 연동이 되는가?

**반영됨 -- 만족.**

홈의 "이번 주 발견" 카드가 `<button>` 요소로 구현되어 있고, 클릭 시 `onNavigateToInsight` 콜백이 호출되어 `setActiveTab('insight')`로 인사이트 탭으로 이동함. "자세히 보기 >" 텍스트도 있어서 탭 전환이 가능하다는 어포던스가 있음. 홈에서 "가장 좋았던 제품: XX (+0.5점)" 요약을 보고 탭해서 인사이트 페이지로 바로 넘어가는 흐름이 자연스러움.

### 3. 스트릭 통일이 되었는가?

**반영됨 -- 부분적으로 만족.**

홈의 스트릭 카운터가 `"{streak}일 연속 기록 중"`으로 표시됨. 그런데 스트릭 계산 로직이 `morningLog && nightLog` 둘 다 있어야만 카운트되는 방식. 즉 아침만 기록하고 밤을 빼먹으면 스트릭이 끊김. 이 기준 자체는 합리적이지만, 사용자에게 "아침+밤 모두 기록해야 연속으로 인정돼요"라는 설명이 없어서 "어제 아침 기록했는데 왜 스트릭이 0이지?" 라고 혼란스러울 수 있음. 스트릭 옆에 기준 설명 툴팁이나 작은 텍스트가 있으면 좋겠음. 그래도 이전 대비 표시 형식이 통일된 건 맞음.

## 새 피드백 (4개)

### 1. 밤 기록에 여전히 메모가 없음 -- 3회 연속 요청

NightLog 타입에 `memo` 필드가 없고, NightLogPage에도 메모 입력 UI가 없음. MorningLog에는 `memo?: string`이 있고 textarea도 구현되어 있는데, NightLog는 `products`와 `variables`만 저장함. "오늘 새 세럼 처음 발랐는데 끈적임", "클렌징 두 번 했음", "마스크팩 30분 올림" 같은 맥락 정보를 남기지 못하면, 나중에 인사이트에서 "이 제품이 왜 점수가 낮지?" 할 때 기억을 되짚을 수가 없음. 밤 루틴의 맥락 메모는 제품 인사이트의 정확도를 높이는 핵심 정보인데, 이 부분이 3번째 반복에서도 빠져 있어 아쉬움.

### 2. 탭바에 아이콘이 여전히 없음

TabBar.tsx를 보면 여전히 `tab.label` 텍스트만 렌더링됨. Icons.tsx에 SunIcon, MoonIcon, LeafIcon, SettingsIcon, CloseIcon 등이 이미 있으니, 홈 아이콘(집 모양), 캘린더 아이콘, 차트 아이콘을 추가해서 탭바에 넣으면 됨. 텍스트 아래에 아이콘 하나 있는 것만으로도 앱의 완성도가 한 단계 올라감. 현재 탭바는 `font-body text-[0.9375rem]` 텍스트만 있어서 웹 느낌이 강하고, 네이티브 앱처럼 느껴지지 않음.

### 3. 홈 "이번 주 발견" 빈 상태 처리 여전히 부족

`bestProduct`와 `worstVariable` 둘 다 null이면 "이번 주 발견" 영역 자체가 렌더링되지 않음 (조건: `{(bestProduct || worstVariable) && ...}`). 첫 주 사용자는 이 기능이 존재하는지도 모름. WeeklySummary에는 빈 상태 시 "기록을 시작하면 여기에 추이가 나와요"라는 메시지가 잘 있고, InsightPage에도 프로그레스 바가 있는데, 홈의 인사이트 요약만 유일하게 빈 상태 처리가 없음. "3일 더 기록하면 제품별 인사이트가 나타나요" 같은 안내 카드를 빈 상태에도 보여줘야 일관성이 맞음.

### 4. 키워드 칩 고정 순서 -- 자주 쓰는 키워드 우선 노출 여전히 미반영

KeywordChips.tsx에서 `ALL_KEYWORDS.map()`으로 항상 동일 순서로 13개 키워드가 나열됨. 건성 피부인 나는 매일 "건조", "당김"을 선택하는데, "촉촉"이 항상 맨 앞에 있고 "당김"은 10번째에 있어서 찾는 데 시간이 걸림. 최근 7일간 가장 자주 선택한 키워드 3개를 "자주 사용" 그룹으로 상단에 분리하거나, records 데이터를 기반으로 빈도순 정렬을 하면 기록 시간이 체감상 절반으로 줄어들 것. 특히 매일 반복하는 기록 앱이라 이런 작은 편의성이 리텐션에 직결됨.
