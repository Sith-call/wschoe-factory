# 피부 일지 (Skin Diary) — User Stories

> 시간축별 정리: 첫 방문 → 첫 기록 → 1주 → 인사이트 발견 → 습관화

---

## Phase 1: 첫 방문 (온보딩)

### US-01. 앱 컨셉 이해

**As a** 첫 방문 유저
**I want to** 이 앱이 뭘 해주는 건지 3초 안에 이해하고 싶다
**So that** 설치/사용을 계속할지 판단할 수 있다

**수용 기준**:
- 인트로 화면에서 "피부는 어제의 결과" 컨셉을 한 문장으로 전달
- 밤 기록 → 아침 기록 → 인사이트 흐름을 시각적으로 보여줌
- 3스텝 이하로 온보딩 완료 가능
- "시작하기" 버튼이 명확하게 보임

**안티패턴**:
- 기능 나열형 온보딩 (5개 이상 슬라이드)
- 회원가입부터 요구
- 퍼미션(알림) 먼저 요청

**페르소나 반응**:
- 서유진: "오 이런 앱 있었으면 했는데. 바로 써볼래"
- 김수빈: "인사이트 화면이 궁금하다. 빨리 기록 시작하고 싶어"
- 이채은: "제품별로 비교할 수 있다고? 딱 내가 필요한 거"

---

### US-02. 피부 타입 설정

**As a** 첫 방문 유저
**I want to** 내 피부 타입을 선택하고 싶다
**So that** 앱이 나에게 맞춤화된 느낌을 준다

**수용 기준**:
- 피부 타입 선택: 건성 / 복합성 / 지성 / 민감성 (복수 선택 가능)
- 선택은 optional (스킵 가능)
- 나중에 설정에서 변경 가능
- 선택 시 탭 한 번으로 완료

**안티패턴**:
- 20개 질문의 피부 진단 퀴즈
- 선택을 강제하는 것
- 결과에 큰 영향이 없는데 과도하게 상세한 입력 요구

**페르소나 반응**:
- 서유진: "복합성 탭. 끝. 간단해서 좋다"
- 김수빈: "건성 + 민감성 둘 다 해당돼서 복수 선택 가능한 게 좋네"
- 이채은: "지성. 모공 옵션도 있으면 좋겠는데... 뭐 일단 넘어가자"

---

### US-03. 초기 제품 등록

**As a** 첫 방문 유저
**I want to** 내가 쓰고 있는 제품을 빠르게 등록하고 싶다
**So that** 기록할 때 바로 선택할 수 있다

**수용 기준**:
- 카테고리 선택 → 제품명 입력 (자유 텍스트)
- 카테고리: 클렌징 / 토너 / 세럼 / 에센스 / 크림 / 아이크림 / 선크림 / 팩 / 기타
- 최소 1개 등록 필수, 최대 제한 없음
- "나중에 추가할게요" 스킵 옵션
- 한 번에 여러 개 연속 등록 가능

**안티패턴**:
- 제품 DB에서 검색하게 하는 것 (MVP에 불필요한 복잡도)
- 브랜드/용량/가격 등 부가 정보 강제
- 등록 없이 진행 불가

**페르소나 반응**:
- 서유진: "토너 하나, 세럼 두 개, 크림 하나. 4개 등록 완료. 1분 걸림"
- 김수빈: "5스텝이라 5개 등록. 카테고리별로 나뉘어 있어서 정리되는 느낌"
- 이채은: "10개... 좀 많은데 일단 자주 쓰는 것만 5개 넣고 나머지는 나중에"

---

## Phase 2: 첫 기록

### US-04. 첫 밤 기록

**As a** 방금 온보딩을 마친 유저
**I want to** 오늘 밤 루틴과 오늘 하루 생활을 기록하고 싶다
**So that** 내일 아침 피부와 연결할 데이터를 남긴다

**수용 기준**:
- 제품 선택: 등록된 제품 목록에서 탭 on/off
- 선택된 제품은 시각적으로 구분 (배경색 변경)
- 생활 변수: 프리셋 칩 탭 on/off (밀가루/매운음식/음주/운동/수면부족/앞머리/스트레스/야근/마스크/생리전)
- 아무것도 선택 안 해도 저장 가능 ("바른 거 없음"도 데이터)
- 저장 후 "내일 아침에 피부 상태를 기록해봐!" 안내 메시지
- 기록 소요 시간 30초 이내

**안티패턴**:
- 제품 사용 순서까지 강제로 정렬하게 하는 것
- 생활 변수를 하나도 안 고르면 경고 띄우는 것
- 저장 버튼이 스크롤해야 보이는 것

**페르소나 반응**:
- 서유진: "오 탭탭탭 끝. 세럼A 체크, 밀가루 체크, 수면부족 체크. 10초 걸림"
- 김수빈: "5개 제품 체크, 야근 체크. 루틴이 고정적이라 매번 같을 듯. '어젯밤과 동일' 기능 있으면 좋겠다"
- 이채은: "오늘은 세럼 두 개 겹쳐 발랐는데 둘 다 체크하면 되는 거지? OK"

---

### US-05. 첫 아침 기록

**As a** 어젯밤 기록을 한 유저
**I want to** 오늘 아침 피부 상태를 기록하고 싶다
**So that** 어젯밤 루틴/생활과 연결된 데이터가 완성된다

**수용 기준**:
- 피부 점수 1~5 탭 (한 번만 탭하면 됨)
- 각 점수에 짧은 라벨: 1=최악 / 2=별로 / 3=보통 / 4=좋음 / 5=꿀피부
- 피부 키워드 복수 선택: 촉촉/건조/트러블/뾰루지/각질/번들/칙칙/화사/붉음/당김/매끈
- 메모 입력 (optional, 한 줄)
- 저장 후 홈 화면 복귀, "오늘 피부: 4점 촉촉" 표시
- 기록 소요 시간 15초 이내

**안티패턴**:
- 점수 슬라이더 (탭이 더 빠름)
- 키워드 최소 1개 강제 선택
- 부위별 상세 입력 강제 (이마/볼/턱/코 각각)

**페르소나 반응**:
- 서유진: "4점, 촉촉. 끝! 와 진짜 빠르다"
- 김수빈: "3점, 건조, 당김. 환절기라 맨날 이런 느낌... 메모에 '볼 각질' 추가"
- 이채은: "2점, 트러블, 뾰루지. 어제 마라탕 먹은 게 문제인 듯. 메모에 '턱 라인 올라옴' 적기"

---

### US-06. 어젯밤과 동일 기록

**As a** 루틴이 거의 고정된 유저
**I want to** 어젯밤과 동일한 루틴을 한 번에 입력하고 싶다
**So that** 매번 같은 걸 탭하는 수고를 줄인다

**수용 기준**:
- 밤 기록 화면 상단에 "어젯밤과 동일" 버튼
- 탭하면 어젯밤 선택했던 제품 + 변수가 그대로 채워짐
- 채워진 후 개별 항목 수정 가능 (오늘은 운동 추가 등)
- 이전 기록이 없으면 버튼 비활성화
- 변경 없이 바로 저장 가능

**안티패턴**:
- "동일" 탭하면 확인 없이 바로 저장
- 제품만 복사하고 변수는 초기화

**페르소나 반응**:
- 서유진: "루틴 바뀌는 날만 직접 고르면 되니까 편하다"
- 김수빈: "이 기능 진짜 좋다. 나는 루틴 거의 안 바뀌거든. 변수만 수정하면 됨"
- 이채은: "나는 매일 다른 거 바르니까 잘 안 쓸 듯. 근데 있으면 좋지"

---

## Phase 3: 1주 사용

### US-07. 주간 피부 추이 확인

**As a** 7일간 기록한 유저
**I want to** 이번 주 피부 점수 변화를 한눈에 보고 싶다
**So that** 피부 상태가 나아지고 있는지 악화되고 있는지 파악한다

**수용 기준**:
- 홈 화면에 최근 7일 피부 점수 라인 차트
- 각 날짜에 점수(1-5) 표시
- 기록 없는 날은 빈 점으로 표시 (선 끊김)
- 날짜 탭하면 해당 날 상세 (루틴 + 변수 + 키워드) 보기
- 주간 평균 점수 표시

**안티패턴**:
- 복잡한 그래프 (다중 축, 범례 가득)
- 데이터 없는 날을 0점으로 처리
- 차트만 보여주고 해석 없음

**페르소나 반응**:
- 서유진: "오 수~목에 확 떨어졌네. 그때 시험 기간이었지..."
- 김수빈: "이번 주 평균 3.7. 지난주보다 나아졌다. 에센스 바꾼 게 효과 있는 건가?"
- 이채은: "월요일 2점에서 금요일 4점까지 올라갔다. 뭐가 달라졌지?"

---

### US-08. 기록 스트릭 확인

**As a** 매일 기록하고 있는 유저
**I want to** 연속 기록 일수를 보고 싶다
**So that** 기록 습관을 유지하는 동기가 된다

**수용 기준**:
- 홈 화면에 "N일 연속 기록 중" 표시
- 밤 + 아침 모두 기록한 날만 카운트
- 하루라도 빠지면 리셋 (0일부터 재시작)
- 최장 스트릭 기록 보관 (설정에서 확인 가능)
- 과도한 알림이나 압박감 없이 가볍게 표시

**안티패턴**:
- 스트릭 끊겼을 때 "아쉬워요!" 같은 죄책감 유발 메시지
- 스트릭 보상 시스템 (MVP 범위 초과)
- 스트릭 끊기면 데이터 손실되는 것처럼 보이는 UI

**페르소나 반응**:
- 서유진: "12일째! 의외로 꾸준히 하게 된다. 누워서 탭탭하면 끝이니까"
- 김수빈: "스트릭이 끊기면 좀 아쉽긴 한데, 압박감은 없어서 좋아"
- 이채은: "20일째. 친구한테 자랑해야지"

---

### US-09. 캘린더에서 월간 패턴 확인

**As a** 2주 이상 기록한 유저
**I want to** 한 달간 피부 상태를 달력으로 보고 싶다
**So that** 특정 시기에 피부가 나빠지는 패턴을 발견한다

**수용 기준**:
- 월간 캘린더 뷰, 날짜 셀에 피부 점수 색상 표시
- 색상 스케일: 1점(살구/로즈 톤) → 3점(중립/회색) → 5점(세이지/그린 톤)
- 기록 없는 날은 빈 셀
- 날짜 탭하면 해당 날 상세 보기 (바텀시트)
- 이전 달로 스와이프 또는 화살표 이동

**안티패턴**:
- 빨강-초록 강렬한 색상 (시각적 스트레스)
- 기록 없는 날에 X 마크나 경고색
- 상세 보기 없이 색상만 보여주는 것

**페르소나 반응**:
- 서유진: "생리 전 일주일이 다 살구색이네... 역시 호르몬 영향이 크구나"
- 김수빈: "월초에 회식 3번 한 주가 확실히 안 좋다. 눈에 바로 보이네"
- 이채은: "이번 달 전반적으로 초록색이 많아졌다! 루틴 잘 잡은 듯"

## Phase 4: 인사이트 발견

### US-10. 제품별 피부 영향 분석

**As a** 7일 이상 기록한 유저
**I want to** 각 제품이 내 피부에 어떤 영향을 주는지 보고 싶다
**So that** 효과 있는 제품을 유지하고 안 맞는 제품을 제외한다

**수용 기준**:
- 인사이트 > 제품 분석 탭
- 제품별 카드: 제품명 / 사용 횟수 / 사용 시 평균 점수 / 미사용 시 평균 점수 / 차이
- 영향력 큰 순으로 정렬 (차이 절대값 기준)
- 긍정 영향: "+0.8" 같이 초록 톤으로 표시
- 부정 영향: "-0.5" 같이 로즈 톤으로 표시
- 차이 0.5 미만: "아직 유의미한 차이 없음"
- 해당 조건 3회 미만: "데이터가 더 필요해요"

**안티패턴**:
- 통계적 유의성 없이 작은 차이를 과장
- 모든 제품을 같은 비중으로 나열
- 그래프만 보여주고 문장형 해석 없음

**페르소나 반응**:
- 서유진: "세럼A: 사용 시 4.2, 미사용 시 3.1. +1.1! 이거 계속 써야겠다 확신이 생겼어"
- 김수빈: "새 에센스: 사용 시 3.8, 이전 에센스 사용 시 3.4. 약간 나은데... 좀 더 써봐야겠다"
- 이채은: "C세럼: -0.7! 이거 바르면 트러블 나는 거였어! 바로 빼야지"

---

### US-11. 생활 변수별 피부 영향 분석

**As a** 7일 이상 기록한 유저
**I want to** 생활 변수가 내 피부에 어떤 영향을 주는지 보고 싶다
**So that** 피부에 안 좋은 습관을 줄이거나, 좋은 습관을 늘린다

**수용 기준**:
- 인사이트 > 변수 분석 탭
- 변수별 카드: 변수명 / 해당 일수 / 해당 시 평균 점수 / 비해당 시 평균 점수 / 차이
- 영향력 큰 순 정렬
- 예시 문장: "밀가루 먹은 다음날, 피부 점수가 평균 1.1점 낮아져요"
- 긍정 변수도 있을 수 있음: "운동한 다음날, 피부 점수가 평균 0.7점 높아져요"

**안티패턴**:
- "밀가루를 끊으세요" 같은 지시형 메시지 (유저 판단에 맡김)
- 데이터 1-2건으로 결론 도출
- 모든 변수를 동일 비중으로 나열

**페르소나 반응**:
- 서유진: "밀가루 -1.1... 라면 좀 줄여야 하나. 근데 운동 +0.7이면 운동하고 라면 먹으면 상쇄되나? ㅋㅋ"
- 김수빈: "음주 -1.5. 역시... 회식 날은 어쩔 수 없지만 수치로 보니까 확실하네"
- 이채은: "매운음식 -0.9. 마라탕 줄여야겠다... 근데 맛있는데..."

---

### US-12. 키워드 빈도 변화 확인

**As a** 특정 피부 고민이 있는 유저
**I want to** 특정 키워드(예: 트러블, 건조)가 줄고 있는지 보고 싶다
**So that** 현재 루틴이 고민 해결에 도움이 되는지 판단한다

**수용 기준**:
- 인사이트 화면에서 제품 카드 탭 시 키워드 상세 표시
- "세럼A 사용 시 '촉촉' 등장 70%, 미사용 시 30%"
- "크림B 사용 시 '건조' 등장 10%, 미사용 시 45%"
- 2주간 키워드 빈도 추이 (감소/증가/유지)

**안티패턴**:
- 모든 키워드를 나열 (유의미한 것만 표시)
- 전문 의학 용어 사용
- 빈도 0-1회인 키워드까지 분석

**페르소나 반응**:
- 서유진: "세럼A 쓸 때 트러블 키워드가 확 줄어드네. 진짜 맞는 제품이었어"
- 김수빈: "이 크림 바꾸고 '건조' 키워드가 반으로 줄었어. 효과 있다!"
- 이채은: "번들 키워드가 나이아신아마이드 세럼 쓰면서 줄어들고 있어"

## Phase 5: 습관화

### US-13. 밤 기록 알림

**As a** 기록을 까먹을 수 있는 유저
**I want to** 밤에 기록 알림을 받고 싶다
**So that** 기록을 빠뜨리지 않고 습관을 유지한다

**수용 기준**:
- 기본 알림 시간: 오후 10시 (설정에서 변경 가능)
- 알림 메시지: "오늘 밤 루틴 기록할 시간이야" (부드러운 톤)
- 이미 기록했으면 알림 안 보냄
- 알림 끄기 옵션 있음
- 알림 탭하면 밤 기록 화면으로 바로 이동

**안티패턴**:
- 매시간 재알림
- "기록 안 하면 스트릭 끊겨요!" 압박형 메시지
- 끄기 옵션 없음

**페르소나 반응**:
- 서유진: "10시에 알림 오면 누워서 바로 기록하게 돼. 타이밍 좋다"
- 김수빈: "11시로 바꿨어. 야근하면 10시에 아직 회사니까"
- 이채은: "알림 없어도 습관이 됐는데, 있으니까 안심"

---

### US-14. 아침 기록 알림

**As a** 아침에 바쁜 유저
**I want to** 아침에 기록 알림을 받고 싶다
**So that** 세수하고 피부 상태 확인한 직후에 기록한다

**수용 기준**:
- 기본 알림 시간: 오전 8시 (설정에서 변경 가능)
- 알림 메시지: "오늘 아침 피부는 어때?" (궁금해하는 톤)
- 이미 기록했으면 알림 안 보냄
- 15초 컷이라는 걸 알림에 암시 ("15초면 끝!")

**안티패턴**:
- 오전 6시 기본 알림 (너무 이름)
- 밤 기록 안 했으면 아침 알림에서 밤 기록도 같이 요구

**페르소나 반응**:
- 서유진: "8시 알림 오면 세수하러 가면서 기록. 이제 루틴 됐어"
- 김수빈: "7시 30분으로 바꿈. 출근 전에 해야 해서"
- 이채은: "세수하고 거울 보면서 바로 기록. 딱 좋은 타이밍"

---

### US-15. 제품 추가/아카이브

**As a** 새 제품을 사거나 다 쓴 유저
**I want to** 제품 목록을 업데이트하고 싶다
**So that** 기록할 때 현재 쓰는 제품만 보인다

**수용 기준**:
- 제품 추가: 이름 + 카테고리 입력으로 즉시 등록
- 제품 아카이브: 안 쓰는 제품은 목록에서 숨기되 기록은 유지
- 아카이브된 제품은 인사이트에서 여전히 분석 가능
- 제품 삭제: 아카이브 상태에서만 삭제 가능 (실수 방지)
- 밤 기록 화면에서도 빠르게 추가 가능 ("+ 제품 추가")

**안티패턴**:
- 삭제하면 관련 기록도 다 사라지는 것
- 제품 수정 불가 (오타 수정도 안 되는 것)
- 아카이브 기능 없이 삭제만 있는 것

**페르소나 반응**:
- 서유진: "올리브영에서 새 세럼 사왔다. 바로 추가하고 오늘부터 기록!"
- 김수빈: "에센스 다 써서 아카이브. 새 거 사면 추가할게"
- 이채은: "제품 10개 넘으니까 안 쓰는 거 아카이브해야겠다. 목록이 깔끔해졌네"

---

### US-16. 커스텀 생활 변수 추가

**As a** 프리셋에 없는 변수를 추적하고 싶은 유저
**I want to** 나만의 생활 변수를 추가하고 싶다
**So that** 내 상황에 맞는 추적을 한다

**수용 기준**:
- 밤 기록 화면 또는 설정에서 "변수 추가"
- 이름만 입력하면 칩으로 추가됨
- 프리셋과 동일하게 on/off 토글 가능
- 인사이트에서도 동일하게 분석됨
- 삭제 가능 (과거 기록은 유지)

**안티패턴**:
- 커스텀 변수 개수 제한 (3개 이하)
- 카테고리 강제 분류
- 추가 과정이 3단계 이상

**페르소나 반응**:
- 서유진: "카페인 추가! 커피 많이 마신 날도 피부 영향 있는 것 같아서"
- 김수빈: "에어컨 추가. 사무실 에어컨이 피부 건조의 원인인 것 같거든"
- 이채은: "핫요가 추가. 일반 운동이랑 다를 것 같아서 분리해서 보고 싶어"

---

### US-17. 데이터 내보내기

**As a** 데이터를 백업하고 싶은 유저
**I want to** 내 기록을 파일로 다운로드하고 싶다
**So that** 데이터를 잃어버리지 않는다

**수용 기준**:
- 설정에서 "데이터 내보내기" 버튼
- JSON 형식으로 전체 기록 다운로드
- 제품 목록 + 변수 목록 + 일별 기록 포함
- 내보내기 완료 시 확인 메시지

**안티패턴**:
- 유료 기능으로 제한
- 내보내기 포맷을 사용자가 선택해야 함 (MVP에서는 JSON만)
- 가져오기(import) 기능까지 구현 (MVP 이후)

**페르소나 반응**:
- 서유진: "혹시 폰 바꾸면 데이터 날아갈까봐 걱정했는데 다운 받아놓을 수 있네"
- 김수빈: "한 달에 한 번 백업해놔야겠다"
- 이채은: "JSON이면 스프레드시트로 열어서 더 분석할 수도 있겠다"

### US-18. 인사이트 데이터 부족 안내

**As a** 기록을 막 시작한 유저 (7일 미만)
**I want to** 인사이트가 언제부터 보이는지 알고 싶다
**So that** 기록을 포기하지 않고 계속한다

**수용 기준**:
- 인사이트 탭 진입 시 "N일 더 기록하면 분석을 볼 수 있어요"
- 프로그레스 바: "3/7일 기록 완료"
- 3일부터 미니 인사이트 (주간 추이, 키워드 빈도)로 맛보기 제공
- 데모 데이터로 인사이트 화면 미리보기 옵션

**안티패턴**:
- 빈 화면에 "데이터 없음" 한 줄만 표시
- 7일 전까지 인사이트 탭 자체를 비활성화
- "매일 기록해야 합니다!" 압박 메시지

**페르소나 반응**:
- 서유진: "3일 만에 미니 인사이트가 나오니까 계속 하고 싶어진다"
- 김수빈: "4일 남았네. 이번 주만 더 하면 된다"
- 이채은: "데모 미리보기 봤는데 진짜 이렇게 나오면 대박... 열심히 기록해야지"

### US-19. 날짜별 상세 기록 확인

**As a** 특정 날의 기록을 확인하고 싶은 유저
**I want to** 캘린더에서 날짜를 탭해서 그날의 전체 기록을 보고 싶다
**So that** "그날 왜 피부가 좋았지/안 좋았지"를 돌아본다

**수용 기준**:
- 캘린더 날짜 탭 → 바텀시트로 상세 표시
- 밤 기록: 사용 제품 목록 + 생활 변수 목록
- 아침 기록: 피부 점수 + 키워드 + 메모
- 기록 수정 가능 (바텀시트에서 "수정" 탭)
- 기록이 없는 날은 "기록 없음" 표시

**안티패턴**:
- 새 화면으로 전환 (바텀시트가 맥락 유지에 좋음)
- 수정 불가 (오타, 잘못 기록한 경우 있음)
- 밤/아침 기록을 따로 보여주는 것 (한 화면에서 연결해서)

**페르소나 반응**:
- 서유진: "지난 화요일 2점이었네. 그날 야식으로 라면 먹었구나..."
- 김수빈: "5점이었던 날 뭐 했는지 보자. 운동 + 일찍 잠 + 에센스. 이 조합이 최고네"
- 이채은: "실수로 3점 눌렀는데 4점이었거든. 수정할 수 있어서 다행"

### US-20. 설정 관리

**As a** 앱을 커스터마이징하고 싶은 유저
**I want to** 알림 시간, 피부 타입, 데이터 관리를 설정하고 싶다
**So that** 앱이 내 생활 패턴에 맞게 동작한다

**수용 기준**:
- 알림 설정: 밤/아침 알림 시간 변경, 알림 on/off
- 피부 타입 변경
- 데이터 내보내기 (JSON)
- 데이터 초기화 (확인 2번 필요)
- 커스텀 변수 관리 (추가/삭제/순서 변경)
- 앱 버전 정보

**안티패턴**:
- 설정 항목이 20개 이상 (MVP에서는 최소한으로)
- 다크 모드 토글 (라이트 모드 고정)
- 언어 설정 (한국어 고정)

**페르소나 반응**:
- 서유진: "알림 시간만 바꾸면 됨. 심플해서 좋다"
- 김수빈: "커스텀 변수 순서 바꿀 수 있어서 자주 쓰는 거 위로 올림"
- 이채은: "설정 깔끔하다. 복잡하지 않아서 좋아"
