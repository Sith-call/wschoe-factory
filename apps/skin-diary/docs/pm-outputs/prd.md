# 피부 일지 (Skin Diary) — Product Requirements Document

> "피부는 어제의 결과다."
> 오늘의 피부 상태는 어제 바른 것 + 어제 먹은 것 + 어제 한 것의 복합 결과.
> 이걸 추적하면 "나에게 맞는 루틴"을 데이터로 찾을 수 있다.

---

## 1. 문제 정의

### 유저가 겪는 현실
- 기초케어를 매번 다르게 바르는데, 다음날 피부 상태가 달라진다
- "이 세럼이 좋은 건지, 어제 밀가루를 먹어서 안 좋은 건지" 원인을 특정할 수 없다
- 머릿속으로만 추적하니까 패턴을 파악할 수 없고, 결국 감으로 루틴을 정한다
- 제품을 바꿔도 "맞는 건지 아닌 건지" 판단하려면 최소 2주는 써봐야 하는데 기억이 안 된다

### 기존 해결책의 한계
| 방법 | 문제 |
|------|------|
| 메모앱에 기록 | 구조화 안 됨, 분석 불가, 귀찮아서 3일이면 그만둠 |
| 뷰티 앱 (화해, 글로우픽) | 제품 리뷰는 있지만 "나의 피부 변화" 추적 기능 없음 |
| 피부 사진 기록 앱 | 사진 찍는 게 귀찮고, 조명/각도 달라서 비교 어려움 |
| 엑셀/스프레드시트 | 입력이 너무 번거로움, 모바일에서 불편 |

### 우리의 해결책
30초 안에 끝나는 밤/아침 2회 기록으로, 제품별/생활변수별 피부 영향을 데이터로 보여준다.

---

## 2. 타겟 유저

**코어 타겟**: 20대 여성, 기초 스킨케어를 하고 있고, 피부 상태 변화에 민감한 사람

**공통 특성**:
- 기초케어 최소 2스텝 이상 하고 있음
- 제품을 자주 바꾸거나, 바꾸고 싶은 욕구가 있음
- "어제 뭘 했길래 오늘 피부가 이런 거지?" 생각해본 적 있음
- 건강/습관 트래커 앱 경험 있음 (수면, 운동 등)

---

## 3. 페르소나

### 페르소나 1: 서유진 (24세, 대학생/취준생)

| 항목 | 내용 |
|------|------|
| 피부 타입 | 복합성, 턱 주변 트러블 반복 |
| 현재 루틴 | 토너 → 세럼 → 크림 (3스텝), 피곤하면 토너만 바르고 잠 |
| 제품 | 저~중가, 올리브영 위주. 세럼을 자주 바꿈 |
| 주요 고민 | "어떤 세럼이 진짜 효과 있는지 모르겠어. 일주일 써보고 별로면 바꾸는데, 그게 제품 탓인지 내 생활 탓인지..." |
| 생활 변수 | 밀가루 자주 먹음 (라면, 빵), 수면 불규칙 (새벽 2-4시), 앞머리 있음, 시험 기간 스트레스 |
| 기대하는 것 | "이 세럼 쓴 날 vs 안 쓴 날 피부 차이를 숫자로 보고 싶어" |
| 기록 패턴 | 밤에 누워서 폰 보다가 기록, 아침에 세수하고 기록 |

**유진이가 만족하는 순간**: 2주 기록 후 "세럼A 사용 시 평균 4.2점, 미사용 시 3.1점"을 보고 "아 이거 계속 써야겠다" 확신을 갖는 순간.

---

### 페르소나 2: 김수빈 (27세, 직장인)

| 항목 | 내용 |
|------|------|
| 피부 타입 | 건성, 환절기에 건조함과 각질이 심해짐 |
| 현재 루틴 | 클렌징 → 토너 → 에센스 → 크림 → 아이크림 (5스텝), 매일 규칙적 |
| 제품 | 중~고가, 쿠팡/시코르. 새 제품 도입에 신중 |
| 주요 고민 | "새 에센스를 2주째 쓰고 있는데 맞는 건지 아닌 건지 판단이 안 돼. 야근이 많아서 피곤한 탓인지 제품 탓인지..." |
| 생활 변수 | 야근 빈도, 회식(음주), 운동 유무, 스트레스, 에어컨/히터 노출 |
| 기대하는 것 | "생활 변수 배제하고 제품 효과만 분리해서 보고 싶어" |
| 기록 패턴 | 루틴이 고정적이라 밤 기록 30초 컷, 아침은 출근 준비하면서 |

**수빈이가 만족하는 순간**: "음주한 다음날 평균 2.3점"을 보고 "야 회식 진짜 피부에 안 좋구나" 하고 근거를 갖는 순간. 동시에 "이 에센스 쓰기 시작한 후 건조 키워드 빈도 40% 감소"를 보고 유지 결정.

---

### 페르소나 3: 이채은 (22세, 뷰티 관심 많은 대학생)

| 항목 | 내용 |
|------|------|
| 피부 타입 | 지성, 모공 넓음, T존 번들거림 |
| 현재 루틴 | 매일 다름. 신제품 마니아. 3-5스텝을 조합하며 실험 |
| 제품 | 다양. 올리브영 신상 + 인스타 광고 제품, 10개 이상 보유 |
| 주요 고민 | "뭘 바르면 좋아지고 뭘 바르면 올라오는지 기록하고 싶어. 제품이 너무 많아서 헷갈려" |
| 생활 변수 | 매운 음식(떡볶이, 마라탕), 운동(필라테스), 마스크 오래 착용 |
| 기대하는 것 | "제품 조합별로 비교하고 싶어. A세럼+B크림 vs C세럼+B크림" |
| 기록 패턴 | 기록 자체를 즐김. 상세하게 적고 싶어함 |

**채은이가 만족하는 순간**: 한 달 기록 후 인사이트에서 "나이아신아마이드 세럼 사용 시 번들 키워드 60% 감소"를 보고 인스타 스토리에 공유하는 순간.

## 4. 핵심 데이터 구조

### 시간축 모델 (이 앱의 핵심)

피부 상태는 **어제의 입력 → 오늘의 출력** 구조다. 따라서 기록을 두 시점으로 나눈다:

```
┌─────────────────────────────────────────────────────┐
│  어젯밤 기록 (Night Log) — 입력 변수                   │
│                                                     │
│  스킨케어 루틴:                                       │
│    [토너A] [세럼B] [크림C]  ← 순서 + 제품명             │
│                                                     │
│  생활 변수:                                           │
│    [밀가루] [매운음식] [음주] [운동] [수면부족]            │
│    [앞머리] [스트레스] [야근] [마스크] [생리전]           │
│    + 커스텀 변수 추가 가능                              │
└─────────────────────────────────────────────────────┘
                        ↓ 하룻밤 경과
┌─────────────────────────────────────────────────────┐
│  오늘 아침 기록 (Morning Log) — 출력 결과               │
│                                                     │
│  피부 점수: 1~5                                       │
│    1=최악  2=별로  3=보통  4=좋음  5=꿀피부              │
│                                                     │
│  피부 키워드 (복수 선택):                               │
│    [촉촉] [건조] [트러블] [뾰루지] [각질]               │
│    [번들] [칙칙] [화사] [붉음] [당김] [매끈]            │
│                                                     │
│  메모: "코 옆에 뾰루지 하나"  ← optional 한 줄          │
└─────────────────────────────────────────────────────┘
```

### 분석 가능한 인사이트

이 구조로 다음을 계산할 수 있다:

- **제품별 영향**: "세럼B 사용 시 피부 점수 평균 4.1 vs 미사용 시 3.2"
- **변수별 영향**: "밀가루 먹은 다음날 평균 2.8점, 안 먹은 날 3.9점"
- **키워드 빈도**: "세럼B 사용 시 '촉촉' 키워드 등장 70%, 미사용 시 30%"
- **복합 분석**: "세럼B + 충분한 수면 → 평균 4.5점 (최고 조합)"
- **트렌드**: "최근 2주 평균 피부 점수 3.8, 이전 2주 3.2 (개선 중)"

## 5. MVP 기능 (P0)

### 5-1. 밤 기록 (Night Log)

**목적**: 오늘 밤 뭘 발랐는지, 오늘 하루 어떤 생활을 했는지 기록

**스킨케어 루틴 입력**:
- 내 제품 목록에서 탭으로 on/off (오늘 바른 제품만 선택)
- 카테고리별 그룹핑: 클렌징 / 토너 / 세럼 / 에센스 / 크림 / 아이크림 / 선크림 / 팩 / 기타
- "제품 추가" 버튼으로 새 제품 등록 (이름 + 카테고리만 입력)
- 자주 쓰는 제품은 상단에 노출 (사용 빈도순)
- "어젯밤과 동일" 버튼 → 한 번에 채우기

**생활 변수 입력**:
- 프리셋 칩: 밀가루 / 매운음식 / 음주 / 운동 / 수면부족 / 앞머리 / 스트레스 / 야근 / 마스크 / 생리전
- 탭으로 on/off (해당하는 것만 선택)
- "변수 추가" 버튼으로 커스텀 변수 등록

**UX 목표**: 30초 이내 완료. 대부분의 경우 탭 5-8회로 끝나야 한다.

---

### 5-2. 아침 기록 (Morning Log)

**목적**: 오늘 아침 피부 상태를 기록

**피부 점수**:
- 1~5 스케일 (탭 한 번)
- 1: 최악 (트러블 폭발, 못 봐줄 수준)
- 2: 별로 (눈에 띄는 문제)
- 3: 보통 (특별히 좋지도 나쁘지도)
- 4: 좋음 (오 오늘 괜찮은데?)
- 5: 꿀피부 (거울 보고 기분 좋아짐)

**피부 키워드**:
- 복수 선택 가능
- 긍정: 촉촉 / 화사 / 매끈
- 부정: 건조 / 트러블 / 뾰루지 / 각질 / 번들 / 칙칙 / 붉음 / 당김
- 키워드는 분석에 활용 (특정 제품 사용 시 특정 키워드 빈도 변화)

**메모**: optional 한 줄 텍스트 (예: "코 옆에 뾰루지", "전반적으로 건조")

**UX 목표**: 15초 이내. 점수 탭 + 키워드 1-3개 탭 = 끝.

---

### 5-3. 홈 화면 (Home)

**상단: 오늘의 기록 상태**
- 기록 전: "오늘 아침 피부는 어때?" → 아침 기록 CTA
- 기록 후: "오늘 피부: 4점 · 촉촉, 화사" 요약 표시
- 밤 기록 미완: "오늘 밤 루틴 기록하기" → 밤 기록 CTA

**중간: 주간 피부 추이**
- 최근 7일 피부 점수 라인 차트
- 날짜별 점수 (1-5) + 키워드 아이콘
- 탭하면 해당 날 상세 보기

**하단: 기록 스트릭**
- "12일 연속 기록 중" (밤+아침 모두 기록한 날 기준)
- 가볍게 동기부여 (과도한 게이미피케이션은 지양)

---

### 5-4. 인사이트 (Insight)

**최소 데이터 요구**: 7일 이상 기록 시 활성화. 미달 시 "N일 더 기록하면 분석을 볼 수 있어요" 안내.

**제품 분석 탭**:
- 제품별 카드 리스트
- 각 카드: 제품명 / 사용 횟수 / 사용 시 평균 점수 / 미사용 시 평균 점수 / 차이
- 정렬: 영향력 큰 순 (차이 절대값)
- 예시: "세럼A → 사용 시 4.2점, 미사용 시 3.1점 (+1.1)"

**변수 분석 탭**:
- 생활 변수별 카드 리스트
- 각 카드: 변수명 / 해당 일수 / 해당 시 평균 점수 / 비해당 시 평균 점수 / 차이
- 예시: "밀가루 → 먹은 다음날 2.8점, 안 먹은 날 3.9점 (-1.1)"

**키워드 분석** (보조):
- 특정 제품 사용 시 어떤 키워드가 많이/적게 나오는지
- 예: "크림B 사용 시 '건조' 키워드 80% 감소"

---

### 5-5. 캘린더 (Calendar)

**월간 히트맵**:
- 달력 형태, 날짜 셀에 피부 점수 색상 표시
- 색상 스케일: 1점(붉은 톤) → 3점(중립) → 5점(건강한 톤)
- 기록 없는 날은 빈 셀

**날짜 탭 시 상세**:
- 해당 날의 밤 기록 (루틴 + 변수) + 아침 기록 (점수 + 키워드 + 메모)
- 바텀시트 또는 상세 화면으로 표시

---

### 5-6. 제품 관리 (Product List)

**내 제품 목록**:
- 카테고리별 섹션: 클렌징 / 토너 / 세럼 / 에센스 / 크림 / 아이크림 / 선크림 / 팩 / 기타
- 제품별: 이름 / 카테고리 / 사용 횟수 / 평균 점수
- 추가: 이름 + 카테고리만 입력 (브랜드, 사진 등은 MVP 이후)
- 삭제: 스와이프 또는 편집 모드
- 아카이브: 안 쓰는 제품은 아카이브로 (기록은 유지, 목록에서 숨김)

## 7. 네비게이션

**하단 3탭**:
- 홈 (오늘) — 기록 + 주간 추이
- 캘린더 — 월간 히트맵
- 인사이트 — 분석

**상단 접근**:
- 제품 관리 — 홈 상단 아이콘
- 설정 — 홈 상단 아이콘

**기록 플로우**:
- 홈에서 CTA 탭 → NightLog 또는 MorningLog → 기록 완료 → 홈 복귀
- 기록 화면은 모달 또는 풀스크린 (탭바 숨김)

## 8. 디자인 방향

### 톤앤매너
- 부드럽고 친근한 톤. "기록하세요" 대신 "오늘 피부는 어때?"
- 과학적이지만 딱딱하지 않게. 데이터를 보여주되 그래프가 아닌 문장형 인사이트 병행.
- 자기관리하는 느낌. 다이어리 감성 + 데이터 분석의 조합.

### 색상 (DESIGN_RULES R1 준수)
- 웰니스/자기관리 앱 → 뮤트 파스텔 또는 자연색 계열
- 제안: 소프트 세이지(#A8B5A2) 또는 로즈 베이지(#C9A99B) 기반
- 라이트 모드 기본 (R9 준수)
- 피부 점수 색상: 1점 로즈/살구 톤 → 5점 건강한 그린 톤 (선명한 빨강/초록 지양)

### 폰트 (DESIGN_RULES R8 준수)
- 한글: Pretendard 또는 Spoqa Han Sans
- 영문: DM Sans 또는 Outfit
- 기존 앱들과 다른 조합 사용

### 컴포넌트 스타일
- 버튼: rounded-lg (pill 아님, sharp 아님). Primary/Secondary 구분 (R16)
- 카드: 큰 radius 없이 간결하게. 카드 네스팅 금지 (R15)
- 칩(토글): 소프트한 배경색 on/off. 선택 시 배경 + 텍스트 색상 변경
- 좌측 정렬 기본 (R14)

### 아이콘 (DESIGN_RULES R11, R13 준수)
- Lucide 또는 Phosphor Icons 사용
- 이모지를 아이콘으로 사용하지 않음

### 애니메이션 (DESIGN_RULES R4 준수)
- 대부분 즉시 표시
- 점수 입력 시 가벼운 스케일 피드백 (0.95 → 1)
- 인사이트 수치 등장 시 카운트업 애니메이션
- fadeInUp 남용 금지

## 9. 기술 고려사항

### 데이터 저장
- MVP: LocalStorage 또는 IndexedDB (서버 없이 로컬 우선)
- 데이터 구조: 날짜 기반 key-value
- 내보내기: JSON 다운로드 (설정에서)

### 분석 로직
- 단순 평균 비교 (제품 사용/미사용 시 점수 평균)
- 최소 샘플 수: 해당 조건 3회 이상이어야 표시
- 차이가 0.5점 미만이면 "유의미한 차이 없음"으로 표시

### 알림
- 밤 기록 알림: 오후 10시 (커스터마이징 가능)
- 아침 기록 알림: 오전 8시 (커스터마이징 가능)
- 웹 푸시 또는 앱 내 리마인더

### 데모 모드
- 첫 진입 시 샘플 데이터 14일분 제공 가능 (인사이트 화면 미리보기용)
- 실제 기록 시작하면 샘플 데이터 제거

## 10. 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 일간 기록률 | 60%+ | 밤+아침 모두 기록한 날 / 전체 활성일 |
| 7일 리텐션 | 50%+ | 첫 기록 후 7일 내 재기록 비율 |
| 인사이트 조회율 | 주 3회+ | 인사이트 탭 방문 수 / 주 |
| 기록 소요 시간 | 밤 30초, 아침 15초 | 기록 시작~완료 시간 |
| 제품 등록 수 | 평균 5개+ | 유저당 등록 제품 수 |

## 11. Out of Scope (MVP 이후)

- 피부 사진 촬영 및 비교
- AI 기반 피부 분석 (사진 → 점수 자동 채점)
- 제품 데이터베이스 연동 (화해, 글로우픽 등)
- 제품 추천 / 쇼핑 연동
- 소셜 기능 (루틴 공유, 커뮤니티)
- 피부과 상담 연동
- 날씨/습도 자동 연동 (기상청 API)
- 생리 주기 트래커 연동
- 복합 분석 (제품 조합별 점수) — 데이터 충분히 쌓인 후
