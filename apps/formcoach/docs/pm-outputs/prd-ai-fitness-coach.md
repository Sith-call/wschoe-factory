# Product Requirements Document: AI 기반 개인 피트니스 코치 앱 "FitAI"

**작성자**: PM Executor
**날짜**: 2026-03-18
**상태**: Draft
**이해관계자**: 제품팀, 엔지니어링팀, 디자인팀, 마케팅팀, 경영진

## 1. Executive Summary

**FitAI**는 AI가 사용자의 체력 수준, 목표, 일정에 맞춰 개인화된 운동 계획을 생성하고, 카메라 기반 실시간 자세 교정과 게이미피케이션 동기부여 시스템을 제공하는 모바일 피트니스 코치 앱이다. 20-40대 헬스장 초보~중급자를 타겟으로 하며, "PT 1회 비용보다 저렴한 월 구독으로 나만의 AI 트레이너"라는 가치를 제안한다. 무료 피트니스 콘텐츠의 범람 속에서 **"내게 맞는 운동의 큐레이션"**이라는 미충족 수요를 해결한다.

## 2. Background & Context

### 문제 공간

20-40대 운동 초보~중급자는 세 가지 핵심 고충을 공유한다:

1. **"뭘 해야 할지 모르겠다"** — 유튜브/인스타에 운동 콘텐츠는 넘치지만, 자기 수준과 목표에 맞는 계획을 선택하지 못하는 정보 과부하 상태
2. **"자세가 맞는지 불안하다"** — 혼자 운동 시 부상 위험에 대한 불안, 피드백 부재로 성장 정체
3. **"작심삼일이다"** — 동기부여 부족으로 평균 2주 내 운동 앱 이탈

### 현재 대안과 한계

| 대안 | 장점 | 한계 |
|------|------|------|
| PT (개인 트레이너) | 개인화, 실시간 피드백 | 월 40-80만원 비용 부담 |
| 유튜브/인스타 운동 영상 | 무료, 다양한 콘텐츠 | 비개인화, 자세 피드백 없음 |
| 기존 피트니스 앱 (Nike Training, 키핏) | 체계적 프로그램 | 개인화 수준 낮음, AI 자세 교정 없음 |

### 시장 기회

- **글로벌 피트니스 앱 시장**: 연평균 21% 성장률 (2025-2030 예측)
- **한국 피트니스 앱 시장**: 스마트폰 보급률 97%+, 헬스장 등록자 수 증가 추세
- **AI 피트니스 차별화 기회**: 기존 앱 대비 MediaPipe/MoveNet 기반 실시간 자세 인식 기술 성숙

### 선행 리서치 요약

- **디스커버리 인터뷰 설계**: 60%+ 사용자가 운동 계획 수립에 불편함 느낄 것으로 가정 (검증 필요)
- **기술 PoC 계획**: 스마트폰 카메라 기반 5개 운동 80%+ 자세 인식 정확도 목표
- **지불 의향 테스트 설계**: 월 9,900~14,900원 가격대, 사전등록 전환율 5%+ 목표

## 3. Objectives & Success Metrics

### Goals (성공의 모습)

1. **MVP 출시 후 3개월 내 MAU 10,000명 달성** — 비치헤드 세그먼트(목표 지향 자기개발러) 중심
2. **Day-7 리텐션 40% 이상 유지** — 기존 피트니스 앱 평균(25-30%) 대비 유의미한 개선
3. **무료→유료 전환율 8% 이상** — 프리미엄 구독 모델의 사업적 유효성 검증
4. **첫 운동 완료율 70% 이상** — 온보딩에서 첫 운동까지의 퍼널 최적화

### Non-Goals (명시적 범위 제외)

1. **식단/영양 관리 기능** — MVP에서 제외. 운동 계획 핵심 가치 검증 이후 Phase 2에서 검토
2. **웨어러블/스마트워치 연동** — 스마트폰 단독 경험에 집중. 연동은 사용자 수요 검증 후 추가
3. **소셜 기능(운동 버디, 커뮤니티)** — MVP에서 제외. 리텐션 데이터 확보 후 Phase 2에서 검토
4. **B2B 헬스장 제휴 모델** — B2C 직접 사용자 검증 우선. B2B는 별도 전략으로 검토
5. **글로벌 출시** — 한국 시장 검증 후 확장. MVP는 한국어 전용

## 4. Target Users & Segments

### Primary 타겟: 목표 지향 자기개발러 (비치헤드 세그먼트)

- **규모**: 전체 타겟의 약 30%
- **특성**: 명확한 목표(여름 전 -5kg, 벤치프레스 80kg 등), 계획적, 데이터 선호
- **대표 페르소나**: 박성훈 (34세, IT 개발자) — 데이터 기반 의사결정, 자세 교정 얼리 어답터
- **지불 의향**: 높음 (월 14,900원+)
- **전환 트리거**: "내 데이터를 분석해서 최적 계획을 만들어준다"
- **선택 이유**: 가장 높은 지불 의향 + 제품 적합도 + 얼리 어답터 가능성 + 입소문 효과

### Secondary 타겟: 재시작 다이어터

- **규모**: 전체 타겟의 약 35% (최대 성장 세그먼트)
- **특성**: 운동 시도 3회 이상, 매번 2주 내 중단, 동기부여가 핵심 니즈
- **대표 페르소나**: 김지현 (28세, 직장인) — 앱 적극 사용하지만 2주 내 삭제
- **지불 의향**: 중간 (월 9,900원)
- **전환 트리거**: "이번에는 다를 거야" 순간 + 친구 추천

### Tertiary 타겟: 시간 부족 홈트러 / 자세 집착 중급자

- Phase 2에서 본격 수용. MVP에서는 기본 지원 수준

## 6. Solution Overview

### 기술 아키텍처

```
┌──────────────────────────────────────────────┐
│                 모바일 앱 (Flutter)              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ 온보딩    │  │ 운동 가이드│  │ 대시보드  │    │
│  │ & 체력테스트│  │ & 타이머  │  │ & 기록   │    │
│  └──────────┘  └──────────┘  └──────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │  카메라 자세 인식 (MediaPipe on-device) │    │
│  └──────────────────────────────────────┘    │
└──────────────────────────────────────────────┘
                      │ API
┌──────────────────────────────────────────────┐
│              백엔드 서버                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ AI 운동   │  │ 사용자    │  │ 게이미    │    │
│  │ 계획 생성  │  │ 프로필/기록│  │ 피케이션  │    │
│  └──────────┘  └──────────┘  └──────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │  LLM 기반 개인화 엔진 (운동 계획 생성)   │    │
│  └──────────────────────────────────────┘    │
└──────────────────────────────────────────────┘
```

### 핵심 설계 결정

1. **카메라 자세 인식은 on-device 처리** — 개인정보 보호 + 실시간성 확보 (MediaPipe Pose)
2. **운동 계획 생성은 서버 사이드 AI** — LLM 기반으로 자연어 설명 포함한 개인화 계획
3. **Flutter 크로스 플랫폼** — iOS/Android 동시 출시, 개발 효율성
4. **자세 교정은 선택적 프리미엄 기능** — 카메라 없이도 핵심 가치(운동 계획) 경험 가능

### 핵심 사용자 흐름

```
앱 설치 → 소셜 로그인 → 목표 선택 → 체력 테스트 (선택) → 주간 계획 생성
    → "오늘의 운동" 시작 → 운동 수행 (자세 교정 선택) → 완료 & 피드백
    → 스트릭 업데이트 → 대시보드 확인 → (반복)
```

### Aha Moment 설계

- **첫 운동 시작 30초 이내**: "당신의 체력 수준에 맞게 설계된 운동이에요" 개인화 메시지
- **첫 자세 교정**: "자세가 좋아요!" 긍정 피드백 → "이 앱은 다르다" 인식
- **첫 주간 리포트**: "이번 주 총 운동 볼륨 1,200kg! 지난주 대비 15% 증가" 성과 가시화

## 8. Timeline & Phasing

### Phase 1: MVP (8주)

```
Week 1-2: 디스커버리 검증 (E1 인터뷰 + E2 기술 PoC 병행)
Week 2-3: Go/No-Go 판단 + UI/UX 디자인
Week 3-6: 핵심 개발
  - 온보딩 & 체력 테스트
  - AI 운동 계획 생성 엔진
  - 운동 가이드 & 타이머
  - 진행 상황 대시보드
  - 게이미피케이션 (스트릭, 배지)
```
