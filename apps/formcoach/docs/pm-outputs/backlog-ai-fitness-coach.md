# Backlog: AI 기반 개인 피트니스 코치 앱 "FitAI"

**형식**: User Stories (유저 스토리)
**총 스토리 수**: 18개 (P0: 11개, P1: 7개)
**추정 총 스토리 포인트**: 104 SP
**스프린트 기준**: 2주 스프린트, 팀 벨로시티 약 25-30 SP 가정 → 약 4 스프린트 (8주)

---

## Epic 1: 인증 & 온보딩

### Story 1: 소셜 로그인 및 회원가입
**사용자로서, 카카오/구글/애플 계정으로 간편하게 가입하고 로그인하고 싶다. 그래야 번거로운 회원가입 없이 바로 시작할 수 있으니까.**

수락 기준:
- [ ] 카카오, 구글, 애플 소셜 로그인 3종 지원
- [ ] 이메일/비밀번호 회원가입 대안 경로 제공
- [ ] 로그인부터 다음 화면 진입까지 15초 이내
- [ ] 자동 로그인 유지 (토큰 기반)
- [ ] 로그아웃 및 계정 삭제 기능

우선순위: **P0** | 포인트: **5 SP** | 의존성: 없음
🏷️ 디자인 필요: 로그인/가입 화면

---

### Story 2: 목표 및 프로필 설정
**신규 사용자로서, 내 운동 목표, 주당 운동 가능 일수, 회당 운동 시간을 설정하고 싶다. 그래야 AI가 나에게 맞는 계획을 만들 수 있으니까.**

수락 기준:
- [ ] 운동 목표 선택: 다이어트 / 근력 증가 / 체력 향상 / 종합 (단일 선택)
- [ ] 주 운동 가능 일수 선택: 3일 / 4일 / 5일 / 6일
- [ ] 회당 운동 시간 선택: 20분 / 30분 / 45분 / 60분
- [ ] 기본 정보 입력: 성별, 나이, 키, 체중 (선택 사항)
- [ ] 설정 완료 후 다음 단계(체력 테스트)로 자동 이동
- [ ] 나중에 프로필 설정에서 변경 가능

우선순위: **P0** | 포인트: **3 SP** | 의존성: Story 1
🏷️ 디자인 필요: 온보딩 플로우 화면 3-4장

---

### Story 3: 온보딩 체력 테스트
**신규 사용자로서, 간단한 체력 테스트로 내 수준을 파악하고 싶다. 그래야 AI가 내 수준에 맞는 운동을 추천해줄 수 있으니까.**

수락 기준:
- [ ] 3분 이내 완료 가능한 3-5개 동작 기반 자가 평가
- [ ] 각 동작에 대해 "쉬웠어요 / 적당해요 / 힘들었어요" 응답 수집
- [ ] 응답 기반으로 초급 / 중급 / 상급 자동 분류
- [ ] 결과 화면에서 체력 수준을 시각적으로 표시 (게이지/그래프)
- [ ] "테스트 건너뛰기" 옵션 제공 → 기본값 "중급" 적용
- [ ] 테스트 결과는 프로필에 저장, 이후 재측정 가능

우선순위: **P0** | 포인트: **5 SP** | 의존성: Story 2
🏷️ 디자인 필요: 체력 테스트 UI, 결과 화면

## Epic 2: AI 운동 계획 생성

### Story 4: AI 주간 운동 계획 자동 생성
**사용자로서, 내 목표/일정/체력에 맞는 주간 운동 계획을 자동으로 받고 싶다. 그래야 "오늘 뭘 해야 하지?" 고민 없이 바로 운동할 수 있으니까.**

수락 기준:
- [ ] 프로필(목표, 일정, 체력 수준) 기반 주간 운동 계획 자동 생성
- [ ] 각 운동일에 운동 목록, 세트 수, 반복 수, 예상 소요 시간 포함
- [ ] 운동 부위 밸런스 고려 (상체/하체/코어 분할)
- [ ] "계획 새로 만들기" 버튼으로 재생성 가능
- [ ] 계획 로딩 5초 이내
- [ ] 오프라인에서도 이전에 생성된 계획 조회 가능

우선순위: **P0** | 포인트: **13 SP** | 의존성: Story 3
🔧 기술 스파이크 필요: LLM 기반 운동 계획 생성 프롬프트 설계 및 비용 측정

---

### Story 5: 오늘의 운동 상세 표시
**사용자로서, 오늘 할 운동 목록을 세부 정보와 함께 보고 싶다. 그래야 헬스장에 가기 전에 오늘 운동을 미리 파악할 수 있으니까.**

수락 기준:
- [ ] 홈 화면에서 "오늘의 운동" 카드 표시 (운동 수, 예상 시간)
- [ ] 운동 목록: 이름, 세트×반복, 무게(해당 시), 휴식 시간
- [ ] 각 운동별 GIF 또는 짧은 영상 가이드 (3-5초)
- [ ] 운동 순서 확인 가능, 탭하면 상세 설명
- [ ] 개별 운동 "건너뛰기" 또는 "대체 운동 요청" 가능

우선순위: **P0** | 포인트: **5 SP** | 의존성: Story 4
🏷️ 디자인 필요: 오늘의 운동 화면, 운동 카드 컴포넌트

---

### Story 6: 적응형 난이도 자동 조정
**사용자로서, 내 운동 피드백과 기록에 따라 다음 운동 강도가 자동 조정되길 원한다. 그래야 항상 내 수준에 맞는 도전적인 운동을 할 수 있으니까.**

수락 기준:
- [ ] 운동 완료 후 "너무 쉬웠어요 / 적당해요 / 너무 힘들었어요" 피드백 수집
- [ ] 피드백 기반 다음 운동 강도(무게, 반복 수, 세트 수) 자동 조정
- [ ] 2회 연속 "쉬웠어요" → 난이도 상향, 2회 연속 "힘들었어요" → 난이도 하향
- [ ] 운동 미완료(건너뛰기) 빈도가 높으면 주간 계획 자동 경량화
- [ ] 조정 내역을 "AI 코치 노트"로 사용자에게 투명하게 표시

우선순위: **P0** | 포인트: **8 SP** | 의존성: Story 4, Story 8

## Epic 3: 운동 수행 가이드

### Story 7: 운동 세션 실행 (타이머 & 진행)
**사용자로서, 운동 세션을 시작하면 세트/반복/휴식을 단계별로 안내받고 싶다. 그래야 운동에만 집중할 수 있으니까.**

수락 기준:
- [ ] "운동 시작" 버튼 → 첫 번째 운동부터 순차 안내
- [ ] 현재 운동: 이름, 세트 번호, 반복 수, 운동 GIF 표시
- [ ] "세트 완료" 버튼 → 휴식 타이머 자동 시작
- [ ] 휴식 타이머: 남은 시간 표시, 조기 종료 가능
- [ ] 운동 건너뛰기, 이전 운동으로 돌아가기 가능
- [ ] 세션 도중 앱 종료 시 → 복귀하면 마지막 위치에서 이어하기
- [ ] 전체 운동 완료 시 "운동 완료!" 축하 화면

우선순위: **P0** | 포인트: **8 SP** | 의존성: Story 5
🏷️ 디자인 필요: 운동 세션 화면, 타이머 UI, 완료 화면

---

### Story 8: 운동 기록 자동 저장
**사용자로서, 운동 세션이 끝나면 기록이 자동 저장되길 원한다. 그래야 수동 기록 없이 내 운동 이력을 관리할 수 있으니까.**

수락 기준:
- [ ] 운동 완료 시 날짜, 운동 목록, 세트/반복/무게, 총 시간 자동 저장
- [ ] 운동 중단 시에도 완료한 세트까지 기록 저장
- [ ] 사용자가 무게를 수동 입력/수정 가능
- [ ] 기록은 서버와 동기화, 오프라인 시 로컬 저장 후 동기화

우선순위: **P0** | 포인트: **5 SP** | 의존성: Story 7

---

## Epic 4: 진행 상황 & 동기부여

### Story 9: 진행 상황 대시보드
**사용자로서, 내 운동 기록과 성장 추이를 한눈에 보고 싶다. 그래야 동기부여가 되고, 내 목표에 얼마나 가까워졌는지 알 수 있으니까.**

수락 기준:
- [ ] 주간/월간 운동 완료 캘린더 (히트맵 스타일)
- [ ] 운동 볼륨(총 무게×반복) 주간 추이 그래프
- [ ] 이번 주 완료 운동 수 / 목표 운동 수 진행률 바
- [ ] 연속 운동일(스트릭) 카운터 메인 표시
- [ ] 최근 4주간 핵심 운동별 무게/반복 변화 표시

우선순위: **P0** | 포인트: **8 SP** | 의존성: Story 8
🏷️ 디자인 필요: 대시보드 화면, 그래프/차트 컴포넌트

---

### Story 10: 스트릭 & 배지 시스템
**사용자로서, 연속 운동일과 달성 배지로 동기부여를 받고 싶다. 그래야 "오늘도 해야지"라는 의지를 유지할 수 있으니까.**

수락 기준:
- [ ] 연속 운동일 스트릭 카운터 (홈 화면 상단 고정)
- [ ] 운동 완료 시 격려 메시지 (5개 이상 변형으로 반복감 감소)
- [ ] 마일스톤 배지: 3일 / 7일 / 14일 / 30일 / 60일 / 100일 스트릭
- [ ] 주간 목표 100% 달성 시 "주간 챔피언" 배지
- [ ] 배지 모아보기 화면 (획득한 배지 + 미획득 배지 잠금 표시)
- [ ] 배지 획득 시 축하 애니메이션 + 사운드

우선순위: **P0** | 포인트: **5 SP** | 의존성: Story 8
🏷️ 디자인 필요: 배지 아이콘 세트, 축하 애니메이션

---

### Story 11: 운동 완료 후 피드백 수집
**사용자로서, 운동이 끝나면 간단하게 오늘 운동에 대한 느낌을 남기고 싶다. 그래야 AI가 다음 운동을 더 잘 조정해줄 수 있으니까.**

수락 기준:
- [ ] 운동 완료 직후 1-tap 피드백: 😓 힘들었어요 / 😊 적당해요 / 💪 쉬웠어요
- [ ] 선택적 텍스트 메모 (자유 입력, 최대 100자)
- [ ] 피드백 건너뛰기 가능 (강제하지 않음)
- [ ] 피드백 데이터는 Story 6 (적응형 난이도 조정)에 연동

우선순위: **P0** | 포인트: **2 SP** | 의존성: Story 7

## Epic 5: 카메라 자세 교정 (P1)

### Story 12: 카메라 자세 인식 기본 설정
**사용자로서, 운동 중 스마트폰 카메라를 켜서 내 자세를 인식시키고 싶다. 그래야 실시간으로 자세 피드백을 받을 수 있으니까.**

수락 기준:
- [ ] 운동 세션 화면에서 "카메라 켜기" 버튼 제공
- [ ] 카메라 권한 요청 및 가이드 메시지 ("1-2m 거리에 스마트폰을 세워주세요")
- [ ] 카메라 프리뷰에 관절 포인트 오버레이 표시 (MediaPipe Pose)
- [ ] "인식 준비 완료" 상태 표시 (전신이 프레임에 들어오면)
- [ ] 카메라 없이도 운동 세션 정상 진행 가능 (선택적 기능)

우선순위: **P1** | 포인트: **8 SP** | 의존성: Story 7
🔧 기술 스파이크 필요: MediaPipe Flutter 플러그인 호환성, 저사양 기기 성능

---

### Story 13: 실시간 자세 피드백 (5개 운동)
**사용자로서, 스쿼트/플랭크/런지/팔굽혀펴기/데드리프트 시 실시간으로 자세 교정 피드백을 받고 싶다. 그래야 부상 없이 올바른 자세로 운동할 수 있으니까.**

수락 기준:
- [ ] 5개 운동별 자세 판정 기준 구현 (각도, 정렬 기반)
- [ ] 올바른 자세: "좋은 자세입니다!" 초록색 표시
- [ ] 자세 오류: 문제 부위 빨간색 하이라이트 + 교정 텍스트 (예: "무릎이 발끝을 넘어갔어요")
- [ ] 자세 정확도 80% 이상 (PoC E2 기준)
- [ ] 세트 완료 후 자세 점수 요약 (예: "이번 세트 자세 점수: 85점")

우선순위: **P1** | 포인트: **13 SP** | 의존성: Story 12

---

## Epic 6: 음성 가이드 & 복귀 시스템 (P1)

### Story 14: 운동 중 음성 안내
**사용자로서, 운동 중 음성으로 세트 시작/종료, 휴식 카운트다운을 안내받고 싶다. 그래야 화면을 보지 않고도 운동에 집중할 수 있으니까.**

수락 기준:
- [ ] 세트 시작: "n세트 시작합니다" 음성
- [ ] 세트 종료: "좋아요! 휴식하세요" 음성
- [ ] 휴식 카운트다운: "10초... 5초... 시작!" 음성
- [ ] 음성 ON/OFF 토글 (설정 화면 + 운동 세션 중)
- [ ] 한국어 TTS, 자연스러운 톤

우선순위: **P1** | 포인트: **3 SP** | 의존성: Story 7

---

### Story 15: 복귀 사용자 격려 시스템
**3일 이상 운동을 쉰 사용자로서, 다시 시작할 수 있도록 격려와 조정된 계획을 받고 싶다. 그래야 "이미 망했다"는 느낌 없이 다시 시작할 수 있으니까.**

수락 기준:
- [ ] 3일+ 미운동 후 앱 실행 시 "괜찮아요! 오늘부터 다시 시작해요" 복귀 메시지
- [ ] 복귀 시 운동 계획 자동 재조정 (강도 20% 하향)
- [ ] 스트릭 리셋 대신 "복귀 스트릭" 별도 카운팅 (긍정 프레이밍)
- [ ] 푸시 알림: 1일차 "오늘 짧은 운동 어때요?", 3일차 "보고 싶었어요!", 7일차 "10분이면 충분해요" (1일 1회 이하)
- [ ] 푸시 알림 설정에서 비활성화 가능

우선순위: **P1** | 포인트: **5 SP** | 의존성: Story 4, Story 10

## Epic 7: 구독 & 결제 (P1)

### Story 16: 무료 체험 & 페이월
**사용자로서, 14일 무료 체험 후 유료 전환 여부를 결정하고 싶다. 그래야 충분히 경험한 뒤 결제할 수 있으니까.**

수락 기준:
- [ ] 신규 가입 시 14일 무료 체험 자동 시작

## 기술 노트 (Cross-cutting Concerns)

| 항목 | 설명 |
|------|------|
| **프레임워크** | Flutter (iOS/Android 동시 지원) |
| **백엔드** | Firebase 또는 자체 서버 (사용자 프로필, 운동 기록, AI 계획) |

| **결제** | RevenueCat 또는 네이티브 IAP 연동 |
| **푸시 알림** | Firebase Cloud Messaging |
| **분석** | Firebase Analytics + 커스텀 이벤트 (리텐션, 퍼널, 기능 사용률) |
| **오프라인** | 로컬 DB (Hive/SQLite)로 운동 계획/기록 캐싱 |
