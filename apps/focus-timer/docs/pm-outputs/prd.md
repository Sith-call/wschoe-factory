# PRD: Focus Timer (집중 타이머)

**버전**: v1.0 MVP
**작성일**: 2026-03-20
**상태**: Draft

---

## 1. 문제 정의

직장인들은 하루 8시간 이상 근무하지만, 실제 깊은 집중 시간은 평균 2-3시간에 불과하다.
메신저 알림, 회의, 멀티태스킹으로 인해 집중이 끊기고, 본인이 얼마나 집중했는지 체감하기 어렵다.
집중/휴식 리듬을 구조화하고, 에너지 상태를 함께 추적하면 업무 효율을 높일 수 있다.

## 2. 타겟 유저

| 항목 | 내용 |
|------|------|
| 연령 | 25-35세 직장인 |
| 니즈 | 업무 집중력 향상, 번아웃 방지 |
| 환경 | 데스크톱/모바일 브라우저 |
| 핵심 동기 | "오늘 얼마나 집중했는지 숫자로 보고 싶다" |

## 3. 핵심 기능 (MVP)

### 3.1 뽀모도로 타이머
- 25분 집중 + 5분 휴식 사이클
- 시작/일시정지/리셋 컨트롤
- 타이머 종료 시 알림음 + 화면 전환

### 3.2 세션 카운터
- 오늘 완료한 집중 세션 수 표시
- 타이머 화면에 항상 노출

### 3.3 에너지 체크
- 각 집중 세션 완료 후 에너지 레벨(1-5) 입력
- 간단한 이모지 또는 바 UI로 빠르게 선택

### 3.4 오늘 통계
- 총 집중 시간 (분)
- 완료 세션 수
- 평균 에너지 레벨

### 3.5 주간 통계
- 최근 7일 집중 시간 바 차트
- 에너지 추이 라인 표시

### 3.6 데모 모드
- 첫 실행 시 샘플 데이터(7일치)로 통계 화면 체험
- "내 데이터로 시작하기" 버튼으로 초기화 후 실사용 전환

## 4. 유저 스토리

### US-1: 집중 타이머 시작
> 직장인으로서, 25분 집중 타이머를 시작할 수 있다.
> 그래야 정해진 시간 동안 한 가지 업무에 몰입할 수 있다.

**인수 조건**:
- 시작 버튼 탭 시 25:00부터 카운트다운
- 진행 중 일시정지/재개 가능
- 리셋 시 25:00으로 복귀

### US-2: 휴식 타이머 자동 전환
> 직장인으로서, 집중 세션이 끝나면 자동으로 5분 휴식 타이머가 시작된다.
> 그래야 적절한 휴식 리듬을 유지할 수 있다.

**인수 조건**:
- 집중 25분 완료 시 알림음 재생
- 에너지 체크 입력 후 5분 휴식 타이머 자동 시작
- 휴식 완료 시 다시 집중 모드로 전환

### US-3: 에너지 레벨 기록
> 직장인으로서, 집중 세션 완료 후 에너지 레벨(1-5)을 기록할 수 있다.
> 그래야 하루 중 에너지 패턴을 파악할 수 있다.

**인수 조건**:
- 집중 타이머 완료 시 에너지 입력 모달 표시
- 1-5 레벨 중 하나를 탭으로 선택
- 선택 후 자동으로 휴식 타이머 전환

### US-4: 오늘 집중 현황 확인
> 직장인으로서, 오늘의 총 집중 시간과 세션 수를 한눈에 볼 수 있다.
> 그래야 하루 업무 집중도를 체감할 수 있다.

**인수 조건**:
- 오늘 통계: 총 집중 시간, 세션 수, 평균 에너지
- 타이머 화면 하단 또는 별도 탭에서 확인

### US-5: 주간 추이 확인
> 직장인으로서, 최근 7일간 집중 시간과 에너지 추이를 차트로 볼 수 있다.
> 그래야 일주일 단위로 집중 패턴을 개선할 수 있다.

**인수 조건**:
- 7일간 일별 집중 시간 바 차트
- 에너지 평균 추이 라인
- 데이터 없는 날은 0으로 표시

### US-6: 데모 모드 체험
> 신규 유저로서, 앱을 처음 열면 샘플 데이터로 모든 기능을 체험할 수 있다.
> 그래야 가입/사용 전에 앱의 가치를 바로 확인할 수 있다.

**인수 조건**:
- 첫 실행 시 7일치 샘플 세션 데이터 로드
- 통계 화면에서 차트/수치 확인 가능
- "내 데이터로 시작하기" 버튼으로 샘플 데이터 삭제 후 실사용 시작

### US-7: 세션 데이터 로컬 저장
> 직장인으로서, 기록한 세션 데이터가 브라우저를 닫아도 유지된다.
> 그래야 매일 이어서 사용할 수 있다.

**인수 조건**:
- 모든 세션 데이터는 localStorage에 저장
- 브라우저 재실행 시 데이터 복원

## 5. 성공 지표

| 지표 | 목표 |
|------|------|
| 일 평균 세션 수 | 4회 이상 |
| 에너지 체크 입력률 | 80% 이상 (세션 완료 대비) |
| 주간 재방문율 | 3일/주 이상 |
| 데모→실사용 전환율 | 60% 이상 |

## 6. 기술 스택

- **프론트엔드**: React + TypeScript + Vite
- **스타일링**: Tailwind CSS (CDN)
- **저장소**: localStorage (서버 없음)
- **배포**: 정적 호스팅

## 7. Out of Scope (MVP 제외)

- 계정/로그인 시스템
- 서버 동기화 및 클라우드 백업
- 팀/그룹 기능
- 커스텀 타이머 시간 설정
- 푸시 알림 (브라우저 Notification API)
- 월간/연간 통계
