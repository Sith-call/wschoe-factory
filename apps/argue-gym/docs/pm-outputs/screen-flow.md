# Screen Flow — Argue Gym

## 화면 목록

1. **S1. 랜딩** — 앱 가치 한 줄 + "샘플로 체험하기" Primary + "내 주장으로 시작" Secondary + 하단 "지난 기록" 텍스트 링크
2. **S2. 주장 입력** — 주장 textarea (120자) + 카테고리 칩 5개 + "반론 받기" Primary 버튼
3. **S3. 세션 (반론 + 재반박)** — 상단 프로그레스(카드 1/3), 반론 카드 본문 + 근거 유형 배지, 하단 재반박 textarea + 글자수, "다음 카드" + "스킵" 버튼
4. **S4. 세션 리포트** — 유형 분포 바, 숫자 지표 3개(글자/완주/회피), 규칙 기반 코멘트, "다시 도전" + "새 주장" + "기록 저장"
5. **S5. 로그 아카이브** — 날짜별 세션 리스트, 탭 시 S4로 이동, 빈 상태 CTA "첫 세션 시작하기"

## 전환

```
S1 랜딩
  ├─[샘플로 체험하기]──▶ S3 (주장 프리필된 샘플 세션)
  ├─[내 주장으로 시작]──▶ S2
  └─[지난 기록]─────────▶ S5

S2 주장 입력
  └─[반론 받기]─────────▶ S3

S3 세션 (카드 1→2→3 선형)
  ├─[다음 카드]×3 완료──▶ S4
  ├─[스킵] (회피 카운트)─▶ 다음 카드
  └─[뒤로]──────────────▶ S2 (경고 없이, 세션은 자동 임시저장)

S4 리포트
  ├─[다시 도전]─────────▶ S3 (동일 주장, 새 카드 3장)
  ├─[새 주장]───────────▶ S2
  └─[기록 저장 → 닫기]──▶ S1

S5 로그
  ├─[세션 탭]───────────▶ S4 (읽기 전용 모드)
  └─[뒤로]──────────────▶ S1
```

## 네비게이션 원칙
- 선형 플로우 앱이므로 **바텀 탭바 없음** (DESIGN_RULES R5 준수)
- 모든 화면에 상단 좌측 뒤로가기 아이콘, 우측 컨텍스트 액션만
- S3는 상단에 "1/3, 2/3, 3/3" 프로그레스 표시 (스피너 금지)

## 빈 상태
- S5 로그가 비었을 때: "아직 훈련 기록이 없어요. 첫 주장을 써볼까요?" + CTA 버튼
