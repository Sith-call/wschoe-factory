# Personas — Argue Gym (Ralph Loop용)

각 페르소나는 서로 다른 니즈·불만·평가 축을 갖는다. Ralph 루프에서 병렬 피드백 시 관점이 겹치지 않도록 설계.

---

## P1. 송지우 (29, 3년차 마케터) — "회의 방어형"

### 배경
- 주 2회 마케팅 전략 회의에서 자기 제안을 발표하고 공격받음
- 말문이 막혀 밤에 "아 그때 이렇게 말할걸" 후회하는 게 루틴

### 니즈
- 실제 업무 맥락에 가까운 반론 (예산·ROI·리소스 관점)
- 짧게 훈련 가능 (출근 전 10분 이내)
- 내 약점 패턴을 정량으로 확인

### 주요 불만 포인트 (이 앱이 실패할 경우)
- "반론이 너무 학술적/뜬구름이면 업무에 안 쓰게 됨"
- "재반박 쓰는 데 30분 걸리면 못 해"
- "리포트가 그냥 글자수만 세면 의미 없음"

### 평가 관점
- **실용성**: 업무 맥락 반영도, 속도, 재반박 작성의 부담
- 만족도 80% 기준: 10분 안에 완주 가능 + 리포트 코멘트가 행동 변화를 유도

---

## P2. 이태훈 (22, 로스쿨 준비생) — "논리 단련형"

### 배경
- LEET/면접 대비로 매일 주장-반론 연습
- 기존 교재는 정답이 주어져 있어 "내 논리를 스스로 점검"하지 못함

### 니즈
- 구조화된 근거 유형(데이터·원칙·사례·감정) 태그
- 재반박 글자 수 제한 없음, 길게 쓸 수 있어야 함
- 세션을 다시 열어 복기 가능

### 주요 불만 포인트
- "반론 카드가 3장으로 너무 적으면 실전 대비 안 됨 → 재도전 기능 필수"
- "근거 유형 라벨이 없으면 내 논리의 약점을 분류 못 함"
- "로그가 없으면 성장 궤적을 못 본다"

### 평가 관점
- **구조·깊이**: 근거 분류 체계, 재도전, 로그 복기, 글자수 자유도
- 만족도 80% 기준: 같은 주장 3번 재도전해도 다른 반론, 로그에서 과거 세션 재열람 자연스러움

---

## P3. 박수연 (36, 프리랜서 에세이 작가) — "글쓰기 워밍업형"

### 배경
- 에세이 원고 쓰기 전 "내 주장에 딴지 걸어보기" 루틴이 필요
- 감각적 글을 쓰기 때문에 논리 강도보다 **다각도 관점 확보**가 중요

### 니즈
- 분위기 있는 UI (라이트 모드, 차분한 타이포)
- 반론 카드가 한 번에 정리되어 읽히고, 재반박은 메모처럼 가볍게
- 글을 "저장"하는 행위 자체에 만족감

### 주요 불만 포인트
- "UI가 게임처럼 가볍거나 촌스러우면 안 열게 됨"
- "강압적으로 '완주하세요'라고 밀어붙이면 거부감"
- "저장 버튼 눌렀을 때 아무 피드백 없으면 불안"

### 평가 관점
- **톤·감성**: 타이포그래피, 여백, 저장 피드백, 선택권 존중
- 만족도 80% 기준: 화면 감성 점수 8+, 스킵/재도전에 압박 없음, 기록 저장 시 토스트/체크 애니메이션

---

## 페르소나 관점 충돌 요약 (디자이너가 알아야 할 것)
- P1은 속도·실용, P2는 구조·깊이, P3는 감성·선택권을 원함
- 셋을 모두 만족시키려면: 라이트 모드 + 차분한 타이포 + 빠른 세션 플로우 + 명확한 근거 유형 태그 + 선택적 재도전/스킵
- **라이트 모드로 가는 것이 핵심** (기존 앱 다수가 다크라 차별화 + P3 니즈 충족)
