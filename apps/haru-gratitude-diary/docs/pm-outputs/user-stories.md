# haru-gratitude-diary User Stories

**앱:** 하루 감사 일기
**Version:** 1.0 (MVP)
**Date:** 2026-04-10
**PRD 참조:** prd.md

---

## P0 -- MVP 필수 스토리

---

### US-01: 오늘의 감사 3가지 기록
**Priority:** P0
**As a** 퇴근 후 지친 직장인 (김지현)
**I want to** 오늘 감사한 일 3가지를 텍스트로 입력하고 저장하고 싶다
**So that** 30초 안에 하루를 마무리하고 차분하게 잠들 수 있다

**Acceptance Criteria:**
- [ ] Entry 화면에 3개의 텍스트 입력 필드가 표시된다
- [ ] 각 필드에 플레이스홀더 힌트가 표시된다 (예: "오늘 감사한 일을 적어보세요")
- [ ] 3개 필드 모두 입력 후 저장 버튼을 누르면 데이터가 저장된다
- [ ] 1~2개만 입력해도 저장이 가능하다 (빈 항목은 빈 문자열로 저장하지 않고 무시)
- [ ] 저장 완료 후 홈 화면으로 자동 이동한다
- [ ] 오늘 날짜가 화면 상단에 표시된다
- [ ] 앱을 열고 3줄 쓰고 저장까지 30초 이내에 완료할 수 있는 UI 구조이다

---

### US-02: 감사 기록 localStorage 저장
**Priority:** P0
**As a** 프라이버시를 중시하는 사용자
**I want to** 내 감사 기록이 브라우저 localStorage에 안전하게 저장되길 원한다
**So that** 가입/로그인 없이도 기록이 유지되고 서버에 데이터가 전송되지 않는다

**Acceptance Criteria:**
- [ ] 감사 기록 저장 시 localStorage에 GratitudeEntry 형태로 저장된다
- [ ] 날짜(YYYY-MM-DD)를 key로 사용하여 하루에 하나의 엔트리만 존재한다
- [ ] 앱을 닫았다 다시 열어도 이전 기록이 유지된다
- [ ] 서버 API 호출이 일절 없다
- [ ] createdAt, updatedAt 타임스탬프가 자동 기록된다

---

### US-03: 홈 화면에서 오늘 기록 상태 확인
**Priority:** P0
**As a** 매일 감사를 기록하는 사용자
**I want to** 홈 화면에서 오늘 감사를 기록했는지 바로 확인하고 싶다
**So that** 오늘 할 일(감사 기록)의 상태를 한눈에 파악하고 다음 행동을 결정할 수 있다

**Acceptance Criteria:**
- [ ] 홈 화면에 오늘 날짜가 표시된다
- [ ] 오늘 기록이 없으면 CTA 버튼 "오늘의 감사 쓰기"가 표시된다
- [ ] 오늘 기록이 있으면 오늘 쓴 감사 3가지 요약이 표시된다
- [ ] 이번 주 기록 현황 (7일 중 N일 기록)이 표시된다
- [ ] 하단 내비게이션에 홈/캘린더 탭이 존재한다

---

### US-04: 캘린더에서 월별 기록 조회
**Priority:** P0
**As a** 감사 습관을 추적하고 싶은 사용자
**I want to** 캘린더에서 이번 달에 며칠이나 감사를 기록했는지 보고 싶다
**So that** 기록 빈도를 확인하고 동기를 유지할 수 있다

**Acceptance Criteria:**
- [ ] 월별 캘린더 그리드가 표시된다
- [ ] 기록한 날은 색상 도트로 시각적 표시된다
- [ ] 미기록 날은 중립적 디자인으로 죄책감을 주지 않는다
- [ ] 이전/다음 월로 이동할 수 있다
- [ ] 이번 달 기록 통계 "N일 기록"이 표시된다
- [ ] 기록이 있는 날짜를 탭하면 Detail 화면으로 이동한다

---

### US-05: 특정 날짜 상세 보기
**Priority:** P0
**As a** 과거 기록을 돌아보고 싶은 사용자
**I want to** 특정 날짜에 어떤 감사를 기록했는지 확인하고 싶다
**So that** 과거의 감사를 다시 읽으며 긍정적 감정을 되새길 수 있다

**Acceptance Criteria:**
- [ ] 해당 날짜가 화면 상단에 표시된다
- [ ] 감사 항목 목록(최대 3개)이 표시된다
- [ ] 수정 버튼이 존재한다
- [ ] 삭제 버튼이 존재한다
- [ ] 뒤로가기 버튼으로 이전 화면(캘린더 또는 홈)으로 돌아갈 수 있다

---

### US-06: 주간 회고 보기
**Priority:** P0
**As a** 자기 발견을 원하는 사용자 (김지현)
**I want to** 일주일 치 감사 기록을 한 화면에 모아서 패턴을 보고 싶다
**So that** "나는 뭘 소중히 여기는 사람인지" 인사이트를 얻을 수 있다

**Acceptance Criteria:**
- [ ] 해당 주의 날짜 범위가 표시된다 (예: 4/1 - 4/7)
- [ ] 일주일간 기록한 모든 감사 항목이 날짜별로 그룹화되어 표시된다
- [ ] 반복 키워드가 하이라이트된다 (빈도 기반)
- [ ] "이번 주 인사이트" 한 줄 요약이 표시된다
- [ ] 기록한 날수/7일 통계가 표시된다
- [ ] 홈으로 돌아가기 버튼이 존재한다
- [ ] 홈 화면에서 주간 회고 진입 버튼이 존재한다

---

### US-07: 감사 기록 수정
**Priority:** P0
**As a** 오타를 수정하거나 내용을 보완하고 싶은 사용자
**I want to** 이전에 기록한 감사 항목을 수정하고 싶다
**So that** 기록의 정확성과 완성도를 높일 수 있다

**Acceptance Criteria:**
- [ ] Detail 화면에서 수정 버튼을 누르면 인라인 편집 모드로 전환된다
- [ ] 기존 텍스트가 입력 필드에 채워진 상태로 편집 가능하다
- [ ] 수정 후 저장하면 updatedAt 타임스탬프가 갱신된다
- [ ] 수정 취소 시 원래 내용으로 복원된다
- [ ] 수정 완료 후 Detail 화면으로 돌아간다

---

### US-08: 감사 기록 삭제
**Priority:** P0
**As a** 특정 날짜의 기록을 지우고 싶은 사용자
**I want to** 감사 기록을 삭제할 수 있다
**So that** 잘못 기록하거나 삭제하고 싶은 기록을 제거할 수 있다

**Acceptance Criteria:**
- [ ] Detail 화면에서 삭제 버튼을 누르면 확인 다이얼로그가 표시된다
- [ ] "정말 삭제하시겠습니까?" 등의 확인 메시지가 포함된다
- [ ] 확인 시 해당 날짜의 기록이 localStorage에서 삭제된다
- [ ] 삭제 후 캘린더 화면으로 이동한다
- [ ] 삭제된 날짜는 캘린더에서 미기록 상태로 변경된다

---

### US-09: 온보딩 (첫 사용)
**Priority:** P0
**As a** 앱을 처음 사용하는 사용자
**I want to** 앱의 핵심 가치와 사용법을 1-2 화면으로 빠르게 이해하고 싶다
**So that** 가입 절차 없이 바로 감사 기록을 시작할 수 있다

**Acceptance Criteria:**
- [ ] 첫 방문 시 온보딩 화면이 자동 표시된다
- [ ] 1-2 화면으로 앱의 핵심 가치를 안내한다 ("매일 감사 3가지, 주간 회고로 나를 발견")
- [ ] 온보딩 완료 버튼을 누르면 홈 화면 또는 Entry 화면으로 이동한다
- [ ] 가입/로그인 절차가 일절 없다
- [ ] 온보딩 완료 여부가 localStorage에 저장되어 재방문 시 표시되지 않는다
- [ ] 온보딩을 건너뛰기할 수 있다

---

### US-10: 빈 상태 CTA (데이터 없을 때 행동 유도)
**Priority:** P0
**As a** 기록이 아직 없는 사용자
**I want to** 빈 화면에서 무엇을 해야 할지 안내받고 싶다
**So that** 빈 페이지 불안 없이 자연스럽게 첫 기록을 시작할 수 있다

**Acceptance Criteria:**
- [ ] 홈 화면에 기록이 없으면 "오늘의 감사 쓰기" CTA 버튼이 명확히 표시된다
- [ ] 캘린더 화면에 기록이 없으면 빈 상태 안내 메시지가 표시된다 ("아직 기록이 없어요. 첫 감사를 시작해보세요")
- [ ] 주간 회고 화면에 데이터가 부족하면 안내 메시지가 표시된다
- [ ] 모든 빈 상태에서 Entry 화면으로 이동할 수 있는 경로가 존재한다
- [ ] 빈 상태에서 에러가 발생하지 않는다

---

### US-11: 데모 모드 (mock 데이터로 체험)
**Priority:** P0
**As a** 앱을 처음 접하는 사용자
**I want to** 샘플 데이터로 주간 회고 등 전체 기능을 체험해보고 싶다
**So that** 직접 데이터를 쌓지 않아도 앱의 가치를 즉시 느낄 수 있다

**Acceptance Criteria:**
- [ ] 온보딩 또는 홈 화면에서 데모 모드 활성화 옵션이 존재한다
- [ ] 데모 모드 활성화 시 최근 7일간의 샘플 감사 기록이 생성된다
- [ ] 샘플 데이터로 캘린더, Detail, 주간 회고 화면을 모두 체험할 수 있다
- [ ] 데모 데이터는 한국어로 작성되어 있다 (예: "맛있는 커피 한 잔", "퇴근 후 산책", "동료의 격려")
- [ ] 데모 모드에서 직접 기록을 추가하거나 수정할 수 있다
- [ ] 데모 데이터를 초기화(삭제)할 수 있는 옵션이 있다

---

### US-12: 하단 내비게이션
**Priority:** P0
**As a** 앱 사용자
**I want to** 하단 내비게이션으로 홈과 캘린더를 쉽게 오갈 수 있다
**So that** 화면 간 이동이 직관적이고 빠르다

**Acceptance Criteria:**
- [ ] 하단에 고정된 내비게이션 바가 존재한다
- [ ] 홈 탭과 캘린더 탭이 포함되어 있다
- [ ] 현재 활성 탭이 시각적으로 구분된다
- [ ] 탭을 누르면 해당 화면으로 즉시 전환된다
- [ ] Entry, Detail, Weekly Reflection 화면에서는 하단 내비게이션 대신 뒤로가기 버튼이 표시된다

## P1 -- MVP 이후 우선 추가

---

### US-13: 힘든 날 모드
**Priority:** P1
**As a** 감사할 것이 떠오르지 않는 날의 사용자 (이서윤)
**I want to** 3가지가 아닌 1가지만 기록하는 모드를 선택하고 싶다
**So that** 힘든 날에도 억지로 3가지를 채우지 않고 한 가지만으로 기록을 이어갈 수 있다

**Acceptance Criteria:**
- [ ] Entry 화면에서 "힘든 날 모드" 토글 또는 버튼이 존재한다
- [ ] 활성화 시 입력 필드가 1개로 변경되고 안내 문구가 바뀐다 ("오늘은 괜찮았던 것 하나만")
- [ ] 1가지만 기록해도 정상적으로 저장된다
- [ ] 캘린더/Detail에서 힘든 날 기록도 동일하게 표시된다

---

### US-14: 부드러운 프롬프트 (선택적 질문 제안)
**Priority:** P1
**As a** 빈 칸 앞에서 막막한 사용자 (이서윤)
**I want to** 감사할 일을 떠올리게 도와주는 질문 힌트를 보고 싶다
**So that** 빈 페이지 불안 없이 자연스럽게 감사를 적을 수 있다

**Acceptance Criteria:**
- [ ] Entry 화면의 각 입력 필드에 매일 다른 플레이스홀더 질문이 표시된다
- [ ] 예시: "오늘 누군가에게 받은 작은 친절은?", "오늘 몸이 편했던 순간은?"
- [ ] 최소 14가지 이상의 프롬프트 풀이 존재한다
- [ ] 프롬프트는 선택적이며 무시하고 자유롭게 작성 가능하다

---

### US-15: 리마인더 알림
**Priority:** P1
**As a** 저녁에 감사 기록을 까먹는 사용자 (박은주)
**I want to** 설정한 시간에 브라우저 알림을 받고 싶다
**So that** 잠들기 전 감사 기록을 잊지 않을 수 있다

**Acceptance Criteria:**
- [ ] 설정 화면에서 리마인더 시간을 설정할 수 있다
- [ ] 브라우저 Notification API를 활용한 알림이 발송된다
- [ ] 알림을 탭하면 Entry 화면으로 이동한다
- [ ] 알림은 하루 1회만 발송된다
- [ ] 알림 해제가 가능하다

---

### US-16: 데이터 내보내기
**Priority:** P1
**As a** 데이터 유실이 걱정되는 사용자
**I want to** 전체 감사 기록을 JSON 또는 CSV 파일로 다운로드하고 싶다
**So that** localStorage 초기화나 브라우저 변경 시에도 기록을 보존할 수 있다

**Acceptance Criteria:**
- [ ] 설정 또는 프로필 영역에서 "데이터 내보내기" 버튼이 존재한다
- [ ] JSON 또는 CSV 형태로 전체 기록이 다운로드된다
- [ ] 다운로드 파일에 모든 날짜의 감사 항목이 포함된다
- [ ] 파일명에 내보내기 날짜가 포함된다 (예: haru-backup-2026-04-10.json)

---

### US-17: 스트릭 시각화
**Priority:** P1
**As a** 습관 형성 동기가 필요한 사용자
**I want to** 이번 달 기록 일수를 부드럽게 확인하고 싶다
**So that** 연속 기록에 대한 성취감을 느끼되 깨져도 죄책감이 없다

**Acceptance Criteria:**
- [ ] 홈 화면에 "이번 달 N일 기록" 형태로 표시된다
- [ ] 연속 기록 일수가 아닌 총 기록 일수를 강조한다
- [ ] 스트릭이 깨져도 부정적 표현이 없다
- [ ] 캘린더 화면과 연동하여 기록 현황이 일치한다

---

## P2 -- 향후 고려

---

### US-18: 오늘 기록 재진입 시 기존 데이터 표시
**Priority:** P2
**As a** 오늘 이미 기록한 사용자
**I want to** 다시 Entry 화면에 진입했을 때 오늘 기록한 내용이 표시되길 원한다
**So that** 중복 기록 없이 수정만 할 수 있다

**Acceptance Criteria:**
- [ ] 오늘 기록이 있는 상태에서 Entry 화면 진입 시 기존 데이터가 채워져 있다
- [ ] 수정 후 저장하면 기존 기록이 업데이트된다 (새 엔트리가 생성되지 않음)
- [ ] updatedAt이 갱신된다

---

### US-19: 커스텀 테마
**Priority:** P2
**As a** 자신만의 분위기를 원하는 사용자
**I want to** 앱의 색상 테마를 변경하고 싶다
**So that** 계절이나 기분에 맞는 시각적 경험을 할 수 있다

**Acceptance Criteria:**
- [ ] 설정에서 2-3가지 테마 중 선택 가능하다
- [ ] 테마 변경 시 전체 앱 색상이 일관되게 변경된다
- [ ] 선택한 테마가 localStorage에 저장되어 유지된다

---

### US-20: 감사 카테고리 자동 태깅
**Priority:** P2
**As a** 감사 패턴을 더 깊이 이해하고 싶은 사용자
**I want to** 내 기록이 "사람", "자연", "음식" 등으로 자동 분류되길 원한다
**So that** 주간/월간 회고에서 어떤 카테고리에 감사가 집중되는지 알 수 있다

**Acceptance Criteria:**
- [ ] 저장 시 키워드 기반으로 자동 카테고리가 태깅된다
