## 2. 스크린 상세

### S-00: Onboarding (온보딩)

**목적:** 첫 방문 사용자에게 앱의 핵심 가치("3줄 쓰고, 1주일 돌아보고, 나를 발견한다")를 전달하고, 가입 없이 바로 작성으로 안내.

**핵심 UI 요소:**
- 앱 로고 / 이름 ("하루 감사 일기")
- 핵심 가치 메시지 (1-2 슬라이드, 최대 2장)
  - 슬라이드 1: "매일 감사한 일 3가지, 30초면 충분해요"
  - 슬라이드 2: "일주일 뒤, 내가 뭘 소중히 여기는지 알게 돼요"
- "시작하기" CTA 버튼 (Primary)
- "건너뛰기" 텍스트 링크 (Secondary -- 상단 우측)
- 페이지 인디케이터 (도트 2개)

**사용자 액션:**
- 좌우 스와이프 또는 도트 탭으로 슬라이드 전환
- "시작하기" 탭 -> S-02 (Entry)로 이동
- "건너뛰기" 탭 -> S-01 (Home)로 이동

**상태 변형:**
- 단일 상태 (첫 방문 시에만 표시)

**전환:**
- -> S-02 (Entry): "시작하기" 탭
- -> S-01 (Home): "건너뛰기" 탭

---

### S-01: Home (홈)

**목적:** 앱의 메인 허브. 오늘의 기록 상태를 보여주고, 가장 중요한 액션(기록하기 / 주간 회고)으로 안내.

**핵심 UI 요소:**
- 상단 헤더: 오늘 날짜 (예: "4월 10일 목요일") + 설정 아이콘 (우측)
- 오늘의 감사 영역:
  - 미완료 시: "오늘의 감사 쓰기" CTA 버튼 (Primary, 큰 사이즈)
  - 완료 시: 오늘 쓴 3가지 요약 표시 (탭하면 S-04 Detail로 이동)
- 이번 주 기록 현황: 7칸 중 기록한 날 표시 (예: "이번 주 4/7일 기록")
- 주간 회고 카드:
  - 활성: "이번 주 회고 보기" 버튼 (7일 중 3일 이상 기록 시)
  - 비활성: "N일 더 기록하면 주간 회고를 볼 수 있어요" 안내
- 하단 내비게이션: 홈 (현재) / 캘린더

**사용자 액션:**
- "오늘의 감사 쓰기" 탭 -> S-02 (Entry)
- 오늘 기록 요약 탭 -> S-04 (Detail, 오늘 날짜)
- "주간 회고 보기" 탭 -> S-05 (Weekly Reflection)
- 캘린더 탭 탭 -> S-03 (Calendar)
- 설정 아이콘 탭 -> S-06 (Settings)

**상태 변형:**

| 상태 | 조건 | 표시 |
|------|------|------|
| 첫 방문 직후 (빈 상태) | 기록 0건 | CTA "첫 감사를 기록해보세요" + 주간 회고 비활성 ("7일 기록하면 회고가 열려요") |
| 오늘 미기록 | 과거 기록 있음, 오늘은 아직 | "오늘의 감사 쓰기" CTA |
| 오늘 기록 완료 | 오늘 기록 존재 | 오늘 쓴 3가지 요약 표시, CTA 숨김 |
| 주간 회고 가능 | 이번 주 3일+ 기록 | "주간 회고 보기" 버튼 활성 |

**전환:**
- -> S-02 (Entry): CTA 버튼 탭
- -> S-03 (Calendar): 하단 내비게이션 "캘린더" 탭
- -> S-04 (Detail): 오늘 기록 요약 탭
- -> S-05 (Weekly Reflection): 주간 회고 버튼 탭
- -> S-06 (Settings): 설정 아이콘 탭

---

### S-02: Entry (감사 기록)

**목적:** 오늘의 감사 3가지를 입력하는 핵심 화면. 30초 이내 완료가 목표. 다른 요소 일절 없음.

**핵심 UI 요소:**
- 상단: 뒤로가기 화살표 (좌측) + 오늘 날짜 (중앙)
- 3개의 텍스트 입력 필드:
  - 각 필드에 부드러운 플레이스홀더 (예: "오늘 고마웠던 사람은?", "작지만 좋았던 순간은?", "당연하지만 감사한 것은?")
  - 각 필드 좌측에 번호 표시 (1, 2, 3)
  - 단일 행 입력 (textarea 아님, 간결함 유지)
- "저장" 버튼 (Primary, 하단 고정)
  - 1개 이상 입력 시 활성화
  - 0개 입력 시 비활성화 (dimmed)

**사용자 액션:**
- 텍스트 필드 탭 -> 키보드 올라옴, 입력 시작
- Tab 또는 엔터로 다음 필드 이동 (빠른 연속 입력)
- "저장" 탭 -> 저장 완료 피드백(체크 애니메이션 + "저장했어요" 토스트) -> S-01 (Home)로 자동 이동
- 뒤로가기 탭 -> S-01 (Home)
  - 입력 내용 있으면 확인 다이얼로그: "작성 중인 내용이 있어요. 나갈까요?"

**상태 변형:**

| 상태 | 조건 | 표시 |
|------|------|------|
| 빈 상태 (신규 입력) | 오늘 기록 없음 | 3개 빈 필드 + 플레이스홀더 |
| 수정 모드 | 오늘 이미 기록함, 홈에서 재진입 | 기존 값 채워진 3개 필드 + "수정" 버튼 |
| 부분 입력 | 1-2개만 입력 | 저장 버튼 활성화, 빈 필드 유지 |

**전환:**
- -> S-01 (Home): 저장 완료 후 자동 이동 / 뒤로가기 탭

**설계 원칙:**
- 이 화면에는 내비게이션 바, 통계, 장식 요소가 없다. 오직 날짜 + 3개 필드 + 저장 버튼.
- 첫 필드에 자동 포커스 -> 앱 열자마자 바로 타이핑 시작 가능.
- 키보드가 올라와도 저장 버튼이 가려지지 않도록 레이아웃 처리.

---

### S-03: Calendar (캘린더)

**목적:** 월별 기록 현황을 시각적으로 확인. 과거 기록 탐색 및 소급 기록 유도.

**핵심 UI 요소:**
- 상단: "캘린더" 타이틀 + 이번 달 표시 (예: "2026년 4월")
- 월 이동: 좌/우 화살표로 이전/다음 달 이동
- 캘린더 그리드 (7열 x 5-6행):
  - 기록한 날: primary 색상 도트 표시
  - 미기록 날 (과거): 빈 상태, 탭 가능
  - 미기록 날 (미래): 비활성화 (탭 불가)
  - 오늘: 테두리 강조
- 이번 달 통계: "이번 달 N일 기록" (하단)
- 하단 내비게이션: 홈 / 캘린더 (현재)

**사용자 액션:**
- 기록된 날짜 탭 -> S-04 (Detail, 해당 날짜)
- 미기록 과거 날짜 탭 -> S-02 (Entry, 해당 날짜로 소급 기록)
- 좌/우 화살표 탭 -> 이전/다음 달로 이동
- 홈 탭 탭 -> S-01 (Home)

**상태 변형:**

| 상태 | 조건 | 표시 |
|------|------|------|
| 빈 상태 (기록 없음) | 이번 달 기록 0건 | 빈 캘린더 + "첫 감사를 기록하면 여기에 표시돼요" CTA |
| 부분 기록 | 일부 날만 기록 | 기록 날에 도트, 나머지 빈 상태 |
| 과거 달 (기록 있음) | 과거 월 탐색 | 기록 날에 도트 표시, 탭하면 Detail |
| 과거 달 (기록 없음) | 과거 월 탐색, 기록 없음 | 빈 캘린더 (CTA 없음 -- 과거 소급은 날짜 탭으로) |

**전환:**
- -> S-04 (Detail): 기록된 날짜 탭
- -> S-02 (Entry): 미기록 과거 날짜 탭 (소급 기록)
- -> S-01 (Home): 하단 내비게이션

---

### S-04: Detail (일별 상세)

**목적:** 특정 날짜의 감사 기록을 확인하고, 필요 시 수정 또는 삭제.

**핵심 UI 요소:**
- 상단: 뒤로가기 화살표 (좌측) + 해당 날짜 (중앙, 예: "4월 7일 월요일")
- 감사 3가지 목록:
  - 각 항목 좌측에 번호 (1, 2, 3)
  - 읽기 모드: 텍스트 표시
  - 편집 모드: 인라인 텍스트 필드로 전환
- 액션 버튼:
  - "수정" 텍스트 버튼 (우측 상단) -> 편집 모드 전환
  - 편집 모드 시 "저장" (Primary) + "취소" (Secondary) 버튼
- "삭제" 텍스트 버튼 (하단, 위험 색상 -- 빨간 텍스트)
  - 탭 시 확인 다이얼로그: "이 날의 기록을 삭제할까요? 되돌릴 수 없어요."

**사용자 액션:**
- "수정" 탭 -> 인라인 편집 모드 전환
- "저장" 탭 -> 수정 내용 저장 + 토스트 "수정했어요" + 읽기 모드로 복귀
- "취소" 탭 -> 변경 사항 버리고 읽기 모드로 복귀
- "삭제" 탭 -> 확인 다이얼로그 -> 삭제 후 이전 화면으로 이동
- 뒤로가기 -> 이전 화면 (S-01 Home 또는 S-03 Calendar)

**상태 변형:**

| 상태 | 조건 | 표시 |
|------|------|------|
| 읽기 모드 | 기본 상태 | 감사 목록 + 수정/삭제 버튼 |
| 편집 모드 | "수정" 탭 후 | 인라인 텍스트 필드 + 저장/취소 버튼 |
| 오늘 상세 | 오늘 날짜의 기록 | 동일하나 "수정" 대신 Entry 화면으로 리다이렉트 가능 |

**전환:**
- -> S-01 (Home): 뒤로가기 (홈에서 진입한 경우)
- -> S-03 (Calendar): 뒤로가기 (캘린더에서 진입한 경우)

---

### S-05: Weekly Reflection (주간 회고)

**목적:** 일주일 치 감사를 모아서 반복 패턴을 보여주는 핵심 차별화 화면. "아하 모먼트" 제공.

**핵심 UI 요소:**
- 상단: 뒤로가기 화살표 (좌측) + 주차 범위 (예: "4/4 - 4/10")
- 주간 요약 카드:
  - 기록한 날수: "7일 중 5일 기록" 
  - 총 감사 항목 수: "15개의 감사"
- 반복 키워드 섹션:
  - 빈도 기반 키워드 태그 (예: "동료" x3, "커피" x2, "산책" x2)
  - 가장 빈도 높은 키워드가 크게 표시 (태그 크기 차별화)
- "이번 주 인사이트" 한 줄:
  - 예: "이번 주, 당신은 '사람'에게서 가장 많은 감사를 느꼈어요"
  - 시각적으로 구분된 영역 (배경색 또는 좌측 보더)
- 일별 감사 목록 (날짜별 그룹):
  - 각 날짜 헤더 + 해당 일의 감사 3가지
  - 미기록 날은 "기록 없음" 표시 (죄책감 없는 톤)
- 주차 이동: 좌/우 화살표로 이전 주 회고 탐색

**사용자 액션:**
- 좌/우 화살표 탭 -> 이전/다음 주 회고로 이동
- 특정 날짜의 감사 항목 탭 -> S-04 (Detail, 해당 날짜)
- 뒤로가기 -> S-01 (Home)

**상태 변형:**

| 상태 | 조건 | 표시 |
|------|------|------|
| 풍성한 회고 | 5일 이상 기록 | 키워드 분석 + 인사이트 + 전체 목록 |
| 부족한 회고 | 1-2일만 기록 | 목록 표시 + "더 기록하면 패턴이 보여요" 안내 |
| 빈 주 | 기록 0건 | "이 주에는 기록이 없어요. 오늘부터 시작해볼까요?" CTA |

**전환:**
- -> S-01 (Home): 뒤로가기
- -> S-04 (Detail): 일별 감사 항목 탭

---

### S-06: Settings (설정)

**목적:** 데모 데이터 생성, 데이터 관리 등 부가 기능.

**핵심 UI 요소:**
- 상단: 뒤로가기 화살표 (좌측) + "설정" 타이틀
- 데이터 관리 섹션:
  - "데모 데이터 생성" 버튼 (주간 회고 체험용 샘플 데이터 7일분 생성)
  - "모든 데이터 삭제" 버튼 (위험 색상) + 확인 다이얼로그
- 앱 정보 섹션:
  - 버전 정보
  - "하루 감사 일기" 한 줄 소개

**사용자 액션:**
- "데모 데이터 생성" 탭 -> 7일분 샘플 데이터 생성 + 토스트 "데모 데이터를 만들었어요"
- "모든 데이터 삭제" 탭 -> 확인 다이얼로그 -> 삭제 후 S-01 (Home, 빈 상태)로 이동
- 뒤로가기 -> S-01 (Home)

**전환:**
- -> S-01 (Home): 뒤로가기

## 3. 내비게이션 모델

### 구조 선택 근거

이 앱은 **2개 메인 섹션 (홈 + 캘린더)** 으로 구성된다. DESIGN_RULES.md R5에 따라, 2-3 섹션 앱에는 바텀 탭바 대신 **하단 미니 내비게이션 (2탭)** 또는 **상단 세그먼트 컨트롤**이 적합하다. 여기서는 모바일에서 엄지 접근성을 고려해 하단 2탭 내비게이션을 채택한다.

Entry, Detail, Weekly Reflection, Settings는 **모달/푸시 스크린**으로, 메인 내비게이션에 포함하지 않고 각 화면에서 뒤로가기로 복귀한다.

### ASCII 플로우 다이어그램

```
                    ┌─────────────┐
                    │  S-00       │
                    │ Onboarding  │
                    │ (첫 방문만)  │
                    └──────┬──────┘
                           │ "시작하기" / "건너뛰기"
                           v
         ┌─────────────────────────────────────┐
         │          메인 내비게이션 (2탭)          │
         │                                     │
         │   ┌───────────┐    ┌───────────┐    │
         │   │  S-01     │<-->│  S-03     │    │
```

## 4. 유저 플로우

### Flow 1: 첫 사용자 플로우 (Onboarding -> 첫 기록)

**목표:** 앱 열고 -> 핵심 가치 이해 -> 첫 감사 기록 완료. 총 60초 이내.

```
1. 앱 첫 실행
   -> S-00 Onboarding 표시

2. 슬라이드 1 확인 ("매일 감사한 일 3가지, 30초면 충분해요")
   -> 스와이프 또는 다음

3. 슬라이드 2 확인 ("일주일 뒤, 내가 뭘 소중히 여기는지 알게 돼요")
   -> "시작하기" 탭

4. S-02 Entry 화면 진입
   -> 첫 필드에 자동 포커스, 키보드 올라옴

5. 감사 3가지 입력 (30초)
   -> "저장" 탭

6. 저장 피드백 (체크 애니메이션 + "저장했어요")
   -> S-01 Home으로 자동 이동

7. Home에서 오늘 기록 요약 확인 + 주간 회고 안내
   ("6일 더 기록하면 주간 회고를 볼 수 있어요")
```

**Time to First Value: 4단계 (앱 실행 -> 온보딩 -> 입력 -> 저장), 약 60초**

**마찰 포인트:**
- 온보딩 슬라이드가 2장을 초과하면 이탈 위험. 반드시 2장 이하 유지.
- 첫 필드 자동 포커스가 없으면 "뭘 해야 하지?" 멈칫 발생.

---

### Flow 2: 일일 기록 플로우 (재방문 사용자)

**목표:** 앱 열고 -> 기록 -> 저장. 총 30초 이내.

```
1. 앱 실행
   -> S-01 Home 표시 (오늘 미기록 상태)

2. "오늘의 감사 쓰기" CTA 탭
   -> S-02 Entry 진입

3. 감사 3가지 입력 (30초 목표)
   -> "저장" 탭

4. 저장 피드백 + S-01 Home 자동 복귀
   -> 오늘 기록 요약 표시
```

**총 단계: 3단계 (홈 -> 입력 -> 저장). 최적 경로.**

**마찰 포인트:**
- CTA 버튼이 스크롤 없이 보여야 한다 (Above the Fold).
- 저장 후 Home 복귀가 자동이어야 한다 (수동 뒤로가기 불필요).

---

### Flow 3: 과거 기록 조회 플로우

**목표:** 특정 날짜의 기록 확인. 3탭 이내.

```
1. S-01 Home -> 하단 "캘린더" 탭
   -> S-03 Calendar 진입

2. 원하는 날짜 탭 (도트가 있는 날)
   -> S-04 Detail 진입

3. 감사 기록 확인
   -> 뒤로가기로 S-03 Calendar 복귀
```

**총 단계: 3단계 (홈 -> 캘린더 -> 상세)**

---

### Flow 4: 주간 회고 플로우

**목표:** 일주일 치 감사 패턴 확인 + 인사이트 획득.

```
1. S-01 Home
   -> "이번 주 회고 보기" 버튼 탭 (3일+ 기록 시 활성)

2. S-05 Weekly Reflection 진입
   -> 주간 요약 + 키워드 + 인사이트 확인

3. (선택) 특정 날짜 항목 탭
   -> S-04 Detail로 이동하여 상세 확인

4. 뒤로가기
   -> S-01 Home 복귀
```

**총 단계: 2단계 (홈 -> 주간 회고). 핵심 가치 도달 최단 경로.**

**이탈 위험:**
- 주간 회고 버튼이 비활성 상태일 때 ("3일 더 기록해야 해요") 동기 부여 메시지가 약하면 이탈 발생.
- 인사이트가 피상적이면 ("감사한 것이 많았어요" 수준) 재방문 동기 소멸.

---

### Flow 5: 기록 수정 플로우

**목표:** 과거 기록 수정. 인라인 편집으로 화면 전환 최소화.

```
1. S-04 Detail (어떤 경로든 도달)
   -> "수정" 텍스트 버튼 탭

2. 인라인 편집 모드 전환
   -> 텍스트 필드 활성화, 기존 값 표시

3. 내용 수정
   -> "저장" 탭

4. 토스트 "수정했어요" + 읽기 모드 복귀
```

**설계 원칙:** 수정은 별도 화면 이동 없이 Detail 화면 내에서 인라인으로 처리. DESIGN_RULES.md R19 (인라인 액션이 페이지 전환을 유발하면 안 됨) 준수.

---

## 5. 빈 상태 (Empty States)

DESIGN_RULES.md R21에 따라 모든 빈 상태에 행동 유도(CTA)를 포함한다.

| 스크린 | 빈 상태 조건 | 표시 내용 | CTA |
|--------|-------------|----------|-----|
| S-01 Home | 기록 0건 (첫 방문 직후) | "아직 기록이 없어요" | "첫 감사를 기록해보세요" 버튼 -> S-02 |
| S-01 Home (주간 회고) | 이번 주 3일 미만 기록 | "N일 더 기록하면 주간 회고가 열려요" | 비활성 버튼 (설명 텍스트로 동기 부여) |
| S-03 Calendar | 이번 달 기록 0건 | 빈 캘린더 그리드 | "첫 감사를 기록하면 여기에 표시돼요" + 오늘 날짜 하이라이트 |
| S-05 Weekly | 해당 주 기록 0건 | "이 주에는 기록이 없어요" | "오늘부터 시작해볼까요?" 버튼 -> S-02 |
| S-05 Weekly | 해당 주 1-2일만 기록 | 기록 표시 + 안내 | "더 기록하면 패턴이 보여요" (키워드 분석 미표시) |

**빈 상태 톤:**
- 죄책감을 주지 않는다. "아직 기록이 없어요"는 OK. "기록을 빠뜨렸어요"는 NO.
- 행동 유도는 부드럽게. "해보세요" 톤. "해야 해요" 톤 금지.
- PRD의 핵심 원칙 반영: 스트릭 깨짐이 자기혐오로 이어지면 안 된다.

## 6. 전환 애니메이션 (Transitions)

DESIGN_RULES.md R4에 따라 fadeInUp 일변도를 피하고, 앱 성격에 맞는 전환을 설계한다.

| 전환 | 애니메이션 | 시간 | 근거 |
|------|----------|------|------|
| Home <-> Calendar | 크로스페이드 (opacity 전환) | 200ms | 탭 전환은 가볍게 |
| Home -> Entry | 아래에서 위로 슬라이드 업 (모달 느낌) | 250ms | 작성은 "집중 모드 진입" 느낌 |
| Entry -> Home (저장 후) | 위에서 아래로 슬라이드 다운 | 200ms | 작성 완료 후 "돌아옴" 느낌 |
| Calendar -> Detail | 우측에서 좌측 슬라이드 (push) | 250ms | 계층 이동 (목록 -> 상세) |
| Detail -> Calendar | 좌측에서 우측 슬라이드 (pop) | 200ms | 계층 복귀 |
| Home -> Weekly | 우측에서 좌측 슬라이드 (push) | 250ms | 계층 이동 |
| Weekly -> Home | 좌측에서 우측 슬라이드 (pop) | 200ms | 계층 복귀 |
| Home -> Settings | 우측에서 좌측 슬라이드 (push) | 250ms | 계층 이동 |
| 저장 피드백 | 체크마크 스케일 (0 -> 1) + 페이드 | 400ms | 성취감 강조 |

**원칙:**
- 탭 내비게이션 전환: 크로스페이드 (수평 이동 아님)
- 계층 이동 (목록 -> 상세): push/pop 슬라이드
- 모달 진입 (Entry): 슬라이드 업/다운
- 피드백 애니메이션: 스케일 + 페이드 (짧고 명확하게)
- 모든 애니메이션은 ease-out 이징. 바운스/스프링 사용하지 않음 (차분한 앱 성격 반영).
