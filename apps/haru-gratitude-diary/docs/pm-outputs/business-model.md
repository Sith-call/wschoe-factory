## 1. Lean Canvas

### 1-1. Problem

1. **한국어 감사 일기 앱 부재:** 한국 시장에 최소주의 감사 일기 앱이 전무. 기존 앱은 영어 기반(Five Minute Journal, Gratitude)이거나 명상 중심(마보, 코끼리).
2. **구독 피로:** Meditopia(USD 60/yr), Five Minute Journal(USD 40/yr) 등 기존 앱의 구독 모델에 대한 저항감 증가.
3. **습관 형성 실패:** 기존 앱이 콘텐츠 입력에만 집중하고, 지속적 습관 형성(회고, 스트릭, 넛지)을 지원하지 않아 이탈률 높음.

### 1-2. Customer Segments

| Segment | Description | Size |
|---------|-------------|------|
| Primary | 25-34세 여성, 한국 거주, 마이크로 웰니스 관심, 저녁 루틴 선호 | ~2.1M |
| Secondary | 20-24세 대학생/사회초년생, 자기계발 관심, 가성비 중시 | ~1.4M |
| Tertiary | 35-44세 직장인, 번아웃 방지, 간단한 일기 습관 원함 | ~1.8M |

### 1-3. Unique Value Proposition

**"30초면 끝. 감사 3가지, 주간 회고, 그것뿐."**

한국어 네이티브 최소주의 감사 일기. 가입 없이 바로 시작, 매일 감사한 일 3가지를 기록하고 주간 회고로 돌아보는 습관 코치. 무료로 핵심 기능 전부 사용 가능.

### 1-4. Solution

1. **즉시 시작:** 가입/로그인 없이 앱 열자마자 오늘의 감사 기록 시작 (localStorage 기반)
2. **3-item 프레임워크:** 매일 정확히 3가지 감사 항목 기록 (선택 아닌 구조화)
3. **주간 회고:** 매주 자동 요약으로 한 주를 되돌아보는 리플렉션 경험
4. **습관 시각화:** 스트릭, 캘린더 뷰, 계절별 성장 시각화로 동기 유지

### 1-5. Channels

| Channel | Type | Priority |
|---------|------|----------|
| 앱스토어 검색 최적화 (ASO) | Organic | P0 |
| Instagram/Threads 감성 콘텐츠 | Organic Social | P0 |
| 블로그/브런치 "감사일기 효과" SEO | Content | P1 |
| 자기계발 유튜브 채널 협업 | Influencer | P1 |
| 네이버 카페/커뮤니티 바이럴 | Community | P2 |

### 1-6. Revenue Streams

| Stream | Model | Price (KRW) |
|--------|-------|-------------|
| 프리미엄 구독 (월간) | Subscription | 2,900/mo |
| 프리미엄 구독 (연간) | Subscription | 19,900/yr |
| 테마 팩 (일회성) | One-time purchase | 1,900-3,900/pack |

### 1-7. Cost Structure

| Cost | Type | Monthly Est. |
|------|------|-------------|
| 호스팅/인프라 | Variable | ~$20 (localStorage 기반이라 최소) |
| 앱스토어 수수료 | Variable | 매출의 15-30% |
| 마케팅/콘텐츠 | Variable | $0-200 (초기 organic 집중) |
| 개발/유지보수 | Fixed | $0 (1인 개발 or 사이드 프로젝트) |

### 1-8. Key Metrics

- DAU / MAU ratio (목표: 40%+)
- D7 / D30 리텐션 (목표: D7 50%, D30 25%)
- 주간 회고 완료율 (목표: 60%+)
- 프리미엄 전환율 (목표: 3-5%)
- 스트릭 7일+ 유지 비율

### 1-9. Unfair Advantage

1. **한국 시장 선점:** 한국어 네이티브 최소주의 감사 일기 앱 최초 진입 (blue ocean)
2. **제로 인프라 비용:** localStorage 기반으로 서버 비용 거의 없음 — 수익성 구조적 우위
3. **구독 피로 역이용:** 핵심 기능 완전 무료 제공으로 경쟁사 대비 획기적 진입 장벽 제거

## 2. Recommended Business Model

### 2-1. 모델 비교

| Model | 장점 | 단점 | 적합도 |
|-------|------|------|--------|
| **Free Core + Optional Premium** | 진입 장벽 제로, 구독 피로 회피, 높은 리텐션 | 수익화 느림 | **최적** |
| Freemium (기능 제한) | 전환 유인 명확 | 무료 버전 불만족 시 이탈, 최소주의 철학과 충돌 | 차선 |
| 광고 기반 | 즉시 수익 | 최소주의 UX 훼손, 감사 일기의 차분한 경험 파괴 | 부적합 |
| 유료 앱 | 단순한 수익 구조 | 한국 시장에서 유료 앱 구매 저항 높음 | 부적합 |

### 2-2. 권장 모델: Free Core + Optional Premium

**핵심 원칙:** 감사 기록의 본질적 기능은 영원히 무료. 프리미엄은 "더 깊은 회고"와 "개인화"에만 적용.

#### 무료 (Free Core)

- 매일 감사 3가지 기록
- 주간 회고 요약
- 스트릭 트래킹 및 캘린더 뷰
- 리마인더 알림
- 데이터 내보내기 (텍스트)
- localStorage 기반 완전 프라이빗

#### 프리미엄 (월 2,900원 / 연 19,900원)

- 월간/분기별 회고 리포트
- 감사 카테고리 자동 분류 및 트렌드 분석
- 커스텀 테마 (계절, 무드별)
- 데이터 클라우드 백업 및 기기 간 동기화
- 데이터 내보내기 (PDF, 이미지)

#### 추가 수익: 테마 팩 (1,900-3,900원 일회성)

- 시즌 한정 테마 (벚꽃, 단풍, 눈 등)
- 아티스트 콜라보 테마
- 별도 구독 없이 영구 소유

### 2-3. 모델 선택 근거

1. **최소주의 철학 일관성:** 광고나 기능 제한은 "심플한 감사 습관"이라는 핵심 가치를 훼손. 핵심 루프를 무료로 유지해야 브랜드 일관성 유지.
2. **구독 피로 대응:** 시장 조사에서 확인된 구독 피로를 정면 공략. "핵심은 무료, 구독은 선택"이 차별화 포인트.
3. **인프라 비용 최소:** localStorage 기반이므로 무료 유저가 늘어도 서버 비용 증가 없음. 전통적 freemium 모델의 "무료 유저 = 비용"이 성립하지 않음.
4. **낮은 가격대:** 경쟁사(Five Minute Journal USD 40/yr, Gratitude USD 30-60/yr) 대비 연 19,900원(~USD 15)으로 가격 저항 최소화.

## 5. Risk Assessment

### 5-1. 리스크 매트릭스

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| **낮은 초기 인지도** | HIGH | HIGH | ASO 최적화, 블로그/SNS 콘텐츠 마케팅 선행, 자기계발 커뮤니티 시딩 |
| **리텐션 드롭** | HIGH | HIGH | 온보딩 최적화, 리마인더 시스템, 스트릭 보상, 주간 회고 넛지 |
| **프리미엄 전환 저조** | MEDIUM | MEDIUM | 무료 체험 기간 제공, 프리미엄 가치 시각적 차별화, 가격 A/B 테스트 |
| **경쟁사 한국어 진출** | MEDIUM | HIGH | 선점 효과 활용, 한국 문화 맞춤 콘텐츠(한국 절기, 명절 회고), 커뮤니티 구축 |
| **localStorage 데이터 유실** | MEDIUM | HIGH | 데이터 내보내기 기능 P0, 클라우드 백업 P1, 유실 경고 안내 |
| **앱스토어 정책 변경** | LOW | MEDIUM | PWA 대안 경로 확보, 웹 버전 병행 운영 |
| **구독 피로 심화** | LOW | LOW | 일회성 테마 팩 수익 비중 확대, 핵심 무료 유지 |

### 5-2. 최대 리스크: 리텐션

감사 일기 앱의 가장 큰 위험은 "작성을 멈추는 것"이다. 습관 앱의 D30 리텐션은 업계 평균 10-15%로 낮다. 25%+ 달성이 비즈니스 전체의 성패를 좌우한다.

**리텐션 방어 전략:**

1. **마찰 제거:** 앱 진입 즉시 오늘의 감사 입력 화면. 탭 1회, 타이핑만으로 완료.
2. **30초 룰:** 하루 기록 완료까지 30초 이내. 길어지면 이탈.
3. **스트릭 심리:** 연속 기록 일수를 시각화하여 "끊기 싫은" 동기 부여.
4. **주간 회고 넛지:** 일요일 저녁 리마인더로 한 주 돌아보기 유도.
5. **계절 시각화:** 기록 누적에 따라 성장하는 시각적 보상 (나무, 정원 등).

### 5-3. 최대 기회: localStorage의 구조적 비용 우위

대부분의 앱 비즈니스가 유저 증가 = 서버 비용 증가인 반면, haru-gratitude-diary는 localStorage 기반이므로 유저가 늘어도 인프라 비용이 증가하지 않는다. 이는:

- 무료 유저 유지에 대한 부담이 없어 generous한 무료 정책 가능
- 손익분기점이 극도로 낮아 실패 리스크 최소화
- 프리미엄 전환 압박 없이 장기 사용자 경험 최적화 가능

## Strategic Summary

**핵심 발견:**
- 한국 감사 일기 앱 시장은 USD 22-26M 규모이나 최소주의 한국어 네이티브 제품이 부재 (blue ocean)
- localStorage 기반 아키텍처가 비용 구조에서 구조적 우위를 제공하여 인디 앱에 최적
- 보수적 시나리오에서도 Year 1부터 흑자 달성 가능 (연간 비용 ~$964 vs 순매출 ~$1,870)

**권장 방향:**
