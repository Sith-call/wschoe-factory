## 1. JTBD Value Proposition (6-Part Template)

### WHO: Target User

25-34세 한국 여성. 직장 생활 3-10년 차로, 일상에서 작은 좋은 것들을 의식적으로 알아차리고 싶지만 복잡한 웰니스 앱이나 빈 노트 앞에서 막막함을 느끼는 사람. 치료사의 권유, 번아웃 후 자기 회복, 혹은 부정적 사고 패턴에서 벗어나고 싶은 계기로 감사 일기를 시작하려 한다. 프라이버시를 중시하며, 구독 피로감이 있고, "딱 이것만 하면 되는" 도구를 원한다.

**핵심 페르소나 특성:**
- 저녁 루틴으로 하루를 마무리하고 싶은 욕구
- 앱에 개인적인 감정을 기록하는 것에 대한 프라이버시 민감성
- 기존 감사 일기 앱을 깔았다가 2주 안에 삭제한 경험 보유
- "마이크로 웰니스" 선호 -- 5분 이내로 끝나는 정신 건강 루틴

### WHY: The Job They Are Hiring For

**Primary JTBD:** "오늘 하루에서 감사한 것 3가지를 빠르게 기록하고, 한 주가 지나면 내가 뭘 소중히 여기는지 패턴으로 돌아보고 싶다."

이 앱을 "고용"하는 진짜 이유는 감사 기록 자체가 아니라 **자기 인식의 축적**이다. 매일 쓰는 3줄은 가볍지만, 일주일이 쌓이면 "내가 실은 사람 관계에서 가장 많이 감사하는구나" 같은 패턴이 보인다. 이 패턴 인식이 "쓸모 있다"는 감각을 만들고, 습관이 지속되는 이유가 된다.

**세부 Job 시나리오:**
1. 잠들기 전 불안한 생각을 멈추고 차분한 마음으로 전환하고 싶을 때
2. 최악의 하루를 보낸 뒤, "전부 다 나쁜 건 아니었다"는 감각을 되찾고 싶을 때
3. 아무것도 나아지지 않는 것 같은 시기에, 주간 회고를 통해 조용한 진전을 확인하고 싶을 때
4. 이전에 감사 일기를 시도했다가 포기한 뒤, 죄책감 없이 다시 시작하고 싶을 때

### WHAT BEFORE: Current Alternatives and Pain

**현재 대안과 한계:**

| 대안 | 사용자가 느끼는 고통 |
|------|---------------------|
| **종이 노트** | 들고 다니기 번거롭고, 검색 불가, 주간 요약 불가. 빈 페이지 앞에서 막막함. 3개월 전에 쓴 감사는 사실상 사라진 것. |
| **Gratitude (Gratefulness.me)** | 비전보드, 확언, 워크시트, 커뮤니티까지 뒤섞여서 "일기 하나 쓰려는데 왜 이렇게 복잡한지" 모르겠음. 프리미엄 유도 팝업에 지침. |
| **Five Minute Journal** | 영어 전용, 한국어 지원 없음. 연 $40 구독료. 주간 회고 기능 부재. |
| **Day One** | 범용 일기 앱이라 감사 구조가 없음. 빈 페이지 = 종이 노트와 같은 문제. |
| **마보, 마인드카페** | 명상/상담 중심. 감사 일기는 부수 기능이거나 아예 없음. 앱 자체가 무겁고 복잡함. |
| **데일리 다이어리, 오늘도 감사** | 감사 일기에 특화되어 있으나, 주간 회고가 약하고, 반복 피로에 대한 해결책이 없음. |

**대안 사용 시 공통적으로 느끼는 감정:**
- "탭이 6개나 있는데 나는 딱 하나만 쓰고 싶은데" (기능 과잉에 대한 좌절)
- "어제도 커피, 날씨 썼는데 오늘도 같은 거 쓰라고?" (반복 피로)
- "힘든 날에 억지로 감사 찾으라니 오히려 기분 나빠진다" (강제 긍정성)
- "매일 썼는데 그래서 뭐가 달라졌는지 모르겠다" (보이지 않는 보상)

### HOW: Our Solution Approach

**핵심 설계 원칙: "3줄 쓰고, 1주일 돌아보고, 나머지는 전부 없앤다"**

1. **극단적 단순함 (Radical Simplicity)**
   - 앱을 열면 오늘 날짜와 3개의 빈 칸만 보인다. 탭 없음, 메뉴 최소화.
   - 가입 없이 바로 시작. 로컬 저장이 기본.
   - 한 칸당 50자 제한 옵션 -- 짧게 써도 되는 심리적 안전망.

2. **주간 회고 (Weekly Lookback) -- 핵심 차별화**
   - 일주일 치 감사를 한 화면에 모아서 보여준다.
   - 반복되는 키워드/테마를 자동 하이라이트 ("사람", "자연", "음식" 등).
   - "이번 주 당신이 가장 많이 감사한 것" 한 줄 인사이트.
   - 이 주간 회고가 매일 쓰는 행위에 의미를 부여하는 피드백 루프.

3. **힘든 날 모드 (Rough Day Grace)**
   - 감사할 게 없는 날을 위한 대안: "오늘은 그냥 괜찮았던 것 하나만" 모드.
   - 건너뛰어도 스트릭이 깨지지 않는 "쉼표" 시스템.
   - 강제 긍정 없이, 솔직한 날도 기록의 일부로 수용.

4. **반복 피로 방지 (Anti-Repetition)**
   - 매일 다른 관점의 부드러운 프롬프트 제안 (사용 선택).
   - "어제와 같은 항목입니다" 알림 대신, "같은 것에 감사하는 것도 괜찮아요" 메시지.
   - 프롬프트 예시: "오늘 누군가에게 받은 작은 친절은?", "오늘 몸이 편했던 순간은?"

5. **차분한 습관 형성 (Calm Habit Building)**
   - 저녁 시간대 단 1회 알림 (사용자 설정).
   - 계절이 바뀌는 나무 또는 달이 차오르는 시각화 -- 조용한 성장 피드백.
   - 연속 기록보다 "이번 달 기록한 날 수"에 초점 -- 실패에 관대한 시스템.

6. **프라이버시 우선 (Privacy First)**
   - 기본 로컬 저장. 계정 생성 불필요.
   - 선택적 클라우드 백업 (iCloud/Google Drive).
   - 감정 데이터가 서버로 전송되지 않는다는 신뢰.

### WHAT AFTER: Desired Outcome

**사용 후 변화 (Before -> After):**

| Before | After |
|--------|-------|
| 잠들기 전 SNS 둠스크롤 | 3분간 감사 기록 후 차분한 마음으로 잠듦 |
| "요즘 좋은 일이 하나도 없어" | 주간 회고에서 매일 작은 감사가 있었음을 확인 |
| 감사 일기 2주 만에 포기, 자기 혐오 | 건너뛴 날도 괜찮은 시스템, 3개월째 느슨하게 지속 |
| 뭘 써야 할지 모르겠어서 앱 닫음 | 부드러운 프롬프트 덕에 30초 만에 작성 완료 |
| 매일 쓰는데 그래서 뭐가 달라졌는지 모름 | 주간 회고에서 "나는 사람 관계에서 힘을 얻는구나" 발견 |
| 복잡한 앱에 질려서 종이로 돌아감 | 앱 열면 빈 칸 3개, 그 이상 아무것도 없음 |

**궁극적 성과:** 사용자가 "감사 일기를 쓰는 사람"이라는 정체성을 갖게 된다. 앱이 아니라 습관이 되고, 습관이 자기 인식이 된다.

### ALTERNATIVES: Why They Won't Go Back

**전환 장벽 (Switching Cost) 구축:**

1. **축적된 주간 회고 데이터** -- 3개월 이상 사용하면 계절별 감사 패턴이 보인다. 이 데이터는 다른 앱에서 재현할 수 없다. "작년 이맘때 나는 뭘 감사했을까?"가 가능해진다.

2. **습관 루틴의 관성** -- 저녁 루틴에 안착한 3분 습관을 옮기는 것은 습관 자체를 깨는 위험을 수반한다. 사용자는 도구를 바꿔서 습관이 깨지는 것을 두려워한다.

3. **단순함의 기준 재설정** -- 하루 감사 일기를 쓴 사용자는 다른 앱을 열었을 때 "왜 이렇게 복잡해?"라고 느낀다. 미니멀리즘 기준이 높아진 상태에서 기능이 많은 앱으로 돌아가기 어렵다.

4. **감정적 애착** -- 힘든 날 "괜찮았던 것 하나"를 기록한 경험, 주간 회고에서 패턴을 발견한 순간 -- 이런 감정적 경험이 앱에 대한 신뢰와 애착을 만든다.

## 4. Emotional Promise

### 감정적 약속 (Emotional Promise)

**"매일 3줄이 모이면, 당신이 보이기 시작합니다."**

하루 감사 일기는 더 긍정적인 사람이 되라고 강요하지 않는다. 대신, 이미 당신의 하루에 존재하는 작은 좋은 것들을 조용히 모아서, 일주일 뒤에 "내가 어떤 사람인지" 보여준다.

**감정적 여정:**
- **Day 1:** "이거 30초면 끝나네. 생각보다 쉽다."
- **Day 3:** "프롬프트 덕에 평소 생각 못 했던 것도 떠올리게 된다."
- **Day 7 (첫 주간 회고):** "나는 사람 관계에서 가장 많이 감사하는구나... 몰랐다."
- **Day 14:** "힘든 날이었는데, 그래도 괜찮았던 거 하나는 있었네."
- **Day 30:** "이제 저녁에 안 쓰면 뭔가 빠진 느낌. 습관이 됐다."
- **Day 90 (첫 계절 회고):** "봄에는 자연에, 겨울에는 사람에게 감사했구나."

**핵심 감정:** 자기 발견의 조용한 기쁨 (quiet joy of self-discovery). 앱이 주는 것은 행복이 아니라, 이미 있었던 감사를 알아차리는 시선이다.

## 5. Strategic Summary

### Key Findings

1. **시장 공백 확인:** 한국 시장에 "감사 특화 + 최소주의" 포지션을 차지한 앱이 없다. 글로벌 앱은 한국어 미지원, 국내 앱은 명상/상담 중심.

2. **차별화 핵심 확보:** 주간 회고를 핵심 기능으로 설계하는 것은 "매일 쓰는데 왜 쓰는지 모르겠다"는 가장 큰 이탈 원인을 직접 해결한다.

3. **감정적 포지셔닝 명확:** "강제 긍정" 앱이 아니라 "자기 발견" 도구. 힘든 날도 수용하는 설계는 기존 감사 일기 앱과 감정적으로 차별화된다.

4. **전환 비용 구조:** 축적된 주간 회고 데이터 + 습관 루틴 관성으로 자연 발생적 락인 효과.

### Recommended Direction

- 주간 회고를 MVP의 P0 기능으로 확정. "3줄 입력 + 주간 회고"가 앱의 전부여야 한다.
- 힘든 날 모드를 V1에 포함. 감사 앱의 가장 큰 이탈 원인인 "나쁜 날의 죄책감"을 정면 해결.
- 한국어 네이티브 + 로컬 저장을 비타협적 원칙으로 유지.
- 무료 핵심 + 저가 프리미엄 (월 2,900원) 모델로 구독 피로 회피.

### Risks to Watch

| Risk | Impact | Mitigation |
|------|--------|------------|
| "단순함 = 무료여야 한다"는 인식 | 수익화 실패 | 핵심 완전 무료, 프리미엄은 부가가치(AI 인사이트, 테마) |
| 주간 회고의 실제 리텐션 효과 미검증 | 핵심 차별점 무력화 | MVP 출시 후 7일/30일 리텐션 A/B 테스트 |
| 대형 앱(마보, 마인드카페)이 감사 기능 추가 | 경쟁 심화 | 미니멀 포지셔닝 유지 -- 대형 앱은 기능을 추가하지 빼지 않음 |
| 초기 사용자 확보 (CAC) | 성장 정체 | 치료사/상담사 채널 + "감사 일기 추천" 검색 SEO |
