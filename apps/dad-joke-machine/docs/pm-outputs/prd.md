# PRD: 아재개그 자판기 (Dad Joke Machine)

## 1. 문제 정의

### 왜 이 앱이 필요한가

일상에서 가볍게 웃을 수 있는 순간이 필요하지만, 기존 유머 앱들은 과도한 UI, 로그인, 광고, 소셜 기능으로 "웃으러 왔는데 피곤해지는" 경험을 준다.

한국 앱스토어에 아재개그 전문 앱은 3~5개 수준이며, 대부분 업데이트가 중단된 방치 앱이다. MZ세대의 아이러니 소비 트렌드(일부러 촌스러운 것을 즐기는 문화)와 맞물려, "아재개그를 진지하게 잘 만든 앱"에 대한 수요가 존재한다.

### 핵심 인사이트

- **원터치 랜덤 UX**: 한 화면, 한 동작, 로그인/온보딩 없음이 최적
- **기대감 딜레이**: 버튼 누른 후 0.5~1초의 짧은 대기가 재미를 극대화
- **오프라인 우선**: 지하철, 엘리베이터 등 네트워크 불안정 상황에서도 동작해야 함

## 2. 핵심 가치 제안

> **"버튼 하나로 즉시 웃음. 로그인도, 광고도, 복잡한 메뉴도 없다."**

- 앱을 열면 버튼 하나만 보인다
- 누르면 0.5~1초 후 아재개그가 "팝!" 하고 등장한다
- 그게 전부다. 다시 누르면 새 개그가 나온다

## 3. 타겟 페르소나

### 페르소나 1: 김대리 (29세, 회사원)

- **직업**: IT 스타트업 마케터
- **사용 시나리오**: 점심 먹고 돌아오는 엘리베이터에서, 오후 회의 전 긴장 풀 때, 팀 슬랙에 아재개그 하나 던질 때
- **페인포인트**: 유머 앱 깔아봤는데 로그인하라고 해서 삭제함. 인스타 밈 계정은 스크롤이 귀찮음. 딱 3초 안에 웃고 싶음
- **핵심 니즈**: 즉시성, 무마찰

### 페르소나 2: 박선생 (45세, 중학교 교사)

- **직업**: 중학교 수학 교사
- **사용 시나리오**: 수업 시작 전 학생들 집중시키려고 아재개그 하나 던짐. 교무실에서 동료 교사에게 한마디. 가족 카톡방에 저녁마다 하나씩
- **페인포인트**: 매번 같은 개그만 치니까 학생들이 "또요?" 함. 새 개그를 찾으려면 검색해야 하는데 귀찮음
- **핵심 니즈**: 개그 레퍼토리 확장, 간편한 접근

### 페르소나 3: 이대학생 (22세, 대학생)

- **직업**: 경영학과 3학년
- **사용 시나리오**: 과팅 분위기 메이커 역할. 친구들이랑 아재개그 배틀. 시험 기간 스트레스 해소
- **페인포인트**: 아재개그를 좋아하는데 매번 구글링하기 귀찮음. 기존 유머 앱은 광고 범벅
- **핵심 니즈**: 깔끔한 UX, 광고 없음, 다양한 개그

## 4. MVP 기능 목록

### P0 (필수 - 출시 차단)

| ID | 기능 | 설명 |
|----|------|------|
| F-001 | 원터치 개그 출력 | 큰 버튼 하나 누르면 랜덤 아재개그 표시 |
| F-002 | 팝 애니메이션 | 개그가 0.5~1초 딜레이 후 "팝!" 효과로 등장 |
| F-003 | 50개 로컬 데이터 | 아재개그 50개를 앱에 내장 (오프라인 동작) |
| F-004 | 중복 방지 | 최근 본 개그를 추적해 연속 중복 방지 |
| F-005 | 미니멀 UI | 한 화면, 버튼 + 개그 텍스트만 표시 |

### P1 (중요 - 출시 후 1주 내)

| ID | 기능 | 설명 |
|----|------|------|
| F-006 | 질문-답 분리 표시 | "왜 바나나가 병원에 갔을까?" → (탭) → "껍질이 벗겨져서" 2단계 표시 |
| F-007 | 햅틱 피드백 | 버튼 탭 시 진동 피드백 (모바일) |
| F-008 | 다크 모드 | 시스템 설정 연동 다크/라이트 모드 |

### P2 (개선 - 출시 후 1달 내)

| ID | 기능 | 설명 |
|----|------|------|
| F-009 | 즐겨찾기 | 마음에 드는 개그를 로컬 저장 |
| F-010 | 본 개그 카운터 | "오늘 N개의 개그를 봤습니다" 통계 |
| F-011 | 사운드 효과 | 팝 등장 시 선택적 효과음 |

## 6. 비기능 요구사항

### 성능

- 앱 콜드 스타트: 1초 이내
- 개그 표시 반응 시간: 딜레이 제외 100ms 이내
- 메모리 사용: 50MB 이하

### 호환성

- 웹: Chrome, Safari, Firefox 최신 2개 버전
- Android: API 24 (7.0) 이상
- Capacitor를 통한 크로스 플랫폼 빌드

### 오프라인

- 100% 오프라인 동작 (네트워크 의존성 없음)
- 모든 개그 데이터 로컬 번들

### 접근성

- 폰트 크기 최소 16px
- 버튼 터치 영역 최소 48x48dp
- 색상 대비 WCAG AA 준수

### 보안/프라이버시

- 수집하는 개인정보 없음
- 네트워크 통신 없음
- 로컬 스토리지만 사용 (즐겨찾기, 본 개그 기록)
