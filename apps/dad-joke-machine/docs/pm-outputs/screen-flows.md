# 화면 플로우: 아재개그 자판기

## 1. 전체 화면 목록

| ID | 화면 이름 | 우선순위 | 설명 |
|----|-----------|----------|------|
| S-001 | 메인 화면 (대기) | P0 | 버튼만 보이는 초기 상태 |
| S-002 | 메인 화면 (로딩) | P0 | 버튼 누른 후 딜레이 상태 |
| S-003 | 메인 화면 (개그 표시) | P0 | 아재개그가 팝 등장한 상태 |
| S-004 | 메인 화면 (질문만) | P1 | 질문형 개그의 질문 표시 상태 |
| S-005 | 메인 화면 (질문+답) | P1 | 질문형 개그의 답 공개 상태 |
| S-006 | 즐겨찾기 목록 | P2 | 저장한 개그 목록 |
| S-007 | 50개 클리어 | P2 | 전체 순회 완료 축하 화면 |

## 3. 화면별 상세

---

### S-001: 메인 화면 (대기 상태)

**진입 조건**: 앱 실행 시, 또는 이전 개그를 닫은 후

**화면 요소**:
- 앱 타이틀 (상단, 작게)
- 메인 버튼 (화면 중앙, 크게)
- 즐겨찾기 아이콘 (우상단, 작게) [P2]
- 오늘 본 개그 카운터 (하단, 작게) [P2]

**인터랙션**:
- 메인 버튼 탭 → S-002 (로딩)으로 전환
- 즐겨찾기 아이콘 탭 → S-006으로 전환 [P2]

**상태**:
- 라이트 모드 / 다크 모드 [P1]

### S-002: 메인 화면 (로딩 상태)

**진입 조건**: 메인 버튼 탭

**지속 시간**: 0.5~1초 (랜덤)

**화면 요소**:
- 앱 타이틀 (유지)
- 메인 버튼 (비활성화, 누르는 중 시각 피드백)
- 로딩 인디케이터 (버튼 내부 또는 버튼 위)

**인터랙션**:
- 버튼 비활성화 (연타 방지)
- 햅틱 피드백 발생 [P1]

**자동 전환**: 딜레이 완료 → S-003 또는 S-004

### S-003: 메인 화면 (일반 개그 표시)

**진입 조건**: 로딩 완료 + 일반형 개그 선택됨

**화면 요소**:
- 앱 타이틀 (유지)
- 개그 텍스트 (화면 중앙, 큰 글씨, 팝 애니메이션으로 등장)
- 메인 버튼 (하단으로 이동, "다음!" 텍스트)
- 즐겨찾기 버튼 (개그 옆, 하트) [P2]

**인터랙션**:
- 메인 버튼 탭 → S-002 (새 개그 로딩)

### S-006: 즐겨찾기 목록 [P2]

**진입 조건**: 메인 화면 우상단 즐겨찾기 아이콘 탭

**화면 요소**:
- 뒤로가기 버튼 (좌상단)
- "즐겨찾기" 타이틀
- 저장된 개그 리스트 (스크롤)
- 비어있을 때: "아직 저장한 개그가 없어요" 안내

**인터랙션**:
- 개그 항목 탭 → 해당 개그 전체 표시
- 좌로 스와이프 → 삭제

                 ▲                                        │

                 │                     일반개그(S-003)      질문형(S-004)
                 │                              │                   │
                 │                              │              탭──► 답공개(S-005)
                 │                              │                   │
                 └──────────── 버튼탭 ◄──────────┴───────────────────┘

## 5. 디자인 원칙

1. **한 화면 원칙**: 메인 경험은 절대 화면을 벗어나지 않는다
2. **원터치 원칙**: 핵심 행동은 항상 탭 한 번으로 완료된다
3. **미니멀 원칙**: 화면에 3개 이상의 요소를 동시에 표시하지 않는다
4. **즉시성 원칙**: 의도적 딜레이(기대감) 외에는 모든 것이 즉각 반응한다
5. **무마찰 원칙**: 로그인, 온보딩, 팝업, 권한 요청 없음
