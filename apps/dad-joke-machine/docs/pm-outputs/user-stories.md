# 유저 스토리 맵: 아재개그 자판기

## 시간축 개요

```
첫 방문 (Day 0)          1주 (Day 1~7)           1달 (Day 8~30)
─────────────────────    ─────────────────────    ─────────────────────
앱 설치 & 첫 경험        반복 사용 & 습관화       깊은 사용 & 애착
```

---

## 첫 방문 (Day 0)

### US-001 | P0 | 즉시 시작

- **스토리**: As a 처음 방문한 사용자, I want 앱을 열자마자 바로 사용할 수 있기를, so that 로그인이나 온보딩에 시간을 낭비하지 않는다.
- **인수 조건**:
  - [ ] 앱 실행 시 로그인, 회원가입, 온보딩 화면 없음
  - [ ] 앱 실행 후 2초 이내에 메인 화면(버튼) 표시
  - [ ] 어떤 권한 요청도 하지 않음

### US-002 | P0 | 첫 번째 개그 보기

- **스토리**: As a 호기심 많은 사용자, I want 큰 버튼을 눌러 아재개그를 받고 싶다, so that 앱의 핵심 경험을 즉시 체험한다.
- **인수 조건**:
  - [ ] 화면 중앙에 명확한 버튼이 표시됨
  - [ ] 버튼을 누르면 0.5~1초 딜레이 후 아재개그가 표시됨
  - [ ] 딜레이 동안 기대감을 주는 시각적 피드백(로딩/애니메이션)이 있음

### US-003 | P0 | 팝 등장 애니메이션

- **스토리**: As a 사용자, I want 개그가 "팝!" 하고 재미있게 등장하기를, so that 텍스트가 뜨는 순간 자체가 즐거움을 준다.
- **인수 조건**:
  - [ ] 개그 텍스트가 스케일 + 페이드 애니메이션으로 등장
  - [ ] 애니메이션 지속 시간 300~500ms
  - [ ] 애니메이션이 자연스럽고 끊김 없음

### US-004 | P0 | 연속 사용

- **스토리**: As a 재미를 느낀 사용자, I want 버튼을 다시 눌러 새로운 개그를 바로 보고 싶다, so that 연속으로 여러 개의 개그를 즐긴다.
- **인수 조건**:
  - [ ] 개그가 표시된 상태에서 버튼을 다시 누르면 새 개그로 교체
  - [ ] 이전 개그가 사라지고 새 개그가 등장하는 전환이 자연스러움
  - [ ] 연속 탭 시 같은 개그가 바로 다시 나오지 않음

### US-005 | P0 | 중복 방지

- **스토리**: As a 여러 번 사용하는 사용자, I want 방금 본 개그가 바로 다시 나오지 않기를, so that 매번 새로운 느낌을 받는다.
- **인수 조건**:
  - [ ] 최근 본 10개 개그는 연속으로 나오지 않음
  - [ ] 50개를 모두 본 후에는 순환 초기화
  - [ ] 앱 재시작 시에도 최근 기록 유지 (로컬 저장)

## 1주 (Day 1~7)

### US-006 | P0 | 오프라인 사용

- **스토리**: As a 지하철 통근 사용자, I want 인터넷 없이도 앱이 동작하기를, so that 출퇴근 지하철에서도 아재개그를 즐긴다.
- **인수 조건**:
  - [ ] 비행기 모드에서도 모든 기능 정상 동작
  - [ ] 네트워크 상태와 무관하게 50개 개그 전부 접근 가능
  - [ ] 오프라인 시 에러 메시지나 로딩 표시 없음

### US-007 | P1 | 질문-답 분리 표시

- **스토리**: As a 개그의 재미를 극대화하고 싶은 사용자, I want 질문이 먼저 나오고 탭하면 답이 나오기를, so that "뭘까?" 하는 기대감이 생긴다.
- **인수 조건**:
  - [ ] 질문형 개그는 질문 먼저 표시
  - [ ] 화면 탭 또는 답 보기 버튼으로 답 표시
  - [ ] 답이 팝 애니메이션으로 등장
  - [ ] 일반형 개그(질문-답 구조가 아닌 것)는 한번에 표시

### US-008 | P1 | 다크 모드

- **스토리**: As a 야간 사용자, I want 어두운 환경에서 눈이 편한 다크 모드를, so that 자기 전에도 편하게 사용한다.
- **인수 조건**:
  - [ ] 시스템 다크 모드 설정에 자동 연동
  - [ ] 다크 모드에서 모든 텍스트 가독성 확보 (WCAG AA)
  - [ ] 모드 전환 시 부드러운 트랜지션

### US-009 | P1 | 햅틱 피드백

- **스토리**: As a 모바일 사용자, I want 버튼을 누를 때 짧은 진동이 느껴지기를, so that 물리적 자판기를 누르는 느낌을 받는다.
- **인수 조건**:
  - [ ] 버튼 탭 시 짧은 햅틱 피드백 (15~30ms)
  - [ ] 개그 팝 등장 시 한 번 더 피드백
  - [ ] 웹에서는 햅틱 미지원 시 무시 (에러 없음)
  - [ ] Capacitor Haptics API 사용

### US-010 | P1 | 빠른 재방문

- **스토리**: As a 재방문 사용자, I want 앱을 다시 열면 바로 버튼이 보이기를, so that 매번 처음처럼 빠르게 시작한다.
- **인수 조건**:
  - [ ] 앱 재실행 시 이전 상태 복원 없이 깨끗한 메인 화면
  - [ ] 콜드 스타트 1초 이내
  - [ ] 스플래시 화면 없음 (또는 최소 300ms)

## 1달 (Day 8~30)

### US-011 | P2 | 즐겨찾기 저장

- **스토리**: As a 단골 사용자, I want 마음에 드는 개그를 저장해두고 싶다, so that 나중에 친구에게 쓸 때 다시 찾을 수 있다.
- **인수 조건**:
  - [ ] 개그 표시 상태에서 즐겨찾기 버튼 (하트/별) 표시
  - [ ] 즐겨찾기한 개그 목록 확인 가능
  - [ ] 로컬 스토리지에 저장 (앱 삭제 전까지 유지)
  - [ ] 즐겨찾기 목록에서 개그 탭 시 전체 표시

### US-012 | P2 | 본 개그 카운터

- **스토리**: As a 꾸준한 사용자, I want 오늘/전체 몇 개의 개그를 봤는지 알고 싶다, so that 소소한 성취감을 느낀다.
- **인수 조건**:
  - [ ] 메인 화면 하단에 "오늘 N개 봄" 카운터 표시
  - [ ] 자정에 일간 카운터 리셋
  - [ ] 누적 카운터는 계속 유지
  - [ ] UI에 방해되지 않는 작은 크기

### US-013 | P2 | 사운드 효과

- **스토리**: As a 감각적 경험을 원하는 사용자, I want 개그 등장 시 "빠밤!" 같은 효과음이 나기를, so that 자판기에서 뭔가 나오는 느낌이 더 강해진다.
- **인수 조건**:
  - [ ] 개그 팝 등장 시 짧은 효과음 재생
  - [ ] 설정에서 사운드 ON/OFF 토글
  - [ ] 기본값: OFF (소리 없이 시작)
  - [ ] 무음 모드/매너 모드에서는 자동 OFF

### US-014 | P2 | 전체 50개 순회 완료

- **스토리**: As a 모든 개그를 본 사용자, I want 50개를 다 봤다는 알림을 받고 싶다, so that 전부 클리어한 성취감을 느낀다.
- **인수 조건**:
  - [ ] 50개 전부 표시 후 축하 메시지/애니메이션
  - [ ] "다시 처음부터" 옵션 제공
  - [ ] 진행률은 별도로 표시하지 않음 (미니멀 유지)

### US-015 | P2 | 접근성

- **스토리**: As a 시각적 불편함이 있는 사용자, I want 큰 글씨와 높은 대비로 개그를 읽고 싶다, so that 누구나 편하게 사용할 수 있다.
- **인수 조건**:
  - [ ] 개그 텍스트 최소 18px
  - [ ] 버튼 터치 영역 최소 48x48dp
  - [ ] 색상 대비 WCAG AA 기준 충족
  - [ ] 스크린 리더 지원 (버튼, 텍스트 aria-label)

### US-016 | P1 | Android 네이티브 빌드

- **스토리**: As a Android 사용자, I want 앱스토어에서 설치할 수 있는 네이티브 앱을 원한다, so that 홈 화면에 추가하고 빠르게 실행한다.
- **인수 조건**:
  - [ ] Capacitor로 Android APK 빌드
  - [ ] Android 7.0 (API 24) 이상 지원
  - [ ] APK 크기 5MB 이하
  - [ ] 기본 앱 아이콘 및 스플래시 적용
