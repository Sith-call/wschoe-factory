## 화면별 디자인 크리틱

### 1. 메인 화면 (대기 상태)
**잘 된 점:**
- 크림 배경(#FBF7F0) 정확히 적용됨. 따뜻한 레트로 톤 살아있음.
- "아재개그 자판기" 타이틀에 Black Han Sans 적용 확인. 레트로 간판 느낌 훌륭.
- "눌러봐!" 버튼의 box-shadow 하단 그림자 → 물리 버튼 입체감 잘 살림.
- 버튼 border(3px solid primary-dark) 레트로 자판기 버튼 느낌 정확.
- 우상단 별 아이콘 → SVG, 이모지 아님 (R13 준수).
- 화면 중앙 정렬 → 인트로/대기 성격이므로 예외 허용 (R14 준수).
- 불필요한 장식 요소 없음. 화이트 스페이스가 자판기 넓은 패널 느낌.

**이슈:**
- 카운터 "오늘 0개 봄" 텍스트가 화면 최하단에 위치하는데, 시각적으로 버튼과의 거리가 너무 멀어 연관성이 약함. 버튼 바로 아래에 위치시키거나 간격을 줄이는 것이 좋겠음.

### 2. 메인 화면 (개그 표시 — 질문형)
**잘 된 점:**
- 개그 카드: 좌측 accent bar(4px solid primary) + 우측 둥근 모서리 → design-system.md 정확히 반영.
- 카드 배경색 #FFF0E6 (primary-light) 정확.
- "탭해서 답 보기" 힌트 텍스트 적절한 tertiary 색상.
- 텍스트 좌측 정렬 (R14 준수).
- 카드 네스팅 없음 (R15 준수).

**이슈:**
- 버튼 텍스트가 "건너뛰기"인데, 이것은 개그를 처음 보는 상태에서의 Primary 액션과 혼동됨. 질문형 개그에서 답을 보기 전 상태일 때 "다음!" 이 아닌 "건너뛰기"가 표시되는 것은 의미상 맞으나, Primary 스타일(솔리드 오렌지)로 "건너뛰기"를 표현하는 것은 버튼 위계(R16)와 약간 충돌. "건너뛰기"는 Secondary 액션에 가까움.

### 3. 메인 화면 (펀치라인 공개 후)
**잘 된 점:**
- 펀치라인 텍스트: Nanum Gothic 700, 1.5rem, primary 색상(#E8651A) → design-system.md 정확 반영.
- 셋업보다 펀치라인이 크고 굵어 시각적 위계 명확.
- 즐겨찾기 별 아이콘 SVG → filled 상태로 primary 색상 변환, 피드백 있음 (R19 준수).
- "다음!" 버튼으로 텍스트 변경 → Primary 액션으로 적절.
- 카운터 "오늘 2개 봄" 업데이트 확인.

**이슈:**
- 한자 병기 "형광등 (蛍光燈)" → 펀치라인 폰트에서 한자가 Nanum Gothic으로 렌더링되는데, 한자 글리프가 이 폰트에서 최적은 아님. 다만 콘텐츠 이슈이지 디자인 시스템 위반은 아님.

### 4. 즐겨찾기 화면 (빈 상태)
**잘 된 점:**
- 좌측 정렬 (R14 준수) — 빈 상태 텍스트도 좌측 정렬로 프로페셔널함.
- 별 아이콘 SVG, secondary 색상 → design-system.md 반영.
- "개그 뽑으러 가기" CTA 버튼 있음 (R21 준수).
- Secondary 버튼 스타일 (아웃라인 녹색) → Primary와 명확 구분 (R16 준수).
- 뒤로가기 화살표 SVG 아이콘, pill 형태 터치 영역.
- "즐겨찾기" 제목: Nanum Gothic 700, 적절한 크기.

**이슈:**
- 없음. 이 화면은 design-system.md를 거의 완벽하게 반영.

### 5. 즐겨찾기 화면 (콘텐츠 있음)
**잘 된 점:**
- 리스트 아이템 좌측 정렬 (R14 준수).
- border-bottom으로 구분 (카드 네스팅 아님, R15 준수).
- 셋업(보조 텍스트 색상) + 펀치라인(primary 색상, 굵게) 위계 명확.
- 삭제 아이콘(휴지통 SVG) 우측 배치, tertiary 색상.
- 아이템 radius 0px (sharp) → design-system.md의 "리스트는 날카롭게" 반영.

**이슈:**
- 삭제 아이콘이 너무 작고 옅어서 터치 타겟으로 인지하기 어려움. 최소 44x44px 터치 영역 권장.

## 개선 제안 (우선순위별)

### 1. [높음] "건너뛰기" 버튼 위계 재조정 (R16)
질문형 개그에서 답을 보기 전 상태일 때, "건너뛰기"가 Primary 스타일(솔리드 오렌지 + box-shadow)로 표시됨. "건너뛰기"는 본질적으로 Secondary 액션이다. 이 상태에서 Primary 액션은 "탭해서 답 보기"이므로, 카드 자체가 Primary CTA 역할을 하고 있다면 하단 버튼은 Secondary 스타일(아웃라인)로 바꿔야 버튼 위계가 명확해진다.

### 2. [중간] 카운터 위치/간격 조정
"오늘 0개 봄" 카운터가 화면 최하단에 고정되어 있어 메인 버튼과의 시각적 연결이 약함. 버튼 바로 아래 16-24px 간격으로 배치하면 "버튼을 누르면 카운터가 올라간다"는 인과관계가 시각적으로 더 명확해짐.

### 3. [중간] 즐겨찾기 삭제 아이콘 터치 영역 확대
현재 삭제 아이콘이 시각적으로 작고 옅음. 모바일 접근성 가이드라인(최소 44x44px 터치 타겟)에 맞게 패딩을 넉넉히 주고, hover/active 상태를 좀 더 명확하게 표현하면 좋겠음.

### 4. [낮음] 앱 타이틀 크기 검토
design-system.md에서 H1 타이틀용으로 3xl(36px/2.25rem)을 지정했으나, 실제 구현은 1.5rem(24px). Black Han Sans 자체가 시각적으로 굵어서 현재도 나쁘지 않지만, 디자인 시스템 문서와의 일관성을 위해 조정을 검토할 수 있음.

### 5. [낮음] 화면 전환 crossfade 미구현
design-system.md에서 메인<->즐겨찾기 전환에 "opacity crossfade 150ms"를 지정했으나, 실제로는 display:none/block 토글(즉시 전환)로 구현됨. 의도적으로 "즉시 표시"를 선택한 것이라면 OK이나, 디자인 시스템 문서와 불일치.

## 디자인 점수: 8/10

**총평:**

디자인 시스템 문서를 매우 충실하게 반영한 구현이다. 색상 팔레트, 폰트, 아이콘, 레이아웃, 애니메이션 모두 design-system.md에서 정의한 토큰을 CSS 변수로 정확하게 구현했고, DESIGN_RULES.md의 안티패턴을 하나도 범하지 않았다. 특히 레트로 자판기 컨셉을 버튼의 box-shadow 입체감, 개그 카드의 좌측 accent bar, 크림 배경 등으로 일관되게 표현한 점이 인상적이다.

감점 요인은 (1) "건너뛰기" 버튼의 위계 혼동, (2) 카운터-버튼 간격의 시각적 단절, (3) 디자인 시스템 문서와의 소소한 불일치(타이틀 크기, 화면 전환 애니메이션)이다. 이들은 사용성에 큰 영향은 없으나, 디자인 시스템의 완결성 측면에서 개선 여지가 있다.

**"이 앱답게"** 만들었는가? — YES. 다른 AI 생성 앱과 구분되는 고유한 비주얼 아이덴티티가 있다.
