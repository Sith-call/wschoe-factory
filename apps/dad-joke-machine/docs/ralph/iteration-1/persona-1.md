# 페르소나 1 피드백: 김대리 (29세, 마케터)

## 화면별 피드백

### 1. 메인 화면 (랜딩)
![메인 화면](/tmp/dad-joke-home.png)

**좋은 점:**
- "눌러봐!" 한 버튼으로 즉시 시작 — 로그인, 온보딩, 튜토리얼 제로. 엘리베이터에서 꺼내자마자 바로 쓸 수 있음
- 화면이 극도로 단순해서 뭘 해야 하는지 0.5초만에 파악 가능
- "오늘 N개 봄" 카운터가 은근히 중독성 있음 — 슬랙에 "나 오늘 10개 봤다" 이런 자랑 가능

**개선점:**
- 화면 상단 절반이 완전히 비어있음. 버튼이 중앙보다 아래에 위치해서 시선이 허전한 빈 공간에 먼저 감. 뭔가 재밌는 일러스트나 캐릭터가 있으면 "이 앱 뭔데?" 하고 호기심 유발 가능
- "아재개그 자판기"라는 타이틀은 있는데, 처음 오는 사람한테 이게 무슨 앱인지 한 줄 설명이 없음. 앱스토어에서 받았으면 알겠지만 링크로 공유받으면 모름
- 배경색(#FFF8F0 계열)이 너무 밋밋함. 자판기 컨셉이면 좀 더 레트로하거나 장난기 있는 비주얼이 기대됨

### 2. 개그 카드 (질문 상태)
![질문 카드](/tmp/dad-joke-next.png)

**좋은 점:**
- "탭해서 답 보기" 힌트가 직관적
- 왼쪽 주황색 바가 카드 영역을 잘 구분해줌
- "건너뛰기" 버튼이 있어서 재미없어 보이는 문제는 패스 가능

**개선점:**
- 카드 탭 영역이 작음. 카드 자체를 탭해야 하는데 터치 타겟이 텍스트 영역에 한정되어 있어서, 카드 바깥 여백을 탭하면 반응 없음
- "건너뛰기"와 "다음!" 버튼이 같은 위치, 같은 스타일이라 답을 보기 전/후의 상태 변화가 버튼 텍스트만으로 구분됨. 시각적 차이가 더 크면 좋겠음
- 카드가 등장할 때 애니메이션이 없어서 "새 개그가 나왔다"는 느낌이 약함. 자판기에서 "탁" 떨어지는 느낌이 있으면 훨씬 재밌을 것

### 3. 개그 카드 (답 공개 상태)
![답 공개](/tmp/dad-joke-answer2.png)

**좋은 점:**
- 답이 주황색 볼드로 강조되어서 눈에 확 들어옴
- 즐겨찾기 별표 아이콘이 바로 옆에 있어서 "이거 슬랙에 써야지" 할 때 바로 저장 가능
- "다음!" 버튼으로 즉시 다음 개그 진행 — 흐름이 끊기지 않음

**개선점:**
- 공유 버튼이 없음! 이게 가장 큰 문제. 팀 슬랙에 개그를 던지고 싶은데, 텍스트를 직접 타이핑해야 함. "복사" 또는 "공유" 버튼이 필수
- 별표를 눌렀을 때 즐겨찾기 페이지로 바로 이동해버림. 토스트("저장됨!") 정도만 뜨고 현재 화면에 머물러야 함. 연속으로 개그를 보다가 흐름이 끊김
- 답이 공개되는 순간 살짝 기다림(서스펜스)이 있으면 더 재밌을 것. 지금은 탭하면 바로 보여서 "두근두근" 느낌이 없음

### 4. 즐겨찾기 페이지 (아이템 있음)
![즐겨찾기](/tmp/dad-joke-favorites.png)

**좋은 점:**
- 깔끔한 리스트 뷰
- 삭제 버튼(휴지통 아이콘)이 직관적
- 뒤로가기 화살표로 메인으로 쉽게 복귀

**개선점:**
- 저장된 개그에서 "복사" 버튼이 없음. 즐겨찾기에 모아놨는데 슬랙에 옮길 방법이 없으면 의미가 반감
- 삭제 시 확인 다이얼로그 없이 바로 삭제됨. 실수로 누르면 복구 불가
- 개그가 많아지면 스크롤만으로 원하는 걸 찾기 어려울 수 있음. 카테고리나 검색은 나중 문제지만, 최소한 날짜별 그룹핑은 있으면 좋겠음

### 5. 즐겨찾기 페이지 (빈 상태)
![빈 즐겨찾기](/tmp/dad-joke-favs-empty.png)

**좋은 점:**
- "아직 저장한 개그가 없어요" 메시지가 친근함
- "개그 뽑으러 가기" CTA로 메인 화면 복귀 유도 — 데드엔드 아님
- 별표 아이콘이 상태를 시각적으로 보여줌

**개선점:**
- 빈 상태 디자인이 너무 심심함. 재밌는 일러스트나 "아직 아재력이 부족하군요!" 같은 위트 있는 문구면 더 좋겠음
- 빈 상태에서의 색상(초록)이 앱 전체 톤(주황/갈색)과 안 맞음

## 개선 제안 (우선순위별)

### P0 (필수 — 없으면 앱의 핵심 가치 훼손)
1. **공유/복사 기능 추가**: 답 공개 후 "텍스트 복사" 또는 "카카오톡/슬랙 공유" 버튼 필수. 아재개그의 핵심 사용처가 "남한테 써먹기"인데 이 경로가 막혀있음
2. **즐겨찾기 추가 시 화면 이동 문제 수정**: 별표 누르면 즐겨찾기 페이지로 이동하지 말고, 현재 화면에서 토스트("저장됨!")만 띄워야 함. 지금은 개그 연속 감상 흐름이 완전히 끊김

### P1 (강력 권장 — UX 품질 크게 향상)
3. **카드 등장 애니메이션**: 새 개그 카드가 위에서 "탁" 떨어지거나 슬라이드인되는 느낌. 자판기 컨셉에 맞는 물리적 피드백
4. **답 공개 서스펜스**: 탭 후 0.3~0.5초 딜레이 + 텍스트 페이드인 or 뒤집히는 애니메이션으로 "두근두근" 연출
5. **메인 화면 비주얼 보강**: 상단 빈 공간에 자판기 일러스트 또는 캐릭터 배치. 첫인상이 "뭔가 재밌겠다"여야 함
6. **삭제 시 확인 또는 언두**: 즐겨찾기 삭제가 너무 즉각적. 최소한 "되돌리기" 토스트 필요

### P2 (있으면 좋음 — 다음 이터레이션)
7. **즐겨찾기에서 복사 기능**: 저장된 개그를 길게 눌러서 텍스트 복사
8. **메인 화면 한 줄 소개**: 처음 오는 유저를 위한 "매일 아재개그 한 잔" 같은 태그라인
9. **빈 상태 디자인 통일**: 즐겨찾기 빈 상태의 초록색 → 앱 전체 톤(주황/갈색)에 맞추기
10. **오늘 카운터 강화**: "오늘 6개 봄" 외에 주간/누적 통계, 또는 "아재 레벨" 같은 게이미피케이션

## 만족도 점수: 62/100

## 만족도 근거

**핵심 니즈 충족도 분석:**

김대리의 3대 사용 시나리오를 기준으로 평가:

1. **"엘리베이터에서 개그 하나 보기"** → **90점**. 앱 열자마자 "눌러봐!" 한 번이면 개그 등장. 로그인 없음, 로딩 없음(21ms). 이건 완벽함.

2. **"슬랙에 개그 하나 던지기"** → **30점**. 개그를 봤는데 공유할 방법이 없음. 텍스트를 눈으로 읽고 직접 타이핑해야 함. 이건 치명적 — 아재개그의 가치는 "써먹기"에 있는데 그 경로가 완전히 막혀있음.

3. **"마음에 드는 개그 저장해두기"** → **45점**. 즐겨찾기 기능은 있지만, 별표 누르면 즐겨찾기 페이지로 이동해버려서 개그 연속 감상 흐름이 끊김. 저장한 개그를 나중에 복사할 방법도 없음.

**무마찰(frictionless) 평가:**
- 첫 접근: 마찰 제로 (로그인, 온보딩 없음) → 만점
- 개그 소비: 마찰 낮음 (탭→답→다음 흐름 자연스러움) → 좋음
- 개그 공유: 마찰 최대 (복사/공유 기능 부재) → 치명적
- 개그 저장: 마찰 중간 (별표 → 화면 이동 → 흐름 끊김) → 개선 필요

**결론:** 앱의 "개그 뽑기" 핵심 루프는 빠르고 직관적이지만, "써먹기"라는 아재개그의 본질적 가치 체인이 끊겨있음. 자판기에서 음료는 잘 나오는데 빨대가 없는 느낌. 공유/복사 기능 추가와 즐겨찾기 UX 수정이 되면 80점 이상 가능.
