# 페르소나 1 피드백 (Iteration 2): 김대리

## Iteration 1 개선사항 반영 확인

### 1. 홈 화면 서브 카피 추가 ✅
- "매일 아재개그 한 잔, 동전은 필요 없어요" 서브 카피가 메인 버튼 위에 잘 배치됨
- 처음 오는 유저도 앱이 뭔지 바로 파악 가능. Iteration 1에서 요구한 "한 줄 소개" 정확히 반영됨
- 톤앤매너가 자판기 컨셉과 잘 맞음

### 2. "건너뛰기" 버튼 Secondary 스타일 ✅
- 답 보기 전 상태에서 "건너뛰기"가 아웃라인(Secondary) 스타일로 변경됨
- "다음!"(Primary, 주황색 solid) vs "건너뛰기"(Secondary, 아웃라인)로 시각적 차이가 명확해짐
- Iteration 1 피드백 "같은 위치, 같은 스타일" 문제 해결됨

### 3. 즐겨찾기 별표 UX — 화면 유지 + "저장됨!" 토스트 ✅
- 별표 누르면 현재 화면에 머무름 (즐겨찾기 페이지로 이동하지 않음!)
- 별 아이콘이 빈 별 → 채워진 별로 토글되며 "저장됨" 텍스트로 변경
- 하단에 "저장 취소" 토스트가 나타남 — Undo 옵션 제공. 실수로 눌러도 되돌릴 수 있어서 좋음
- Iteration 1의 최대 불만("흐름 끊김") 완전히 해결됨

### 4. 클립보드 복사 버튼 ✅
- 답 공개 후 "복사" 버튼이 저장 버튼 옆에 배치됨
- 아이콘 + "복사" 텍스트 조합으로 직관적
- (헤드리스 환경에서 "복사 실패" 토스트가 뜨지만, 실제 모바일에서는 정상 작동할 것으로 예상)

### 5. 즐겨찾기 삭제 터치 영역 확대 ✅
- 삭제 버튼 44x44px (min-width/min-height 모두 44px)
- Apple HIG 최소 터치 타겟(44pt) 정확히 충족

### 6. 카운터 자정 리셋 — 확인 불가 ⚠️
- "오늘 6개 봄" 카운터는 정상 동작 확인
- 자정 리셋은 테스트 시간대(오후)에서 확인 불가. 코드 로직 기준으로는 구현된 것으로 보임

## 화면별 피드백

### 1. 메인 화면 (랜딩)
![메인 화면](/tmp/home-screen.png)

**좋아진 점:**
- 서브 카피 추가로 첫 방문자도 즉시 이해 가능
- "오늘 N개 봄" 카운터 하단 유지 — 은근 중독성
- 여전히 로그인/온보딩 제로. 엘리베이터 문 열리기 전에 개그 하나 뽑을 수 있음

**아쉬운 점:**
- 상단 절반 여백은 여전히 비어있음. Iteration 1에서 언급한 자판기 일러스트/캐릭터는 아직 미반영
- 서브 카피가 버튼 바로 위에 있어서 시선 흐름이 "중간부터" 시작됨. 상단에서 자연스럽게 내려오는 시각 흐름이면 더 좋겠음

### 2. 개그 카드 (질문 → 답 공개)
![질문 카드](/tmp/joke4.png) → ![답 공개](/tmp/joke4-answer.png)

**좋아진 점:**
- "건너뛰기" Secondary 스타일로 답 보기 전/후 상태 구분이 명확해짐
- 답 공개 후 복사/저장 버튼이 바로 접근 가능
- "탭해서 답 보기" 힌트가 자연스러움

**아쉬운 점:**
- 카드 등장 애니메이션이 여전히 없음. "두구두구..." 로딩은 있지만, 카드 자체가 "탁" 떨어지는 물리적 피드백이 부족
- 답 공개 시 서스펜스 연출(딜레이 + 페이드인)이 여전히 없음. 탭하면 즉시 답이 보임
- 복사/저장 버튼이 카드 안에 있어서 카드 디자인이 다소 복잡해진 느낌. 카드 아래에 별도 액션 바로 분리하면 더 깔끔할 수 있음

### 3. 즐겨찾기 페이지
![즐겨찾기](/tmp/all-interactive.png)

**좋아진 점:**
- 삭제 터치 영역 44x44px로 모바일 UX 충족
- 저장된 개그 리스트가 질문+답 형태로 잘 보임

**아쉬운 점:**
- 즐겨찾기 목록에서 "복사" 버튼이 여전히 없음. 저장해둔 개그를 슬랙에 옮기려면 일일이 텍스트를 선택해야 함
- 삭제 시 확인/Undo 없이 즉시 삭제 + 빈 상태면 홈으로 자동 이동. 실수로 마지막 아이템을 삭제하면 돌이킬 수 없음
- 빈 상태의 초록색 별 아이콘이 앱 전체 톤(주황/갈색)과 여전히 불일치

### 4. 전체 흐름 (엘리베이터 시나리오)
앱 열기 → "눌러봐!" → 두구두구 로딩(~1.5초) → 질문 카드 → 탭 → 답 공개 → 복사 → 슬랙에 붙여넣기

이 흐름이 **5초 이내**에 가능함. Iteration 1 대비 "슬랙에 개그 던지기" 시나리오가 완전히 해결됨!

## 개선 제안 (우선순위별)

### P1 (강력 권장)
1. **즐겨찾기 목록에서 복사 기능**: 저장해둔 개그를 나중에 꺼내 쓰는 게 즐겨찾기의 존재 이유. 각 아이템에 복사 버튼 또는 탭해서 복사
2. **즐겨찾기 삭제 Undo**: 삭제 후 "되돌리기" 토스트 (저장 시 "저장 취소" 토스트가 있듯이, 삭제에도 같은 패턴 적용)
3. **카드 등장 애니메이션**: 자판기에서 떨어지는 느낌의 슬라이드다운 + 바운스. 앱의 재미 요소에서 핵심

### P2 (있으면 좋음)
4. **답 공개 서스펜스 연출**: 탭 후 0.3초 딜레이 + 답 텍스트 페이드인/슬라이드업
5. **메인 화면 상단 비주얼**: 자판기 캐릭터 또는 일러스트로 첫인상 강화
6. **빈 즐겨찾기 색상 통일**: 초록 별 아이콘 → 주황 톤으로 통일
7. **즐겨찾기에서 삭제 시 빈 상태 전환 애니메이션**: 마지막 아이템 삭제 시 갑자기 홈으로 이동하지 않고, 빈 상태 화면을 잠시 보여준 후 자연스럽게 전환

## 만족도 근거

**Iteration 1 대비 핵심 개선 (62점 → 78점, +16점):**

김대리의 3대 사용 시나리오 재평가:

1. **"엘리베이터에서 개그 하나 보기"** → **92점** (90→92). 서브 카피 추가로 첫 방문 경험 소폭 개선. 여전히 즉시성 만점.

2. **"슬랙에 개그 하나 던지기"** → **75점** (30→75, **+45점!**). 복사 버튼 추가로 핵심 불만 해결! 답 공개 후 바로 복사 → 슬랙 붙여넣기 가능. 다만 즐겨찾기 목록에서의 복사는 아직 미지원.

3. **"마음에 드는 개그 저장해두기"** → **70점** (45→70, **+25점**). 별표 UX 대폭 개선 — 화면 유지 + 저장됨 피드백 + Undo 토스트. 다만 즐겨찾기에서 꺼내 쓰기(복사)가 안 되는 건 여전히 아쉬움.

**무마찰(frictionless) 재평가:**
- 첫 접근: 마찰 제로 → 변동 없음 (만점)
- 개그 소비: 마찰 낮음 → 변동 없음 (좋음)
- 개그 공유: ~~마찰 최대~~ → **마찰 낮음** (복사 버튼으로 해결!)
- 개그 저장: ~~마찰 중간~~ → **마찰 낮음** (화면 유지 + Undo)
- 저장 개그 재활용: 마찰 중간 (즐겨찾기에서 복사 미지원)

**80점 미달 사유:**
- 즐겨찾기 목록에서 복사 불가 (-8점): 저장의 최종 목적이 "나중에 써먹기"인데 이 경로가 여전히 불편
- 카드 애니메이션/서스펜스 부재 (-7점): 기능은 완비되었으나 "재미" 요소가 부족. 아재개그 앱에서 재미 연출은 핵심
- 즐겨찾기 삭제 Undo 미지원 (-5점): 저장에는 Undo가 있는데 삭제에는 없는 비일관성
- 빈 즐겨찾기 색상 불일치 (-2점): 사소하지만 앱 완성도에 영향

**결론:** Iteration 1의 치명적 문제(공유 불가, 저장 시 화면 이동)가 모두 해결되어 앱의 핵심 루프가 완성됨. "자판기에서 음료 뽑기 → 빨대 꽂기 → 마시기"까지 가능해졌다. 남은 과제는 "재미 연출"과 "저장 개그 재활용" — 이 두 가지가 해결되면 85점 이상 가능.
