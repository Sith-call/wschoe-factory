## Critical lessons from Plan 2 execution

1. **Text-signal leakage was the #1 spec violation.** Legacy orchestrator prose contains phrases like "emit PM_STAGE_COMPLETE" that subagent implementers instinctively copied. Caught 2 instances (T1 release-prep, T7 pm-orchestrate) — both fixed before merge. **Always `grep -r STAGE_COMPLETE plugins/*/skills/*/SKILL.md` after writing any skill.**

2. **Dual gate in spec is intentional, not a bug.** `§6.3 Stage 4→5` uses `design_score ≥ B` (outer gate, trusts Ralph report); `§8.7` uses `≥ B+` (inner Phase 3 judgment). Spec reviewer flagged this as inconsistency — it isn't. Each gate must be quoted from its own source section.

3. **Subagent-driven execution worked well.** 14 tasks, all implementers DONE on first try. Spec reviewer caught 1 real violation + 1 false positive. Worth keeping the 2-stage review flow.
